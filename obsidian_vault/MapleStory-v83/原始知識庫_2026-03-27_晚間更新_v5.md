## 🆕 Maple_Pshark GitHub 最新確認 (2026-03-27)

**GitHub**: https://github.com/obstriker/Maple_Pshark

### README 原文
> "Mapleroyals packet logger.
> This is a Maplestory v83 packet logger for private servers.
> This piece of software hooks to the encryption function and let you intercept messages as they go."

### 核心功能確認
1. **Intercepting Recv messages** — 攔截客戶端接收封包
2. **Modify messages** — 修改封包內容
3. **Injecting packets for send or recv** — 注入自訂封包
4. **Rule based packet manipulation/filtering** — 規則化封包操作

### 技術原理
- Hook 到加密函數，直接截獲明文
- 不需要知道 AES Key 或 IV
- 適用於 MapleRoyals 等私人伺服器

### 專案結構
```

├── Maple_Pshark/    # 主程式碼
├── Debug/           # 調試構建
├── Maple_PShark.sln # Visual Studio 方案檔
└── README.md
```

## 🆕 今日學習重點摘要 (2026-03-27 晚間)

### 1) 登入器/客戶端修改
- Ezorsia V2 + Hendi's Client 是 2026 年最佳組合
- 4G Flag 修改：偏移 0x0000013E，0x0F → 0x2F
- Magpie 可實現無損全螢幕投射
- STREDIT 仍是標準 IP 修改工具

### 2) 抓包分析
- Maple_Pshark 是 v83 專用封包截獲工具
- 第一個封包（Hello Packet）永遠是明文
- 三層加密：MapleCrypto + Shanda + AES-OFB
- Wireshark 過濾：`tcp.port == 7575 || tcp.port == 8484`

### 3) Opcode 結構
- v83 Opcode 幾乎全部被映射（開源專案已完整提供）
- 唯一需要的反向工程是修改客戶端時
- CClientSocket::ProcessPacket 是所有封包入口
- IDA + MaplePacketPuller 是標準分析流程

### 4) WZ 修改
- HaRepacker-resurrected v8.0.0 是最新工具
- GMS (old) 加密格式是 v83 標準
- **黃金法則**：永遠不要用伺服器 XML 覆蓋客戶端 WZ
- NX 格式是高效替代方案，但需確認音源支援

### 5) NPC 腳本編寫
- status 狀態機是核心概念
- `cm.` 函數是跨音源相容性最好的方式
- `importPackage` 在 JDK 7+ 已廢棄
- 動態商店腳本可避免 SQL 修改
