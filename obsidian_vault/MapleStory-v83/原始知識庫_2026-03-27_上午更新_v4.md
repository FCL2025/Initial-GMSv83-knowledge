**可編輯內容**:

- 忘記 ID/密碼 URL

**使用方法**（v118 以下版本）:
```
1. 開啟 STREDIT
2. 開啟 localhost.exe
3. 當 STREDIT 詢問 "Is this v118+?" → 選擇「No」
4. 當 STREDIT 詢問 "IP Search?" → 選擇「Yes」
5. 點擊 "Force to a IP" 按鈕
6. 輸入目標 IP
7. 儲存
```

**v83 專用 STREDIT 版本差異**:

#### HxD Hex 編輯 IP 步驟（最通用方法）

```
1. HxD 開啟 localhost.exe（先複製備份）
2. Ctrl+F，選擇 "Text-string"，搜尋 "127.0.0.1"
3. 會找到三個相連的 127.0.0.1
4. 在第一個 "127" 前面游標置入，覆蓋輸入目標 IP
5. 三個 IP 都完成替換
6. Ctrl+S 儲存
```

**注意**: IP 長度可能不同（如 "192.168.1.100" 比 "127.0.0.1" 長），需注意覆蓋範圍。

### 3) Opcode 結構

#### Send vs Recv Opcode 核心區分

> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another." — MapleRef Wiki

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送，客戶端接收 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送，伺服器接收 | Client Opcode Set |

**⚠️ 重要**: 同一個 Opcode 數字在不同方向代表**完全不同**的含義！

#### 登入相關 Opcode 對照

| Opcode (Hex) | S→C 名稱 | C→S 名稱 |
|-------------|---------|---------|
| 0x00 | LOGIN_STATUS（登入結果） | - |
| 0x01 | - | LOGIN_PASSWORD（登入請求） |
| 0x02 | ACCEPT_LOGIN | - |
| 0x03 | CHARLIST（角色列表） | - |
| 0x06 | - | SERVERLIST_REQUEST |
| 0x07 | - | CHARLIST_REQUEST |
| 0x08 | - | CHAR_SELECT |
| 0x0B | - | PLAYER_DROP |

#### CClientSocket::ProcessPacket 封包分發流程

```
網路接收 (TCP)
    ↓
CClientSocket::ProcessPacket()
    ↓  根據 MapleCrypto header 解密
    ↓  提取 opcode (2 bytes, little endian)
    ↓
switch(opcode)
    ↓
CLogin::OnPacket()      // 登入畫面狀態
    或
CField::OnPacket()      // 遊戲中狀態
    ↓
呼叫具體的封包處理函數
```

#### MapleCrypto 頭部混淆公式（v83 核心）

**加密端 (getPacketHeader)**:
```java
public byte[] getPacketHeader(int opcode, int length, int iv) {
    int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
    verify = (verify << 3) | (verify >>> 5);
    verify &= 0xFFFF;
    return new byte[] {
        (byte) (verify & 0xFF),
        (byte) ((verify >>> 8) & 0xFF),
        (byte) (length & 0xFF),
        (byte) ((length >>> 8) & 0xFF)
    };
}
```

**解密端 (retrieveOpcode)**:
```java
public int retrieveOpcode(byte[] header, int iv) {
    int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
    verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
    return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
}
```

#### v83 三層加密架構

```
┌─────────────────────────────────────────────────────┐
│ Layer 1: MapleCrypto (頭部混淆)                     │
│   - 4 bytes header (混淆 opcode + length)           │
│                                                        │
│ Layer 2: Shanda Encryption (對稱加密)               │
│   - v83 仍在使用                                     │
│   - v118+ 已移除                                     │
│                                                        │
│ Layer 3: AES-OFB (主加密)                           │
│   - 256-bit key, 4-byte IV                          │
│   - 版本決定的靜態金鑰                              │
└─────────────────────────────────────────────────────┘

加密流程 (C→S):
明文 → AES-OFB → Shanda → MapleCrypto header → 網路

解密流程 (S→C):
網路 → MapleCrypto 解混淆 → Shanda 解密 → AES-OFB → 明文
```

### 5) NPC 腳本編寫

#### NPC 腳本狀態機核心

```javascript
var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
        return;
    }
    if (mode == 0 && type == 1) {
        cm.dispose();
        return;
    }
    
    if (mode == 1)
        status++;
    else
        status--;
    
    if (status == 0) {
        cm.sendNext("第一頁對話");
    } else if (status == 1) {
        cm.sendSimple("選擇：\r\n#b#L0#選項1#l\r\n#L1#選項2#l");
    } else if (status == 2) {
        cm.dispose();
    }
}
```

**Mode 參數含義**:
| Mode 值 | 意義 |
|---------|------|
| 1 | 玩家點擊了"下一步"或選項 |
| 0 | 玩家點擊了"返回" (back button) |
| -1 | 玩家關閉了對話框 (ESC 或 X) |

#### sendAcceptDecline / sendGetNumber / sendGetText

**sendAcceptDecline (Type = 12)**:
| Mode 值 | 意義 |
|---------|------|
| `mode == -1` | 玩家關閉對話框 |
| `mode == 1` | 玩家點擊「接受」(Accept) |
| `mode == 0` | 玩家點擊「拒絕」(Decline) |

**sendGetNumber**:
```javascript
// 參數：提示文字、預設值、最小值、最大值
cm.sendGetNumber("請輸入你要購買的數量：", 1, 1, 99);
```

#### 動態 NPC 商店腳本源（所有音源通用）

```javascript
/*
 * 動態 NPC 商店 - 不需要 SQL
 * NPC ID: 9010009
 */
var status = 0;
var items = Array(
    Array(1302000, 10000, 1),
    Array(1302001, 20000, 1),
    Array(1302002, 30000, 1)
);

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) { cm.dispose(); return; }
    if (mode == 0 && type == 1) { cm.dispose(); return; }
    if (mode == 1) status++; else status--;

    if (status == 0) {
        var msg = "歡迎光臨！\r\n請選擇商品：\r\n";
        for (var i = 0; i < items.length; i++) {
            msg += "#L" + i + "##i" + items[i][0] + "# - " + items[i][1] + " 金幣#l\r\n";
        }
        cm.sendSimple(msg);
    } else if (status == 1) {
        var item = items[selection];
        if (cm.getMeso() >= item[1]) {
            if (cm.canHold(item[0])) {
                cm.gainMeso(-item[1]);
                cm.gainItem(item[0], item[2]);
                cm.sendOk("購買成功！");
            } else {
                cm.sendOk("背包空間不足！");
            }
        } else {
            cm.sendOk("金幣不足！");
        }
        cm.dispose();
    }
}
```

## 🆕 v83 完整架設流程摘要 (RaGEZONE 2025-2026)

### 環境需求
| 軟體 | 版本 | 必要性 |
|------|------|--------|
| JDK | 8 | 必要 |
| JCE | 8 | 必要（加密） |
| MySQL | 5.5+ | 必要 |
| v83 localhost 客戶端 | - | 必要 |
| HxD | - | 修改 IP 用 |

### 完整架設流程
```
1. 安裝 JDK 8 + JCE 8
2. 安裝 MySQL Server 5.5+
3. 下載 v83 localhost 客戶端
4. 架設 Java 伺服器（NetBeans / IntelliJ）
5. 執行 SQL 初始化資料庫
6. 設定 ServerConstants.java 中的 IP 和埠口
7. HxD 或 STREDIT 修改客戶端 IP（三個 127.0.0.1 全部替換）
8. 啟動客戶端和伺服器
```

### LAN 多人連線必檢查清單
- [ ] 確認兩台 PC 在同一 LAN 網段
- [ ] 確認伺服器 listen 0.0.0.0（而非 127.0.0.1）
- [ ] 確認防火牆開啟 7575 和 8484 埠口
- [ ] 若錯誤碼 0x80004005 → 防火牆/權限問題
- [ ] 若錯誤碼 -2147467259 → 同上
