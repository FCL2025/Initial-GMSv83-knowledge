### 重點學習領域更新

#### 登入器/客戶端修改
- **Ezorsia V2** (2023-11-16 發布) 是第一個真正的獨立 dinput8.dll 客戶端
- **Kimmy's Archive** 包含所有可直接使用的 v83 localhost 客戶端
- **Hendi 4G Client** 需手動 Hex 編輯 `0000013E`: `0f` → `2f`

#### 抓包分析
- **Maple_Pshark** 是 v83 專用的即時封包 Logger，Hook 加密函數直接截獲明文
- **MaplePacketPuller + SpiritIDAPlugin** 離線分析封包結構
- **Wireshark** 仍是基本工具，過濾：`tcp.port == 7575 || tcp.port == 8484`

#### Opcode 結構
- v83 Opcode 已幾乎全部映射，開源專案 (Cosmic, HeavenMS) 完整提供
- **MapleRef Wiki** 是最完整的封包結構參考：mapleref.fandom.com/wiki/Packets
- IDA 分析價值在於理解客戶端內部運作，而非尋找 Opcode

#### WZ 修改
- **HaRepacker-resurrected v8.0.0** (2026-03-05) 新增 AI 地圖編輯器對話功能
- **WzImg-MCP-Server** 74 tools 供 AI 助手直接互動 WZ/IMG 檔案
- **WZ 黃金法則**：客戶端 WZ → 直接編輯 → 導出 "Private Server" XML → 伺服器端
- **千萬不要**：用伺服器 XML 覆蓋客戶端 WZ

#### NPC 腳本編寫
- **cm. 函數**是最普遍支援的介面（Cosmic、Kinoko、HeavenMS 都支援）
- **避免 importPackage**（JDK 7+ 已廢棄）
- **NPC TYPE 0** = 普通 NPC（需要 .js 腳本），**TYPE 1** = 商店 NPC
- 動態商店腳本可避免每次修改都要重啟伺服器

### 2026-03-27 最新論壇討論重點

#### NPC TYPE 值最新定義
| Type 值 | 客戶端行為 | 腳本需求 | 範例 |
|---------|-----------|---------|------|
| `0` | 普通 NPC，點擊後發送 NPC_TALK | ✅ 需要 `.js` | 任務 NPC |
| `1` | 商店 NPC，點擊後開啟商店介面 | 可有可無 | 商人、鐵匠 |
| `2~6` | 特殊功能 NPC（拍賣、保險箱等） | ❌ 不需要 | 拍賣管理員 |

#### LAN 多人連線錯誤排查
- 錯誤碼 `-2147467259` (0x80004005) = Access Denied，**非網路問題，是防火牆問題**
- 排查順序：確認同一 LAN → 確認 listen 0.0.0.0 → 檢查 Windows 防火牆
- 快速測試：`netsh advfirewall set allprofiles state off`

#### Quest LvMax 移除 Regex

```regex
取代：（留空）
```
