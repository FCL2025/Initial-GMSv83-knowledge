## 一、客戶端修改 (Client Modification)

### 1.1 Hex 編輯客戶端 - 詳細流程

根據 HeavenMS 官方文檔及多個論壇教程，v83 客戶端修改的核心是透過十六進制編輯器修改 `localhost.exe` 中的 IP 位址。

**所需工具：**
- Neo Hex Editor (Free Hex Editor Neo)
- OllyDbg (調試用)
- Cheat Engine (輔助)

**Hex 編輯步驟：**

1. **定位 IP 位址：**
   - 在十六進制編輯器中以 "String" 類型搜索 `127.0.0.1`
   - v83 localhost.exe 通常包含 3 處 IP 地址需要修改

2. **修改流程：**
   ```
   1. 使用 Neo Hex Editor 打開 "localhost.exe"
   2. 搜索字符串 "127.0.0.1" (共3處)
   3. 將每處替換為目標 IP (如 Hamachi IP 或公網 IP)
   4. 保存並退出
   ```

3. **兼容性問題 (Windows 8/10)：**
   - 若客戶端無法啟動，需設置兼容模式：
     - 運行模式：Windows 7
     - 分辨率：640 x 480
     - 以管理員身份運行
   - 若仍不行，可使用 `lolwut.exe` 繞過

### 1.2 客戶端修改的進階方法

**使用 STRedit.exe：**
- 專用於 MapleStory 的字符串編輯工具
- 比 Hex 編輯更直觀，專門處理字串替換
- 適用於修改伺服器 IP、端口等字符串

**去除保護殼 (Themida)：**
- 早期 v83 客戶端使用 Themida 加殼
- 需使用脫殼工具處理後才能正常調試/修改
- 脫殼後才能使用 Cheat Engine 等工具

**重要安全提示：**
> HeavenMS 官方指出：這些 localhost.exe 被防病毒軟件標記為潛在惡意軟件，這是由於應用了逆向工程方法。但這些軟件已使用多年，確認安全。

## 二、抓包分析 (Packet Capture)

### 2.1 MapleStory v83 封包加密機制

v83 版本使用兩層加密：

**第一層：AES-OFB 加密**
- 每個連接會話使用兩個密碼器 (ciphers)
- 每個密碼器包含一個 IV (Initialization Vector)
- 用於 AES-OFB 模式加密

**第二層：Shanda 加密 (MapleStory 自有加密)**
- v83 仍使用 Shanda 加密
- 較新版本已移除此層

**密鑰交換流程：**
```
客戶端連接 → 伺服器發送 Hello 封包 (明文) 
→ 包含 recv_iv 和 send_iv (兩組 IV)
→ 客戶端使用這些 IV 初始化 AES 加密
→ 此後所有封包均加密
```

### 2.2 抓包工具

**MaplePshark (v83 專用)：**
- GitHub: obstriker/Maple_Pshark
- 功能：鉤子到加密函數，攔截往來消息
- 專為 MapleRoyals v83 私服設計

**Wireshark：**
- 通用網絡封包分析工具
- 需要理解 TCP 協議和加密機制
- 適合分析登入流程和Opcode

**Packet Editor (DLL 版本)：**
- Kitterz Packet Editor
- 可修改往來封包
- 支持 opcode + AES key 修改

### 2.3 登入流程封包分析

**Hello 封包結構 (v83)：**
```
0E 00 23 00 01 00 [字節: 31 46 72 7A 52 52 30 78 73 04]
recv_iv: 70, 114, 122, 82
send_iv: 82, 48, 120, 115
```

**登入請求封包：**
- 經 AES 解密後再經 Shanda 解密
- Opcode 通常為 `01 00` (登入請求)
- 包含帳號、密碼等敏感信息

## 三、Opcode 結構 (Opcode Structure)

### 3.1 Opcode 基本概念

**什麼是 Opcode：**
- 操作碼，封包的第一部分
- 指示伺服器/客戶端這個封包的類型
- 每個遊戲動作對應一個 Opcode

**v83 Opcode 特性：**
- 使用 2 位元組無符號整數 (0x0000 - 0xFFFF)
- 客戶端發送和伺服器發送的 Opcode 各自獨立
- 需要對應的密鑰進行加密/解密

### 3.2 常用 Opcode 列表

**客戶端 → 伺服器：**
| Opcode | 名稱 | 說明 |
|--------|------|------|
| 0x01 | LOGIN | 登入請求 |
| 0x06 | CHARLIST | 請求角色列表 |
| 0x07 | CHARSELECT | 選擇角色 |
| 0x08 | PLAYERLOGIN | 進入遊戲 |

**伺服器 → 客戶端：**
| Opcode | 名稱 | 說明 |
|--------|------|------|
| 0x00 | PING | 心跳包 |
| 0x13 | SERVERLIST | 伺服器列表 |
| 0x14 | CHARLIST | 角色列表 |
| 0x16 | SERVERSTATUS | 伺服器狀態 |

### 3.3 動態 Opcode

v83 使用動態 Opcode 系統：
- Opcode 會根據遊戲狀態動態變化
- 需要正確的密鑰才能正確解析
- 不同版本 (GMS, KMS, CMS) 的 Opcode 有所不同

## 四、WZ 修改 (WZ Modification)

### 4.1 WZ 文件結構

WZ 是 MapleStory 的資源文件格式：
- **Map.wz** - 地圖資源
- **Skill.wz** - 技能數據
- **Item.wz** - 物品數據
- **Mob.wz** - 怪物數據
- **NPC.wz** - NPC 數據

### 4.2 WZ 編輯工具

**HaRepacker-resurrected (推薦)：**
- 開源 WZ 編輯器
- 支持 GMS (old) 加密方式
- 可進行節點的複製、粘貼、編輯

**編輯流程：**
1. 使用 HaRepacker 打開 .wz 文件
2. 在節點層次結構中找到要修改的內容
3. 進行所需更改
4. 保存覆蓋原文件
5. 重新導出為 "Private Server" 格式 (XML)

### 4.3 客戶端 vs 伺服器 WZ

**重要區別：**
- **客戶端 WZ**：包含客戶端渲染所需的資源 (圖片、精靈等)
- **伺服器 WZ (XML)**：純數據，不包含圖形資源
- 兩者格式不同，不能直接互换

**修改原則：**
1. 編輯客戶端 .wz 文件
2. 使用 HaRepacker 導出為 Private Server 格式
3. 將導出的 XML 覆蓋到伺服器的 wz 目錄
4. 始終從客戶端 WZ 導出到伺服器，而非反向

### 4.4 動態 NPC 商店腳本

動態 NPC 商店可通過腳本實現，無需修改數據庫：
- 伺服器可通過修改支持動態商店
- 每次玩家與 NPC 對話時讀取腳本
- 方便快速修改商店內容

## 五、NPC 腳本編寫 (NPC Scripting)

### 5.1 NPC 腳本基礎

v83 私服 NPC 腳本基於 JavaScript 或服務器內置腳本語言。

**基本結構：**
```javascript
// 標準 NPC 對話腳本
var status = -1;

function start() {
    cm.sendNext("歡迎來到我的商店！");
}

function action(mode, type, selection) {
    if (mode == 1) {
        status++;
    } else {
        cm.dispose();
        return;
    }
    
    if (status == 0) {
        cm.sendYesNo("你想要購買物品嗎？");
    } else if (status == 1) {
        // 處理購買
        cm.gainItem(4000000, 1);  // 給予物品
        cm.sendOk("謝謝惠顧！");
        cm.dispose();
    }
}
```

### 5.2 NPC 腳本常見功能

| 功能 | 方法 | 說明 |
|------|------|------|
| 發送消息 | sendNext(), sendOk(), sendYesNo() | 與玩家對話 |
| 給予物品 | gainItem(itemId, quantity) | 給玩家物品 |
| 扣除物品 | gainItem(itemId, -quantity) | 從玩家移除物品 |
| 給予金錢 | gainMeso(amount) | 給玩家金幣 |
| 給予經驗 | gainExp(exp) | 給玩家經驗值 |
| 判斷物品 | hasItem(itemId) | 檢查玩家是否擁有物品 |
| 開始任務 | startQuest(questId) | 開始任務 |
| 完成任務 | completeQuest(questId) | 完成任務 |

### 5.3 任務 NPC 腳本

任務 NPC 需要更複雜的狀態管理：

```javascript
var questId = 1000;
var questStatus = cm.getQuestStatus(questId);

if (questStatus == 0) {
    // 任務未開始
    cm.sendNext("你聽說過 #b黑暗森林#k 的怪物嗎？");
} else if (questStatus == 1) {
    // 任務進行中
    if (cm.hasItem(4001000, 10)) {
        // 完成條件
        cm.gainItem(4001000, -10);
        cm.gainExp(1000);
        cm.completeQuest(questId);
        cm.sendOk("太好了！你收集夠了！");
    } else {
        cm.sendOk("我还需要 #b10個#k 黑暗精華。");
    }
} else {
    // 任務已完成
    cm.sendOk("謝謝你，已經完成了！");
}
cm.dispose();
```

### 5.4 腳本存放位置

在 HeavenMS/MapleSolaxiaV2 等服務端：
```
source/scripts/npc/
  ├── 9000000.js  (測試NPC)
  ├── 9000001.js  (商店NPC)
  └── ...
```
