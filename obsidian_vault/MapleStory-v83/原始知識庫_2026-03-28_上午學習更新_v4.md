## 1. 登入器 / 客戶端修改 — Hex 位址修改流程

### 1.1 localhost.exe IP 修改 (v83 通用流程)

**工具**：HxD Hex Editor / Hex Edit Workshop 6

**修改步驟**：
1. 以 16 進制模式開啟 `localhost.exe`
2. CTRL+F 搜尋 `127.0.0.1`（ASCII 或 Hex `31 32 37 2E 30 2E 30 2E 31`）
3. 替換為目標 IP（Hamachi / WAN IP / VPS IP）
4. **注意**：需修改全部三處出現位置

**範例（ExcellZone 論壇）**：
```
1. Hex Edit Workshop 6 開啟 localhost.exe
2. CTRL+F 搜尋 "127.0.0.1"
3. 全部替換為目標 IP（例如 84.25.81.181）
```

## 2. 抓包分析 — 工具詳解與實戰

### 2.1 MaplePE — 32-bit 封包編輯器 (2025 新工具)

**GitHub**：https://github.com/zhyonc/MaplePE

**核心技術架構**：
```
MaplePE.exe ──UDP server──> Packet.dll (injected)
     │                            │
     │                     hooks CInPacket::*Decode*
     │                     hooks COutPacket::*Encode*
     │                            │
     └────<── UDP client ──────────┘
```

**需要 Hook 的函數地址**：
| 函數 | 用途 |
|------|------|
| `CInPacket::Decode1` | 讀取 1 byte |
| `CInPacket::Decode2` | 讀取 2 bytes |
| `CInPacket::Decode4` | 讀取 4 bytes |
| `CInPacket::Decode8` | 讀取 8 bytes（可選）|
| `CInPacket::DecodeStr` | 讀取字串 |
| `CInPacket::DecodeBuffer` | 讀取緩衝區 |
| `COutPacket::Encode1` | 寫入 1 byte |
| `COutPacket::Encode2` | 寫入 2 bytes |
| `COutPacket::Encode4` | 寫入 4 bytes |
| `COutPacket::Encode8` | 寫入 8 bytes（可選）|
| `COutPacket::EncodeStr` | 寫入字串 |
| `COutPacket::EncodeBuffer` | 寫入緩衝區 |
| `COutPacket::MakeBufferList` | 封包組裝 |
| `CClientSocket::ProcessPacket` | 封包分發 |
| `CClientSocket::SendPacket` | 封包發送 |

**已知版本位址表**：https://docs.google.com/spreadsheets/d/1-LqA-OWgOuT-Xy3i73Jy5F_uVmogaGvmYpH4mLDSPhg

**使用前提條件（重要）**：
> "Before sending a packet via MaplePE.exe, the server must respond with at least one packet to ensure that the CClientSocket instance can be correctly specified."

**BB 版本後防鉤檢測**：
> "Since some versions after BB add hook-detection in CClientSocket::SendPacket, Hook MakeBufferList instead of SendPacket to avoid client crash"

**v83 封包發送崩潰修復（Cheat Engine NOP）**：
```
需禁用以下六個崩潰檢查點：
1. retaddr < 4198400 檢查
2. m_hSocket 相關鎖定檢查 ×4
3. CClientSocket::Flush 鎖定檢查
```

## 3. Opcode 結構 — v83 封包通信底層

### 3.1 Opcode 基本結構

**Opcode 定義（MapleRef）**：
> "Actions that are handled by packets on MapleStory are defined by an identifier called an Operation Code (opcode for short), which is the first 2 bytes of the packet (a short)"

**Little Endian 注意**：
> "when that value is put in Little Endian format it becomes 30 11 00 00."

**Opcode 命名規範**：
- C→S (Client to Server)：`LOGIN_REQUEST`, `CHECK_PASSWORD`, `MOVE_PLAYER`
- S→C (Server to Client)：`LOGIN_STATUS`, `SERVERLIST`, `ENTER_PORTAL`

### 3.2 MapleCrypto 加密頭部格式

**頭部 4 bytes**：
```
[2 bytes: 封包長度][2 bytes: 混淆後 opcode/verify值]
```

**版本號處理**：
```python
version_outgoing = -(version + 1)  # 發送到客戶端
version_incoming = version          # 發送到伺服器

# verify 值計算
verify = (opcode ^ iv[0]) + (iv[1] << 8)
verify = (verify << 3) | (verify >> 13)  # rotate left 3 bits
```

### 3.3 第一個封包特性（Botnored）

> "The first packet sent / received will always be in plain-text (not encrypted)."

**意義**：
- 可用 Wireshark 直接查看
- 用於交換加密 IV（金鑰協商）
- 之後所有封包經 MapleCrypto + AES-OFB 加密

### 3.4 封包分發流程 (CClientSocket::ProcessPacket)

```
CClientSocket::ProcessPacket
    ↓
    ├─ CLogin::OnPacket       → 登入相關 opcode
    ├─ CWvsContext::OnPacket  → warp2map 之前
    └─ CField::OnPacket       → 遊戲中所有其餘 opcode
```

### 3.5 更新 Opcode 的方法 (RaGEZONE)

**原文**：
> "All calls that have a parameter of a2 (or, iPacket as the official variable is named) is a function that is reading bytes from the packet structure sent from the server. So in OnCheckPassword you will see a lot of CInPacket calls using the a2 parameter. This is the packet structure that follows AFTER your 2-byte packet header."

**操作**：
1. IDA 找到 `OnCheckPassword` 等 handler
2. 分析 `CInPacket` 函數調用（如 `Decode1`, `Decode2`）
3. 對照封包結構文檔

### 5.3 腳本狀態機制

**基本模式**：
```javascript
function action(mode, selection) {
    if (mode == 1) {           // 玩家選擇
        status++;
    } else {
        status--;
    }
    
    if (status == 0) {
        // 第一頁
    } else if (status == 1) {
        // 第二頁
    }
    
    cm.dispose();  // 結束對話
}
```

**常見 status 值**：
| status 值 | 意義 |
|----------|------|
| 0 | 初始對話 |
| 1 | 第一選項處理 |
| 2 | 第二選項處理 |
