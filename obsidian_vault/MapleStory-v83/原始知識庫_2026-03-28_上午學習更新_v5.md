## 1. 登入器 / 客戶端修改 — 新發現

### 1.1 Ezorsia V2 Admin 提權移除重要發現（第七十五次研究新增）

**問題**：編輯 Admin elevation（提權）後，客戶端仍然要求管理員權限

**根本原因**：當執行檔命名為預設名稱 `MapleStory.exe` 時，Windows UAC 會繞過自訂的 `asInvoker` 設定

**解決方案**：
1. 使用 Hendi's Hex Editor 工具包編輯 `asInvoker` 部分
2. **將執行檔改名為 `MapleStory2.exe` 或其他名稱**
3. Windows 就會正確應用 `asInvoker` 設定，不再要求管理員權限

## 3. Opcode 結構 — 最新發現

### 3.1 MapleRef Wiki Send/Recv Opcode 核心概念

**來源**: https://mapleref.fandom.com/wiki/Operation_Code

### Send vs Receive Opcode 區分
> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

**⚠️ 重要**：同一個 Opcode 數字可能代表不同含義，視方向而定！

### 3.2 v83 封包 Header 核心發現

> "Packet headers are 4 bytes long, and are generated with the use of the send IV, packet length (without header) and the version of MapleStory. The version is -(version + 1) when sending a packet to the client, otherwise just the plain version."

### 3.3 MapleCrypto 頭部混淆公式

```java
// 加密端 (getPacketHeader)
int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
verify = (verify << 3) | (verify >>> 5);  // rotate left 3 bits
verify &= 0xFFFF;
return new byte[] {
    (byte) (verify & 0xFF),
    (byte) ((verify >>> 8) & 0xFF),
    (byte) (length & 0xFF),
    (byte) ((length >>> 8) & 0xFF)
};

// 解密端 (retrieveOpcode)
int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
```

### 3.4 MapleCrypto 頭部長度公式

```java
// 加密端
lengthEncrypted = length ^ ((iv >> 16) & 0xFFFF);

// 解密端
length = lengthEncrypted ^ ((iv >> 16) & 0xFFFF);
```

### 3.5 v83 AES User Key & 加密金鑰 (CMS 預設值)

```javascript
// AES User Key (8 bytes)
[0x13, 0x08, 0x06, 0xB4, 0x1B, 0x0F, 0x33, 0x52]

// IV Transformation Key
```

**注意**: 不同私服可能使用不同的金鑰，需與伺服器端匹配。

## 4. WZ 修改 — 最新論壇動態

### 4.1 HaRepacker WZ 導出/導入最佳實踐

**RaGEZONE 最新討論**: https://forum.ragezone.com/threads/community-wz-img-repack-for-v83.1221835/

**導出步驟**：
1. 用 HaRepacker 開啟 WZ 檔案（選擇 **GMS (old)** 加密格式）
2. 選擇要導出的節點
3. 點擊 Export → 選擇 XML (Private Server) 格式
4. 導出後可使用文字編輯器修改
5. 重新導入 WZ

**⚠️ 重要警告**：
- v83 WZ 加密格式需選擇 `GMS (old)`
- 不要直接刪除/重新導入節點，可能造成數據丟失
- 建議使用 HaRepacker UI 直接編輯數值

### 4.2 WZ 編輯黃金法則（MapleSolaxiaV2 README）

> "DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs."

**正確流程**：
1. HaRepacker 4.2.4 開啟 **GMS (old)** 加密的客戶端 WZ
2. **使用 UI 直接修改數值**（禁止刪除/重新導入節點！）
3. 儲存 WZ
4. 在 HaRepacker 中選擇 **"Private Server..."** 導出
5. 將導出的 XML 覆蓋到伺服器的 WZ.XML 目錄

```javascript
// JavaScript 基本寫法 - 適用所有音源
var status = 0;

function start() {
    cm.sendNext("你好！");
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
        return;
    }
    if (mode == 1) status++;
    else status--;
    
    if (status == 0) {
        cm.sendSimple("選擇：\r\n#b#L0#選項1#l\r\n#L1#選項2#l");
    } else if (status == 1) {
        if (selection == 0) {
            cm.gainItem(1302000, 1);
            cm.sendOk("獲得物品！");
        }
        cm.dispose();
    }
}
```

### 5.5 Quest NPC 與 MigrateIn 封包機制

**問題**：如何讓 Quest NPC 出現在 "Other" 選項中？

**解決方案**：
- 需要發送 `PlayerMigrateInPacket` 封包
- 封包包含：NPC ID 列表 + FileTime
- 這個封包讓客戶端知道哪些 NPC 有自訂腳本

**實現位置**（以 HeavenMS 為例）：
```java
// 在 PlayerLoggedInHandler 或 similar handler 中
int fileTime = 自定義腳本結束時間; // 毫秒時間戳
List<Integer> npcIds = Arrays.asList(9000000, 9000001, 9000002);
sendPacket(PlayerMigrateInPacket.encode(npcIds, fileTime));
```

### 5.6 Fast Travel NPC 完整腳本源（Z41N, RaGEZONE 2020）

```javascript
/*
 * 作者: Z41N
 * NPC 名稱: Sunny
 * 地圖: 前往 Ludi 的車站 (200000121)
 * 功能: 傳送到 Ludibrium，支援快速傳送票
 */
var status = 0;

function start() {
    cm.sendYesNo("嘿，我是 Sunny！你想傳送到 Ludibrium 嗎？");
}

function action(mode, type, selection) {
    status++;
    if (mode != 1) {
        if(mode == 0 && type != 1)
            status -= 2;
        else if(type == 1 || (mode == -1 && type != 1)) {
            if(mode == 0) {
                cm.sendOk("嗯... 看來你還有事要在這裡處理？");
                cm.dispose();
                return;
            }
        }
    }

    if (status == 1) {
        if (cm.haveItem(4310000))
            cm.sendNext("#b檢測到快速傳送票#k。\r\n看來你有這張票！");
        else
            cm.sendNext("當然可以！我帶你去 Ludibrium。");
    } else if (status == 2) {
        cm.warp(220000000);  // Ludi
        cm.dispose();
    }
}
```
