## 三、抓包分析

### 3.1 MaplePacketPuller — IDA 偽代碼分析工具

**GitHub**: https://github.com/Bratah123/MaplePacketPuller
**Python 3.8+ | IDA Pro 7.0+**

這是一個 Python 工具，用於分析 IDA 生成的 C 偽代碼，提取封包結構。

#### 使用流程

```
1. IDA Pro 對封包處理函數進行反編譯
2. 複製偽代碼到 .txt 文件（位於 MaplePacketPuller/IDA Maple Script/Functions/）
3. cd MaplePacketPuller/IDA Maple Script/src/main/python
4. python main.py
5. 選擇分析類型
6. 程式輸出封包結構到 FuncOutput/
```

#### 限制
- 不處理 `while()` 迴圈嵌套（設 `GET_ALL_DECODES = true` 可改善但有美觀缺點）
- 不顯示 switch case（需用戶自行添加）
- IDA 6.8 不相容（`os.scandir()` 在 Python 3.6+ 才有）

#### 配套工具：SpiritIDAPlugin
- GitHub: `Bratah123/SpiritIDAPlugin`
- 自動格式化 IDA 輸出為伺服器友好的格式

#### 輸入/輸出
- **輸入**：IDA 產生的 C 偽代碼（.txt）
- **輸出**：封包結構檔案（.txt）+ 主控台輸出

### 3.3 封包加密三層架構（v83）

```
┌──────────────────────────────────────────────┐
│ 1. MapleCrypto (頭部混淆)                    │
│    Header 4 bytes = 長度 + 混淆Opcode        │
│                                              │
│ 2. Shanda Encryption (IOBufferManipulator)   │
│    在 AES 之前應用                           │
│    v118+ 已移除                              │
│                                              │
│ 3. AES-OFB (主加密)                          │
│    主要資料加密，IV 每次更新                  │
└──────────────────────────────────────────────┘
```

**重要**：連接建立後第一個封包為**明文**（未加密），用於交換 AES IV。

## 四、Opcode 結構

### 4.1 v83 Opcode 特點

- **長度**: 2 bytes (little endian)
- **位置**: Header 之後
- **混淆**: MapleCrypto 頭部混淆
- **文檔化程度**: v83 Opcode 幾乎全部被映射

### 4.2 MapleCrypto 頭部混淆原理

```
原始Opcode: 0x0001
IV: [0x7A, 0x52, ...]

Step 1: XOR + SHIFT
  verify = (opcode ^ iv[0]) + (iv[1] << 8)
        = (0x0001 ^ 0x7A) + (0x52 << 8)
        = 0x007B + 0x5200
        = 0x527B

Step 2: rotate left 3 bits
  verify = (verify << 3) | (verify >> 13)
        = 0x527B << 3 | 0x527B >> 13
        = 0x29BD8 | 0x00
        = 0x29BD8

Step 3: 取低位 16 bits
  verify = 0x29BD8 & 0xFFFF = 0x9BD8
```

### 4.3 第一個封包（Hello Packet）

- 連接建立後第一個封包為**明文**（未加密）
- 用於交換 AES 初始化向量 (IV)
- 客戶端和伺服器各自生成隨機密鑰

### 6.2 動態 NPC 商店腳本（v83）

**來源**: https://forum.ragezone.com/threads/dynamic-npc-shop-script-v83.1089979/

特點：
- 不需要每次修改資料庫並重啟伺服器
- 使用 JavaScript 腳本動態生成商店
- 可靈活調整商品和價格

```javascript
var status = 0;
var items = [2000002, 2000003, 2000004]; // 物品ID列表
var prices = [1000, 500, 800];            // 對應價格

function start() {
    var sel = "#b#n=== 我的商店 ===#k\n\n";
    for (var i = 0; i < items.length; i++) {
        sel += "#L" + i + "##i" + items[i] + "# - " + prices[i] + " 楓幣#l\n";
    }
    cm.sendSimple(sel);
}

function action(mode, type, selection) {
    if (mode == 1) {
        var item = items[selection];
        var price = prices[selection];
        if (cm.getMeso() >= price) {
            cm.gainMeso(-price);
            cm.gainItem(item, 1);
            cm.sendNext("謝謝購買！");
        } else {
            cm.sendNext("楓幣不足！");
        }
    }
    cm.dispose();
}
```

### 6.3 NPC 腳本源相容性問題

**問題**：
- 不同音源（Cosmic、HeavenMS）的 NPC 腳本解析器版本不同
- 較舊的腳本在新解析器可能不相容

**原因**：
- `importPackage` 和 `SavedLocationType` 處理差異
- JDK 7+ 已廢棄 `importPackage`

**解決方案**：
1. 使用普遍支援的 `cm.` 函數
2. 避免使用特定音源專有的 Package 路徑
3. **避免 `importPackage`**（改用直接引用）
4. 先在本地測試再部署
