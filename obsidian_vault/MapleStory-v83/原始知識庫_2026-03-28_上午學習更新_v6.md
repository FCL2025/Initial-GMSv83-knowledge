### 1.1 MapleClientEditTemplate — C++ 客戶端修改框架（第七十六次研究新增）

**GitHub**: https://github.com/MapleStory-Archive/MapleClientEditTemplate
**作者**: Minimum Delta (Erik A)
**開發狀態**: Alpha，主動開發中
**YouTube 教學**: https://www.youtube.com/watch?v=Yr5IOhc5sPk

#### 設計理念
這是一個**越來越通用化的 MapleStory 客戶端編輯框架**，用於製作客戶端重定向/修改。

**核心特點**：
- 適用於任何版本（版本特定的 anticheat 和 CRC bypass 需要調整）
- 所有 Windows library hooks 已抽象化，可在 `commonconfig.h` 中開關
- v83 等未虛擬化的 localhost 可**禁用 WinSock hooks**，直接進行記憶體編輯
- 詳細除錯日誌可用 DebugView 查看

#### 架構說明
```
Common 專案：所有 Windows API hooks（非版本特定）
  - 在 injection 後立即執行（CreateMutexA hook 觸發 MainFunc）
  - 建議將所有 hooking/memory editing 放在 MainFunc 中
  - MapleStory function hooks：版本特定（尚未加入模板，需自行擴展）
```

#### 編譯設定（重要）
| 設定 | 說明 |
|------|------|
| 必須編譯為 **x86** | x64 會有 detour linker errors |
| detours.lib 放入 Debug 資料夾 | 必需 |
| `$(SolutionDir)Common;%(AdditionalIncludeDirectories)` | 加入 C/C++ → Additional Include Directories |

#### 兩種注入方式

| 方式 | 設定 | 說明 |
|------|------|------|
| **ijl15 proxy 自動注入** | 定義 `MAPLE_INJECT_USE_IJL` | MapleStory 啟動時自動注入，需將 DLL 改名為 `ijl15.dll` |
| **GenericLauncher 手動注入** | 普通注入 | 啟動 MapleStory 並注入 `LEN.dll` |

#### 教學影片章節
1. 框架介紹與環境架設
2. Common.h 設定詳解
3. 記憶體修改實例
4. WinSock Hook 教學（適用於需要攔截網路連線的版本）
5. 實際應用：製作 localhost enabler

### 1.3 v83 新版本物品導入技術（RaGEZONE, 2025 年 12 月最新）

**URL**: https://forum.ragezone.com/threads/maplestory-v83-newer-items-import.1256083/

#### 常見問題
將高版本（如 v176）的物品導入 v83 時：
- 物品圖像顯示正常
- 但**無法裝備**
- 系統提示錯誤或無反應

#### 問題根源
高版本物品的 WZ 結構比 v83 更複雜，包含 v83 不支援的額外欄位。

#### 解決方案
1. **不要直接移植整個 img 節點**
2. **替換法**：選擇一個閒置的 v83 物品（如 Long Sword 1302007），用 HaRepacker 開啟高版本物品的 img
3. **複製內容替換**：選擇高版本物品的內容，覆蓋到 v83 物品上（而非創建新物品 ID）
4. **重新導出測試**：導出後在客戶端測試是否能正常裝備

**具體步驟**：
```
1. HaRepacker 開啟 v176 Character.wz（或對應的 WZ）
2. 找到目標物品（如 Death's Scythe, ID: 1702565）
3. 選擇所有子節點，右鍵 → Export → XML
4. 在 v83 Character.wZ 中找到閒置物品（如 1302007 Long Sword）
5. Export v83 該物品為 XML
6. 用文本編輯器將 v176 內容合併到 v83 結構中
7. Import 回 v83 Character.wz
8. 測試裝備
```

#### Reskin 技術要點
> "replace a current weapon you won't use, like Long Sword(1302007) in your v83 character.wz with the Death's Scythe contents"

**核心思想**：不改變物品 ID，只改變物品的視覺和屬性內容。

## 2. 抓包分析 — 新發現

### 2.1 MaplePacketPuller + SpiritIDAPlugin 離線分析工作流（第七十六次研究新增）

**GitHub**:
- MaplePacketPuller: https://github.com/Bratah123/MaplePacketPuller
- SpiritIDAPlugin: https://github.com/Bratah123/SpiritIDAPlugin

#### 完整工作流程
```
Step 1: IDA Pro 分析
  → 用 IDA 打開 MapleStory.exe
  → 生成目標函數的 C-style pseudocode
  → 複製 pseudocode 到 .txt 文件

Step 2: SpiritIDAPlugin 格式化
  → 將 .txt 檔案放入 MaplePacketPuller/IDA Maple Script/src/main/python/Functions/
  → 執行 main.py
  → 選擇 Analysis type
  → 輸出格式化後的封包結構

Step 3: MaplePacketPuller 解析
  → 將格式化後的結構輸入 MaplePacketPuller
  → 生成對應的程式碼
```

#### SpiritIDAPlugin 已知限制
> "while() loops aren't properly handled, i.e. whether a decode is in the scope of one well. Possible circumvention: may be rectified if GET_ALL_DECODES = true"

while() 迴圈處理不完善。建議設 `GET_ALL_DECODES=true` 來提高準確度。

#### MaplePacketPuller 支援環境
| 環境 | 支援狀態 |
|------|---------|
| Python 3.8.5 | ✅ 測試通過 |
| Python 3.6.12 & 3.8.6 | ✅ 測試通過 |
| IDA Pro 7.0 (32-bit) | ✅ 測試通過 |
| IDA Pro 7.5 | ✅ 測試通過 |

| 特性 | Maple_Pshark | Wireshark |
|------|-------------|-----------|
| 抓包方式 | Hook 加密函數，直接明文 | 被動 TCP 監聽（需解密） |

| 操作難度 | 低（直接可用） | 高（需自行解密） |

#### 使用場景
1. **研究新 Opcode**: 當伺服器發送未知封包時，直接看到明文
2. **除錯 NPC 腳本**: 觀察伺服器和客戶端的實際通訊

2. 複製整個技能 img 節點

```
Skill.wz/
├── 0XXX.img  (新手技能)
├── 1XXX.img  (見習職業)
├── 2XXX.img  (一轉)
├── 3XXX.img  (二轉)
└── ...
```

### 4.2 Quest NPC 與 MigrateIn 封包機制（第七十六次研究更新）

**問題**：如何讓 Quest NPC 出現在 "Other" 選項中？

**解決方案**：
- 需要發送 `PlayerMigrateInPacket` 封包
- 封包包含：NPC ID 列表 + FileTime
- 這個封包讓客戶端知道哪些 NPC 有自訂腳本

**實現位置**（以 HeavenMS 為例）：
```java
// 在 PlayerLoggedInHandler 或 similar handler 中
int fileTime = 自定義腳本結束時間; // 毫秒時間戳
List<Integer> npcIds = Arrays.asList(9000000, 9000001, 9000002);
sendPacket(PlayerMigrateInPacket.encode(npcIds, fileTime));
```
