## 一、重點更新摘要

### 1. MapleClientEditTemplate — C++ 客戶端修改框架 (GitHub, oung/Minimum Delta)

**GitHub**: https://github.com/oung/MapleClientEditTemplate

這是一個**越來越通用化的 MapleStory 客戶端編輯框架**，用於製作客戶端重定向/修改。

**核心特色**：
- 包含所有 Windows library hooks，適用於任何版本
- **抽象化的 Hook 系統**：可在 `commonconfig.h` 中開關
- 支援 WinSock hooks（用於 localhost 重定向）
- 可直接記憶體編輯，無需等待客戶端解包（v83 已是解包狀態）

**編譯設定（重要）**：
```
1. 將 detours.lib 放入 Debug 資料夾
2. 在專案屬性加入：$(SolutionDir)Common;%(AdditionalIncludeDirectories)
3. 必須編譯為 x86（x64 會有 detour linker errors）
```

**兩種注入方式**：
| 方式 | 設定 | 說明 |
|------|------|------|
| **ijl15 proxy 自動注入** | 定義 `MAPLE_INJECT_USE_IJL` | MapleStory 啟動時自動注入，需將 DLL 改名為 `ijl15.dll` |
| **GenericLauncher 手動注入** | 普通注入 | 啟動 MapleStory 並注入 `LEN.dll` |

**架構說明**：
```
Common 專案：所有 Windows API hooks（非版本特定）
  - 可在 commonconfig 中開關
  - 在 injection 後立即執行（CreateMutexA hook 觸發）
  - 建議將所有 hooking/memory editing 放在 MainFunc 中

MapleStory function hooks：版本特定，尚未加入模板
```

### 2. 封包抓包與 IDA 分析技巧 (RaGEZONE)

**IDA 分析要點**：

所有客戶端接收的封包都經過：
```
CClientSocket::ProcessPacket
  → 根據 header 分發到各 handler
  → CField::OnPacket / CLogin::OnPacket 等
```

這就是為什麼找到 client-recv packet handlers 那麼容易 — **大多數 packet handler 的名稱都是自解釋的**。

**IDA xref 分析**：
- `COutPacket::Encode` 函數的 xref 可找到所有 client-send opcode
- 使用 IDAPython 腳本：`Show all address xref from COutPacket with value!`
- 技巧：命名 disassembled function 後，MaplePacketPuller 可自動解析

**MaplePacketPuller** (GitHub: Bratah123)：
- Python 封包結構分析工具
- 假設你在 IDA 中已經命名了要分析目標函數
- 自動提取封包結構

### 3. 封包加密移除 (v83)

**流程**：
```
明文資料
  → IOBufferManipulator.Encrypt()  (Shanda 對稱加密)
  → AES-OFB 加密                   (主加密)
  → MapleCrypto 頭部混淆            (4-byte header)
  → 網路傳輸
```

**本地移除加密方法**：
1. 用 IDA 分析客戶端
2. 找出 `IOBufferManipulator.Encrypt` 和 `AES-OFB` 加密函數
3. NOP 掉這些函數的調用
4. 伺服器端同步修改以發送未加密封包

### 5. NPC Type 值詳細解析 (RaGEZONE, 2026-03)

**來源**: https://forum.ragezone.com/threads/npc-type.1262188/

| Type 值 | 名稱 | 客戶端行為 | 腳本需求 |
|---------|------|-----------|---------|
| `0` | 普通 NPC | 點擊後發送 NPC_TALK 封包 | 需要 `.js` 腳本 |
| `1` | 商店 NPC | 點擊後直接開啟商店介面 | 可能需要 `.js` |
| `2-6` | 拍賣 NPC | 點擊後開啟拍賣功能 | 不需要腳本 |

在 HaRepacker 中查看：`NPC.wz` → `XXX.img` → `info` → `type`

**音源相容性警告** (RaGEZONE, 2026-03)：
- 不同音源（Cosmic、Kinoko、HeavenMS）的 NPC 腳本解析器版本不同
- 較舊腳本可能不相容
- **最佳實踐**：
  - ✅ 使用普遍支援的 `cm.` 函數
  - ❌ 避免使用特定音源專有的 Package 路徑
  - ❌ 避免 `importPackage`（JDK 7+ 已廢棄）

### 登入器/客戶端修改
```
方案比較：
HxD Hex 編輯        → 傳統，適合簡單 IP 修改

```

### 抓包分析
```
工具鏈：
Wireshark (tcp.port == 7575 || tcp.port == 8484)
  → Maple_Pshark (v83 專用命令行工具)
  → MaplePacketPuller (Python，IDA 命名後自動解析)
  → MaplePE (可主動發送封包)
```

### Opcode結構
```
所有 client-recv 封包入口：CClientSocket::ProcessPacket
所有 client-send 封包：COutPacket::Encode (xref IDA 分析)
三層加密：MapleCrypto → Shanda (IOBufferManipulator) → AES-OFB
第一個封包永遠是明文（Hello Packet）
```

### WZ修改
```
工具：HaRepacker-resurrected v8.0.0（熱插拔 + AI 地圖編輯）

社群資源：Community WZ repack（RaGEZONE，有地圖/物品/NPC/音樂）
```

### NPC腳本編寫
```
✅ cm. 函數（普遍支援）

❌ importPackage（廢棄）

Type 0 = 普通 NPC（需要 .js），Type 1 = 商店 NPC
```
