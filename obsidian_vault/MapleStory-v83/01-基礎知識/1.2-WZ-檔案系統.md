# 1.2 WZ 檔案系統

---

## 📦 WZ 是什麼？

WZ 是 MapleStory 使用的**遊戲資源檔案格式**，類似於 XML 但經過加密和壓縮。

```
MapleStory.exe ←→ WZ 檔案 ←→ 遊戲資源（地圖、怪物、道具、腳本等）
```

---

## 📁 常見 WZ 檔案

| 檔案 | 內容 |
|------|------|
| `Map.wz` | 地圖資料 |
| `Mob.wz` | 怪物資料 |
| `Npc.wz` | NPC 資料 |
| `Item.wz` | 道具資料 |
| `Skill.wz` | 技能資料 |
| `String.wz` | 字串顯示 |
| `UI.wz` | 介面資源 |
| `Sound.wz` | 音效 |
| `Quest.wz` | 任務資料 |
| `Character.wz` | 角色外觀/造型 |
| `Cash.wz` | 現金道具 |

## 🔧 WZ 編輯工具

### 1. HaRepacker-resurrected（推薦）

GitHub: https://github.com/lastbattle/Harepacker-resurrected

| 功能 | 說明 |
|------|------|
| WZ 解包/打包 | 支援幾乎所有 MapleStory 版本 |
| 加密格式 | 支援 GMS (old)、GMS (new)、KMST、CN 等 |
| XML 導出 | 可將 WZ 內容導出為 XML 修改後重新導入 |
| IMG 編輯 | 支援直接編輯 .img 檔案內容 |

**加密格式選擇（v83）：**
- 使用 `GMS (old)` 加密（大多數 v83 私人伺服器客戶端）
- 如果是較新的客戶端，嘗試 `GMS (new)`

### 2. WzComparerR2-NTMS

GitHub: https://github.com/HCTOrganization/WzComparerR2-NTMS

用途：WZ 檔案比較、差異分析，適合升級/移植內容時使用。

### 3. NX 格式

NX 是 WZ 的高效替代格式：
- **效能更好**：載入速度比 WZ 快
- **格式新穎**：部分新版 MapleStory 使用
- **工具支援**：HaRepacker 也支援 NX 格式

> 如果客戶端同時有 `.wz` 和 `.nx` 檔案，建議使用 NX。

## 📝 WZ 修改實戰

### 實戰 1：移除任務等級限制（lvmax）

**目標**：讓低等級玩家也能接取高等級任務

**工具**：HaRepacker-resurrected + Notepad++

**步驟**：
```
1. 用 HaRepacker-resurrected 打開 Quest.wz
2. 導航到：Quest/Check.img.xml
3. Tools → Export selection → XML → Classic（用 Classic，New 會用新版加密）
4. 用 Notepad++ 開啟導出的 XML
5. 搜尋所有 lvmax 屬性並移除它們
   例如：<quest lvmax="30"> → <quest>
6. 存檔
7. 用 HaRepacker-resurrected 重新導入（Tools → Import XML）
8. 將修改後的 Quest.wz 放回客戶端
```

**來源**：RaGEZONE (2025-07) - https://forum.ragezone.com/threads/maplestory-v83-method-to-disable-quest-level-restrictions.1250107/

### 實戰 2：新版道具導入 v83（物品無法裝備問題）

**問題**：從新版（v100+）導入道具到 v83，spawn 出來後無法裝備。

**原因分析**：
1. `.img` 檔案結構不兼容（新版本可能有多餘欄位）
2. 缺少對應的 Sound.wz / Quest.wz 依賴
3. Character.wz 中缺少造型定義

**解決方案**：
```
1. 確保導入完整的 .img 文件及所有依賴
2. 檢查 Item.wz 中該道具的 info 節點：
   - 需要有 correctSkill 欄位（某些裝備需要對應技能）
   - 需要有 Tüü（升級等級上限）欄位
3. 在 Character.wz 中確認對應的目錄存在
4. 重新打包後測試
```

**來源**：RaGEZONE (2025-11) - https://forum.ragezone.com/threads/maplestory-v83-newer-items-import.1256083/

### 實戰 3：版本號修改（將 v98 WZ 用於 v83 伺服器）

```
1. 用 HaRepacker 打開 Map.wz (v98)
2. 修改版本號（通常是檔案內的版本數值）
3. 另存新檔為 Map.wz
4. 對其他 WZ 檔案重複同樣步驟
```

**可轉換的檔案**：Map.wz、Mob.wz、Item.wz、String.wz
**不建議轉換**：Skill.wz（有結構變化）、UI.wz（部分模組不相容）

## 🗂️ WZ 資料夾結構

MapleStory 客戶端的 WZ 資料夾結構：

```
MapleStory/
├── WZ/
│   ├── Map.wz          # 地圖
│   ├── Mob.wz          # 怪物
│   ├── Npc.wz          # NPC
│   ├── Item.wz         # 道具
│   ├── Skill.wz        # 技能
│   ├── String.wz       # 字串（ID→名稱對照）
│   ├── UI.wz           # 介面
│   ├── Sound.wz        # 音效
│   ├── Quest.wz        # 任務
│   ├── Character.wz    # 角色外觀
│   ├── Etc.wz          # 雜項
│   └── Base.wz         # 基礎資源
├── Patch/              # 可選：自定義補丁（優先於 WZ）
│   ├── Character.wz    # 優先於 WZ/Character.wz
│   └── Item.wz
└── ...
```

### Patch 資料夾（重要！）

**工作原理**：遊戲優先讀取 Patch 資料夾的內容來覆蓋原始 WZ。

```
優勢：
- 不需要修改原始 WZ 檔案
- 方便管理多個自定義補丁
- 出錯時只需刪除 Patch 內容即可還原
```

**用途**：自定義新道具、新地圖、新功能時，可直接放入 Patch 資料夾。

## 🆕 WZ 編輯完整工作流程 (Cosmic README, 2026-03 更新)

**來源**: https://github.com/P0nk/Cosmic

### 核心原則
> **永遠從 client WZ 匯出到 server XML，不要反向操作！**

### 正確流程
```
1. HaRepacker 開啟 client WZ（encryption: GMS (old)）
2. 使用 UI 直接修改數值（❗不要刪除/重新導入節點！）
3. 儲存 client WZ
4. HaRepacker → "Private Server..." 導出
5. 覆蓋伺服器的 WZ XML 目錄
```

### 為什麼不能反向操作？
- client WZ 包含 sprite 資源（圖片、音效等）
- server XML 只有邏輯數據（文字、數值）
- 從 server XML 匯入 client WZ 會造成 sprite 丢失

### v83 WZ 加密格式選擇
| 格式 | 用途 | 風險 |
|------|------|------|
| **GMS (old)** | v83 私人伺服器 | ✅ 正確 |
| GMS (new) | 現代 MapleStory 客户端 | ❌ 不相容 |
| KMS | 韓版客户端 | ❌ 不相容 |

### WZ to NX 轉換（效能優化）
- NX 格式比 WZ/XML 載入更快、檔案更小
- 工具：WzToNx.exe（RaGEZONE forum）
- 使用方式：將 .wz 檔案拖曳到工具上，自動轉換
- 轉換後 Lua 工具可直接讀取 .nx 檔案

### Cosmi 長期目標：擺脫 Custom WZ
Cosmic 專案正在逐步移除對 custom WZ 的依賴，最終目標是：
- 只使用 vanilla WZ（官方原始檔案）
- 不再需要修改客戶端 WZ
- 從客戶端和伺服器兩端同時實現 vanilla 支援

## WZ 加密機制

### 加密原理

WZ 檔案使用 XOR 加密，根據字元類型（ASCII/Unicode）應用不同的偏移公式。

#### ASCII 字元加密
```
encrypted_char = char ^ (0xAA + offset) ^ key[offset]
```

#### Unicode 字元加密
```
encrypted_char = char ^ (0xAAAA + offset) ^ ((key[offset * 2 + 1]) << 8 + key[offset * 2])
```

**參數說明**：
| 參數 | 說明 |
|------|------|
| `char` | 原始字元 |
| `offset` | 在資料流中的位置（從 0 開始）|
| `key` | 版本決定的加密金鑰 |

**來源**：mapleref.fandom.com

## 🆕 mob.wz 怪物屬性速查 (RaGEZONE 新增)

**URL**: https://forum.ragezone.com/threads/community-wz-img-repack-for-v83.1221835/

### 怪物屬性欄位
| 屬性 | 說明 | 範例值 |
|------|------|--------|
| `type` | M = Monster, N = NPC | M |
| `x`, `y` | 出生座標 | 100, 200 |
| `rx0`, `rx1` | 巡邏範圍（左/右邊界） | -100, 100 |
| `speed` | 移動速度 | 150 |
| `acc` | 命中率 | 100 |
| `MADamage` | 魔法攻擊傷害 | 50 |
| `PADamage` | 物理攻擊傷害 | 50 |
| `jump` | 跳躍力 | 100 |
| `bodyAttack` | 是否可被身體攻擊碰撞 | 0/1 |

### 怪物無敵修改方法
| 方法 | 原理 |
|------|------|
| `bodyAttack = 0` | 禁止身體碰撞攻擊 |
| `removeAfter = 999999` | 怪物不死（永不解召） |
| `HP = MAX` | 將 HP 設為最大值 |

## 🆕 WZ 編輯黃金法則 (MapleSolaxiaV2 README, 2026-03-28 新增)

> **DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs.**

### 正確 WZ 編輯流程

```
1. HaRepacker 4.2.4 開啟 WZ（使用 GMS (old) 加密）
2. 使用 UI 直接修改數值（禁止刪除/重新導入節點！）
3. 儲存 WZ
4. 在 HaRepacker 中選擇 "Private Server..." 導出
5. 將導出的 XML 覆蓋到伺服器的 WZ.XML 目錄
```

### HaRepacker GMS (old) 加密支援版本

| 加密類型 | 支援版本 |
|---------|---------|
| GMS (old) | v83 ~ v120 左右的版本 |
| GMS (new) | 較新版本 |
| EMS / KMS | 歐版 / 韓版 |

