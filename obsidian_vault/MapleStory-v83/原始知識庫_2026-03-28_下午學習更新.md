## 🆕 MapleStory v83 完整架設指南 (RaGEZONE, Apr 2025)

**來源**: https://forum.ragezone.com/threads/video-guide-how-to-make-a-maplestory-v83-private-server-–-full-setup-tutorial.1243566/

### 完整架設環境需求

| 軟體 | 版本 | 必要性 |
|------|------|--------|
| JDK | 8 | 必要 |
| JCE | 8 | 必要（Java Cryptography Extension） |
| MySQL | 5.5 | 必要 |
| v83 localhost 客戶端 | - | 必要 |
| HxD | - | 修改 IP 用 |

### 架設流程摘要

```
1. 安裝 JDK 8 + JCE 8
2. 安裝 MySQL Server 5.5
3. 下載 v83 localhost 客戶端（Kimmy's Archive）
4. Clone Cosmic 或 HeavenMS 源碼
5. Import SQL 資料庫（db_database.sql + db_drops.sql）
6. 修改 ServerConstants.java 的 IP 和埠口
7. 修改 MapleAESOFB 中的版本 key（如需要）
8. HxD 修改客戶端 IP（三個 127.0.0.1 全部替換）
9. 啟動客戶端和伺服器
```

### LAN 多人連線必檢查清單

- [ ] 確認兩台 PC 在同一 LAN 網段
- [ ] 確認伺服器 listen `0.0.0.0`（而非 `127.0.0.1`）
- [ ] 確認防火牆開啟 7575 和 8484 埠口
- [ ] 若錯誤碼 `0x80004005` → 防火牆/權限問題
- [ ] 若錯誤碼 `-2147467259` → 同上

### 架設後抓包應用

- 確認 Hello Packet 能正常交換（驗證版本匹配）
- 分析 AES 握手是否成功
- 排查連線問題

## 🆕 v83 商場封包 Dupe 分析 (PDFCOFFEE)

**來源**: https://pdfcoffee.com/v83-dupe-pdf-free.html

### Opcode `0x6F` (SHOP_BUY) 完整結構

```
6F 00 20 00 FF FF FF 80 00
```

| 位元組 | 意義 |
|--------|------|
| `6F 00` | Opcode (little endian = 0x006F) |
| `20 00` | 商店ID (0x0020 = 32) |
| `FF FF FF 80` | 物品索引 + 最大 Mesos 標誌 |
| `00` | 結束標記 |

> ⚠️ 僅供學習研究用途，請勿用於作弊

## 🆕 MapleRef Wiki — 封包結構參考 (v83)

**URL**: https://mapleref.fandom.com/wiki/Packets

### Opcode 核心原則

> "Actions that are defined by an identifier called an Operation Code (opcode for short), which is the first 2 bytes of the packet (a short)"

### Send vs Receive Opcode 區分

| 方向 | 說明 |
|------|------|
| **S→C (Send)** | 伺服器發送，客戶端接收 |
| **C→S (Recv)** | 客戶端發送，伺服器接收 |

**重要**：同一個 Opcode 數字在不同方向代表**完全不同**的含義！

### 使用方式

```
1. 前往 https://mapleref.fandom.com/wiki/Packets
2. 選擇目標版本 (v83)
3. 查看 Send (S→C) 或 Recv (C→S) 封包結構
4. 配合 Wireshark 抓包分析
```

## 🆕 登入器/客戶端修改學習重點總結

### 1. 登入器原理

```
原始流程：
  MapleStory.exe → connect("203.78.xx.xx:7575") → Nexon 伺服器

使用登入器：
  MapleStory.exe → Hook connect/WSAConnect
                  → 攔截並修改目標 IP/Port
                  → connect("127.0.0.1:7575") → 私人伺服器
```

### 2. IP 修改方案比較

| 方案 | 難度 | 穩定性 | HD 支援 | LAN 支援 | 推薦度 |
|------|------|--------|---------|---------|--------|
| HxD Hex 編輯 | ⭐ | ⭐⭐⭐⭐ | 需額外 | ✅ 直接 | ⭐⭐⭐ |
| Ezorsia v2 DLL | ⭐⭐ | ⭐⭐⭐⭐⭐ | 內建 | ⚠️ 需設定 | ⭐⭐⭐⭐⭐ |
| 第三方 localhost | ⭐ | ⭐⭐⭐ | 各版本不同 | ✅ 直接 | ⭐⭐⭐ |

### 3. 抓包分析重點

**第一個封包永遠是明文** — 這是分析的起點

```
Hello Packet 結構：
- 版本號 (int)
- SendIV (byte[4])
- RecvIV (byte[4])
```

### 4. Opcode 結構

- Opcode = 2 bytes (v83)
- 方向不同，Opcode 集合不同（S→C vs C→S）
- MapleRef Wiki 有完整列表

### 5. WZ 修改要點

- 使用 HaRepacker 4.2.4 編輯 GMS (old) 加密
- **禁止刪除/重新導入節點**
- 正確流程：HaRepacker UI 修改 → Private Server 導出 → XML 覆蓋伺服器

### 6. NPC 腳本編寫

- 語言：JavaScript (ECMAScript)，不是 Java
- 引擎：Nashorn (JDK 8+) — **Cosmic 最活躍**
- **避免 `importPackage`**（JDK 7+ 已廢棄）
- 使用 `cm.` 函數，最廣泛相容
