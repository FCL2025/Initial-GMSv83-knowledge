
3. .IMG 檔放 Data 資料夾（僅 IMG 用戶）
4. `iniReader` 無法讀取 → 避免延伸注釋（`;` 後內容）

## 二、伺服器開發 — Cosmic 最新動態（GitHub 2026-03 完整抓取）

**GitHub**: https://github.com/P0nk/Cosmic

### 專案現況（2026-03 確認）
- **仍然是最活躍的 v83 開源伺服器**
- 基於 HeavenMS（2019），可追溯至 OdinMS（2008）
- Java 21（建議 Amazon Corretto）
- 嚴格遵循 semantic versioning（MAJOR.MINOR.PATCH）
- GitHub Actions 全自動化 CI/CD
- PR merge → 自動創建 git tag

### Semantic Versioning 規則
| 變更類型 | 版本範例 | 說明 |
|---------|---------|------|
| Bug fix | 1.2.3 → 1.2.4 | PATCH |
| 一般改進 | 1.2.3 → 1.3.0 | MINOR |
| 重大變更 | 1.2.3 → 2.0.0 | MAJOR |

### Docker 部署（新增資訊）
```bash
# 最簡單的啟動方式
docker compose up

# 需重建映像
docker compose up --build
```
- 需提供 `wz-path` 系統屬性指向 WZ 目錄
- `launch.bat` 自動提供此參數

### WZ 處理（重要更新）
> ⚠️ Cosmic 目前仍需要自訂 WZ 檔案，但**打算最終移除這個需求**，回歸原生 WZ。

**編輯流程**：
```
1. HaRepacker（加密：GMS old）開啟客戶端 WZ
2. 編輯（僅使用 copy/paste，盡量避免刪除重導）
3. 儲存 WZ（覆蓋客戶端資料夾）
4. 使用 "Private Server" 導出選項
5. 將導出 XML 覆蓋到伺服器 wz 目錄
```

**⚠️ 黃金法則（Cosmic 官方強調）**：
> Editing the client .wz without exporting to the server may lead to strange behavior.

> The server does not make use of the sprites — this is the motivation for different kinds of exporting.

- 客戶端單獨存放：https://github.com/P0nk/Cosmic-client

## 三、WZ 修改 — HaRepacker-resurrected 新動態

### WzImg MCP Server（2024 新功能）

**dotnet publish 指令**：
```bash
git clone https://github.com/lastbattle/Harepacker-resurrected.git
git submodule update --init --recursive

dotnet publish .\WzImg-MCP-Server\WzImgMCP\WzImgMCP.csproj -c Release -r win-x64 --self-contained false
```

**這是什麼？**
- MCP = Model Context Protocol（AI 代理上下文協議）
- 可讓 AI agent（如 Claude）直接操作 WZ 文件
- 實現 AI 驅動的 WZ 自動修改
- 適用於自動化腳本編寫、批量修改等場景

## 四、NPC腳本編寫 — Kinoko 音源相容性動態

### RaGEZONE 熱門討論（2026-03）

**主題**: Kinoko NPC Scripts  
**URL**: https://forum.ragezone.com/threads/kinoko-npc-scripts.1261538/

**核心問題**：如何建立缺失的 NPC 腳本？

**社區回答要點**：
1. **腳本參考來源**：
   - 官方腳本 dump
   - 其他模擬器的腳本（如 Cosmic、HeavenMS）
   - 從零開始按照 Kinoko 腳本模式編寫

2. **常見問題**：Job Advancement（職業進階）腳本缺失導致無法轉職

3. **解決方案**：
   - 補全對應的職業進階 NPC 腳本
   - 參考其他伺服器的等效腳本

### ES6 Nashorn 寫法確認（IntransigentMS）

**官方示例**：
```javascript
// ✅ 正確：使用 Java.type
const MapleCharacter = Java.type("net.sf.odinms.client.MapleCharacter");
const SkillFactory = Java.type("net.sf.odinms.server.maps.SkillFactory");

function start() {
    print("NPC 啟動");
    cm.sendNext("歡迎來到我的商店！");
}

// ✅ 模板字串（ES6）
const name = cm.getPlayer().getName();
cm.sendNext(`歡迎, ${name}!`);
```

**❌ 錯誤：使用 importPackage**
```javascript
// ❌ 已廢棄（JDK 7+ 不支援）
importPackage(Packages.net.sf.odinms.client);
```

### NPC Type 值對照表（持續更新）

| Type 值 | 名稱 | 客戶端行為 | 腳本需求 |
|---------|------|-----------|---------|
| `0` | 普通 NPC | 點擊後發送 NPC_TALK 封包 | 需要 `.js` 腳本 |
| `1` | 商店 NPC | 點擊後直接開啟商店介面 | 可能需要 `.js` |
| `2-6` | 拍賣/功能 NPC | 點擊後開啟特定功能 | 不需要腳本 |

**GitHub**: https://github.com/obstriker/Maple_Pshark

**5 大核心功能**：
```
1. Intercepting Recv messages（攔截客戶端接收封包）
2. Modify messages（修改封包內容）
3. Injecting packets for send or recv（注入自訂封包）
```
