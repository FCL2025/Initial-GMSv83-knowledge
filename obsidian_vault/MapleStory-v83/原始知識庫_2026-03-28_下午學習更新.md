### 1) 登入器/客戶端修改 (Login Client Modification)

**localhost.exe 修改流程：**

1. 安裝 MapleStory 客戶端 (通過 ManagerMsv83.exe)
2. 刪除以下文件：HShield資料夾、ASPLnchr.exe、MapleStory.exe、Patcher.exe
3. 用修改過的 localhost.exe 替換原執行檔
4. 如果伺服器 IP 不是 127.0.0.1，需要用 Hex 編輯器修改 localhost.exe 中的 IP 位址

**Hex 編輯 IP 步驟 (使用 Neo Hex Editor)：**
- 搜尋 "Type: String" "127.0.0.1" 
- 找到後替換為目標 IP
- 通常需要修改 3 處 IP 位址

- 需要鉤入加密函數才能看到加密後的內容

### 3) Opcode 結構 (Opcode Structure)

**封包結構：**
```
[Header (4 bytes)][Opcode (2 bytes)][Data...]
```

**Header 組成：**
- 4 bytes 長
- 使用 Send IV、封包長度（不含header）、MapleStory 版本生成
- 發送到客戶端時版本 = -(version + 1)

**Opcode 歷史：**
- 早期版本：Opcode 是 1 byte (0-255)
- 後來改為 short (2 bytes)，範圍 0-65535
- v83 使用 2 bytes Opcode

**加密機制 (v83)：**
1. **Shanda 加密**：先進行 Shanda 解密/加密
2. **AES OFB 加密**：之後再進行 AES 加密

**封包方向：**
- Client → Server：使用 Send IV
- Server → Client：使用 Receive IV

**重要參考資源：**
- MapleStory Reference Wiki (Fandom): https://mapleref.fandom.com/wiki/Packets
- 包含各版本 opcode 列表

### 4) WZ 修改 (WZ Modification)

**主要 WZ 文件功能：**

| WZ 文件 | 用途 |
|---------|------|
| Base.wz | 武器、技能、音效等資訊 |
| Character.wz | 角色相關精靈和裝備 |
| Effect.wz | 傷害數字、墓碑、升級等特效 |
| Etc.wz | 任務、Buff、詛咒、角色創建等資訊 |
| Item.wz | 所有物品/裝備的數據和圖片 |
| Map.wz | 地圖視覺 |
| Mob.wz | 怪物數據（速度、技能、HP、EXP 等） |
| Npc.wz | NPC 數據 |
| Quest.wz | 任務數據 |
| Reactor.wz | 可破壞物件（如箱子、入口） |
| Skill.wz | 所有職業和怪物的技能 |
| Sound.wz | 音效和歌曲 |
| String.wz | 服務端資訊（無法編輯） |
| UI.wz | 遊戲圖標 |

**常用 WZ 編輯工具：**
- **HaRepacker 3.5**：支援 BMS 格式，通用 WZ 編輯
- **deMSwZ**：支援批量修改
- **GM Studio**：專業 WZ 編輯

**Mob.wz 修改示例（GodMode 等）：**
- acc = 0 → Miss GodMode
- acc = 1, MADamage = 1, PADamage = 1 → Jesus GodMode
- 攻擊間隔修改：attackAfter = 65535, effectAfter = 65535

**Map.wz 重要概念：**
- Foothold：怪物的站立點
- Spawn Point：角色出生點
- Portal：傳送門設置

**WZ 編輯注意事項：**
- 備份原始文件
- 客戶端和伺服器端 WZ 必須匹配
- 某些修改可能導致客戶端崩潰

### 5) NPC 腳本編寫 (NPC Script Writing)

**腳本檔案位置：**
- 腳本通常位於 `scripts > npc` 目錄
- 檔案名稱對應 NPC ID

**基本腳本結構：**
```javascript
function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();  // 關閉對話
    } else {
        if (mode == 0 && status == 0) {
            cm.dispose();  // 選擇"否"時關閉
        }
        if (mode == 1) {
            status++;
        } else {
            status--;
        }
        
        // 狀態機制處理對話流程
        if (status == 0) {
            cm.sendYesNo("你想要一把免費的劍嗎？");
        } else if (status == 1) {
            cm.gainItem(1302000, 1);
            cm.sendOk("拿去吧！");
        }
    }
}
```

**常用 cm 方法：**
| 方法 | 用途 |
|------|------|
| cm.sendNext(text) | 顯示下一頁對話 |
| cm.sendYesNo(text) | 顯示是和否選項 |
| cm.sendSimple(text) | 顯示選項列表 (#L0#...#l) |
| cm.sendOk(text) | 顯示確定按鈕 |
| cm.dispose() | 關閉對話 |
| cm.gainItem(id, amount) | 給予物品 |
| cm.getPlayer() | 獲取玩家物件 |
| cm.getParty() | 獲取隊伍物件 |
| cm.warp(mapId) | 傳送玩家到地圖 |
| cm.startQuest(questId) | 開始任務 |
| cm.completeQuest(questId) | 完成任務 |

**選項格式（sendSimple）：**
```
#b#L0#第一個選項#l
#L1#第二個選項#l
#L2#第三個選項#l
```
- #b = 藍色文字
- #k = 黑色文字
- #vXXXX# = 物品圖示（XXXX為物品ID）
- \r\n = 換行

**狀態機模式：**
- status = -1：對話結束
- status = 0, 1, 2...：逐步遞進
- mode = 1：下一頁/確認
- mode = 0：返回/否
- mode = -1：關閉對話框

**動態商店腳本：**
- 可用純腳本實現商店功能，無需重啟伺服器
- 從 JSON 或其他資料來源讀取商品列表

**職業進階 NPC 示例：**
```javascript
function action(mode, type, selection) {
    if (mode == 1) status++;
    else status--;
    
    if (status == 0) {
        cm.sendSimple("選擇你的職業：\r\n#b#L0#烈火使者#l\r\n#L1#寒冰使者#l\r\n#L2#聖魂使者#l");
    } else if (status == 1) {
        if (selection == 0) {
            cm.sendNext("烈火使者擅長火系法術...");
        } else if (selection == 1) {
            cm.sendNext("寒冰使者擅長冰系法術...");
        } else if (selection == 2) {
            cm.completeQuest(100000);  // 假設的任務ID
            cm.sendOk("恭喜你成為聖魂使者！");
        }
        cm.dispose();
    }
}
```
