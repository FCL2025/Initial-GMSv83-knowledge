## 一、客戶端修改 / 登入器修改

### 1.1 核心概念

MapleStory v83 客戶端修改涉及對 `MapleStory.exe` 和相關 DLL 的十六進制(Hex)編輯。常見修改目標：

- **去除伺服器IP驗證** — 將客戶端連接的目標IP/PORT替換為本地或私人伺服器
- **去除版本檢查** — 繞過版本號比對讓舊客戶端連接新伺服器
- **去除加密校驗** — 移除或修改遊戲安全檢測

### 1.2 工具

| 工具 | 用途 |
|------|------|
| OllyDbg / x64dbg | 動態調試器，用於找到關鍵記憶體位址 |
| Cheat Engine | 記憶體掃描、修改 |
| IDA Pro | 反編譯神器，用於靜態分析 opcode 和函數結構 |
| Wireshark / WPE Pro | 網路封包攔截 |
| CFF Explorer | PE 編輯工具，修改匯入表等 |
| Themida | 加殼工具 (脫殼見 2.4 章節) |

### 1.3 登入器修改流程

1. **定位連接字串**
   - 在客戶端使用 IDA 字串搜尋，查找 `"localhost"`、`"127.0.0.1"`、或伺服器域名
   - 或使用 OllyDbg 附加進程，在網路連接函數下斷點

2. **修改 IP / Port**
   - 找到位址後使用 Hex 編輯器（如 FlexHex）修改
   - 確保位元組長度不變（IP：`0xC0A80101` = 127.0.0.1）

3. **去除版本校驗**
   - 在 IDA 中搜索 `version`、`patch`、`check` 相關字串
   - 找到版本號比對函數，NOP掉跳轉或直接修改版本值

4. **去除 CRC/完整性檢查**
   - 使用 Themida 等殼需先脫殼
   - 修改 `.text` 段 CRC 校驗跳轉

### 1.4 參考資源

- **MPGH Tutorial**: https://www.mpgh.net/forum/showthread.php?t=1546661 — How to find offsets
- **maplesecrets hex editing guide**: http://maplesecrets.blogspot.com/2013/04/how-to-hex-edit-clients-easily.html

## 二、抓包分析 (Packet Capture)

### 2.1 MapleStory 封包基礎

- **傳輸層**: TCP Socket
- **首包特徵**: 第一個發送/接收的封包**永遠是明文**（未加密）
- **加密方式**: 
  - 第一層: Shanda (MapleSeed) 對稱加密
  - 第二層: AES-128-CBC (v97+)
- **Opcode 位置**: 在 v83 中，opcode 是封包的**第一個 2 bytes**（short，little-endian）
- **封包格式**: Little Endian

### 2.2 抓包工具

| 工具 | 用途 |
|------|------|
| **MaplePshark** (GitHub: obstriker/Maple_Pshark) | 專門的 Maplestory v83 封包記錄工具 |
| **MaplePacketPuller** (GitHub: Bratah123/MaplePacketPuller) | Python 封包結構分析工具 |
| **Wireshark** | 通用 TCP 流量監控 |
| **WPE Pro** | 遊戲封包攔截與修改 |

### 2.4 MaplePacketPuller 使用 (IDA 配合)

這是一個 Python 腳本，用於分析 IDA 反編譯输出的 C-伪代码，提取封包結構：

**輸入**: IDA 生成的 `.txt` 伪代码文件
**輸出**: 格式化後的封包結構文本

使用方法：
1. 在 IDA 中打開 MapleStory client
2. 找到封包處理函數（如 `SendPacket`, `ProcessPacket`）
3. 生成該函數的 Pseudocode (F5)
4. 複製到 `MaplePacketPuller/IDA Maple Script/Functions/` 目錄下的 `.txt` 文件
5. 執行 `main.py`，選擇分析模式
6. 結果輸出到 `FuncOutput/`

**功能特點**:
- 自動識別 InHeader opcodes
- 格式化封包結構成友好格式
- 適用於 IDA Pro 7.0+ (32-bit)

### 2.5 封包結構分析方法

1. **靜態分析 (IDA)**:
   - 搜索 `SendPacket`, `RecvPacket` 函數
   - 分析封包構建邏輯
   - 找到每個 opcode 對應的結構

2. **動態分析 (調試器 + 抓包)**:
   - 在關鍵函數下斷點
   - 搭配 Wireshark 對照觀察

3. **特徵識別**:
   - 登入階段 opcode 通常是 `0x01`, `0x02` 等低位值
   - 遊戲階段 opcode 是動態的，需对照 MapleStory 原始碼

## 三、Opcode 結構

### 3.1 Opcode 基礎

> Opcode (Operation Code) 是封包的第一個 2 bytes (v83)，用於識別封包類型。

**重要Opcode (v83)**:

| Opcode (Hex) | 名稱 | 說明 |
|-------------|------|------|
| 0x01 | `PONG` | 客戶端回應伺服器心跳 |
| 0x0E | `LOGIN_REQUEST` | 登入請求 |
| 0x10 | `CHARLIST_REQUEST` | 角色列表請求 |
| 0x11 | `CHAR_SELECT` | 選擇角色 |
| 0x14 | `ENTER_PORTAL` | 進入傳送門 |
| 0x15 | `MOVE_PLAYER` | 玩家移動 |
| 0x25 | `CLOSE_RANGE_ATTACK` | 近距離攻擊 |
| 0x2F | `RANGED_ATTACK` | 遠程攻擊 |
| 0x37 | `MAGIC_ATTACK` | 魔法攻擊 |
| ... | ... | ... |

### 3.2 IDA 中找 Opcode

**RaGEZONE 教程**: https://forum.ragezone.com/threads/getting-packet-structures-opcodes-using-ida.792436/

方法：
1. IDA 載入客戶端
2. 找 `CLogin::OnPacket` 或類似函數
3. 在 Hex View 中搜索 `0F 84 ?? ?? ?? ?? 8B` 模式
4. 交叉引用找到 switch-case 表

**更現代的方法**: 使用 MaplePacketPuller + SpiritIDAPlugin

### 3.3 Opcode 映射表

私人伺服器需要維護自己的 Opcode 映射表。常見做法：
- 在伺服器原始碼中定義枚舉
- 客戶端和伺服器的 Opcode 必須完全一致

## 五、NPC 腳本編寫

### 5.1 腳本引擎

v83 私人伺服器使用 JavaScript 引擎（如 Rhino/Nashorn）執行 NPC 腳本。

### 5.2 核心對象: `cm` (CashShop/MapleMessenger)

```javascript
// 發送文字對話框
cm.sendNext("內容");           // 下一個按鈕
cm.sendPrev("內容");           // 上一步按鈕
cm.sendOk("內容");             // 確定按鈕
cm.sendYesNo("內容");          // 是/否選項
cm.sendSimple("選項\n #b#L0#選項0#l\n #L1#選項1#l");
cm.sendStyle("內容", styleId); // 風格選擇
cm.sendGetNumber("內容", default, min, max);
cm.sendGetText("內容");        // 文字輸入
```

### 5.3 狀態機模式 (傳統寫法 — Cosmic, HeavenMS)

```javascript
var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1 || mode == 0) {
        cm.dispose();
        return;
    }
    if (mode == 1) status++;
    else status--;
    
    switch(status) {
        case 0:
            cm.sendNext("歡迎來到 shop!");
            break;
        case 1:
            cm.sendSimple("選擇:\n #b#L0#購買藥水#l\n #L1#購買裝備#l");
            break;
        case 2:
            if (selection == 0) {
                // 處理購買
                cm.gainItem(2000002, 100);
                cm.sendOk("已給予100個藥水!");
            }
            cm.dispose();
            break;
    }
}
```

### 5.4 新型 Kinoko 寫法 (無需 status)

```javascript
// Kinoko 引擎 — 不需要 status 管理
function start() {
```

### 5.5 常用腳本函數

```javascript
// 玩家狀態
cm.getPlayer()              // 獲取玩家對象
cm.getPlayerId()            // 玩家ID
cm.getPlayer().getLevel()   // 等級
cm.getJob()                 // 職業
cm.getFame()                // 人氣

// 物品操作
cm.gainItem(itemId, quantity)    // 給予物品
cm.hasItem(itemId, quantity)    // 檢查物品
cm.removeItem(itemId, quantity)  // 移除物品

// 金幣/經驗
cm.gainMeso(amount)          // 給予金幣
cm.gainExp(amount)           // 給予經驗
cm.getMeso()                 // 當前金幣

// 任務
cm.completeQuest(questId)    // 完成任務
cm.startQuest(questId)        // 開始任務
cm.getQuestStatus(questId)   // 獲取任務狀態

// 地圖/傳送
cm.warp(mapId)               // 傳送到地圖
cm.warpParty(mapId)          // 整隊傳送

// NPC 管理
cm.dispose()                 // 結束對話
cm.getNpc()                  // 獲取NPC ID

// 檢查/判斷
cm.canHold(itemId)           // 背包是否有空位
cm.haveEquipped(id)          // 是否已裝備
```

### 5.6 腳本檔案位置

- 腳本檔案放在伺服器原始碼的 `scripts/npc/` 目錄
- 檔案名稱 = NPC ID（如 `9000001.js`）
- 伺服器啟動時自動加載

### 5.7 範例：商店NPC

```javascript
var status = 0;
var items = [2000002, 2000003, 2000004]; // 藥水ID列表

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1 || mode == 0) {
        cm.dispose();
        return;
    }
    if (mode == 1) status++;
    
    switch(status) {
        case 0:
            cm.sendNext("歡迎來到 #bHealth Shop#k! 我有各種恢復藥水。");
            break;
        case 1:
            var selStr = "選擇你要購買的物品:\n";
            for (var i = 0; i < items.length; i++) {
                selStr += "#b#L" + i + "#" + items[i] + " - 1000  Meso#l\n";
            }
            cm.sendSimple(selStr);
            break;
        case 2:
            var price = 1000;
            if (cm.getMeso() >= price) {
                if (cm.canHold(items[selection])) {
                    cm.gainMeso(-price);
                    cm.gainItem(items[selection], 1);
                    cm.sendOk("購買成功！");
                } else {
                    cm.sendOk("背包空間不足！");
                }
            } else {
                cm.sendOk("金幣不足！");
            }
            cm.dispose();
            break;
    }
}
```
