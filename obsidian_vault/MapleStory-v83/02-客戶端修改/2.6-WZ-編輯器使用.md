# 2.6 WZ 編輯器使用教學

## HaRepacker 介紹
HaRepacker 是一款專門用於編輯 MapleStory WZ 文件的工具。

## 版本選擇
- **HaRepacker 4.2.4**: 經典版本，穩定
- **HaRepacker-resurrected** (推薦): 現代版本，持續更新

## 下載地址
GitHub: https://github.com/lastbattle/Harepacker-resurrected

## 支援的加密格式
v83 使用 **GMS (old)** 加密格式

## WZ 編輯流程

### 步驟 1：開啟 WZ 檔案
1. 執行 HaRepacker
2. 選擇 File → Open
3. 瀏覽到 WZ 檔案位置
4. 選擇加密類型：**GMS (old)**
5. 點擊 OK

### 步驟 2：進行修改
⚠️ **重要**：使用 UI 直接編輯，不要刪除/重新匯入節點！

可修改內容：
- 數值屬性 (如物品價格、怪物血量)
- 字串內容 (如 NPC 名稱、物品描述)

## ⚠️ Server XML vs Client WZ 重要警告

> **DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate some kind of bugs afterwards.**

**原因**：
- 客戶端 WZ 有特殊的加密格式
- 伺服器 WZ.xml 經過重新導出
- 混用會導致資料同步問題

**正確流程**：
```
1. 用 HaRepacker 打開 WZ，encryption: GMS (old)
2. 使用 UI 直接修改客戶端 WZ
3. 儲存客戶端 WZ
4. Export XML → 伺服器端 WZ.xml（最後才做）
```

## 🆕 WZ 導出為 XML 步驟 (RaGEZONE 2023)

```
1. 用 HaRepacker 打開 WZ 檔案
2. 選擇要導出的節點
3. 點擊 Export → 選擇 XML (Private Server) 格式
4. 導出後可使用文字編輯器修改
5. 重新導入 WZ
```

**注意**：

- 不要直接刪除/重新導入節點，可能造成數據丟失
- 建議使用 HaRepacker UI 直接編輯數值
- 圖片節點

### 步驟 4：導出到伺服器
1. 選擇要導出的 WZ 檔案
2. File → Export → **Private Server...**
3. 選擇導出位置 (伺服器的 wz 目錄)

## 客戶端 vs 伺服器 WZ

| 類型 | 用途 | 格式 |
|------|------|------|
| 客戶端 WZ | 玩家電腦上的資源檔 | 加密的 .wz |
| 伺服器 XML | 伺服器讀取的數據 | .xml 格式 |

## 重要原則

### ⚠️ 千萬不要
- 不要將伺服器 XML 匯入客戶端 WZ！
- 不要刪除 WZ 中的節點後重新匯入
- 不要混用不同來源的 WZ 檔案

### ✅ 正確流程
```
客戶端修改：
編輯客戶端 WZ → 遊戲測試 → 滿意後 → 導出為 Private Server XML → 覆蓋伺服器 XML

伺服器修改：
從客戶端 WZ 導出 → 修改 XML → 測試
```

## 🆕 匯出格式詳解

### Private Server 格式
當你編輯完客戶端 WZ 後，需匯出為「Private Server」格式才能給伺服器使用：
```
編輯客戶端 WZ → 測試遊戲 → 滿意後 → File → Export → Private Server
```

### GMS (old) 加密 vs GMS (new)
| 加密格式 | 版本 | 用途 |
|---------|------|------|
| GMS (old) | v83 ~ v120 | 舊版本客戶端 |
| GMS (new) | v121+ | 新版本客戶端 |

## ⚠️ 重要警告

### 千萬不要
- ❌ 將伺服器 XML 匯入客戶端 WZ
- ❌ 刪除 WZ 節點後重新匯入（數據會丟失）
- ❌ 混用不同來源的 WZ 檔案
- ❌ 直接修改原版 WZ（先備份！）

### 正確流程
```
客戶端 WZ 修改流程：
1. 備份原始 WZ
2. 用 HaRepacker-resurrected 編輯客戶端 WZ
3. 遊戲中測試效果
4. 滿意後 → File → Export → Private Server XML
5. 將 XML 覆蓋到伺服器的 wz 目錄
6. 重啟伺服器或使用 @reload 命令

錯誤修正流程：
1. 恢復客戶端 WZ 備份
2. 重新編輯
3. 重新匯出
```

## 🛠️ 疑難排除

### Q: 打開 WZ 失敗
**A**: 確認選擇正確的加密格式 (GMS old)；確認 WZ 版本匹配

### Q: 保存後客戶端崩潰
**A**: 可能修改了不該修改的節點（長度、類型不符）；恢復備份

### Q: 導出 XML 有錯誤
**A**: 確保目標資料夾有寫入權限；確認磁碟空間充足

### Q: 客戶端載入 WZ 錯誤
**A**: WZ 版本與客戶端不匹配；嘗試用 HaRepacker 重新保存

### Q: 看不到修改效果
**A**: 清除客戶端快取（WZ 文件快取在記憶體）；確認已重新載入

## 🆕 新版物品 Import 到 v83 教學 (RaGEZONE, 2025-11)

RaGEZONE: https://forum.ragezone.com/threads/maplestory-v83-newer-items-import.1256083/

**問題描述**：
- 嘗試將新版物品 Import 到 v83
- 物品可以生成但無法裝備

**可能原因**：
1. 物品的 `.img` 檔案 Import 時結構不完整
2. Character.wz 中缺少對應的裝備定義
3. 裝備限制屬性（Level, STR, DEX 等）在 Item.wz 中設置不正確
4. 物品依賴的新屬性在 v83 客戶端中不存在

**解決方案**：
1. 確保導入完整的 .img 文件及其所有依賴
2. 檢查物品的 lvlm、STR、DEX 等屬性
3. 確認 WZ 結構版本匹配
4. 嘗試從同版本 WZ 提取而非跨版本

## 🆕 2026-03-25 新增：fonts folder 問題修復

**問題徵兆**：
- 客戶端啟動後文字無法顯示
- 只有背景圖片和框架
- UI 文字全部變成方塊或空白

**根本原因**：
- 缺少 fonts folder
- 字體檔案未正確複製
- 某些改過的客戶端移除了 fonts

**解決方案**：
```
Step 1: 找到正常的客戶端
  - 從備份或其他正常運作的客戶端複製

Step 2: 複製 fonts folder
  - 將 fonts folder 粘貼到目標客戶端目錄
  - 確認路徑：MapleStory/fonts/

Step 3: 檢查權限
  - 確認 fonts folder 有讀取權限
  - 嘗試以管理員身份執行客戶端

Step 4: 若問題持續
```

## 🆕 2026-03-25 新增：WZ 編輯安全守則更新

### 編輯前必做檢查清單
```
□ 備份原始 WZ 檔案
□ 確認使用正確的加密格式 (GMS old for v83)
□ 不要刪除節點，只修改數值
□ 每次修改只改一個項目
□ 測試後再繼續
□ 記錄所有修改項目
□ 建立修改日誌
```

### 編輯後必做檢查清單
```
□ 客戶端是否正常啟動
□ 目標功能是否生效
□ 是否有異常錯誤
□ 遊戲是否穩定
□ 確認無崩潰風險後再部署
```

### 常見錯誤與解決
| 錯誤 | 原因 | 解決方案 |
|------|------|---------|
| 客戶端崩潰 | 修改了長度/類型不符的節點 | 恢復 WZ 備份 |
| 讀取 WZ 失敗 | 加密格式選擇錯誤 | 確認使用 GMS old |
| 看不到修改效果 | WZ 快取未清除 | 重啟客戶端 |
| 匯入後數據丟失 | 刪除並重新匯入 | 不要刪除節點 |
| 物品無法使用 | 缺少必要屬性 | 檢查必備屬性清單 |
