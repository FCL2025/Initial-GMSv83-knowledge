# 2.6 WZ 編輯器使用教學

## HaRepacker 介紹
HaRepacker 是一款專門用於編輯 MapleStory WZ 文件的工具。

## 版本選擇
- **HaRepacker 4.2.4**: 經典版本，穩定
- **HaRepacker-resurrected** (推薦): 現代版本，持續更新

## 下載地址
GitHub: https://github.com/lastbattle/Harepacker-resurrected

## 支援的加密格式
v83 使用 **GMS (old)** 加密格式

## WZ 編輯流程

### 步驟 1：開啟 WZ 檔案
1. 執行 HaRepacker
2. 選擇 File → Open
3. 瀏覽到 WZ 檔案位置
4. 選擇加密類型：**GMS (old)**
5. 點擊 OK

### 步驟 2：進行修改
⚠️ **重要**：使用 UI 直接編輯，不要刪除/重新匯入節點！

可修改內容：
- 數值屬性 (如物品價格、怪物血量)
- 字串內容 (如 NPC 名稱、物品描述)
- 圖片節點

### 步驟 3：保存
1. File → Save (Ctrl+S)
2. 覆蓋原來的 WZ 檔案

### 步驟 4：導出到伺服器
1. 選擇要導出的 WZ 檔案
2. File → Export → **Private Server...**
3. 選擇導出位置 (伺服器的 wz 目錄)
4. 確認導出

## 客戶端 vs 伺服器 WZ

| 類型 | 用途 | 格式 |
|------|------|------|
| 客戶端 WZ | 玩家電腦上的資源檔 | 加密的 .wz |
| 伺服器 XML | 伺服器讀取的數據 | .xml 格式 |

## 重要原則

### ⚠️ 千萬不要
- 不要將伺服器 XML 匯入客戶端 WZ！
- 不要刪除 WZ 中的節點後重新匯入
- 不要混用不同來源的 WZ 檔案

### ✅ 正確流程
```
客戶端修改：
編輯客戶端 WZ → 遊戲測試 → 滿意後 → 導出為 Private Server XML → 覆蓋伺服器 XML

伺服器修改：
從客戶端 WZ 導出 → 修改 XML → 測試
```

## 常見 WZ 檔案

| 檔案 | 內容 |
|------|------|
| `String.wz` | 所有字串、NPC 名稱、物品描述 |
| `Item.wz` | 物品屬性、圖標 |
| `Skill.wz` | 技能數據 |
| `Mob.wz` | 怪物屬性 |
| `Map.wz` | 地圖數據 |
| `Npc.wz` | NPC 圖片和屬性 |
| `Quest.wz` | 任務數據 |

## 🆕 匯出格式詳解

### Private Server 格式
當你編輯完客戶端 WZ 後，需匯出為「Private Server」格式才能給伺服器使用：
```
編輯客戶端 WZ → 測試遊戲 → 滿意後 → File → Export → Private Server
```

### GMS (old) 加密 vs GMS (new)
| 加密格式 | 版本 | 用途 |
|---------|------|------|
| GMS (old) | v83 ~ v120 | 舊版本客戶端 |
| GMS (new) | v121+ | 新版本客戶端 |

v83 使用 **GMS (old)** 加密格式。

## ⚠️ 重要警告

### 千萬不要
- ❌ 將伺服器 XML 匯入客戶端 WZ
- ❌ 刪除 WZ 節點後重新匯入（數據會丟失）
- ❌ 混用不同來源的 WZ 檔案
- ❌ 直接修改原版 WZ（先備份！）

### 正確流程
```
客戶端 WZ 修改流程：
1. 備份原始 WZ
2. 用 HaRepacker-resurrected 編輯客戶端 WZ
3. 遊戲中測試效果
4. 滿意後 → File → Export → Private Server XML
5. 將 XML 覆蓋到伺服器的 wz 目錄
6. 重啟伺服器或使用 @reload 命令

錯誤修正流程：
1. 恢復客戶端 WZ 備份
2. 重新編輯
3. 重新匯出
```

## 📊 WZ 數據客戶端 vs 伺服器區分

### 客戶端 WZ（客戶端讀取）
| WZ 檔案 | 用途 |
|---------|------|
| `UI.wz` | 介面資源 |
| `Sound.wz` | 音效 |
| `Character.wz` | 角色外觀 |
| `Map.wz` | 地圖圖片 |
| `Npc.wz` | NPC 圖片 |
| `Mob.wz` | 怪物圖片 |
| `Item.wz` | 物品圖標 |

### 伺服器 XML（伺服器讀取）
| XML 目錄 | 用途 |
|---------|------|
| `wz/Item.xml` | 物品屬性邏輯 |
| `wz/Skill.xml` | 技能屬性邏輯 |
| `wz/Mob.xml` | 怪物屬性邏輯 |
| `wz/Map.xml` | 地圖屬性邏輯 |

### 重要原則
> 視覺效果修改客戶端 WZ，邏輯屬性修改伺服器 XML

## 🔧 常用 WZ 修改範例

### 範例 1：修改物品價格
```
1. 開啟 Item.wz (GMS old)
2. 導航到 Item/Consume/1222000.img
3. 找到 price 節點
4. 改為新價格
5. 保存 → 匯出到 Private Server XML
6. 覆蓋伺服器 XML
```

### 範例 2：修改怪物屬性
```
1. 開啟 Mob.wz (GMS old)
2. 導航到 Mob/100100.img (巖石巨人)
3. 修改 info/maxHP, info.atk 等節點
4. 保存 → 匯出
```

### 範例 3：新增 NPC 名稱字串
```
1. 開啟 String.wz
2. 導航到 Npc/npc9000000
3. 修改 name 節點
4. 保存
```

## 🛠️ 疑難排除

### Q: 打開 WZ 失敗
**A**: 確認選擇正確的加密格式 (GMS old)；確認 WZ 版本匹配

### Q: 保存後客戶端崩潰
**A**: 可能修改了不該修改的節點（長度、類型不符）；恢復備份

### Q: 導出 XML 有錯誤
**A**: 確保目標資料夾有寫入權限；確認磁碟空間充足

### Q: 客戶端載入 WZ 錯誤
**A**: WZ 版本與客戶端不匹配；嘗試用 HaRepacker 重新保存

### Q: 看不到修改效果
**A**: 清除客戶端快取（WZ 文件快取在記憶體）；確認已重新載入

## 🆕 2025-2026 WZ 修改新知

### Cosmic 專案 WZ 需求
Cosmic (P0nk) 明確說明：
> ⚠️ 這個伺服器需要自訂 WZ 文件，無法和原版 WZ 完全相容。
> 意圖是未來能完全在原始 WZ 上運行，但目前仍需使用改過的 WZ。

詳見：https://github.com/P0nk/Cosmic

### Skill.wz 修改需要組合語言
根據 Reddit (2024) 討論：
- 修改 Skill.wZ 需要使用 x86 組合語言
- 推薦工具：Ollydbg（免費 x86 調試器）
- 需具備組合語言基礎或相關課程背景

Reddit: https://www.reddit.com/r/mapleservers/comments/19cz6eu/v83_skillwz_editing/

### WZ 編輯風險提醒
- 大多數 WZ 編輯是允許的（如 UI 和技能修改）
- 因為不會給玩家不公平優勢
- **Cash Shop 相關 WZ 修改要小心**，可能導致自動封號
- **Packet 編輯涉及不公平優勢**，可能違反規則
