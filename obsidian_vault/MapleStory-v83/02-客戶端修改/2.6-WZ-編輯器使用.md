# 2.6 WZ 編輯器使用教學

## HaRepacker 介紹
HaRepacker 是一款專門用於編輯 MapleStory WZ 文件的工具。

## 版本選擇
- **HaRepacker 4.2.4**: 經典版本，穩定
- **HaRepacker-resurrected** (推薦): 現代版本，持續更新

## 下載地址
GitHub: https://github.com/lastbattle/Harepacker-resurrected

## 支援的加密格式
v83 使用 **GMS (old)** 加密格式

## WZ 編輯流程

### 步驟 1：開啟 WZ 檔案
1. 執行 HaRepacker
2. 選擇 File → Open
3. 瀏覽到 WZ 檔案位置
4. 選擇加密類型：**GMS (old)**
5. 點擊 OK

### 步驟 2：進行修改
⚠️ **重要**：使用 UI 直接編輯，不要刪除/重新匯入節點！

可修改內容：
- 數值屬性 (如物品價格、怪物血量)
- 字串內容 (如 NPC 名稱、物品描述)
- 圖片節點

### 步驟 3：保存
1. File → Save (Ctrl+S)
2. 覆蓋原來的 WZ 檔案

### 步驟 4：導出到伺服器
1. 選擇要導出的 WZ 檔案
2. File → Export → **Private Server...**
3. 選擇導出位置 (伺服器的 wz 目錄)

## 客戶端 vs 伺服器 WZ

| 類型 | 用途 | 格式 |
|------|------|------|
| 客戶端 WZ | 玩家電腦上的資源檔 | 加密的 .wz |
| 伺服器 XML | 伺服器讀取的數據 | .xml 格式 |

## 重要原則

### ⚠️ 千萬不要
- 不要將伺服器 XML 匯入客戶端 WZ！
- 不要刪除 WZ 中的節點後重新匯入
- 不要混用不同來源的 WZ 檔案

### ✅ 正確流程
```
客戶端修改：
編輯客戶端 WZ → 遊戲測試 → 滿意後 → 導出為 Private Server XML → 覆蓋伺服器 XML

伺服器修改：
從客戶端 WZ 導出 → 修改 XML → 測試
```

## 🆕 匯出格式詳解

### Private Server 格式
當你編輯完客戶端 WZ 後，需匯出為「Private Server」格式才能給伺服器使用：
```
編輯客戶端 WZ → 測試遊戲 → 滿意後 → File → Export → Private Server
```

### GMS (old) 加密 vs GMS (new)
| 加密格式 | 版本 | 用途 |
|---------|------|------|
| GMS (old) | v83 ~ v120 | 舊版本客戶端 |
| GMS (new) | v121+ | 新版本客戶端 |

v83 使用 **GMS (old)** 加密格式。

## ⚠️ 重要警告

### 千萬不要
- ❌ 將伺服器 XML 匯入客戶端 WZ
- ❌ 刪除 WZ 節點後重新匯入（數據會丟失）
- ❌ 混用不同來源的 WZ 檔案
- ❌ 直接修改原版 WZ（先備份！）

### 正確流程
```
客戶端 WZ 修改流程：
1. 備份原始 WZ
2. 用 HaRepacker-resurrected 編輯客戶端 WZ
3. 遊戲中測試效果
4. 滿意後 → File → Export → Private Server XML
5. 將 XML 覆蓋到伺服器的 wz 目錄
6. 重啟伺服器或使用 @reload 命令

錯誤修正流程：
1. 恢復客戶端 WZ 備份
2. 重新編輯
3. 重新匯出
```

## 🛠️ 疑難排除

### Q: 打開 WZ 失敗
**A**: 確認選擇正確的加密格式 (GMS old)；確認 WZ 版本匹配

### Q: 保存後客戶端崩潰
**A**: 可能修改了不該修改的節點（長度、類型不符）；恢復備份

### Q: 導出 XML 有錯誤
**A**: 確保目標資料夾有寫入權限；確認磁碟空間充足

### Q: 客戶端載入 WZ 錯誤
**A**: WZ 版本與客戶端不匹配；嘗試用 HaRepacker 重新保存

### Q: 看不到修改效果
**A**: 清除客戶端快取（WZ 文件快取在記憶體）；確認已重新載入

2. 檢查物品的 lvlm、STR、DEX 等屬性
3. 確認 WZ 結構版本匹配
4. 嘗試從同版本 WZ 提取而非跨版本

## 🆕 跨版本 WZ 物品 Import 失敗詳細分析 (RaGEZONE, 2025-11)

**來源**: https://forum.ragezone.com/threads/maplestory-v83-newer-items-import.1256083/

**問題描述**：
- 嘗試將新版（v100+）物品 Import 到 v83
- 物品可以在遊戲中生成，但**無法裝備**

**根本原因分析**：

| 原因 | 說明 |
|------|------|
| **.img 結構不完整** | 較新版本的物品 .img 包含 v83 不支援的屬性節點 |
| **Character.wz 缺少定義** | 裝備需要在 Character.wz 中有對應的造型定義 |
| **新屬性不存在** | v83 客戶端無法識別某些 v100+ 才引入的屬性（如裝備潛能卷軸相關） |
| **屬性值不合規範** | `lvlm`、`STR`、`DEX` 等屬性在 v83 有不同範圍限制 |

**正確流程**：
```
1. 從同版本（v83）WZ 提取物品，而非跨版本
2. 如果必須跨版本：
   - 提取完整的 .img 文件及其所有依賴
   - 移除 v83 不支援的屬性節點
   - 確認 lvlm、STR、DEX 在合理範圍內
3. 在 Character.wz 中新增對應的裝備造型定義
4. 在 String.wz 中新增物品名稱
```

**v83 裝備必備屬性檢查清單**：
- [ ] `info/lvlm` - 等級上限（需 ≤ v83 支援的最大值）
- [ ] `info/STR` - 力量需求
- [ ] `info/DEX` - 敏捷需求
- [ ] `info/INT` - 智力需求（法師裝備）
- [ ] `info/LUK` - 幸運需求
- [ ] `info/tuc` - 可強化的次數
- [ ] `info/cursed` - 詛咒標誌
- [ ] `info/only` - 職業限制

## 🆕 Quest 等級限制移除（LvMax Removal）完整步驟 (RaGEZONE, 2025-07)

**來源**: https://forum.ragezone.com/threads/maplestory-v83-method-to-disable-quest-level-restrictions.1250107/

**目的**：移除任務的等級上限限制，讓任意等級都能接任務

**使用場景**：
- 私人伺服器想要讓新手任務不被高等級玩家跳過
- 想要讓高等級玩家能夠重溫低等內容

**詳細步驟**：
```
1. 用 Harepacker 開啟 Quest.wz，encryption: GMS (old)
2. 導航到要修改的任務目錄
3. Tools → Export selection → XML → Classic
   ⚠️ 重要：選擇 Classic，否則會使用新版加密導致不相容
4. 用 Notepad++ 開啟 XML，搜尋所有 lvmax 屬性並移除
5. 存檔
6. 用 Harepacker 重新導入覆蓋
```

**XML 修改範例**：
```xml
<!-- 修改前 -->
<quest id="1000" lvmin="1" lvmax="30">
    ...
</quest>

<!-- 修改後（移除 lvmax） -->
<quest id="1000" lvmin="1">
    ...
</quest>
```

**批量移除所有 lvmax 的 Notepad++ Regex**：
```
尋找：lvmax="\d+"\s*
取代：（留空）
```
