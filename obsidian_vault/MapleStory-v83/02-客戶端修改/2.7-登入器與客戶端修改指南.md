## 📝 STREDIT — localhost IP 修改工具

**來源**: https://forum.ragezone.com/threads/release-stredit-localhost-editor.810762/

### 功能
- 修改 localhost.exe 內的加密 IP 字串
- 支援 v83（含加密字串）
- 可修改：IP 位址、廣告 URL、忘記密碼 URL

### 標準修改流程（v83）
```
1. 下載並開啟 STREDIT
2. File → Open → 選擇 localhost.exe
3. 當詢問「是否 v118+」→ 選「No」（v83 適用）
4. 當詢問「IP 搜尋模式」→ 選「Yes」
5. 點擊「Force to a IP」按鈕
6. 輸入目標 IP（如 127.0.0.1 或 VPS IP）
7. 保存修改後的 localhost.exe
```

### STREDIT 版本歷程
| 版本 | 更新內容 |
|------|---------|
| 2.9.2.0 | 支援 IP 修改（2012年） |
| 最新版 | 修復 v83 加密相容性 |

### 疑難排解
- **找不到 IP**：嘗試 Hex Editor 直接搜尋 `127.0.0.1` 位元組
- **編輯後無法連線**：確認版本（v118+ vs v83）選擇正確

## 🌐 LAN / 公開伺服器 IP 修改流程

### 完整 IP 修改清單
架設公開或 LAN 伺服器時，以下位置**全部需要修改**：

| 位置 | 工具 | 說明 |
|------|------|------|
| `Channel.properties` | 文字編輯器 | 登入端口 |
| `World.properties` | 文字編輯器 | 世界伺服器 IP |
| `localhost.exe` | STREDIT | 客戶端 IP |
| `config.yaml` | 文字編輯器 | （部分音源） |
| Hacked Client | Hex Editor | 如使用修改版客戶端 |

### LAN 架設（Hamachi / Radmin VPN）
```
1. 建立 Hamachi 網路或 Radmin VPN
2. 取得 Hamachi IP（如 25.64.92.1）
3. 修改所有 IP 指向該 Hamachi IP
4. 確認雙方都在同一 VPN 網路
5. 防火牆開啟 7575 和 8484 端口
```

### 公開架設（VPS）
```
1. 取得 VPS 公網 IP
2. 修改 Channel.properties: host=你的VPS_IP
3. 修改 localhost.exe IP 為 VPS_IP
4. 設定防火牆：TCP 7575, 8484
5. 建議關閉管理員權限要求（原廠客戶端）
```

## 🛡️ Themida 保護殼處理

### Themida 是什麼？
Themida 是**軟體保護殼**，用於加密和保護執行檔：
- 防止逆向工程
- 防止調試分析
- 防止修改程式碼

### 脫殼環境需求
| 需求 | 說明 |
|------|------|
| 作業系統 | Windows 7 x86 (32-bit) SP1 |
| 調試器 | OllyDbg + 隱藏插件 |
| 工具 | OreansUnvirtualizer |
| 記憶體 | 脫殼後需 `VirtualProtect` 修改 |

### 脫殼步驟概述
```
1. 使用 OreansUnvirtualizer 處理 VM 保護
2. 手動修復 IAT (Import Address Table)
3. 驗證脫殼後檔案可用 STRedit 開啟
```

### Themida 脫殼工具比較
| 工具 | 優點 | 缺點 |
|------|------|------|
| OreansUnvirtualizer | 自動處理 VM 保護 | 需手動修復 IAT |
| x64dbg + ScyllaHide | 可繞過反調試 | 需要較多設定 |
| Themida 官方卸載器 | 官方支援 | 僅支援舊版本 |

### v83 Themida 脫殼後注意事項
- Hendi's 客戶端是 Themida 脫殼版本，**適合修改**
- 原廠 HMS 客戶端內建 CRC bypass
- 脫殼後需要 `VirtualProtect` 才能修改記憶體

## 🆕 2026-03-27 下午 最新更新：Ezorsia V2 4G Flag 詳細設置

**GitHub**: https://github.com/444Ro666/MapleEzorsia-v2

### 4G Flag (Large Address Space Aware) 設置步驟

**目的**：允許客戶端使用最高 4GB 記憶體，而非預設的 2GB

**工具需求**：
- Hendi's Hex Editor 工具包（推薦）
  - 下載: https://drive.google.com/drive/folders/1hgnb92MGL6xqEp9szEMBh0K9pSJcJ6IT
- 或任何 Hex Editor（如 HxD、Neo）

**修改步驟**：
```
1. 使用 Hex Editor 打開 MapleStory.exe（原廠或 Hendi's 客戶端）
2. 找到 PE Header 中的特定偏移
   - 目標偏移: 0x0000013E
   - 預設值: 0x0F
   - 修改為: 0x2F
3. 保存並關閉
```

**4G Flag 作用**：
- 開啟後客戶端可以使用最大 4GB 虛擬記憶體
- 對於大型 WZ 文件（如 HD 客戶端）非常有用
- 避免記憶體不足導致的崩潰

### 🆕 Ezorsia V2 Admin 提權移除重要發現（2026-03-28 新增）

**來源**: https://github.com/444Ro666/MapleEzorsia-v2/wiki

**問題**：編輯 Admin elevation（提權）後，客戶端仍然要求管理員權限

**根本原因**：當執行檔命名為預設名稱 `MapleStory.exe` 時，Windows UAC 會繞過自訂的 `asInvoker` 設定

**解決方案**：
1. 使用 Hendi's Hex Editor 工具包編輯 `asInvoker` 部分
2. 編輯完成後，**將執行檔改名為 `MapleStory2.exe` 或其他名稱**
3. 這樣 Windows 就會正確應用 `asInvoker` 設定，不再要求管理員權限

## 🆕 LAN 多人連線常見錯誤排查 (RaGEZONE, 2026-03)

**來源**: https://forum.ragezone.com/threads/need-help-v83-lan-server-works-on-pc1-but-pc2-gets-error-2147467259.1262187/

### 錯誤碼分析
| 錯誤碼 | Hex 值 | 意義 | 根本原因 |
|--------|--------|------|---------|
| `-2147467259` | `0x80004005` | Access Denied | 防火牆/權限問題（非網路問題）|

### 排查檢查清單
- [ ] 確認兩台 PC 在同一 LAN 網段
- [ ] 確認伺服器 listen 0.0.0.0（而非 127.0.0.1）
- [ ] 確認防火牆開啟 7575 和 8484 埠口
- [ ] 若錯誤碼 0x80004005 → 防火牆/權限問題
- [ ] config.yaml HOST 設定為 LAN IP（不是 127.0.0.1）
- [ ] WZ 檔案在兩台 PC 上完全一致

### 端口配置要點 (RaGEZONE, Maple83 GitHub)
**來源**: https://github.com/jonnylin13/Maple83

- MapleSolaxia 需要開放大端口：`7575 + (channel數)` 和 `8484`
- 建立新的自訂服務，啟用 TCP/UDP 兩種協定
- 預設配置：
  - Login Port: 8484
  - Channel 1: 7575
  - Channel 2: 7576
  - ... 依此類推

### 防火牆快速設定
```cmd
netsh advfirewall firewall add rule name="MapleStory Login" dir=in action=allow protocol=tcp localport=7575
netsh advfirewall firewall add rule name="MapleStory Game" dir=in action=allow protocol=tcp localport=8484
```

## 🆕 MaplePE — 32-bit MapleStory 封包編輯器（GitHub README 直接抓取，2026-03-28 新增）

**GitHub**: https://github.com/zhyonc/MaplePE

### 原文 README 功能列表

| 功能 | 原文說明 |
|------|---------|
| 封包解析 | Log and format packet structures as list items |
| 自訂封包發送 | Edit packet data and send it to the client |
| 目標進程提取 | Extract InPacket and OutPacket information from the target process |
| Opcode 過濾 | Filter specific opcodes to avoid logging certain packets |
| 程式碼生成 | Generate simple code from package structures |

### 技術架構（原文）

> "MaplePE.exe runs a UDP server to receive packet information. Packet.dll hooks functions related to writing and reading packets to detect packet structures."

技術原理：
- MaplePE.exe 本身是一個 UDP 伺服器，用於接收封包資訊
- 使用 CreateRemoteThread + LoadLibraryW 將 Packet.dll 注入目標進程
- Packet.dll Hook 封包讀寫相關函數以檢測封包結構
- Packet.dll 使用 UDP client 發送封包資訊

### 封包發送前提條件（原文）

> "Before sending a packet via MaplePE.exe, the server must respond with at least one packet to ensure that the CClientSocket instance can be correctly specified."

在使用 MaplePE.exe 發送封包前，伺服器必須至少回應過一個封包，以確保 CClientSocket instance 能被正確指定。

### BB+ 版本崩潰問題（原文）

> "Since some versions after BB add hook-detection in CClientSocket::SendPacket"

BB 版本後的 CClientSocket::SendPacket 有 hook 檢測機制。

**解決方案**：Hook `MakeBufferList` 而非 `SendPacket` 來避免 DLL 注入後發送 OutPacket 時崩潰。使用 Cheat Engine 將崩潰條件位址填為 `90` (NOP) 來禁用。

> "Some version addresses can be found here: https://docs.google.com/spreadsheets/d/1-LqA-OWgOuT-Xy3i73Jy5F_uVmogaGvmYpH4mLDSPhg"

Google Sheets 包含各版本函數位址參考表。
