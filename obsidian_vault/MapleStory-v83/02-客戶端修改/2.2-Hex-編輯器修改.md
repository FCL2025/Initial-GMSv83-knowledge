## 📝 修改步驟

### Step 1：開啟檔案

1. 下載並開啟 HxD
2. 拖曳 `MapleStory.exe` 到編輯器
3. 或使用 `File → Open`

### Step 2：搜尋 IP

1. 按下 `Ctrl + F` 開啟搜尋
2. 選擇 **Text** (文字搜尋)
3. 輸入 `127.0.0.1`（或要修改的 IP）
4. 點擊 **OK**

### Step 3：修改 IP

找到後，直接在文字上輸入新 IP：

```
原始：127.0.0.1 (7 bytes)
新：  192.168.1.100 (15 bytes)

⚠️ 重要：長度必須相同！
```

### ⚠️ 長度問題

如果新 IP 比原 IP 長，有以下解決方案：

**方案 1：使用 Ezorsia v2**
- Ezorsia v2 內建 IP 導向功能，無需修改 hex
- GitHub: https://github.com/444Ro666/MapleEzorsia-v2
- 只要將 `dinput8.dll` 放入遊戲目錄即可

**方案 2：使用 STRedit（長度不同的修改）**
- 參考 [[2.3-STRedit-字串修改|STRedit 教學]]

**方案 3：使用 Cheat Engine**
- 在記憶體中動態修改 IP，無需改變檔案長度

## 🔍 進階技巧

### 如何找到正確的 IP？

有時候 IP 不是以 `127.0.0.1` 形式存在，可能是：

| 形式 | 範例 |
|------|------|
| 文字 | `127.0.0.1` |
| 十六進位 | `7F 00 00 01` (127.0.0.1) |
| 分段儲存 | `127` + `.0` + `.0` + `.1` |

### 十六進位計算

```
127.0.0.1 = 7F 00 00 01
192.168.1.1 = C0 A8 00 01
```

用 HxD 的 **Hex 搜尋** 模式：
1. `Ctrl + F` → 選擇 **Hex**
2. 輸入 `7F 00 00 01`
3. 替換為 `C0 A8 00 01`（需確認長度一致）

## 🌐 LAN 多人連線：IP 修改完整流程

### 問題描述

RaGEZONE (2週前)：PC1（主機）正常遊玩，PC2 連線時出現錯誤碼 **-2147467259**

### 原因分析

| 可能原因 | 說明 |
|---------|------|
| 防火牆阻擋 | 7575/8484 端口被阻擋 |
| IP 修改不完整 | 客戶端未正確指向主機 IP |
| 端口未轉發 | 路由器未設定端口轉發 |
| IV 不同步 | AES 加密 IV 問題 |

### 解決步驟

**Step 1：確認兩台 PC 在同一網段**
```
檢查方法：
- PC1 IP: 192.168.1.100
- PC2 IP: 192.168.1.101
兩台 PC 都應該能 ping 到對方
```

**Step 2：修改 PC1（主機）的客戶端 IP**
```
1. 用 HxD 打開 MapleStory.exe
2. 搜尋 "127.0.0.1" 並替換為 "192.168.1.100"
3. 注意長度要一致！
4. 存檔
```

**Step 3：開放防火牆**
```
Windows 防火牆：
1. 控制台 → 系統與安全性 → Windows Defender 防火牆
2. 點擊「允許應用程式通過防火牆」
3. 新增 java.exe 和 MapleStory.exe
4. 確認 7575 和 8484 端口已開放

或直接關閉防火牆測試：
- 控制台 → 系統與安全性 → Windows Defender 防火牆
- 點擊「開啟或關閉 Windows Defender 防火牆」
```

**Step 4：確認伺服器 IP 設定**
```
在伺服器設定中確認：
- 監聽 IP 為 0.0.0.0（接受所有連線）
- 不要綁定 127.0.0.1
```

**Step 5：測試連線**
```
1. 在 PC2 的瀏覽器輸入：http://192.168.1.100:7575
   如果能看到回應，說明伺服器正常
2. 啟動 PC2 客戶端，嘗試連線
3. 如果仍失敗，嘗試雙方都關閉防火牆測試
```

### 快速排查清單

| 檢查項目 | PC1 (主機) | PC2 (客戶端) |
|---------|-----------|-------------|
| 防火牆關閉/開放端口 | ✅ | ✅ |
| IP 修改為主機 IP | ✅ | ✅ |
| 兩台 PC 同一網段 | N/A | ✅ |
| 伺服器正常運行 | ✅ | N/A |
| 客戶端版本 v83 | ✅ | ✅ |

## 🆕 Hex 編輯 localhost.exe 詳解 (HeavenMS 指南)

根據 HeavenMS README (https://github.com/ronancpl/HeavenMS)：

### 修改步驟
```
1. 使用 Neo Hex Editor 開啟 localhost.exe
2. 搜尋 "Type: String" + "127.0.0.1" 的位址
3. 在 3 個不同位址處覆寫 IP（**不要用取代功能**）
4. 儲存並離開編輯器
```

### ⚠️ 重要注意事項
- **不要使用取代/Replace 功能**，必須手動一個一個輸入
- 如果使用 Windows 8/10 有相容性問題，搭配 `lolwut.exe` 解決
- 某些客戶端需要管理員權限才能執行

### Windows 8/10 相容性問題解決
```
1. 下載 lolwut-v0.01.rar
2. 解壓 lolwut.exe 到 MapleStory 客戶端資料夾
3. localhost.exe 屬性設定：
   - 相容模式：Windows 7
   - 取消：減少色彩模式
   - 解析度：640 x 480
   - 取消：停用高 DPI 設定
   - 執行身分：系統管理員
4. 開啟 lolwut.exe 執行
```
