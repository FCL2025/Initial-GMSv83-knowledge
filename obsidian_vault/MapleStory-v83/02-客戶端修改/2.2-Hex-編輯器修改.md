## 📝 修改步驟

### Step 1：開啟檔案

1. 下載並開啟 HxD
2. 拖曳 `MapleStory.exe` 到編輯器
3. 或使用 `File → Open`

### Step 2：搜尋 IP

1. 按下 `Ctrl + F` 開啟搜尋
2. 選擇 **Text** (文字搜尋)
3. 輸入 `127.0.0.1`（或要修改的 IP）
4. 點擊 **OK**

### Step 3：修改 IP

找到後，直接在文字上輸入新 IP：

```
原始：127.0.0.1 (7 bytes)
新：  192.168.1.100 (15 bytes)

⚠️ 重要：長度必須相同！
```

### ⚠️ 長度問題

如果新 IP 比原 IP 長，有以下解決方案：

**方案 1：使用 Ezorsia v2**
- Ezorsia v2 內建 IP 導向功能，無需修改 hex
- GitHub: https://github.com/444Ro666/MapleEzorsia-v2
- 只要將 `dinput8.dll` 放入遊戲目錄即可

**方案 2：使用 STRedit（長度不同的修改）**

## 🔍 進階技巧

### 如何找到正確的 IP？

有時候 IP 不是以 `127.0.0.1` 形式存在，可能是：

| 形式 | 範例 |
|------|------|
| 文字 | `127.0.0.1` |
| 十六進位 | `7F 00 00 01` (127.0.0.1) |
| 分段儲存 | `127` + `.0` + `.0` + `.1` |

### 十六進位計算

```
127.0.0.1 = 7F 00 00 01
192.168.1.1 = C0 A8 00 01
```

用 HxD 的 **Hex 搜尋** 模式：
1. `Ctrl + F` → 選擇 **Hex**
2. 輸入 `7F 00 00 01`
3. 替換為 `C0 A8 00 01`（需確認長度一致）

## 🌐 LAN 多人連線：IP 修改完整流程

### 問題描述

RaGEZONE (2週前)：PC1（主機）正常遊玩，PC2 連線時出現錯誤碼 **-2147467259**

### 原因分析

| 可能原因 | 說明 |
|---------|------|
| 防火牆阻擋 | 7575/8484 端口被阻擋 |
| IP 修改不完整 | 客戶端未正確指向主機 IP |
| 端口未轉發 | 路由器未設定端口轉發 |
| IV 不同步 | AES 加密 IV 問題 |

### 解決步驟

**Step 1：確認兩台 PC 在同一網段**
```
檢查方法：
- PC1 IP: 192.168.1.100
- PC2 IP: 192.168.1.101
兩台 PC 都應該能 ping 到對方
```

**Step 2：修改 PC1（主機）的客戶端 IP**
```
1. 用 HxD 打開 MapleStory.exe
2. 搜尋 "127.0.0.1" 並替換為 "192.168.1.100"
3. 注意長度要一致！
4. 存檔
```

**Step 3：開放防火牆**
```
Windows 防火牆：
1. 控制台 → 系統與安全性 → Windows Defender 防火牆
2. 點擊「允許應用程式通過防火牆」
3. 新增 java.exe 和 MapleStory.exe
4. 確認 7575 和 8484 端口已開放

或直接關閉防火牆測試：
- 控制台 → 系統與安全性 → Windows Defender 防火牆
- 點擊「開啟或關閉 Windows Defender 防火牆」
```

**Step 4：確認伺服器 IP 設定**
```
在伺服器設定中確認：
- 監聽 IP 為 0.0.0.0（接受所有連線）
- 不要綁定 127.0.0.1
```

**Step 5：測試連線**
```
1. 在 PC2 的瀏覽器輸入：http://192.168.1.100:7575
   如果能看到回應，說明伺服器正常
2. 啟動 PC2 客戶端，嘗試連線
3. 如果仍失敗，嘗試雙方都關閉防火牆測試
```

### 快速排查清單

| 檢查項目 | PC1 (主機) | PC2 (客戶端) |
|---------|-----------|-------------|
| 防火牆關閉/開放端口 | ✅ | ✅ |
| IP 修改為主機 IP | ✅ | ✅ |
| 兩台 PC 同一網段 | N/A | ✅ |
| 伺服器正常運行 | ✅ | N/A |
| 客戶端版本 v83 | ✅ | ✅ |

## 🆕 localhost.exe Hex Edit 完整指南 (HeavenMS, 2019 更新)

**重要來源**：https://github.com/ronancpl/HeavenMS/issues/437

### 為什麼是 localhost.exe 而不是 MapleStory.exe？

localhost.exe 是一個專門修改過的啟動程式，專門用於私人伺服器：
- 已經過脫殼和簡化
- 包含簡化的 IP 連線邏輯
- 更容易修改

### Hex Edit 三步驟（適用於所有 v83 localhost.exe）

```
Step 1: 使用 HxD 開啟 localhost.exe

Step 2: Ctrl+F → 選擇 "Text-string" → 搜尋 "127.0.0.1"
        → 會找到 **三個連續出現** 的 "127.0.0.1"

Step 3: 將三處 "127.0.0.1" 都替換為目標 IP
        → 存檔
```

### ⚠️ IP 長度一致性原則

**重要**：修改後的 IP 長度必須與原來的 "127.0.0.1" (9字元) 相同！

| 原 IP | 目標 IP | 長度 | 可行性 |
|-------|---------|------|--------|
| 127.0.0.1 | 127.0.0.1 | 9 → 9 | ✅ |
| 127.0.0.1 | 192.168.1.1 | 9 → 13 | ⚠️ 需補零或用 hex |
| 127.0.0.1 | 10.0.0.1 | 9 → 9 | ✅ |

**長度不同時的解決方案**：
1. 在 IP 字串後面加 `\x00`（NULL）填充
2. 或使用數字格式 `7F000001`（127.0.0.1 的 hex 表示）

### 常見問題排查

**Q: HxD 搜尋不到 "127.0.0.1"？**
```
可能原因：
1. IP 以 split 形式儲存（如 "127" + ".0.0.1" 分開）
2. 使用 hex 格式儲存（如 7F 00 00 01）
3. 使用了不同的 localhost.exe 版本

解決方案：
→ 嘗試搜尋 "127" 或 "localhost"
→ 或搜尋 hex: 31 32 37 2E 30 2E 30 2E 31
```

**Q: 修改後客戶端無法連線？**
```
檢查清單：
1. ✅ 確認目標伺服器 IP + Port 是否正確
2. ✅ 確認伺服器已啟動
3. ✅ 確認防火牆已開啟對應 Port
4. ✅ 確認 WZ 版本與伺服器匹配
5. ✅ 嘗試重啟客戶端（有時快取問題）
```
