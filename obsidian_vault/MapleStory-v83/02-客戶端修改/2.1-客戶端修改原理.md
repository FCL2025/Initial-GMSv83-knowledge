# 2.1 客戶端修改原理

---

## 🎯 為什麼要修改客戶端？

MapleStory 客戶端出廠時設定為連接到**官方伺服器**：

```
正常流程：
客戶端 ──→ 官方伺服器 (login.maplestory.nexon.com:7575)
                    ↑
              這是我們要改掉的！
```

為了讓客戶端連接到**我們的私人伺服器**，必須修改客戶端內的 IP 位址。

---

## 🔍 修改的基本原理

### IP 位址在哪裡？

客戶端中儲存了官方伺服器的 IP 位址，通常以 **字串** 形式存在：

```
原始內容：[127.0.0.1] 或 [203.78.xx.xx]
         ↓
修改後：[你的伺服器 IP 或網域]
```

### 修改位置

| 位置 | 說明 |
|------|------|
| `MapleStory.exe` | 主執行檔，包含連線 IP |
| `login.cfg` | 某些版本有獨立的設定檔 |

---

## 🌐 連線流程解析

```
1. 啟動客戶端
2. 客戶端讀取設定的 IP (如 127.0.0.1)
3. 嘗試連接到該 IP 的 port 7575
4. 連線成功 → 進入登入畫面
5. 發送帳密到伺服器認證
```

---

## ⚠️ 常見問題

### Q: 修改後無法連接？
- 檢查 IP 是否正確
- 確認伺服器已啟動
- 檢查防火牆是否阻擋

### Q: 多個 IP 要修改？
- 通常只需修改**第一個**出現的 IP
- 其餘 IP 可能是備用或內網 IP

## 🆕 v83 LAN 多人遊戲架設 - 錯誤排查 (RaGEZONE, 2026-03)

**來源**：https://forum.ragezone.com/threads/need-help-v83-lan-server-works-on-pc1-but-pc2-gets-error-2147467259.1262187/

**問題**：PC1 (伺服器主機) 可正常遊玩，PC2 連線時出現錯誤碼 -2147467259

**可能原因與解決方案**：

| 問題 | 解決方案 |
|------|---------|
| 客戶端 IP 指向 127.0.0.1 | 需指向 PC1 的 LAN IP（如 192.168.1.x）而非 localhost |
| 防火牆未開放端口 | `netsh advfirewall firewall add rule name="MapleStory" dir=in action=allow protocol=tcp localport=8484` |
| MySQL 未授權遠端 | `GRANT ALL PRIVILEGES ON *.* TO 'root'@'%' IDENTIFIED BY 'password'; FLUSH PRIVILEGES;` |

**正確 LAN 架設流程**：
```
1. 伺服器主機 PC1：
   - 啟動 MySQL、JDK 伺服器
   - 確認 LAN IP（如 192.168.1.100）
   
2. 客戶端 PC2：
   - 使用 HxD 修改客戶端 IP 為 PC1 的 LAN IP
   - 確認防火牆已開放 8484 端口
   
3. 測試：
   - PC1 上 localhost 測試確認正常
   - PC2 連線到 192.168.1.100:8484
```

## 🆕 2026-03-25 新增：v83 客戶端與音源版本匹配重要性

```
為什麼版本匹配很重要？
├── AES Key 不同 → 封包加密失敗
├── Opcode 表不同 → 封包無法正確解析
├── WZ 格式不同 → 客戶端崩潰或顯示異常
└── 音源假設客戶端版本 → 邏輯判斷錯誤

如何確認匹配？
1. 檢查客戶端的 VERSION (如 0x0053 = 83)
2. 檢查音源的 ServerConstants.java
3. 確認 WZ 檔案來源一致
4. 測試 Hello Packet 握手是否成功
```
