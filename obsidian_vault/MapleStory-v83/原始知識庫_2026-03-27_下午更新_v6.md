## 二、MapleEzorsia-v2 — HD 客戶端革命性方案

### 2.1 核心原理

```
傳統方案：
  官方客戶端 → 刪除 HShield/、ASPLnchr.exe、MapleStory.exe、Patcher.exe
  → 放入 localhost.exe → 手動替換 WZ

MapleEzorsia-v2 方案：
  官方客戶端 → 只放入 dinput8.dll + config.ini → 直接啟動遊戲
  → 無需刪除任何原始檔案！

| 特性 | localhost.exe 方案 | Ezorsia-v2 dinput8.dll |

| 客戶端完整性 | 破壞性修改 | 非破壞性（dll 注入） |

### 2.3 技術實現細節

```
dinput8.dll 鉤子函數：
  1. 掛鉤 DirectInput 創建函數

## 三、MapleCrypto 加密機制深度解析

### 3.1 雙密鑰系統

```
每個連線會話使用兩組密鑰：

發送密鑰（Send Key）  → 加密客戶端發出的封包
接收密鑰（Recv Key）  → 解密伺服器發來的封包

密鑰生成流程：
1. 客戶端連線時隨機生成兩個 32-byte 密鑰
2. 通過第一個未加密的握手封包發送給伺服器
3. 伺服器使用這兩個密鑰初始化加密狀態
4. 後續所有通訊都使用 MapleCrypto + AES 加密
```

### 3.2 握手封包結構（第一個明文封包）

```
Offset  Size  說明
------  ----  ----
0x00    4     封包標頭（Length，經過 MapleCrypto 混淆）
0x04    2     Opcode（0x0001 = HANDSHAKE）
0x06    4     客戶端版本（v83 = 83）
0x0A    16    發送密鑰（Send IV / AES IV）
0x1A    16    接收密鑰（Recv IV / AES IV）
...     ...   其他標頭數據
```

### 3.3 MapleCrypto + AES 兩層加密

```
加密流程（發送端）：
  原始資料 → [Layer 1: MapleCrypto] → [Layer 2: AES-OFB] → 網路傳輸

解密流程（接收端）：
  網路接收 → [Layer 2: AES-OFB] → [Layer 1: MapleCrypto] → 原始資料

AES 模式：OFB（Output Feedback）
- IV（初始化向量）每次傳遞後會根據上次密文更新
- 這就是為什麼同一封包每次出現的密文都不同
```

### 3.4 AES 參數（v83 CMS 參考值）

```
AES 金鑰（AES User Key）：0x13, 0x08, 0x06, 0xB4, 0x1B, 0x0F, 0x33, 0x52
IV 轉換常量：0xC65053F2
版本遮罩：left = (version >> 8) & 0xFF, right = (version << 8) & 0xFF00
```

### 3.5 MapleCrypto（Shanda）混淆層

```
目的：簡單的 XOR/shift 混淆，防止明文靜態分析

原理（v83 具體實現）：
1. 維護一個 32-bit 狀態值（IV）
2. 每處理 1 byte：狀態 = f(狀態, 金鑰)
3. 輸出 = 輸入 ^ (狀態的低 8 位)

注意：新版 MapleStory（v140+）已移除 MapleCrypto 層
```

### 3.6 MapleShark 解密原理

```
MapleShark 能在工具內自動解密的原因：
1. 握手封包為明文，包含完整 IV
2. 使用 MapleCrypto 實現逆向混淆
3. 使用 AES-OFB 實現解密

使用前提：
- 必須能截獲第一個握手封包（明文）
- 知道正確的版本號
- 工具已內建 v83 的 MapleCrypto 實現
```

## 四、IDA Pro 抓 Opcode 實戰流程

### 4.1 核心概念：CWvsContext::OnPacket

```
客戶端封包處理調用鏈：

CClientSocket::ProcessPacket()
    ↓ 解析 Header（長度）
    ↓ 解密（AES + MapleCrypto）
    ↓ 呼叫 CWvsContext::OnPacket(opcode, data)
         ↓
    根據 opcode 進入 switch 語句
         ↓
    NPC_TALK        → CField::OnNPCTalk
    CHANGE_MAP      → CField::OnChangeMap
    MOVE_PLAYER     → CField::OnMovePlayer
    ...
```

### 4.2 IDA 找到 OnPacket 的步驟

```
Step 1：打開 IDA Pro，載入 MapleStory.exe
Step 2：等待自動分析完成（約 10-30 分鐘）
Step 3：搜尋字串 "OnPacket" 或 "CLogin"
Step 4：找到 CClientSocket::ProcessPacket 或 CWvsContext::OnPacket
Step 5：在函數內找到 switch(opcode) 語句
Step 6：每個 case 分支就是一個 opcode handler
Step 7：使用 IDC Script 批量導出所有 case
```

### 4.3 IDC Script 自動導出 Opcode

```c
// MapleStory Opcode 導出腳本（IDC）
// 使用方法：在 IDA Console 中執行

#include <idc.idc>

static main(void) {
    auto ea, func, xref;
    
    // 找到 CWvsContext::OnPacket 函數
    func = get_name_ea(BADADDR, "CWvsContext__OnPacket");
    if (func == BADADDR) {
        msg("CWvsContext__OnPacket not found!\n");
        return;
    }
    
    msg("Found OnPacket at: 0x%08X\n", func);
    
    // 遍歷所有交叉引用
    for (xref = RfirstB(func); xref != BADADDR; xref = RnextB(func, xref)) {
        msg("Referenced from: 0x%08X\n", xref);
    }
}
```

### 4.4 IDA 分析 recv 和 send opcode 的差異

| 目標 | 關鍵函數 | 特徵 |
|------|---------|------|
| **Recv (S→C)** | `CClientSocket::ProcessPacket` | 進入 switch 前有解密 call |
| **Send (C→S)** | `COutPacket::Encode2` | 在發送前有加密 call，Encode2 參數即 opcode |

### 4.5 IDA 找 opcode 的另一條路：CInPacket::Decode2

```
在 CInPacket::Decode2 中：
  this->opcode = *(short*)(packet + 4)  // 第4 byte 開始是 opcode

在 COutPacket::Encode2 中：
  *(short*)(packet + 4) = opcode         // 寫入 opcode

在這些函數下斷點，遊戲運行時即可看到實時 opcode 值
```

### 4.6 IDA 找 CInPacket::Decode2 的技巧

```
1. IDA 字串搜尋 "CInPacket"
2. 或搜尋 "Decode2"
3. 或交叉引用 "short" 類型讀取
4. 下斷點後，Decode2 的第二個參數就是 opcode
```

### 4.7 IDA 找到 Send Opcode 表

```
方法1：搜尋靜態字串 "SND_" 或 "OP_SEND"
方法2：在 COutPacket::Encode2 中找到 switch(opcode) 表
方法3：交叉引用 Data segment 中的 opcode 陣列
```

### 4.8 實用 IDC Script：導出 recv opcode 結構

```c
// 批量命名 IDA 中的 opcode 分支
#include <idc.idc>

static name_opcodes() {
    auto ea, opcode;
    
    // 從 0x0000 開始遍歷
    for (opcode = 0; opcode < 0x100; opcode++) {
        ea = find_binary(ea + 1, SEARCH_DOWN, "01 00"); // 測試 pattern
        if (ea == BADADDR) break;
        // 命名：OP_RECV_XXXX
        set_name(ea, sprintf("OP_RECV_%04X", opcode), SN_NOWARN);
    }
}
```

### 4.9 MaplePacketPuller — Python 自動化方案

```
GitHub: https://github.com/Bratah123/MaplePacketPuller

功能：
- Python 自動化抓取 packet 結構
- 配合 IDA 使用（需先在 IDA 中命名目標函數）
- 自動生成 packet 結構報告

原理：
1. IDA 中手動命名感興趣的函數（如 CWvsContext__OnPacket）
2. 運行 MaplePacketPuller 指定函數名
3. 腳本遍歷所有 case 分支，提取 packet 結構
4. 輸出格式化報告
```

## 五、抓包工具進階 — MapleShark 實戰

### 5.1 MapleShark 完整工作流程

```
Step 1：環境準備
  → 安裝 WinPcap 或 Npcap
  → 確認網路介面（127.0.0.1 或實際網卡）
  → 載入 v83 封包腳本（.cs 檔）

Step 2：開始捕獲
  → File → Start Capture
  → 選擇網路介面
  → 設定過濾器：tcp.port == 8484

Step 3：觸發封包
  → 啟動 MapleStory 客戶端
  → 進行遊戲操作（移動、對話等）

Step 4：分析封包
  → 點擊封包 → 右側顯示樹狀結構
  → 自動解析 opcode、名稱、欄位

Step 5：匯出數據
  → Write → 匯出日誌
  → Export → 匯出 Packet 定義
```

### 5.2 v83 MapleShark 腳本核心 API 進階

```csharp
// 陣列處理
void AddArray(string name, int fixedCount)
    // 固定數量陣列

void AddConditionalArray(string name, string counterName)
    // 條件數量陣列，counterName 是前面某個欄位

void StartArray(string name)
void AddArrayItem(string name)
void EndArray()
    // 手動控制動態陣列

// 枚舉

    // bitWidth: 1=bool, 2=short, 4=int

// 結構嵌套
void StartNode("SubPacket")    // 開始嵌套結構
    AddInt("x");
    AddInt("y");
void EndNode()
```

### 5.3 封包解析示例：MOVE_PLAYER (0x0013)

```csharp
// MOVE_PLAYER Send (C→S) — 玩家移動
void MOVE_PLAYER() {
    AddByte("x");              // 發送次數（通常為 1）
    StartNode("MoveData");
        AddShort("x");         // 目標 X
        AddShort("y");         // 目標 Y
        AddByte("newMap");     // 是否換地圖
        AddShort("variance");  // 速度
        AddByte("foot");       // 腳步
        // 可變長度：攻擊路徑座標
        StartArray("path");
            AddByte("ox");     // 原 X
            AddByte("oy");     // 原 Y
            // ... 更多路徑點
        EndArray();
    EndNode();
}
```

## 六、WZ 修改黃金法則（補充）

### 6.1 WZ 加密類型識別

| 加密類型 | 識別特徵 | 工具支援 |
|---------|---------|---------|
| **GMS (Old)** | 原始 Nexon 加密 | HaRepacker 4.x ✅ |
| **GMS (New)** | 2010+ 新加密 | HaRepacker 4.2.4 ✅ |
| **EMS/JMS** | 歐/日/韓版 | 需特定版本 |
| **CMS** | 中國版 | 特殊工具 |

v83 使用 **GMS (Old)** 加密，兼容性最好。

### 6.2 HaRepacker 4.2.4 完整操作流程

```
正確流程（客戶端 WZ）：
1. 啟動 HaRepacker 4.2.4
2. File → Open → 選擇 WZ 檔（如 Mob.wz）
3. 導航樹中選擇要修改的節點
4. 右鍵 → Edit Value
5. 輸入新數值
6. File → Save（保持 GMS Old 加密）
7. 關閉 HaRepacker

錯誤：不要用「Private Server Export/Import」來編輯客戶端 WZ！
```

### 6.3 WZ 編輯安全清單

```
✅ 可以安全編輯：
  - Mob.wz → 怪物屬性（HP、MP、AD、攻擊間隔等）
  - Npc.wz → NPC 名稱（在 Quest.wz 中編輯更安全）
  - Item.wz → 物品屬性（攻擊力、魔力等）
  - Skill.wz → 技能屬性（冷卻時間、MP消耗等）
  - Map.wz → 地圖物件（！但 foothold 需專用工具）
  - Reactor.wz → 反應堆屬性

⚠️ 需特別小心：
  - String.wz → 直接編輯可能崩潰
  - Character.wz → 精靈圖直接修改需專業工具
  - UI.wz → UI 框架修改複雜

❌ 禁止操作：
  - 刪除 WZ 中的任何父節點
  - 新增非標準格式的節點
  - 使用錯誤的加密類型重新儲存
```
