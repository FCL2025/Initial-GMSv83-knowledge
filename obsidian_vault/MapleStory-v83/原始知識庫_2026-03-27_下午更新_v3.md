## 三、抓包分析工具 — MapleShark 深度解析

### 3.1 MapleShark 簡介

**GitHub**: https://github.com/diamondo25/MapleShark

**作者**: diamondo25

**定位**：MapleStory 專用封包嗅探器，使用 **SharpPcap** (.NET Pcap library)

### 3.2 核心功能

| 功能 | 說明 |
|------|------|
| 封包截獲 | 截取客戶端↔伺服器所有 TCP 流量 |
| 封包解析 | 自動解析封包頭部、Opcode、資料 |
| 封包過濾 | 按 Opcode、方向、大小過濾 |
| 腳本定義 | 使用 S# (ScriptDotNet) 定義封包結構 |
| 匯出功能 | 將封包日誌寫入文字檔 |

### 3.3 MapleShark 腳本系統 (ScriptDotNet / S#)

MapleShark 的核心亮點是**腳本化的封包結構定義**：

```csharp
// S# 語法示例（定義 NPC_TALK 封包）
int AddByte(string name)        // 添加 1 byte 欄位
ushort AddUShort(string name)   // 添加 2 bytes 欄位
int AddInt(string name)          // 添加 4 bytes 欄位
float AddFloat(string name)      // 添加 4 bytes 浮點數
string AddPaddedString(string name, int length)  // 添加固定長度字串
void StartNode(string name)     // 開始子節點
void EndNode(bool expand)       // 結束子節點
void Write(string file, string line)  // 寫入日誌檔
```

### 3.4 MapleShark vs Maple_Pshark 對比

| 功能 | MapleShark | Maple_Pshark |
|------|-----------|-------------|
| 平台 | Windows (.NET) | Windows |
| 捕獲方式 | SharpPcap (WinPcap) | DLL Hook |
| 封包解密 | 需腳本定義 | 自動解密 (v83) |
| 腳本系統 | S# (ScriptDotNet) | 內建規則 |
| 開源 | ✅ | ✅ |
| 維護狀態 | 已停止 (2016) | 持續 |

### 3.5 MapleShark 腳本編寫教學

**完整 S# 函數列表**：


## 四、MaplePE — 32-bit MapleStory 封包編輯器 (2025 GitHub 新工具)

### 4.1 專案資訊

**GitHub**: https://github.com/zhyonc/MaplePE

**定位**：專為 MapleStory 設計的 32-bit 封包編輯器

### 4.2 核心功能

| 功能 | 說明 |
|------|------|
| 封包解析 | 自動解析封包結構 |
| 自訂封包發送 | 可發送自訂封包到客戶端 |
| 目標進程提取 | 從目標進程提取 InPacket 和 OutPacket 函數位址 |
| Opcode 過濾 | 過濾特定 Opcode 以最佳化分析 |
| 程式碼生成 | 從封包結構生成 C++/C# 程式碼 |

### 4.3 架構：MaplePE.exe + Packet.dll

MaplePE 由兩部分組成：
- **MaplePE.exe**：UI 主程式，UDP 伺服器接收封包資訊
- **Packet.dll**：注入目標進程，Hook 封包讀寫函數

### 4.4 需要 Hook 的函數位址

**接收端 (CInPacket)**：
| IDA 函數 | MaplePacketWriter 方法 | 讀取大小 |
|---------|----------------------|---------|
| `CInPacket::Decode1` | `mplew.write` | 1 byte |
| `CInPacket::Decode2` | `mplew.writeShort` | 2 bytes |
| `CInPacket::Decode4` | `mplew.writeInt` | 4 bytes |
| `CInPacket::DecodeStr` | `mplew.writeMapleAsciiString` | 可變 |
| `CInPacket::DecodeBuffer(x, 8)` | `mplew.writeLong` | 8 bytes |

**發送端 (COutPacket)**：
| IDA 函數 | MaplePacketWriter 方法 | 寫入大小 |
|---------|----------------------|---------|
| `COutPacket::Encode1` | `mplew.write` | 1 byte |
| `COutPacket::Encode2` | `mplew.writeShort` | 2 bytes |
| `COutPacket::Encode4` | `mplew.writeInt` | 4 bytes |
| `COutPacket::EncodeStr` | `mplew.writeMapleAsciiString` | 可變 |
| `COutPacket::MakeBufferList` | - | 構造緩衝區 |

**Socket 函數**：
| IDA 函數 | 用途 |
|---------|------|
| `CClientSocket::ProcessPacket` | 接收封包處理入口 |
| `CClientSocket::SendPacket` | 發送封包函數 |

### 4.5 使用流程

```
Step 1: 設定
  → 點擊 SettingView 圖示
  → 填入 CInPacket/COutPacket 函數位址
  → 儲存設定

Step 2: 啟動遊戲
  → 使用 launcher 啟動 MapleStory 客戶端
  → 每個現有 process 只注入一次

Step 3: 注入 DLL
  → 點擊 Patch 按鈕定位目標 process ID
  → 點擊 InjectDLL 注入 Packet.dll

Step 4: 分析封包
  → 選擇封包項目查看完整 bytes
  → 編輯 bytes 並發送到客戶端

Step 5: 發送自訂封包
  → 雙擊封包項目開啟 FormatView
  → 編輯數值或序列並發送
```

### 4.9 SendPacket 崩潰防護（BB+ 版本）

BB 版本後的 `CClientSocket::SendPacket` 有 hook 檢測。可用 Cheat Engine 將以下位址填為 `90` (NOP) 來禁用崩潰條件：

```
(int)&loc_7FFFFE + 4198402 範圍外的 retaddr → JUMPOUT(0)  // 第1次崩潰
retaddr == m_pLock → MEMORY[0](this, a2)                   // 第2次崩潰
retaddr == m_pLock → MEMORY[0]()                           // 第3~6次崩潰
```

## 九、Opcode 結構最新資源

### 9.1 MapleStory Reference Wiki — Packet 完整文檔

**URL**: https://mapleref.fandom.com/wiki/Packets

### 9.2 Opcode 定義基礎

- Opcode 是封包的第一個 2 bytes（v83）
- 使用 MapleCrypto 混淆，需 AES key 解密
- 不同私服可能自訂義 opcode 表

### 9.3 v83 封包格式速查

```
[Header: 4 bytes][Opcode: 2 bytes][Data: 變長]

Header 格式：
  [2 bytes: 封包長度][2 bytes: 混淆後 opcode]
```

### 9.4 登入相關 Opcode (v83)

| Opcode | 名稱 | 方向 | 說明 |
|--------|------|------|------|
| 0x01 | LOGIN_REQUEST | C → S | 登入請求 |
| 0x02 | LOGIN_RESPONSE | S → C | 登入回應 |
| 0x03 | CHECK_PASSWORD | C → S | 密碼驗證 |
| 0x04 | CHECK_PASSWORD_RESULT | S → C | 密碼驗證結果 |

### 9.5 地圖相關 Opcode (v83)

| Opcode | 名稱 | 方向 | 說明 |
|--------|------|------|------|
| 0x0B | CHANGE_MAP | C → S | 換地圖請求 |
| 0x0C | CHANGE_MAP_RESULT | S → C | 換地圖結果 |
| 0x2E | SPAWN_PORTAL | S → C | 生成傳送門 |

### 9.6 NPC 對話 Opcode (v83)

| Opcode | 名稱 | 方向 | 說明 |
|--------|------|------|------|
| 0x4F | NPC_TALK | S → C | NPC 對話 |
| 0x50 | NPC_TALK_MORE | C → S | NPC 繼續對話 |
| 0x51 | NPC_ACTION | C → S | NPC 動作 |
