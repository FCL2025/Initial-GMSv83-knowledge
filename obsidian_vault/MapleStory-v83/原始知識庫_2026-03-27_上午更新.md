### 3. Opcode結構

#### Send vs Recv Opcode 核心區分
> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."
> — MapleRef Wiki

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送，客戶端接收 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送，伺服器接收 | Client Opcode Set |

**重要**：同一個數字在不同方向代表不同含義！

#### MapleCrypto 頭部混淆公式
```java
// 加密端
int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
verify = (verify << 3) | (verify >>> 5);
verify &= 0xFFFF;

// 解密端
int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
```

#### CInPacket 封包接收完整流向
```
CClientSocket::ProcessPacket()
  → 讀取 4-byte MapleCrypto header
  → 解密提取 opcode (2 bytes) + length
  → switch(opcode)
    → CLogin::OnPacket()    // 登入畫面
    → CField::OnPacket()    // 遊戲中
```

#### IDA 中 Decode 函數對照
| IDA 函數 | Java 實現 | 讀取大小 |
|---------|-----------|---------|
| `Decode1` | `mplew.write(1)` | 1 byte |
| `Decode2` | `mplew.writeShort(1)` | 2 bytes |
| `Decode4` | `mplew.writeInt(1)` | 4 bytes |
| `Decode8` | `mplew.writeLong(1)` | 8 bytes |
| `DecodeStr` | `mplew.writeMapleAsciiString()` | 可變 |
| `DecodeBuffer(x, n)` | `mplew.write(n bytes)` | n bytes |

#### ✅ 相容寫法（所有音源通用）
```javascript
// ❌ 不相容 (JDK 7+ 已廢棄)
importPackage(Packages.client);

// ✅ 相容寫法 - 直接使用 cm. 函數即可
var status = 0;
function action(mode, type, selection) {
    if (mode == -1) { cm.dispose(); return; }
    if (mode == 1) status++; else status--;
    if (status == 0) { cm.sendNext("你好！"); }
    // ...
}
```

#### 狀態機核心
| Status | 意義 |
|--------|------|
| -1 | 初始調用 start() |
| 0 | 第一個 sendNext/sendSimple |
| 1 | 玩家回應後 |
| ... | 依此類推 |

| Mode | 意義 |
|------|------|
| 1 | 點擊「下一步/是」 |
| 0 | 點擊「返回」|
| -1 | 關閉對話框 |

#### NPC TYPE 欄位含義
| Type 值 | 行為 |
|---------|------|
| 0 | 普通對話 NPC |
| 1 | 商店 NPC |
