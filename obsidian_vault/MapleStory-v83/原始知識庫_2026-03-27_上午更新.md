#### MapleRef Wiki 封包標頭格式
> "Packet headers are 4 bytes long, and are generated with the use of the send IV, packet length (without header) and the version of MapleStory."

**格式**：
- 4 bytes: MapleCrypto 混淆頭部
- Opcode 通常為 2 bytes

#### 第一個封包（明文 Hello Packet）
- MapleStory 連線時，**第一個封包是明文**（未加密）
- 之後所有封包都經過 Shanda + AES 雙重加密
- 這是客戶端與伺服器交換加密金鑰（IV）的時機

### 3. Opcode結構

#### Send vs Recv Opcode 核心區分
> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."
> — MapleRef Wiki

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送，客戶端接收 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送，伺服器接收 | Client Opcode Set |

**重要**：同一個數字在不同方向代表完全不同含義！

#### 登入結果碼（Login Status Code）

| 結果碼 | 意義 |
|--------|------|
| 0 | 登入成功 |
| 5 | 帳號或密碼錯誤 |
| 7 | 帳號已停用 |
| 13 | 刪除中（冷靜期） |
| 23 | 登入太頻繁（防機器人延遲） |

#### CInPacket 封包接收完整流向
```
CClientSocket::ProcessPacket()
  → 讀取 4-byte MapleCrypto header
  → 解密提取 opcode (2 bytes) + length
  → switch(opcode)
    → CLogin::OnPacket()    // 登入畫面
    → CField::OnPacket()    // 遊戲中
```

#### CInPacket 類別成員函數對照 (RaGEZONE, 2016)

| C++ 函數 (IDA) | Java 實現 (伺服器端) | 讀取大小 |
|---------------|---------------------|---------|
| `Decode1` | `mplew.write(1)` | 1 byte (byte) |
| `Decode2` | `mplew.writeShort(1)` | 2 bytes (short) |
| `Decode4` | `mplew.writeInt(1)` | 4 bytes (int) |
| `Decode8` | `mplew.writeLong(1)` | 8 bytes (long) |
| `DecodeBuffer(x, n)` | `mplew.write(n bytes)` | n bytes |
| `DecodeStr` | `mplew.writeMapleAsciiString()` | 可變 |
| `DecodePos` | `mplew.writePos()` | 特殊：座標編碼 |

**來源**: MapleSolaxiaV2 README

> "DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate some kind of bugs afterwards."

**正確流程**：
```
HaRepacker 4.2.4 → 開啟客戶端WZ → UI修改 → 儲存 → "Private Server..."匯出 → 覆蓋伺服器XML
```

#### WZ XML 導出流程
```
1. HaRepacker → 開啟 WZ 檔案
2. 選擇 GMS (old) 加密格式
3. 導航到目標節點
4. 修改數值（禁止刪除/重新導入節點！）
5. 儲存
6. 選擇 "Private Server..." 導出為 XML
7. 覆蓋到伺服器的 WZ.XML 目錄
```

```javascript
function action(mode, type, selection) {
    if (mode == -1) { cm.dispose(); return; }
    if (mode == 1) status++; else status--;
    if (status == 0) { cm.sendNext("你好！"); }
    // ...
}
```

#### NPCConversationManager 核心方法速查

**對話發送**：
```javascript
cm.sendNext("文字")        // 顯示「下一步」
cm.sendYesNo("文字")        // 顯示「是/否」
cm.sendSimple("文字\r\n#L0#選項0#l")  // 選擇選單
cm.sendOk("文字")           // 顯示「確定」
cm.sendStyle("文字", styleId)  // 髮型選擇
cm.sendGetNumber("文字", default, min, max)  // 數字輸入
cm.sendGetText("文字")      // 文字輸入框
```

**玩家狀態**：
```javascript
cm.getLevel()              // 等級
cm.getJob()                // 職業
cm.getMesos()              // 楓幣
cm.haveItem(itemId, count) // 檢查物品
cm.getQuestStatus(questId) // 任務狀態 (0=未接, 1=進行中, 2=完成)
```

**玩家操作**：
```javascript
cm.gainItem(itemId, count)   // 給予物品
cm.gainMesos(amount)          // 給予/收取楓幣
cm.changeJob(jobId)           // 轉職
cm.startQuest(questId)        // 開始任務
cm.completeQuest(questId)     // 完成任務
cm.openShop(shopId)           // 開啟商店
cm.warp(mapId)                // 傳送地圖
cm.dispose()                  // 結束腳本
```
