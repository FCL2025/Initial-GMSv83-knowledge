## 🆕 v83 LAN 多人連線錯誤排查 (RaGEZONE, 2026-03 最新)

**URL**: https://forum.ragezone.com/threads/need-help-v83-lan-server-works-on-pc1-but-pc2-gets-error-2147467259.1262187/

### 錯誤碼分析

| 錯誤碼 | Hex 值 | 意義 | 根本原因 |
|--------|--------|------|---------|
| `-2147467259` | `0x80004005` | Access Denied | 防火牆/權限問題（非網路問題）|

### 排查檢查清單 (2026-03 更新)

```
□ 確認兩台 PC 在同一 LAN 網段
□ 確認伺服器 listen 0.0.0.0（而非 127.0.0.1）
□ 確認防火牆開啟 7575 和 8484 埠口
□ config.yaml HOST 設定為 LAN IP（不是 127.0.0.1）
□ WZ 檔案在兩台 PC 上完全一致
□ localhost.exe IP 已修改為正確 LAN IP
```

### 快速防火牆設定

```cmd
netsh advfirewall firewall add rule name="MapleStory Login" dir=in action=allow protocol=tcp localport=7575
netsh advfirewall firewall add rule name="MapleStory Game" dir=in action=allow protocol=tcp localport=8484
```

### 端口配置要點

- Login Port: 8484
- Channel 1: 7575
- Channel 2: 7576
- ... 依此類推

> ⚠️ 若錯誤碼是 0x80004005，這是防火牆/權限問題，不是網路連線問題

## 🆕 v83 新版本物品導入技術 (RaGEZONE, 2025-12)

**URL**: https://forum.ragezone.com/threads/maplestory-v83-newer-items-import.1256083/

### 問題描述

將高版本物品導入 v83 時，物品顯示正常但**無法裝備**。

### 解決方案：Reskin 法

```
1. HaRepacker 開啟高版本 WZ（如 v176 Character.wz）
2. 找到目標物品（如 Death's Scythe, ID: 1702565）
3. Export 為 XML
4. 在 v83 Character.wz 中選擇一個閒置物品（如 Long Sword 1302007）
5. 用文本編輯器將高版本內容覆蓋到 v83 物品結構中
6. Import 回 v83
7. 測試裝備
```

> **核心思想**：不改變物品 ID，只改變物品的視覺和屬性內容。

### 為什麼無法裝備？

| 原因 | 說明 |
|------|------|
| 物品類型不相容 | 高版本物品類型在 v83 不存在 |
| 職業限制 | v83 職業系統與新版本不同 |
| 屬性欄位差異 | 屬性數量或類型不匹配 |

## 🆕 登入器/客戶端修改要點 (v83)

### STREDIT IP 修改流程

```
1. 下載並開啟 STREDIT
2. File → Open → 選擇 localhost.exe
3. 當詢問「是否 v118+」→ 選「No」（v83 適用）
4. 當詢問「IP 搜尋模式」→ 選「Yes」
5. 點擊「Force to a IP」按鈕
6. 輸入目標 IP（如 127.0.0.1 或 VPS IP）
7. 保存修改後的 localhost.exe
```

### Hex Editor IP 修改

- `127.0.0.1` → Hex: `31 37 32 2E 30 2E 30 2E 31`
- 需修改 3 處 IP 位址

## 🆕 WZ 編輯重要原則

### 禁止事項

> **禁止**使用伺服器的 XML 重新導入客戶端 WZ

### 正確流程

```
1. HaRepacker 4.2.4 + GMS (old) 加密
2. 使用 UI 直接編輯客戶端 WZ
3. 保存到客戶端
4. 導出 XML 到伺服器端 WZ.XML
```

### WZ 匹配的重要性

> "This is not a purely v83 emulator (cannot directly use Nexon/Wizet's original WZ files). You must use the provided custom WZ files."

**原因**：伺服器端的 wz.xml 和客戶端的 WZ 必須完全匹配，否則 CRC 校驗失敗。

## 🆕 Opcode 結構核心概念

### v83 Opcode 特性

| 特性 | 說明 |
|------|------|
| **格式** | 2 bytes (little endian) |
| **加密** | MapleCrypto + AES-OFB + Shanda |
| **映射** | 幾乎全部被映射 |
| **工具支援** | SpiritIDAPlugin + MaplePacketPuller |

### 封包頭部格式

```
[封包頭部 (4 bytes)][Opcode (2 bytes)][資料...]

Header 格式：
[2 bytes: 封包長度][2 bytes: 混淆後的 Opcode]
```

### 腳本位置

| Quest NPC | `scripts/quest/` |

### 傳統 status 寫法（Cosmic, HeavenMS）

```javascript
var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1 || mode == 0) {
        cm.dispose();
        return;
    }
    
    if (mode == 1) {
        status++;
    } else {
        status--;
    }
    
    switch(status) {
        case 0:
            cm.sendNext("你好！");
            break;
        case 1:
            cm.sendSimple("選擇:\n #b#L0#購買#l\n #L1#出售#l");
            break;
    }
}
```
