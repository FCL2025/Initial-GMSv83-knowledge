### 客戶端修改流程 (HxD Hex Editor)

**核心步驟**：
```
1. 使用 HxD 打開 GMSv83.exe
2. 搜尋 "127.0.0.1"（所有實例）
3. 替換為目標伺服器 IP（如 192.168.1.100）
4. 儲存檔案
```

**LAN 多人連線關鍵**（RaGEZONE, 2026-03）:
- 將所有 `127.0.0.1` 替換為伺服器 LAN IP
- 確認兩台 PC 在同一網段
- 防火牆需開放 7575 (login) 和 8484 (game)

**最新錯誤排查**：
| 錯誤碼 | Hex 值 | 意義 | 解決方案 |
|--------|--------|------|---------|
| `-2147467259` | `0x80004005` | Access Denied | 防火牆/權限問題 |

### localhost.exe IP Redirector 使用方式

```
localhost.exe [目標IP] [目標端口]

範例：
localhost.exe 127.0.0.1 8480
→ 監聽 port 8480，轉發到 127.0.0.1:8484
```

|-------------|---------|------|
| `Decode1` | `mplew.write` | 寫入 1 byte |

### MapleRef Wiki Send/Recv Opcode 核心概念

| 方向 | 說明 |
|------|------|
| **S→C (Send)** | 伺服器發送到客戶端 |
| **C→S (Recv)** | 客戶端發送到伺服器 |

**重要**：同一個 Opcode 數字可能代表不同含義，視方向而定！

## 🛠️ 4. WZ修改 - 實戰技巧和注意事項

### HaRepacker WZ 編輯流程

```
1. 使用 HaRepacker 打開 WZ 檔案
   → 選擇 "GMS (old)" 加密格式（v83 適用）

2. 直接在 UI 中修改數值
   ⚠️ 禁止刪除/重新導入節點！可能造成數據丟失

3. 儲存 WZ

4. 若需導出 XML：
   → 選擇 "Private Server..." 格式導出
   → 覆蓋到伺服器的 WZ.XML 目錄
```

### WZ 編輯黃金法則

> **DO NOT USE** the server's XMLs for reimporting into the client's WZ, it **WILL** generate bugs.

### Quest Level Restrictions 移除 (RaGEZONE, July 2025)

**Notepad++ Regex 批量移除 lvmax**：
```
尋找：lvmax="\d+"\s*
取代：（留空）
```

### 客戶端 vs 伺服器 WZ 差異

| WZ 來源 | 用途 | 編輯方式 |
|---------|------|---------|
| 客戶端 WZ | 顯示資源（圖片、字串） | HaRepacker 直接修改 |
| 伺服器 WZ | 遊戲邏輯（數值、腳本） | XML 修改後覆蓋 |

### ✅ 相容寫法（所有音源通用）

```javascript
// 萬用寫法 - 適用所有音源
var status = 0;

function start() {
    cm.sendNext("你好！");
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
        return;
    }
    if (mode == 1) status++;
    else status--;

    if (status == 0) {
        cm.sendSimple("選擇：\r\n#b#L0#選項1#l\r\n#L1#選項2#l");
    } else if (status == 1) {
        if (selection == 0) {
            cm.gainItem(1302000, 1);
            cm.sendOk("獲得物品！");
        }
        cm.dispose();
    }
}
```

### ❌ 不相容寫法（JDK 7+ 已廢棄）

```javascript
// ❌ 不相容 (JDK 7+ 已廢棄)
importPackage(Packages.client);
importPackage(Packages.server);

// ✅ 相容寫法 - 完全不要使用 importPackage
```

### NPC TYPE 欄位含義

| Type 值 | 客戶端行為 | 腳本需求 |
|---------|-----------|---------|
| `0` | 普通 NPC，點擊後發送 NPC_TALK 封包 | 需要 `.js` 腳本 |
| `1` | 商店 NPC，點擊後直接開啟商店介面 | 可能需要 `.js` 腳本 |
| `2~6` | 特殊功能 NPC（拍賣、保險箱等） | 不需要腳本 |

### Quest NPC MigrateIn 封包機制

**問題**：如何讓 Quest NPC 出現在 "Other" 選項中？

**解決方案**：
- 需要發送 `PlayerMigrateInPacket` 封包
- 封包包含：NPC ID 列表 + FileTime
- 這個封包讓客戶端知道哪些 NPC 有自訂腳本
