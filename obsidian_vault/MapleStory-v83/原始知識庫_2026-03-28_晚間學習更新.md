### 1. MapleSolaxiaV2 README 完整架設流程（GitHub 直接抓取）

**來源**: https://github.com/skhzhang/MapleSolaxiaV2

MapleSolaxiaV2 是截至 2026-03 仍被廣泛參考的 v83 開源專案。以下是 README 原文精華：

#### 客戶端安裝完整步驟
```
1. 使用 ManagerMsv83.exe 安裝 MapleStory 到 C:\Nexon\MapleStory
2. 刪除以下檔案：
   - HShield (資料夾)
   - ASPLauncher.exe
   - MapleStory.exe
   - patcher.exe
3. 用 client_wz 資料夾中的 WZ 檔案覆蓋原版
4. 用 localhost.exe 替換
5. 若 IP 不是 127.0.0.1 → Hex Editor 修改 localhost.exe
```

#### 伺服器架設完整步驟
```
1. 安裝 JDK 7 或 8（必須）
2. 安裝 MySQL Server 5.5
3. 安裝 JCE Unlimited Strength Policy（覆蓋 6 個位置！）
4. NetBeans 8.0.2 開啟專案
5. 執行 SQL：
   - db_database.sql（建立資料庫結構）
   - db_drops.sql（怪物掉落資料）
   - db_shopupdate.sql（可選，更新商店商品）
6. 在 accounts 資料表手動建立第一個帳號
7. 設定 configuration.ini 中的 IP
8. Clean and Build → launch.bat 啟動
```

#### WZ 編輯黃金法則
> "DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate some kind of bugs afterwards."

**正確流程**：
1. HaRepacker 4.2.4 + **GMS (old)** 加密開啟客戶端 WZ
2. **使用 UI 直接修改數值**（禁止刪除/重新導入節點！）
3. 保存到客戶端
4. 在 HaRepacker 中選擇 **"Private Server..."** 導出 XML
5. 將導出的 XML 覆蓋到伺服器的 wz.xml 目錄

#### 為什麼 MapleSolaxiaV2 不能用原廠 WZ？
> "This is not a purely v83 emulator (cannot directly use Nexon/Wizet's original WZ files). You must use the provided custom WZ files."

原因：MapleSolaxiaV2 的伺服器端對 WZ 結構有自訂義修改，必須配套使用。

### 2. RustMS — 2026-03 重啟，AES-OFB + Shanda 完整加密實現

**GitHub**: https://github.com/neeerp/RustMS

#### 專案現況（2026-03 重啟）
- 使用 AI coding assistant 輔助開發
- 目前僅有 Login Server（監聽 localhost:8484）
- 大量遊戲邏輯待實現
- 非 JVM 方案（Rust 編譯後無需 JVM）

#### 完整加密棧（RustMS 實作）
| 層次 | 演算法 | 說明 |
|------|--------|------|
| 第1層 | MapleCrypto | Header 混淆（版本特定 XOR + 移位） |
| 第2層 | Shanda | 對稱加密（v83 早期版本殘留） |
| 第3層 | AES-OFB | 主要加密，256-bit key，4-byte IV |

#### RustMS 握手流程
```
1. 客戶端連線 → 發送版本號 (明文)
2. 伺服器驗證版本 → 回傳 serverIV (明文)
3. 客戶端生成 clientIV → 回傳 (明文)
4. 建立加密會話
5. 之後所有封包：MapleCrypto → Shanda → AES-OFB
```

**URL**: https://pkg.go.dev/github.com/defaultmagi/maplelib

```

// JavaScript 基本寫法 - 適用所有音源
var status = 0;

function start() {
    cm.sendNext("你好！");
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
        return;
    }
    if (mode == 1) status++;
    else status--;

    if (status == 0) {
        cm.sendSimple("選擇：\r\n#b#L0#選項1#l\r\n#L1#選項2#l");
    } else if (status == 1) {
        if (selection == 0) {
            cm.gainItem(1302000, 1);
            cm.sendOk("獲得物品！");
        }
        cm.dispose();
    }
}
```

### 6. WZ↔XML 同步工作流（HaRepacker 最佳實踐）

#### MapleSolaxiaV2 README 原文強調
> "DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate some kind of bugs afterwards."

#### 正確流程（MapleSolaxiaV2 + RaGEZONE 2023-11）
```
1. HaRepacker 4.2.4 + GMS (old) 加密
   → 開啟客戶端 WZ（如 Character.wz）

2. 使用 UI 直接修改數值
   ⚠️ 禁止刪除/重新導入節點！可能造成數據丟失

3. 保存到客戶端 WZ

4. 在 HaRepacker 中選擇 "Private Server..."
   → 導出 XML 到伺服器 wz.xml 目錄

5. 覆蓋伺服器端對應的 XML 檔案
```

#### 為什麼不能直接用伺服器 XML？
- 伺服器 wz.xml 經過特定處理，可能包含客戶端不需要的額外資料
- 直接混用會導致 CRC 校驗失敗或客戶端崩潰
- WZ 和 XML 結構略有差異

#### Community WZ/IMG Repack for v83（RaGEZONE 2023-11）
**來源**: https://forum.ragezone.com/threads/community-wz-img-repack-for-v83.1221835/

HaRepacker 導出時選擇 **"Private Server"** 格式，可確保：
- XML 格式相容伺服器 wz.xml
- 精簡掉客戶端特定的 sprite 資料
