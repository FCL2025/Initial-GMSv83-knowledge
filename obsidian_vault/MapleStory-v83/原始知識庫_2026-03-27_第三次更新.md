## 🌟 HeavenClient — 開源 MapleStory v83 自訂客戶端 (ryantpayton, GitHub 2026)

**GitHub**: https://github.com/ryantpayton/MapleStory-Client

### 專案定位
HeavenClient 是 HeavenMS 官方出品的**從零開始重寫**的 MapleStory v83 客戶端，專為配合 HeavenMS 伺服器使用。

### 技術架構
| 組件 | 技術 |
|------|------|
| 平台 | Windows (VS 2026, SDK 8.1, Toolset v140) |
| 圖形 | GLFW3, GLEW, FreeType |
| 音頻 | Bass |
| 網路 | Asio (可選) |
| 資源格式 | NX (NoLifeNx) 或 WZ |
| 加密 | AES + MapleCrypto |
| 構建 | CMake + Visual Studio |

### 版本相容性
| 平台 | 客戶端 | 測試版本 |
|------|--------|---------|
| 標準 | HeavenClient (主分支) | v229.2 + HeavenMS |
| Switch | HeavenClientNX | - |
| Linux | HeavenClient (linux 分支) | - |
| Web | MapleStory WASM | - |

### 核心功能表 (MapleStory.h)
| 巨集 | 說明 |
|------|------|
| `USE_ASIO` | 使用 Asio 替代原生 Winsock |
| `USE_CRYPTO` | 啟用加密（伺服器通訊） |
| `USE_NX` | 使用 NX 格式替代 WZ |
| `USE_DEBUG` | 抑制 Settings 檔案生成 |

### 建置流程
```
1. Visual Studio 2026 Community Edition (v18.2.1)
2. Windows SDK 8.1
3. Platform Toolset v140
4. Open MapleStory.sln → Build > Build Solution (Ctrl+Shift+B)
5. Run: Debug > Start Debugging (F5)
```

### 必須檔案
- 所有 WZ 檔案需從 v229.2 官方客戶端轉換為 NX 格式
- 放在執行檔的父目錄
- 詳細轉換說明：NoLifeWzToNx (GitHub)

### Settings 檔案
- 首次執行後自動生成
- 編輯 `MapleStory.h` 或 `Settings` 檔案效果相同
- 刪除 `Settings` 會回覆預設值

### 必備 NX 檔案
詳見 `NxFiles.h` 完整列表

### Dependencies
| 類別 | 函式庫 |
|------|--------|
| NX/WZ | NoLifeNx |
| 圖形 | GLFW3, GLEW, FreeType |
| 音頻 | Bass |
| 網路 | Asio (可選) |

### RustMS 搭配 HeavenClient
RustMS (https://github.com/neeerp/RustMS) 可完全支援 HeavenClient，兩者結合實現：
- 從零開始的 Rust 登入伺服器
- C++ 重寫的客戶端
- 全程開源，無需 JVM

**建置 RustMS 客戶端**：
```bash
cd HeavenClient
./build-deps.sh
cmake -S . -B cmake-build
cmake --build cmake-build -j"$(nproc)"
```

## 🌟 MapleSolaxiaV2 最新 v83 架設指南 (RaGEZONE 2025-11)

**URL**: https://forum.ragezone.com/threads/video-guide-how-to-make-a-maplestory-v83-private-server-%E2%80%93-full-setup-tutorial.1243566/

### 環境架設摘要
```
Step 1: 安裝 JDK 8 + JCE 8
Step 2: 安裝 MySQL Server 5.5
Step 3: 下載 v83 localhost 客戶端
Step 4: 使用 HxD Hex 修改客戶端 IP (127.0.0.1 → 目標 IP)
Step 5: 架設 Java 伺服器 (NetBeans / IntelliJ)
Step 6: 執行 SQL 初始化資料庫
Step 7: 啟動測試
```

### LAN 多人連線必檢查清單
- [ ] 確認兩台 PC 在同一 LAN 網段
- [ ] 確認伺服器 listen 0.0.0.0（而非 127.0.0.1）
- [ ] 確認防火牆開啟 7575 和 8484 埠口
- [ ] 錯誤碼 0x80004005 → 防火牆/權限問題
- [ ] 錯誤碼 -2147467259 → 同上

### 錯誤碼排查對照
| 錯誤碼 | Hex | 原因 |
|--------|-----|------|
| Access Denied | 0x80004005 | 防火牆阻擋或權限不足 |
| 連線失敗 | -2147467259 | 防火牆/網路設定問題 |
| 版本不符 | - | 客戶端與音源版本不匹配 |

## 📋 本次學習重點摘要

### 1. 登入器/客戶端修改
- **HeavenClient**: 從零重寫的開源客戶端，支援 v229.2 + HeavenMS
- **MapleEzorsia v2**: dinput8.dll 方案，自動完成所有修改
- **HxD**: 傳統 Hex 修改 IP 方法，仍是主流
- **RustMS + HeavenClient**: 從零開始的非 JVM 方案

### 2. 抓包分析
- **Maple_Pshark**: v83 即時封包攔截工具，Hook 加密函數
- **MaplePacketPuller + SpiritIDAPlugin**: IDA Pro 離線分析
- **Wireshark**: tcp.port == 7575 || tcp.port == 8484

### 3. Opcode 結構
- **v83 已是高度成熟版本**: Opcode 幾乎全部被映射
- **MapleCrypto**: 頭部混淆 (4 bytes header)
- **Shanda + AES-OFB**: 三層加密架構
- **HeavenMS/Cosmic**: 最完整的 Opcode 實現參考

### 4. WZ 修改
- **HaRepacker-resurrected**: 官方推薦 WZ 編輯器
- **WZ → NX**: NoLifeWzToNx 轉換工具
- **lvmax removal**: Notepad++ Regex 批量處理
- **NPC.wz**: 自訂 NPC 需要 WZ + JS 腳本配合

### 5. NPC 腳本編寫
- **Nashorn**: JDK 8+ 標準 JavaScript 引擎
- **ES6 支援**: Cosmic/IntransigentMS 完全支援
- **importPackage**: 已廢棄，改用 `Java.type()` 或完整路徑
- **cm. 函數**: 最通用的相容寫法
- **配方系統**: 動態商店腳本，無需 SQL
