### 技術原理
MaplePE 由兩部分組成：
- **MaplePE.exe**：UI 主程式，UDP 伺服器接收封包資訊
- **Packet.dll**：注入目標進程，Hook 封包讀寫函數

### 常見注入失敗原因
| 錯誤 | 原因 | 解決方案 |
|------|------|---------|
| Permission denied | 權限不足 | 以系統管理員身份執行 MaplePE.exe |
| Missing Packet.dll | 安全軟體刪除 | 暫時關閉安全軟體 |
| Injection protection | Launcher 有記憶體保護 | 使用其他方式啟動遊戲 |

## 🆕 MapleShark ScriptDotNet (S#) 腳本引擎

**GitHub**: https://github.com/diamondo25/MapleShark

MapleShark 的腳本引擎使用 **ScriptDotNet (S#)**，用於自訂封包解析：

```csharp
// ScriptDotNet 語法範例
byte AddByte(string name)        // 添加 1 byte 無符號欄位
short AddShort(string name)      // 添加 2 bytes 有符號欄位
int AddInt(string name)         // 添加 4 bytes 整數欄位
float AddFloat(string name)     // 添加 4 bytes 浮點數欄位
string AddPaddedString(string name, int length)  // 固定長度字串
void StartNode(string name)     // 建立子節點（直到 EndNode）
void EndNode(bool expand)       // 結束節點
void Write(string file, string line)  // 追加文字到檔案
int Remaining()                 // 返回剩餘未處理位元組數
```

**使用範例**（解析登入封包）：
```csharp
int opcode = AddShort("Opcode");
string username = AddPaddedString("Username", 13);
string password = AddPaddedString("Password", 13);
```

## 🆕 RustMS — Rust 實現 MapleStory v83 登入伺服器（2026-03 重啟）

**GitHub**: https://github.com/neeerp/RustMS

### 專案特點
- **2026 年 3 月重啟**，使用 AI coding assistant 輔助開發
- 目前僅包含基本可用的 **Login Server**（監聽 localhost:8484）
- 完全使用 Rust 語言從頭編寫
- 支援加密協商（不需要禁用加密）
- 特色：編譯後無需 JVM，記憶體佔用更少

### RustMS 加密實現
- `MapleAES (OFB 模式)` - 負責封包加密
- `Shanda 加密` - 對稱加密演算法
- `bcrypt` - 密碼雜湊

### RustMS 登入流程
```
1. 客戶端連線到 localhost:8484
2. 伺服器發送 handshake（含版本號 + IV）
3. 客戶端回應（更換 IV）
4. 之後所有通訊使用 Maplestory 自訂加密 + AES
```

### RustMS 教學價值
1. 清晰的代碼結構（Rust ownership 模型）
2. 完整加密實現開源
3. 非 JVM 方案
4. 學習現代 Rust（async/await、ORM、密碼學）
