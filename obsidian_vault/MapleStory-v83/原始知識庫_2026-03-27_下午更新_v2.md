## 四、Opcode 結構

### 4.1 MapleCrypto 頭部混淆 — 核心公式（2026-03 更新）

#### 加密端（發送封包）
```java
public byte[] getPacketHeader(int opcode, int length, int iv) {
    int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
    verify = (verify << 3) | (verify >>> 5);
    verify &= 0xFFFF;
    return new byte[] {
        (byte) (verify & 0xFF),
        (byte) ((verify >>> 8) & 0xFF),
        (byte) (length & 0xFF),
        (byte) ((length >>> 8) & 0xFF)
    };
}
```

#### 解密端（接收封包）
```java
public int retrieveOpcode(byte[] header, int iv) {
    int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
    verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
    return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
}

public int retrieveLength(byte[] header, int iv) {
    int length = (header[2] & 0xFF) | ((header[3] << 8) & 0xFF00);
    length ^= (iv >> 16) & 0xFFFF;
    return length;
}
```

### 4.4 Send vs Recv Opcode 核心區分（2026-03 整理）

> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送，客戶端接收 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送，伺服器接收 | Client Opcode Set |

**⚠️ 重要**：同一個 Opcode 數字在不同方向代表**完全不同**的含義！

#### 登入相關 Opcode 對照
| Opcode (Hex) | S→C 名稱 | C→S 名稱 |
|-------------|---------|---------|
| 0x00 | LOGIN_STATUS（登入結果） | — |
| 0x01 | — | LOGIN_PASSWORD（登入請求） |
| 0x02 | ACCEPT_LOGIN | — |
| 0x03 | CHARLIST（角色列表） | — |
| 0x06 | — | SERVERLIST_REQUEST |
| 0x07 | — | CHARLIST_REQUEST |
| 0x08 | — | CHAR_SELECT |
| 0x0B | — | PLAYER_DROP |

### 4.5 IDA 分析 Opcode 進階技巧

#### IDAPython 相對位址計算（E8 Call 指令）
```
指令格式: E8 C5 90 D8 FE (Call near)
計算步驟:
1. 取後4位元組: C5 90 D8 FE
2. 反轉位元組序: FED890C5
3. 轉2's complement: ~0xFED890C5 + 1 = 0x01276F3B
4. 目標地址 = 當前地址 - 0x01276F3B
```

### 6.4 動態 NPC 商店腳本（v83, RaGEZONE 2016）

**來源**: https://forum.ragezone.com/threads/dynamic-npc-shop-script-v83.1089979/

#### 傳統商店 vs 動態商店

| 方案 | 優點 | 缺點 |
|------|------|------|
| SQL 資料庫商店 | 統一管理 | 需重啟伺服器才能更新 |
| 動態 NPC 腳本商店 | 即時更新，不需重啟 | 需編寫 JavaScript 腳本 |

#### 動態 NPC 商店腳本範例

```javascript
var items = [
    [1302000, 1000, 1],   // [物品ID, 價格, 數量]
    [1302001, 2000, 1],
    [1302002, 3000, 1],
    [1302003, 4000, 1]
];

var status = -1;

function start() {
    status = 0;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
        return;
    }
    
    if (mode == 0) {
        status--;
        if (status == -1) {
            cm.dispose();
            return;
        }
    }
    
    if (mode == 1) {
        status++;
    }
    
    if (status == 0) {
        var shopMenu = "歡迎來到我的商店！\r\n";
        for (var i = 0; i < items.length; i++) {
            shopMenu += "#L" + i + "# #z" + items[i][0] + "# - " + items[i][1] + " 楓幣#l\r\n";
        }
        cm.sendSimple(shopMenu);
    } else if (status == 1) {
        var selectedItem = items[selection];
        if (cm.getMesos() >= selectedItem[1]) {
            cm.gainItem(selectedItem[0], selectedItem[2]);
            cm.gainMesos(-selectedItem[1]);
            cm.sendOk("購買成功！");
        } else {
            cm.sendOk("楓幣不足！");
        }
        cm.dispose();
    }
}
```
