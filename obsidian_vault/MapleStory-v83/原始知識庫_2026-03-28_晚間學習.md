## 🆕 v83 客戶端修改：localhost.exe Hex Edit 詳解 (MapleSolaxiaV2 README 原文)

### 為什麼要修改 localhost.exe？

v83 客戶端默認連接到官方伺服器。localhost.exe 是修改過的啟動器，連接到本地私人伺服器（127.0.0.1）。

### HxD Hex Edit 步驟

```
目標：將 localhost.exe 中的 "127.0.0.1" 替換為你的伺服器 IP

工具：HxD Hex Editor（免費）或 Neo Hex Editor

步驟：
1. 用 HxD 打開 localhost.exe
2. 搜尋 "127.0.0.1"（作為 Text String）
3. 找到後，確認周圍的 bytes
   - 通常有 3 處需要修改
4. 全部替換為你的 IP（如 "192.168.1.100"）
5. 儲存

注意：
- IP 字串長度不同時，會影響其他資料
  （可用 127.0.0.1 作為墊字，或重新調整偏移）
- 建議先備份原始檔案
```

### 替換後驗證

替換完成後，重新開啟 localhost.exe，確認：
1. 連線到正確的伺服器 IP
2. login.bat 中的 port 設定正確（7575 / 8484）

### localhost.exe 找不到 "127.0.0.1" 的原因

某些版本使用加密或封包頭部，需要先確認：
```
1. 用十六進位視圖搜尋：31 32 37 2E 30 2E 30 2E 31
   （127.0.0.1 的 ASCII hex）
2. 若找不到，確認 IP 格式是否為別的形式
```

| 函數 | 用途 |

| `COutPacket::Encode2` | 寫入 2 bytes |

### 使用流程（原文）

```
Step 1: 設定
  → 點擊 SettingView 圖示
  → 填入函數位址並儲存

Step 2: 啟動遊戲
  → 使用 launcher 啟動遊戲客戶端
  → 每個現有 process 只注入一次

Step 3: 注入
  → 點擊 Patch 按鈕定位目標 process ID
  → 點擊 InjectDLL 注入 Packet.dll

Step 4: 分析
  → 選擇封包項目查看完整 bytes
  → 編輯 bytes 並發送到客戶端

Step 5: 發送自訂封包
  → 雙擊封包項目開啟 FormatView
  → 編輯數值或序列並發送
```

## 🆕 CClientSocket::ProcessPacket 封包分發流程 (RaGEZONE, 2019)

**來源**: https://forum.ragezone.com/threads/how-does-one-find-a-structure-of-a-packet-via-ida.1162654/

### 封包分發架構

所有客戶端接收封包都經過以下函數鏈：

```
CClientSocket::ProcessPacket
    ↓
根據 opcode header 分發到:
    ├── CLogin::OnPacket         (登入相關 opcode)
    ├── CWvsContext::OnPacket    (warp2map 之前的 opcode)
    ├── CField::OnPacket         (遊戲中所有其餘 opcode)
    └── CCashShop::OnPacket      (現金商店 opcode)
```

### 為什麼了解分發流程重要？

1. **除錯**：知道在哪個函數中斷點
2. **理解封包來源**：確認某個功能是由哪個模組處理
3. **客戶端修改**：修改 UI 或去除限制時，知道該找哪個函數

### v83 函數位址參考

| 函數 | 說明 |
|------|------|
| `CClientSocket::ProcessPacket` | 封包分發入口（所有封包都經過這裡） |
| `CLogin::OnPacket` | 登入相關 opcode 的處理 |
| `CWvsContext::OnPacket` | warp 前的一些全域 opcode |
| `CField::OnPacket` | 遊戲中所有其餘 opcode（最大範圍） |
| `CCashShop::OnPacket` | 現金商店 opcode |

### 分析方法

**靜態分析**：
1. IDA 中找到 `CClientSocket::ProcessPacket`
2. 觀察 switch-case 跳轉表
3. 每個 case 對應一個 opcode

**動態分析**：
1. 在 `CInPacket::Decode2` 設斷點
2. 每次讀取 opcode 時觀察值
3. 配合 CField/CLogin OnPacket 交叉引用確認

## 🆕 IDA 封包結構分析：IDAPython 指令碼 (RaGEZONE, 2015)

**來源**: https://forum.ragezone.com/threads/idapython-show-all-address-xref-from-coutpacket-with-value.1060540/

### COutPacket Opcode 分析技巧

**Opcode E8 (CALL) 分析示例**：

```
Address 0x168B8F6: E8 C5 90 D8 FE
  = Call from address 004149C0

計算方式：
1. E8 = Call near amd relative call (read next 4 byte then do 2's complement)
2. 後 4 bytes: C5 90 D8 FE
3. Reverse: FED890C5
4. 2's complement: 0xFED890C5 xor 0xFFFFFFFF = 0x01276F3A
   + 1 = 0x01276F3B
5. 減去當前位址: 0x168B8F6 - 0x01276F3B = 0x4149BB
6. 加上 5 bytes (E8 + 4 bytes): 0x4149BB + 5 = 0x4149C0 ✓
```

### IDAPython 快速指令

```python
import idautils
import idc

# 找出所有 COutPacket 的交叉引用
for xref in XrefsTo(idc.get_name_ea_simple("COutPacket__Encode")):
    print(f"Reference at: {hex(xref.frm)}")

# 列出所有 Decode 函數
for func in idautils.Functions():
    name = idc.get_func_name(func)
    if "Decode" in name:
        print(f"{hex(func)}: {name}")
```

## 🆕 v83 Opcode 結構總結 (MapleRef + RaGEZONE 2026)

### 封包格式

```
[Header (4 bytes)][Opcode (2 bytes)][Data (變長)]
```

- Header: MapleCrypto 混淆頭部（包含長度 + 加密 opcode）
- Opcode: 2 bytes (v83)，little endian
- Data: 變長，視封包類型而定

### 主要 Opcode 流向

| 方向 | 說明 |
|------|------|
| C → S | 客戶端發送到伺服器（CLIENT_SEND） |
| S → C | 伺服器發送到客戶端（SERVER_SEND） |

### v83 Opcode 特性

| 特性 | 說明 |
|------|------|
| 大小 | 2 bytes (早期版本為 1 byte) |
| 順序 | Little endian（低位在前） |
| 加密 | MapleCrypto 混淆 + AES-OFB |
| 文檔化程度 | 幾乎全部被映射 |

### CClientSocket 封包分發

```
所有 S→C 封包都經過：
CClientSocket::ProcessPacket
    ├── CLogin::OnPacket         (登入相關)
    ├── CWvsContext::OnPacket    (warp 之前)
    ├── CField::OnPacket         (遊戲中大部分)
    └── CCashShop::OnPacket      (現金商店)
```
