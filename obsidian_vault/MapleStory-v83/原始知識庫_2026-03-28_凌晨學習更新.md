## 一、重點更新摘要

### 1. MapleEzorsia-v2 最新動態 (GitHub 2026)

**設計哲學**：
- 完全相容任何客戶端或 localhost
- **不需要刪除原廠任何檔案**
- 僅需放入 `dinput8.dll` 即可
- x86 平台限定

**config.ini 警告**：
> "try to avoid extending the comments (behind ;comment) inside of it or INIReader will be unable to read from it and crash you"

### 2. MaplePE 技術架構 (GitHub 2026-03)

**技術棧**：
| 組件 | 技術 |
|------|------|
| UI | C++ MFC + MFC-GridCtrl |
| 網路 | WinSock2 |
| Hook | Detours (微軟官方) |
| 編譯 | Visual Studio 2022 (x86 & x64) |

### 3. RustMS 專案重啟 (2026-03)

專案狀態：
- 最初為學習 Rust 而建立，後被放棄
- **2026 年 3 月重啟**，使用 AI coding assistant 輔助
- 目前僅包含基本可用的 **Login Server**（監聽 localhost:8484）
- 仍在早期階段

### 4. WZ 編輯 — HaRepacker v8.0.0 革命性功能 (2026-03-05)

- **Hot-swap for .img files**: 可直接拖放替換遊戲數據

- **Git 友好**: 提取後易於 diff、review 和版本控制
- **AI 地圖編輯器**: 對話式迭代編輯

## 二、學習重點複習

### 登入器/客戶端修改
```
三大方案：
1. HxD Hex 編輯 → 傳統但有效
2. Ezorsia v2 DLL → 2026年最新，免修改
3. MapleClientEditTemplate → 可自訂框架
```

### 抓包分析
```
工具鏈：
Wireshark → Maple_Pshark → MaplePacketPuller → MapleRef Wiki
         → MaplePE (可發送自訂封包)
```

### Opcode結構
```
三層加密：MapleCrypto → Shanda → AES-OFB
第一個封包永遠是明文（Hello Packet）
```

### WZ修改
```
黃金法則：編輯客戶端 WZ → 測試 → 導出 Private Server XML → 覆蓋伺服器
千萬不要：直接用伺服器 XML 覆蓋客戶端 WZ
```

### NPC腳本編寫
```
✅ 使用 cm. 函數
❌ 避免 importPackage（JDK 7+ 已廢棄）
✅ Nashorn ES6 寫法（Cosmic / IntransigentMS）
```
