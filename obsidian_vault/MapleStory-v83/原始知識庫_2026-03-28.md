## 一、登入器/客戶端修改 — 本次新獲取

### MapleEzorsia-v2 GitHub 原始 README 重點（2026-03-28 抓取）

**GitHub**: https://github.com/444Ro666/MapleEzorsia-v2

**原始目標**：讓 4:3 的老遊戲能在 16:9 現代螢幕上運行，同時盡可能保留原始遊戲體驗。

**設計原則**：
- 這是一個**獨立的 DLL 客戶端**
- 可與**任何客戶端或 localhost 搭配使用**
- 包括預設安裝的原廠客戶端
- **不需要刪除原廠任何檔案**
- 僅需放入 dinput8.dll 即可

**技術實現**：
- 對原廠、封裝的 v83 MapleStory 客戶端進行記憶體位址修改
- 修改遊戲視窗和畫布解析度為 HD
- 遊戲內元件邊界也會随自定義解析度縮放

**已知限制**：
- 只能使用 **x86 平台**編譯
- 不支援 x64

**編譯設置參考**：
- Platform: x86
- 編譯設置截圖: https://www.mediafire.com/view/9ssrqg8eiwbh0k1/buildsettings2.png
- 若編譯後 DLL 無法正常運作，嘗試: https://www.mediafire.com/view/6xk7b9bf8qv3gda/buildsettingsEzorsia.png

**第三方 WZ 編輯相容性**：
- 建議將所有 WZ 編輯放在 `EzorsiaV2_UI.wz` 或 `MapleEzorsiaV2wzfiles.img` 中
- 這樣可避免與原廠 WZ 衝突，並便於追蹤修改

**可選工具 — Magpie（全螢幕無損投射）**：
- GitHub: https://github.com/Blinue/Magpie
- 將 1280x720 的遊戲畫面無損投射到全螢幕
- 避免全螢幕拉伸
- 保持 16:9 比例
- 原始畫質無損耗
- 建議搭配 1280x720 分辨率使用

**config.ini 注意事項**：
- 包含在 Release 中，預設包含在遊戲資料夾
- 盡量避免延伸注釋（在 `;` 後面的內容）
- 否則 INIReader 可能無法讀取導致崩潰

**疑難排解**：
1. 遊戲未開啟 → 手動下載 config 和 WZ/IMG 檔案，手動插入適當位置
2. 除了 .IMG 外的所有檔案都放在遊戲資料夾
3. .IMG 檔案只給使用 .IMG 檔案的人，放在 Data 資料夾

### MaplePE — 32-bit MapleStory 封包編輯器（GitHub 抓取）

**GitHub**: https://github.com/zhyonc/MaplePE

**核心功能**：
| 功能 | 說明 |
|------|------|
| 封包解析 | 自動解析封包結構 |
| 自訂封包發送 | 可發送自訂封包到客戶端 |
| 目標進程提取 | 從目標進程提取 InPacket 和 OutPacket 資訊 |
| Opcode 過濾 | 過濾特定 Opcode 以最佳化分析 |
| 程式碼生成 | 從封包結構生成簡單程式碼 |

**技術原理**：
- `MaplePE.exe`：UI 主程式，本身是 UDP 伺服器接收封包資訊
- `Packet.dll`：注入目標進程，Hook 封包讀寫函數

**需要 Hook 的函數**：

| 接收端 (CInPacket) | 發送端 (COutPacket) | Socket 函數 |
|-------------------|-------------------|------------|
| `CInPacket::Decode1` | `COutPacket::Encode1` | `CClientSocket::ProcessPacket` |
| `CInPacket::Decode2` | `COutPacket::Encode2` | `CClientSocket::SendPacket` |
| `CInPacket::Decode4` | `COutPacket::Encode4` | |
| `CInPacket::Decode8` | `COutPacket::Encode8` | |
| `CInPacket::DecodeStr` | `COutPacket::EncodeStr` | |
| `CInPacket::DecodeBuffer` | `COutPacket::EncodeBuffer` | |
| | `COutPacket::MakeBufferList` | |

**版本位址參考**：
- Google Sheets: https://docs.google.com/spreadsheets/d/1-LqA-OWgOuT-Xy3i73Jy5F_uVmogaGvmYpH4mLDSPhg

**使用流程**：
```
Step 1: 設定
  → 點擊 SettingView 圖示
  → 填入函數位址並儲存

Step 2: 啟動遊戲
  → 使用 launcher 啟動遊戲客戶端
  → 每個現有 process 只注入一次

Step 3: 注入
  → 點擊 Patch 按鈕定位目標 process ID
  → 點擊 InjectDLL 注入 Packet.dll

Step 4: 分析
  → 選擇封包項目查看完整 bytes
  → 編輯 bytes 並發送到客戶端

Step 5: 發送自訂封包
  → 雙擊封包項目開啟 FormatView
  → 編輯數值或序列並發送
```

**注入失敗原因**：
| 錯誤 | 原因 | 解決方案 |
|------|------|---------|
| Permission denied | 權限不足 | 以系統管理員身份執行 MaplePE.exe |
| Missing Packet.dll | 安全軟體刪除 | 暫時關閉安全軟體 |
| Injection protection | Launcher 有記憶體保護 | 使用其他方式啟動遊戲 |

**BB+ 版本 SendPacket 崩潰防護**：
- BB 版本後的 `CClientSocket::SendPacket` 有 hook 檢測
- 使用 Cheat Engine 將以下位址填為 `90` (NOP) 來禁用崩潰條件
- 範圍: `(int)&loc_7FFFFE + 4198402` 外的 retaddr → `JUMPOUT(0)`

## 二、Opcode結構 — 本次新獲取

### MaplePacketPuller + SpiritIDAPlugin（GitHub 抓取）

**GitHub**: 
- https://github.com/Bratah123/MaplePacketPuller
- https://github.com/Bratah123/SpiritIDAPlugin

**SpiritIDAPlugin** 是 MaplePacketPuller 的 IDA Pro 插件版，可直接在 IDA 中提取封包結構。

**支援環境**：
| 環境 | 支援狀態 |
|------|---------|
| Python 3.8.5 | ✅ 測試通過 |
| Python 3.6.12 & 3.8.6 | ✅ 測試通過 |
| IDA Pro 7.0 (32-bit) | ✅ 測試通過 |
| IDA Pro 7.5 | ✅ 測試通過 |
| IDA Pro 6.8 | ✅ 測試通過 |
| Python 2.7 | ❌ 不相容 (os.scandir) |

**注意**：while() 迴圈處理不完善，建議設 `GET_ALL_DECODES=true` 來提高準確度。

**完整工作流程**：
```
Step 1: IDA Pro 反編譯
  → 在 IDA 中找到 CField::OnPacket 或 CLogin::OnPacket
  → 產生 pseudocode (.p64 或 .txt)

Step 2: 複製 pseudocode
  → 在 Functions/ 目錄建立 .txt 檔案
  → 貼上 IDA 產生的 C-style pseudocode

Step 3: Python 分析
  → cd MaplePacketPuller/IDA\ Maple\ Script/src/main/python
  → python main.py
  → 輸入分析類型（完整/標準）
  → 輸入要分析的檔案名稱

Step 4: 輸出結果
  → 封包結構自動寫入 FuncOutput/ 目錄
  → 控制台顯示 Decode1/Decode2/Decode4/DecodeStr 等欄位
```

### IntransigentMS 腳本源庫（GitHub 抓取）

**GitHub**: https://github.com/NoetherEmmy/intransigentms-scripts

**腳本引擎**：Nashorn（JDK 8+ 內建），完全符合 ECMAScript 6 標準

**主要特點**：
- 所有腳本都使用 `Java.type()` 而非 `importPackage`
- 可使用 `print()` 輸出到伺服器控制台除錯
- 涵蓋 `npc/`、`quest/`、`reactor/`、`portal/`、`pet/` 等腳本類型

**ES6 寫法範例**：
```javascript
// ✅ ECMAScript 6 寫法
const MapleCharacter = Java.type("net.sf.odinms.client.MapleCharacter");
const SkillFactory = Java.type("net.sf.odinms.server.maps.SkillFactory");

function start() {
    print("NPC 啟動 - 開始對話");  // 輸出到伺服器 console
    cm.sendNext("歡迎來到我的商店！");
}

// 模板字串
const name = cm.getPlayer().getName();
cm.sendNext(`歡迎, ${name}!`);
```

## 五、WZ修改 — 本次補充

### WZ 編輯最佳實踐（持續更新）

**Gold Standard 原則**：
1. 使用 **HaRepacker** 開啟客戶端 WZ（選擇 `GMS (old)` 加密格式）
2. **使用 UI 直接修改數值**（禁止刪除/重新導入節點！）
3. 儲存 WZ
4. 選擇 **"Private Server..."** 導出 XML
5. 將導出的 XML 覆蓋到伺服器的 WZ.XML 目錄

**⚠️ 重要提醒**：
> DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs.

**自訂 WZ 檔案管理**（配合 Ezorsia V2）：
- 建議將所有自訂修改放在 `EzorsiaV2_UI.wz` 中
- 避免與原廠 WZ 衝突
- 便於版本控制和備份

### Quest Level Restrictions 移除（RaGEZONE 2025-07）
