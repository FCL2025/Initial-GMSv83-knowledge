# 4.3 NPC 腳本編寫

---

## 🎯 NPC 腳本是什麼？

NPC 腳本控制與 NPC 的對話流程和事件觸發。

---

## 📝 基本結構

```javascript
// 範例 NPC 腳本
var status = -1;

function start() {
    cm.sendNext("你好，我是 NPC！");
}

function action(mode, type, selection) {
    if (mode == 1) {
        status++;
    } else {
        status--;
    }
    
    if (status == 0) {
        cm.sendYesNo("要接受任務嗎？");
    } else if (status == 1) {
        cm.gainItem(4000000, 1);
        cm.dispose();
    }
}
```

## 🔧 常用函數

| 函數 | 用途 |
|------|------|
| `cm.sendNext(msg)` | 發送訊息 |
| `cm.sendYesNo(msg)` | 發送 Yes/No 選項 |
| `cm.gainItem(id, count)` | 給予物品 |
| `cm.dispose()` | 結束對話 |

## 🏪 NPC 腳本類型

### 1. 基礎對話型
最基本的 NPC 僅提供文字對話和選項選擇：
```javascript
function start() {
    cm.sendNext("歡迎來到 MapleStory！");
}

function action(mode, type, selection) {
    if (mode == 1) status++;
    else status--;
    
    if (status == 1) {
        cm.sendNext("祝你旅途愉快！");
    } else if (status == 2) {
        cm.dispose();
    } else {
        cm.dispose();
    }
}
```

### 2. 任務型 NPC
提供任務接取和完成功能：
```javascript
var questId = 1000;  // 任務 ID
var status = -1;

function start() {
    if (cm.getQuestStatus(questId) == 0) {
        cm.sendNext("幫我收集 10 個 #b怪物掉落物#k！");
    } else if (cm.getQuestStatus(questId) == 1) {
        // 任務進行中
        if (cm.haveItem(4000000, 10)) {
            cm.sendNext("太好了！你完成任務了！");
        } else {
            cm.sendNext("還沒收集完嗎？");
            cm.dispose();
        }
    } else {
        cm.sendNext("你已經完成過這個任務了。");
        cm.dispose();
    }
}

function action(mode, type, selection) {
    if (mode == 1) status++;
    else status--;
    
    if (status == 0) {
        if (cm.getQuestStatus(questId) == 0) {
            cm.startQuest(questId);
            cm.dispose();
        }
    } else if (status == 1) {
        if (cm.haveItem(4000000, 10)) {
            cm.gainItem(4000000, -10);
            cm.gainMeso(10000);
            cm.completeQuest(questId);
        }
        cm.dispose();
    } else {
        cm.dispose();
    }
}
```

### 3. 商店型 NPC
販賣物品的商店腳本：
```javascript
function start() {
    cm.sendNext("想要買些什麼嗎？");
}

function action(mode, type, selection) {
    if (mode == 1) {
        status++;
    } else {
        status--;
    }
    
    if (status == 0) {
        cm.sendSimple("請選擇：\r\n#L0#購買武器#l\r\n#L1#購買防具#l");
    } else if (status == 1) {
        if (selection == 0) {
            cm.openShop(1);  // 開啟商店 ID 1
        } else {
            cm.openShop(2);
        }
        cm.dispose();
    } else {
        cm.dispose();
    }
}
```

### 4. 進階腳本（帶驗證）
檢查玩家條件後才給予獎勵：
```javascript
var questId = 2000;
var requiredLevel = 30;
var requiredMeso = 50000;

function start() {
    if (cm.getPlayer().getLevel() < requiredLevel) {
        cm.sendNext("你的等級需要達到 " + requiredLevel + " 級！");
        cm.dispose();
        return;
    }
    
    if (cm.getMeso() < requiredMeso) {
        cm.sendNext("你需要至少 " + requiredMeso + " 金幣！");
        cm.dispose();
        return;
    }
    
    cm.sendYesNo("確定要花費 " + requiredMeso + " 金幣購買 #t4000000# 嗎？");
}

function action(mode, type, selection) {
    if (mode == 1) {
        if (cm.getMeso() >= requiredMeso) {
            cm.gainMeso(-requiredMeso);
            cm.gainItem(4000000, 1);
            cm.sendNext("購買成功！");
        }
    }
    cm.dispose();
}
```

### 5. 動態商店腳本 (v83 進階)
不依賴 SQL，動態定義商品：
```javascript
// Dynamic NPC Shop Script (v83)
var status = 0;
var shopId = -1;
var items = Array(
    Array(1302000, 100000, 1),  // itemId, price, quantity
    Array(1302001, 150000, 1),
    Array(1312000, 120000, 1)
);

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
        return;
    }
    
    if (mode == 0) {
        status--;
        if (status < 0) {
            cm.dispose();
            return;
        }
    } else {
        status++;
    }
    
    if (status == 0) {
        var selStr = "歡迎光臨！請選擇商品：\r\n";
        for (var i = 0; i < items.length; i++) {
            selStr += "#L" + i + "##i" + items[i][0] + "# - " + items[i][2] + "個 - " + items[i][1] + " 金幣#l\r\n";
        }
        cm.sendSimple(selStr);
    } else if (status == 1) {
        var selected = items[selection];
        if (cm.getMeso() < selected[1]) {
            cm.sendNext("金幣不足！");
            cm.dispose();
        } else {
            cm.gainMeso(-selected[1]);
            cm.gainItem(selected[0], selected[2]);
            cm.sendNext("購買成功！");
            cm.dispose();
        }
    }
}
```

## 📚 常用 API 參考

### 對話函數
| 函數 | 說明 |
|------|------|
| `cm.sendNext(msg)` | 發送下一句對話 |
| `cm.sendPrev(msg)` | 發送上一句（返回） |
| `cm.sendNextPrev(msg)` | 上下句對話 |
| `cm.sendYesNo(msg)` | Yes/No 選項 |
| `cm.sendAcceptDecline(msg)` | 接受/拒絕選項 |
| `cm.sendSimple(msg)` | 簡單列表選擇 |
| `cm.sendGetNumber(msg, def, min, max)` | 數字輸入框 |
| `cm.sendGetText(msg)` | 文字輸入框 |
| `cm.sendOk(msg)` | 確定對話框 |
| `cm.askYesNo(msg)` | 提問 Yes/No |
| `cm.askMenu(msg)` | 選單選擇 |
| `cm.askAvatar(msg)` | 角色造型選擇 |

### 玩家狀態函數
| 函數 | 說明 |
|------|------|
| `cm.getPlayer()` | 獲取玩家物件 |
| `cm.getPlayer().getLevel()` | 玩家等級 |
| `cm.getPlayer().getJob()` | 玩家職業 ID |
| `cm.getPlayer().getFame()` | 人氣值 |
| `cm.getPlayer().getGender()` | 性別 (0=男, 1=女) |
| `cm.getMeso()` | 玩家金幣 |
| `cm.haveItem(id)` | 檢查物品數量 |
| `cm.haveItem(id, count)` | 檢查物品數量（含數量） |
| `cm.getQuestStatus(questId)` | 任務狀態 (0=未接, 1=進行中, 2=已完成) |
| `cm.getJobId()` | 職業 ID |
| `cm.getNpcId()` | NPC ID |

### 動作函數
| 函數 | 說明 |
|------|------|
| `cm.gainItem(id, count)` | 給予/移除物品 (count 負數為移除) |
| `cm.gainMeso(amount)` | 給予/移除金幣 |
| `cm.startQuest(questId)` | 開始任務 |
| `cm.completeQuest(questId)` | 完成任務 |
| `cm.forfeitQuest(questId)` | 放棄任務 |
| `cm.openShop(shopId)` | 開啟商店 |
| `cm.openShop(shopId, npcId)` | 開啟指定 NPC 商店 |
| `cm.warp(mapId)` | 傳送到指定地圖 |
| `cm.warpParty(mapId)` | 傳送整個隊伍 |
| `cm.dispose()` | 結束對話 |
| `cm.reloadChar()` | 重新載入角色 |
| `cm.getPlayer().dropMessage(msg)` | 顯示訊息 |
| `cm.getEventInstance()` | 獲取事件實例 |
| `cm.getMap()` | 獲取當前地圖物件 |

### 條件判斷函數 (進階)
```javascript
// 檢查玩家是否為 GM
cm.getPlayer().isGM()

// 檢查是否為特定職業
cm.getJob() >= 100 && cm.getJob() <= 132

// 檢查背包空間
cm.canHold(itemId)  // 是否能容納新物品

// 檢查角色是否在特定地圖
cm.getMapId() == 100000000

// 檢查日期/時間
var date = new Date();
var dayOfWeek = date.getDay();  // 0=週日, 1=週一...

// 亂數判斷
Math.random() < 0.5  // 50% 機率
```

## 🔧 NPC 調試技巧

### 常見錯誤
```
1. 忘記呼叫 dispose() → 對話框無法關閉
2. status 邏輯錯誤 → 選項順序混亂
3. 物品 ID 不存在 → 客戶端顯示錯誤
4. 任務 ID 衝突 → 任務無法正常觸發
5. 無限迴圈 → 腳本卡死
6. 變數未初始化 → 腳本崩潰
```

### 調試方法
```javascript
// 添加調試輸出
cm.sendNext("DEBUG: Quest Status = " + cm.getQuestStatus(questId));
cm.sendNext("DEBUG: Player Level = " + cm.getPlayer().getLevel());
cm.sendNext("DEBUG: Meso = " + cm.getMeso());
cm.sendNext("DEBUG: Job = " + cm.getJob());
cm.sendNext("DEBUG: NpcId = " + cm.getNpcId());
cm.sendNext("DEBUG: MapId = " + cm.getMapId());
```

### 腳本測試技巧
```javascript
// 測試模式 - 直接給予物品而不檢查條件
var TEST_MODE = true;

function action(mode, type, selection) {
    if (TEST_MODE) {
        cm.gainItem(4000000, 1);
        cm.sendOk("測試模式：已給予物品");
        cm.dispose();
        return;
    }
    // 正常邏輯...
}
```

## 📝 腳本檔案命名規則

v83 腳本存放位置：`scripts/npc/`
- 檔案名 = NPC ID（不含副檔名 `.js`）
- 例如：`9000001.js` 對應 NPC ID 9000001

### 特殊 NPC 類型
| ID 範圍 | 類型 |
|---------|------|
| 9000000+ | 普通 NPC |
| 9010000+ | 系統 NPC |
| 9200000+ | 商店 NPC |
| 9900000+ | 特殊 NPC |
