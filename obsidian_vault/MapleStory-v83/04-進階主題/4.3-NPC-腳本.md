# 4.3 NPC 腳本編寫

## 腳本語言
v83 模擬器使用 **JavaScript** 編寫 NPC 腳本。

## 腳本位置
通常位於：`source/scripts/npc/`
每個 NPC 一個 `.js` 檔案，檔名為 NPC ID。

## 基本模板

```javascript
var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        // 對話結束
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        
        if (mode == 1)
            status++;
        else
            status--;
        
        // 根據 status 處理不同對話
        if (status == 0) {
            cm.sendNext("你好！歡迎來到我的商店！");
        } else if (status == 1) {
            cm.sendSimple("你想做什麼？\r\n#b#L0#購買物品#l\r\n#L1#出售物品#l\r\n#L2#離開#l");
        } else if (status == 2) {
            if (selection == 0) {
                cm.openShop(1);
                cm.dispose();
            } else if (selection == 1) {
                cm.sendOK("抱歉，收購服務暫時關閉。");
                cm.dispose();
            } else {
                cm.dispose();
            }
        }
    }
}
```

## 常用函數

| 函數 | 功能 |
|------|------|
| `cm.sendNext(text)` | 下一個對話框 |
| `cm.sendPrev(text)` | 上一个對話框 |
| `cm.sendSimple(text)` | 選擇對話框 |
| `cm.sendYesNo(text)` | 是/否對話框 |
| `cm.sendOk(text)` | 確定對話框 |
| `cm.openShop(id)` | 打開商店 |
| `cm.gainItem(id, qty)` | 給予物品 |
| `cm.gainMeso(qty)` | 給予金幣 |
| `cm.getPlayer()` | 獲取玩家對象 |
| `cm.dispose()` | 關閉對話 |
| `cm.getJob()` | 獲取職業 |
| `cm.getLevel()` | 獲取等級 |

## 對話框類型

### sendNext
```javascript
cm.sendNext("這是下一個對話框");
```

### sendSimple (帶選項)
```javascript
cm.sendSimple("請選擇：\r\n#b#L0#選項1#l\r\n#L1#選項2#l");
```

### sendYesNo (確認)
```javascript
cm.sendYesNo("確定要繼續嗎？");
```

## 常見錯誤處理

### importPackage 錯誤
JDK 7+ 不再支援 `importPackage`：
```javascript
// 錯誤
importPackage(Packages.client);

// 正確 - 直接使用完整路徑
var MapleCharacter = Packages.client.MapleCharacter;
```

### SavedLocationType 錯誤
更新腳本使用正確的枚舉值。

## 調試技巧

1. **查看日誌**：伺服器 Bat 會顯示加載狀態
2. **找不到腳本**：日誌顯示「NPC xxx.js is not coding yet..」
3. **測試命令**：`@npc [npcId]` 在遊戲中測試

## 🔄 動態 NPC 商店腳本 (v83)

### 傳統商店的問題
- 需要修改 SQL 資料庫
- 每次修改需重啟伺服器

### 動態商店腳本
```javascript
/*
 * 動態 NPC 商店 - 不需要 SQL
 * NPC ID: 9010009
 */
var status = 0;
var items = Array(
    Array(1302000, 10000, 1),   // [物品ID, 價格, 庫存]
    Array(1302001, 20000, 1),
    Array(1302002, 30000, 1),
    Array(1302003, 50000, 1)
);

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        
        if (mode == 1)
            status++;
        else
            status--;
        
        if (status == 0) {
            var msg = "歡迎光臨！\r\n請選擇商品：\r\n";
            for (var i = 0; i < items.length; i++) {
                msg += "#L" + i + "##i" + items[i][0] + "# - " + items[i][1] + " 金幣#l\r\n";
            }
            cm.sendSimple(msg);
        } else if (status == 1) {
            var item = items[selection];
            if (cm.getMeso() >= item[1]) {
                if (cm.canHold(item[0])) {
                    cm.gainMeso(-item[1]);
                    cm.gainItem(item[0], item[2]);
                    cm.sendOk("購買成功！");
                } else {
                    cm.sendOk("背包空間不足！");
                }
            } else {
                cm.sendOk("金幣不足！");
            }
            cm.dispose();
        }
    }
}
```

## 🎯 進階腳本：任務 NPC

### 完整任務腳本範例
```javascript
/*
 * 任務 NPC - 從怪物身上收集物品
 * NPC ID: 9000001
 * 任務 ID: 1000
 */
var status = 0;
var questId = 1000;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        
        if (mode == 1)
            status++;
        else
            status--;
        
        // 檢查任務狀態
        var quest = cm.getQuestStatus(questId);
        
        if (status == 0) {
            if (quest == 0) {
                // 任務未開始
                cm.sendNext("冒險家，我需要你的幫助！#r黑木妖#k正在侵擾村莊，請幫我消滅並收集 #b10個黑木妖的葉子#k。");
            } else if (quest == 1) {
                // 任務進行中
                var leafCount = cm.getQuestProgress(questId);
                cm.sendNext("你目前收集了 #b" + leafCount + "#k 個葉子。加油！");
            } else {
                // 任務已完成
                cm.sendOk("感謝你！村莊得救了。");
            }
        } else if (status == 1) {
            if (quest == 0) {
                // 開始任務
                cm.startQuest(questId);
                cm.sendOk("去吧！收集 #b10個黑木妖的葉子#k 後回來找我。");
            } else if (quest == 1) {
                // 檢查是否完成
                if (cm.getQuestProgress(questId) >= 10) {
                    cm.completeQuest(questId);
                    cm.gainItem(4000000, 1); // 獎勵物品
                    cm.gainMeso(5000);
                    cm.sendOk("太棒了！這是你的獎勵。");
                } else {
                    cm.sendOk("還沒收集夠呢... 繼續努力！");
                }
            }
            cm.dispose();
        }
    }
}
```

## 🆕 NPC 對話格式標籤完整對照 (elitepvpers)

**來源**：https://www.elitepvpers.com/forum/maplestory/597399-guide-npc-scripting-tutorial.html

| 標籤 | 功能 | 範例 |
|------|------|------|
| `#b` | 藍色文字 | `#b藍色文字` |
| `#c[itemid]` | 顯示玩家持有該物品的數量 | `#c1302000#` → "5"（若有5個） |
| `#d` | 紫色文字 | `#d紫色文字` |
| `#e` | 粗體文字 | `#e粗體` |
| `#f[imagelocation]` | 顯示 WZ 中的圖片 | `#fimg/item/1302000.img/0/0#` |
| `#g` | 綠色文字 | `#g綠色文字` |
| `#h #` | 顯示玩家名稱 | `#h#` → "快樂貓" |
| `#i[itemid]` | 顯示物品圖片（聊天框內） | `#i1302000#` |
| `#k` | 黑色文字 | `#k黑色文字` |
| `#l` | 選項標記（用於 sendSimple） | `#L0#選項1#l` |
| `#n` | 正常字體（取消粗體） | `#n` |
| `#r` | 紅色文字 | `#r紅色文字` |
| `#s` | 黃色文字 | `#s黃色文字` |
| `\r\n` | 換行 | 多行對話 |

**使用範例**：
```javascript
cm.sendNext("你好 #h#！\r\n" +
    "#b#L0#我想購買武器#l\r\n" +
    "#r#L1#我想購買防具#l\r\n" +
    "#i1302000# #t1302000# x #c1302000#");
```
顯示：玩家名稱（藍色）、武器選項（藍色可選）、防具選項（紅色可選）、以及該武器的圖示+名稱+持有數量。

## 🐾 NPC 腳本狀態機詳解

### Status 狀態流程
```
status = -1 → 初始調用 start()
  ↓
status = 0 → 第一次 sendNext/sendSimple
  ↓
status = 1 → 玩家回應後
  ↓
status = 2 → 第二次回應後
  ↓
... 依此類推
```

### Mode 參數含義
| Mode 值 | 意義 |
|---------|------|
| 1 | 玩家點擊了"下一頁"或選項 |
| 0 | 玩家點擊了"返回" (back button) |
| -1 | 玩家關閉了對話框 (ESC 或 X) |

### Type 參數含義
| Type 值 | 意義 |
|---------|------|
| 0 | 普通對話框回應 |
| 1 | 當 mode=0 時，通常是強制關閉 |

## ⚠️ 常見錯誤與解決

### 錯誤 1：無窮對話迴圈
```javascript
// 錯誤：每次都返回 status=0
if (status == 0) {
    cm.sendNext("你好");
    // 忘記 status++
}

// 正確
if (status == 0) {
    cm.sendNext("你好");
    status = 1;  // 手動增加
}
```

### 錯誤 2：忘記 dispose()
```javascript
// 錯誤：最後一個分支忘記 dispose
if (status == 1) {
    cm.sendOk("完成");
    // 沒有 cm.dispose();
}

// 正確
if (status == 1) {
    cm.sendOk("完成");
    cm.dispose();
}
```

### 錯誤 3：importPackage 已廢棄 (JDK 7+)
```javascript
// 錯誤 (JDK 7+ 不支援)
importPackage(Packages.client);

// 正確
var MapleCharacter = Packages.client.MapleCharacter;
```

## 🆕 NPC 腳本與 WZ 編輯的結合 (2025-2026)

### 創建自訂 NPC 流程
1. 使用 HaRepacker 導出 NPC WZ 為 XML
2. 在 XML 中新增自訂 NPC 的 frame 和 origin 設定
3. 複製 `0001337.img` 作為模板，修改尺寸
4. 將修改後的 img 放入 `NPC.WZ`
5. 在伺服器端編寫對應的 `.js` 腳本

### HaRepacker 導出 WZ 為 XML (RaGEZONE, 2023)
來源: https://forum.ragezone.com/threads/community-wz-img-repack-for-v83.1221835/

```
導出步驟：
1. 用 HaRepacker 打開 WZ 檔案
2. 選擇要導出的節點
3. 點擊 Export → 選擇 XML (Private Server) 格式
4. 導出後可使用文字編輯器修改
5. 重新導入 WZ
```

**注意**：
- v83 WZ 加密格式需選擇 `GMS (old)`
- 不要直接刪除/重新導入節點，可能造成數據丟失
- 建議使用 HaRepacker UI 直接編輯數值

## 🆕 2025-2026 常見錯誤修復

### importPackage 錯誤（JDK 7+ 廢棄）
```javascript
// 錯誤寫法 (JDK 6)
importPackage(Packages.client);

// 正確寫法 - 移除 importPackage，直接使用完整路徑
var MapleCharacter = Packages.client.MapleCharacter;
```

### SavedLocationType 未定義錯誤
```
Error: ReferenceError: "SavedLocationType" is not defined
原因: 某些 HeavenMS/MapleSolaxia 腳本使用 JDK 6 編譯
解決: 註解掉或移除相關程式碼
```

### 常見錯誤排查
| 錯誤訊息 | 原因 | 解決方案 |
|---------|------|---------|
| `importPackage is not defined` | JDK 7+ 不支援 | 移除 importPackage 行 |
| `SavedLocationType is not defined` | 變數未定義 | 檢查來源版本相容性 |
| `NPC xxx.js is not coding yet` | 腳本路徑錯誤 | 確認檔案在正確目錄 |
| `NullPointerException` | 物件為空 | 檢查 `cm.getPlayer()` 返回值 |

## 🆕 NPC 腳本源間相容性問題 (RaGEZONE, 2026-03)

RaGEZONE: https://forum.ragezone.com/threads/kinoko-npc-scripts.1261538/

**問題**：
- 不同音源（Cosmic、Kinoko、HeavenMS）的 NPC 腳本格式存在差異
- 較舊的腳本在新解析器可能不相容
- 原因：腳本解析器版本不同

**解決方案**：
1. 在 source 中 include 較舊版本的 script reader
2. 或使用相容格式編寫腳本（避免使用特定版本專有的函數）
3. 測試時在目標音源上驗證

**最佳實踐**：
- 使用普遍支援的 `cm.` 函數
- 避免使用特定音源專有的 Package 路徑
- 先在本地測試再部署到正式伺服器
