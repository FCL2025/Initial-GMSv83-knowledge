# 4.2 Opcode 結構分析

---

## 🎯 Opcode 是什麼？

Opcode (Operation Code) 是封包的**辨識碼**，用來區分不同的遊戲操作。

```
[Opcode: 5] [帳號] [密碼]  →  登入請求
[Opcode: 6] [結果]        →  登入回應
```

## 🔄 v83 Opcode 結構深度分析

### 封包格式

v83 封包結構：
```
[Header: 4-6 bytes][Opcode: 2 bytes][Data: variable]
```

### 重要 Opcode 列表 (v83)

| Opcode (Hex) | Opcode (Dec) | 名稱 | 方向 |
|-------------|-------------|------|------|
| 0x01 | 1 | 保留 | - |
| 0x02 | 2 | Ping | 雙向 |
| 0x05 | 5 | 登入請求 | C→S |
| 0x06 | 6 | 登入回應 | S→C |
| 0x08 | 8 | 伺服器列表 | S→C |
| 0x09 | 9 | 選擇伺服器 | C→S |
| 0x0A | 10 | 選擇角色 | C→S |
| 0x0B | 11 | 角色資料 | S→C |
| ... | ... | ... | ... |

### 登入流程 Opcode

```
C → S: 0x05 (登入請求)      [帳號][密碼][用戶端版本]
S → C: 0x06 (登入回應)      [結果][錯誤碼]
S → C: 0x08 (伺服器列表)    [伺服器數量][名稱][IP][端口]
C → S: 0x09 (選擇伺服器)    [伺服器ID]
C → S: 0x0A (選擇角色)      [角色索引]
S → C: 0x0B (進入遊戲)      [角色資料][遊戲伺服器資訊]
```

### 遊戲內常見 Opcode

| Opcode | 名稱 | 說明 |
|--------|------|------|
| 0x12 | NPC對話 | 開啟 NPC 對話框 |
| 0x14 | 移動 | 角色移動 |
| 0x15 | 普通攻擊 | 近距離攻擊 |
| 0x16 | 技能攻擊 | 使用技能攻擊 |
| 0x1E | 使用物品 | 使用道具 |
| 0x23 | 玩家的資訊 | 角色屬性更新 |
| 0x2E | 聊天 | 一般/大喊/悄悄話 |

## 📝 實戰：分析怪物生成封包

```
假設我們抓到一個怪物生成封包：
[00][01][15][xx][yy][zz][mob_id][...]

解析：
- 00 01 = Header
- 15 = Opcode (怪物生成)
- xx yy zz = 位置座標
- mob_id = 怪物ID (WORD)
```
