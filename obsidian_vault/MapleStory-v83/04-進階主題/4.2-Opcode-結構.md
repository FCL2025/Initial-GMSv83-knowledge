# 4.2 Opcode 結構分析（更新版）

## 什麼是 Opcode？
Opcode (Operation Code) 是網路封包中用於標識操作類型的唯一識別符。

## v83 Opcode 特性
- **已高度文檔化**：v83 的 Opcode 幾乎全部被映射
- **使用 AES 加密**：需配合客戶端金鑰
- **高度客製化**：每個私服可能使用不同設定
- v83 Opcode 格式：**2 bytes** (little endian)

## Opcode 結構
```
[封包頭部 (4 bytes)][Opcode (2 bytes)][資料...]
```

### 封包頭部格式
```
偏移  長度   內容
0x00  4      MapleCrypto 混淆頭部
      (包含長度 + 混淆 opcode)
```

### v83 Opcode 格式
- Opcode 為 **2 bytes** (little endian)
- 例如：Opcode `0x0001` 在封包中顯示為 `01 00`

## 🔑 MapleCrypto 頭部混淆機制

### 混淆原理
```
原始 opcode: 0x0001 (例如 LOGIN_PASSWORD)
       IV: [0x7A, 0x52, ...]

Step 1: XOR + SHIFT
  verify = (opcode ^ iv[0]) + (iv[1] << 8)
        = (0x0001 ^ 0x7A) + (0x52 << 8)
        = 0x007B + 0x5200
        = 0x527B

Step 2: 位旋轉 (rotate left 3 bits)
  verify = (verify << 3) | (verify >> 13)
        = 0x527B << 3 | 0x527B >> 13
        = 0x29BD8 | 0x00
        = 0x29BD8

Step 3: 取低位 16 bits
  verify = 0x29BD8 & 0xFFFF = 0x9BD8
```

## 🆕 CClientSocket::ProcessPacket 封包分發流程 (RaGEZONE, 2019)

**來源**: https://forum.ragezone.com/threads/how-does-one-find-a-structure-of-a-packet-via-ida.1162654/

### 封包分發架構

所有 S→C（伺服器發送到客戶端）封包都經過以下函數鏈：

```
CClientSocket::ProcessPacket
    ↓
根據 opcode header 分發到:
    ├── CLogin::OnPacket         (登入相關 opcode)
    ├── CWvsContext::OnPacket    (warp2map 之前的 opcode)
    ├── CField::OnPacket         (遊戲中所有其餘 opcode，最大範圍)
    └── CCashShop::OnPacket      (現金商店 opcode)
```

### 為什麼了解分發流程重要？

1. **除錯**：知道在哪個函數中斷點
2. **理解封包來源**：確認某個功能是由哪個模組處理
3. **客戶端修改**：修改 UI 或去除限制時，知道該找哪個函數

### v83 函數位址參考

| 函數 | 說明 |
|------|------|
| `CClientSocket::ProcessPacket` | 封包分發入口（所有封包都經過這裡） |
| `CLogin::OnPacket` | 登入相關 opcode 的處理 |
| `CWvsContext::OnPacket` | warp 前的一些全域 opcode |
| `CField::OnPacket` | 遊戲中所有其餘 opcode（最大範圍） |
| `CCashShop::OnPacket` | 現金商店 opcode |

### 分析方法

**靜態分析**：
1. IDA 中找到 `CClientSocket::ProcessPacket`
2. 觀察 switch-case 跳轉表
3. 每個 case 對應一個 opcode

**動態分析**：
1. 在 `CInPacket::Decode2` 設斷點
2. 每次讀取 opcode 時觀察值
3. 配合 CField/CLogin OnPacket 交叉引用確認

## 🆕 IDAPython 快速指令 (RaGEZONE, 2015)

**來源**: https://forum.ragezone.com/threads/idapython-show-all-address-xref-from-coutpacket-with-value.1060540/

### COutPacket Opcode 分析技巧

**Opcode E8 (CALL) 計算示例**：
```
Address 0x168B8F6: E8 C5 90 D8 FE
  = Call from address 004149C0

計算方式：
1. E8 = Call near amd relative call (read next 4 bytes)
2. 後 4 bytes: C5 90 D8 FE → Reverse: FED890C5
3. 2's complement: 0xFED890C5 xor 0xFFFFFFFF + 1 = 0x01276F3B
4. 減去當前位址: 0x168B8F6 - 0x01276F3B = 0x4149BB
5. 加上 5 bytes: 0x4149BB + 5 = 0x4149C0 ✓
```

### IDAPython 快速指令

```python
import idautils
import idc

# 找出所有 COutPacket 的交叉引用
for xref in XrefsTo(idc.get_name_ea_simple("COutPacket__Encode")):
    print(f"Reference at: {hex(xref.frm)}")

# 列出所有 Decode 函數
for func in idautils.Functions():
    name = idc.get_func_name(func)
    if "Decode" in name:
        print(f"{hex(func)}: {name}")
```
