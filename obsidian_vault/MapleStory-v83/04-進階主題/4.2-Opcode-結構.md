# 4.2 Opcode 結構分析（更新版）

## 什麼是 Opcode？
Opcode (Operation Code) 是網路封包中用於標識操作類型的唯一識別符。

## v83 Opcode 特性
- **已高度文檔化**：v83 的 Opcode 幾乎全部被映射
- **使用 AES 加密**：需配合客戶端金鑰
- **高度客製化**：每個私服可能使用不同設定
- v83 Opcode 格式：**2 bytes** (little endian)

## Opcode 結構
```
[封包頭部 (4 bytes)][Opcode (2 bytes)][資料...]
```

### 封包頭部格式
```
偏移  長度   內容
0x00  4      MapleCrypto 混淆頭部
      (包含長度 + 混淆 opcode)
```

### v83 Opcode 格式
- Opcode 為 **2 bytes** (little endian)
- 例如：Opcode `0x0001` 在封包中顯示為 `01 00`

## 🔑 MapleCrypto 頭部混淆機制

### 混淆原理
```
原始 opcode: 0x0001 (例如 LOGIN_PASSWORD)
       IV: [0x7A, 0x52, ...]

Step 1: XOR + SHIFT
  verify = (opcode ^ iv[0]) + (iv[1] << 8)
        = (0x0001 ^ 0x7A) + (0x52 << 8)
        = 0x007B + 0x5200
        = 0x527B

Step 2: 位旋轉 (rotate left 3 bits)
  verify = (verify << 3) | (verify >> 13)
        = 0x527B << 3 | 0x527B >> 13
        = 0x29BD8 | 0x00
        = 0x29BD8

Step 3: 取低位 16 bits
  verify = 0x29BD8 & 0xFFFF = 0x9BD8
```

## 🔬 IDA 分析技巧（更新）

### IDA 函數命名對應（v83）
| IDA 解碼函數 | 伺服器對應 | 說明 |
|-------------|---------|------|
| `Decode1` | `mplew.write` | 寫入 1 byte |
| `Decode2` | `mplew.writeShort` | 寫入 2 bytes (short) |
| `Decode4` | `mplew.writeInt` | 寫入 4 bytes (int) |
| `DecodeStr` | `mplew.writeMapleAsciiString` | 寫入字串 |
| `DecodeBuffer(x, 8)` | `mplew.writeLong` | 寫入 8 bytes (long) |
| `DecodeBuffer(x, 13)` | `mplew.writeAsciiString` | 寫入固定長度 ASCII |

### IDA 封包分析方法（RaGEZONE 2024）

**方法 1：使用 SpiritIDAPlugin 自動分析**

GitHub: https://github.com/Bratah123/SpiritIDAPlugin

這是一個 IDAPython 插件，提供 MapleStory 客戶端反向工程工具。

**功能：**
- 分析任何函數的封包結構
- 自動搜尋所有 OutPacket Header
- 自動搜尋 InPacket 結構
- 將分析結果寫入文字檔 (C:\SpiritIDA)
- 右鍵點擊函數即可使用工具

**支援版本：**
- IDA Pro 7.0（目標版本）
- Python 2.7 (IDAPython 原生)
- 目前不相容 IDA 7.5 Pro

**使用方法：**
```
1. 將插件檔案複製到 IDA plugins 目錄
2. 修改 init.py 中的路徑常數指向 SpiritMS.png
3. 啟動 IDA，輸出視窗顯示 "Successfully Loaded"
4. 右鍵點擊函數，存取插件工具
```

**方法 2：MaplePacketPuller（命令列工具）**

GitHub: https://github.com/Bratah123/MaplePacketPuller

Python 工具，從 IDA 產生的虛擬碼分析封包結構。

**輸入：** IDA 產生的 C 虛擬碼 (.txt 檔案)
**輸出：** 封包結構 + console 輸出

**使用步驟：**
```
1. 在 IDA 中對目標函數產生 pseudocode
2. 複製並貼上到 MaplePacketPuller/IDA Maple Script/Functions/ 目錄下的 .txt 檔案
3. 在 CLI 中執行 main.py
4. 選擇分析類型
5. 輸入檔案名稱
6. 分析結果輸出至 FuncOutput/ 目錄
```

**已知限制：**
- while() 迴圈處理不完全（建議启用 GET_ALL_DECODES=true 精確模式）
- 不顯示 switch case（需用戶自行添加）
- 需預先對 decode 函數命名

### 使用 IDA 找封包結構（RaGEZONE, 2019）

**步驟 1：** 在 IDA 中找到 SendPacket 函數
**步驟 2：** 找到 packet header（例：0x1C 為 Inventory Operation）
**步驟 3：** 追蹤 decode 函數調用
**步驟 4：** 確認 packet structure 是否改變

```
// v83 InventoryOperation 封包結構範例
0x1C [header]
Decode1 [類型: 0=add, 1=remove, etc.]
Decode4 [item unique ID]
Decode4 [item ID]
Decode1 [position]
...（視類型不同而有不同欄位）
```

## ⚠️ 注意事項

1. v83 的封包結構已經非常完整，不需要重新追蹤大多數 Opcode
2. 唯一需要反向工程的可能是客戶端修改（如修改 UI、去除限制）
3. 使用 IDA 時，確保已對 Decode 函數進行正確命名
4. 不同私服可能使用不同的 Opcode 值集，需與伺服器原始碼同步
