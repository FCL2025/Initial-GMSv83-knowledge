# 4.2 Opcode 結構分析

---

## 🎯 Opcode 是什麼？

Opcode (Operation Code) 是封包的**辨識碼**，用來區分不同的遊戲操作。

```
[Opcode: 2 bytes] [Data: variable]
```

在 v83 中，Opcode 是 **2 bytes (16-bit short)**，幾乎所有常用 opcode 都已由社群完整映射。

## 🔍 IDA 分析技巧

### 必備快捷鍵

| 快捷鍵 | 功能 |
|--------|------|
| F5 | 反編譯為偽代碼 |
| X | 查找所有引用 (交叉參照) |
| ALT+T | 搜尋文字 |
| ALT+B | 搜尋位元組 |
| G | 跳轉到指定地址 |

### 關鍵方法

| 方法 | 用途 |
|------|------|
| `CWvsContext::OnPacket` | 處理登入相關 opcode (< 140) |
| `CField::OnPacket` | 處理遊戲內 opcode (≥ 140) |
| `CLogin::OnPacket` | 處理登入介面 opcode |
| `CInPacket::GetOpcode` | 取得封包 opcode |
| `COutPacket::Encode2` | 寫入 opcode |
| `CClientSocket::ProcessPacket` | 所有客戶端接收封包的入口 |

### IDA 分析要點
```
1. 找到 CClientSocket::ProcessPacket
2. 分析跳轉表 (switch case)
3. 交叉參照每個 opcode handler
4. 記錄每個 opcode 的處理函數名稱
```

## 📝 v83 封包格式

### 封包結構
```
[Header: 4-6 bytes][Opcode: 2 bytes][Data: variable]

Header 格式：
  [Length: 2 bytes]  // 封包總長度（含Header）
  [Verify: 2 bytes]  // 混淆後的校驗值 (MapleCrypto)
```

### 解密流程（伺服器視角）
```
1. 讀取 4 bytes header
2. MapleCrypto 解混淆 → 取得長度和 opcode
3. 剩餘內容用 AES-OFB 解密
4. 交付給 handler 處理
```

| 代碼 | 意義 |
|------|------|
| 0x00 | 登入成功 |
| 0x05 | 帳號或密碼錯誤 |
| 0x0D | 帳號被禁用 |
| 0x13 | 帳號鎖定（多次失敗） |
| 0x14 | 連線過多 |

## 🛠️ Opcode 分析工具

### IDA Pro (最推薦)
```
1. 反編譯 MapleStory.exe
2. 搜尋字串 "CClientSocket" 或 "OnPacket"
3. 分析 CClientSocket::ProcessPacket 函數
4. 每個 opcode 的 handler 通常命名為 On_Opcode_XXXX
```

### 交叉參照 (XREF) 使用
```
1. 在 IDA 中選中 opcode handler 函數
2. 按 X 查看所有交叉參照
3. 確認哪些函數呼叫了此 handler
4. 追蹤封包流向
```

### Opcode 映射表來源
- **HeavenMS** 開源碼：`src/tools/Integer_M_AString_Map.java`
- **Cosmic** 開源碼：`src/client/command/` 或 `src/net/server/`
- **RaGEZONE**: v83 Opcode 討論帖（需註冊）
- **MapleRef Wiki**: https://mapleref.fandom.com/wiki/Operation_Code

## 💡 實戰：分析怪物生成封包

### 抓包分析
```
假設我們抓到一個怪物生成封包：
00 01 15 00 xx xx yy yy zz zz mob_id_low mob_id_high ...

解析：
- 00 01 = Length (little-endian) = 272 bytes
- 15 00 = Opcode 0x15 (怪物生成)
- xx xx = X 座標 (short)
- yy yy = Y 座標 (short)
- zz zz = stance/frame (short)
- mob_id_low mob_id_high = 怪物 ID (little-endian short)
```

### IDA 分析
```
1. 在 IDA 中搜尋 "OnMobSpawn" 或 "CWvsContext::OnMobSpawn"
2. 分析函數的參數結構（CInPacket）
3. 比對每個 Decode1/Decode2/Decode4 的意義
```

## 📝 v83 Opcode 與其他版本差異

| 版本 | Opcode 長度 | 加密方式 |
|------|------------|---------|
| v62 ~ v83 | 2 bytes | AES-OFB + MapleCrypto |
| v90 ~ v117 | 2 bytes | AES-OFB + 動態金鑰 |
| v118+ | 4 bytes | AES-GCM (新加密) |

> v83 的好處是 opcode 已被完整映射，不需要像新版本那樣自行追蹤。
