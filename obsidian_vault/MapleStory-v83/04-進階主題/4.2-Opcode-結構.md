# 4.2 Opcode 結構分析

## 什麼是 Opcode？
Opcode (Operation Code) 是網路封包中用於標識操作類型的唯一識別符。

## v83 Opcode 特性
- **已高度文檔化**：v83 的 Opcode 幾乎全部被映射
- **使用 AES 加密**：需配合客戶端金鑰
- **高度客製化**：每個私服可能使用不同設定

## Opcode 結構
```
[封包頭部][Opcode (2-4 bytes)][資料...]
```

### 封包頭部格式
```
偏移  長度   內容
0x00  4      MapleCrypto 混淆頭部
      (包含長度 + 混淆 opcode)
```

### v83 Opcode 格式
- Opcode 為 **2 bytes** (little endian)
- 例如：Opcode `0x0001` 在封包中顯示為 `01 00`

## 🔑 MapleCrypto 頭部混淆機制

### 混淆原理
```
原始 opcode: 0x0001 (例如 LOGIN_PASSWORD)
       IV: [0x7A, 0x52, ...]

Step 1: XOR + SHIFT
  verify = (opcode ^ iv[0]) + (iv[1] << 8)
        = (0x0001 ^ 0x7A) + (0x52 << 8)
        = 0x007B + 0x5200
        = 0x527B

Step 2: 位旋轉 (rotate left 3 bits)
  verify = (verify << 3) | (verify >> 13)
        = 0x527B << 3 | 0x527B >> 13
        = 0x29BD8 | 0x00
        = 0x29BD8

Step 3: 取低位 16 bits
  verify = 0x29BD8 & 0xFFFF = 0x9BD8
```

## 📊 IDA 分析技巧

### 1. 找到關鍵函數
使用 IDA 分析 MapleStory.exe，搜尋：
- `CWvsContext::OnPacket`
- `SendPacket`
- `DecryptPacket`
- `MapleAESOFB` (加密類)

### 2. IDA 字串搜尋技巧
- 搜尋 `"opcode"` 或 `"Packet"` 相關字串
- 查找 StringPool 引用 (`push opcode 68`)
- AOB (Array of Bytes) 搜尋可找到 opcode 表

### 3. 分析流程
```
1. 找到 SendPacket/RecvPacket 函數
2. 分析封包加密邏輯 (MapleCrypto)
3. 提取 Opcode 表
4. 對比伺服器代碼
```

### 4. 靜態分析 vs 動態分析
- **靜態分析 (IDA)**：分析二進位碼，不執行
- **動態分析 (OllyDbg/CE)**：在執行時追蹤

## 🐾 MapleCrypto Java 實現參考
```java
// Cosmic / HeavenMS 的 MapleCrypto 實現關鍵片段
public class MapleCrypto {
    private final byte[] key;
    private final int iv;

    public byte[] getPacketHeader(int opcode) {
        int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
        verify = (verify << 3) | (verify >>> 5);
        verify &= 0xFFFF;
        return new byte[] {
            (byte) (verify & 0xFF),
            (byte) ((verify >>> 8) & 0xFF),
            (byte) (length & 0xFF),
            (byte) ((length >>> 8) & 0xFF)
        };
    }
}
```

## 🆕 IDA 分析Opcode - 進階技巧 (2024-2025)

### 使用 IDA 找到封包處理函數
RaGEZONE 教程：https://forum.ragezone.com/threads/getting-packet-structures-opcodes-using-ida.792436/

**分析關鍵**：
1. 找 `CClientSocket::ProcessPacket` - 客戶端接收封包的總入口
2. 從這裡根據 header 分發到 `CField::OnPacket` 或 `CLogin::OnPacket`
3. 這就是為什麼有 opcode 清單就能輕鬆找到客戶端接收封包的處理函數

### IDA 分析封包結構
RaGEZONE：https://forum.ragezone.com/threads/how-does-one-find-a-structure-of-a-packet-via-ida.1162654/

**分析流程**：
```
1. 找到 SendPacket / RecvPacket 函數
2. 分析封包加密邏輯 (MapleCrypto)
3. 提取 Opcode 表
4. 對比伺服器端代碼
```

### AOB (Array of Bytes) 搜尋
- 在 IDA 中使用 Hex View 搜尋已知 opcode 的 bytes 模式
- 可找到 opcode 表的記憶體位置
- 適用於：已知某個 opcode，想找它的引用位置

### 靜態分析 vs 動態分析
| 方法 | 工具 | 優點 | 缺點 |
|------|------|------|------|
| 靜態分析 | IDA Pro | 不需要執行，可看完整程式碼 | 無法追蹤執行時行為 |
| 動態分析 | OllyDbg, Cheat Engine | 可追蹤執行流程 | 需要脫殼後執行 |
