## 🔐 登入器 (Login Redirector) 原理

### 什麼是登入器？

登入器是一個 DLL 或外部程式，攔截客戶端的網路連線，將其導向私人伺服器，而非官方伺服器。

### 原理架構
```
原始流程：
  MapleStory.exe → connect("203.78.xx.xx:7575") → Nexon 伺服器

使用登入器：
  MapleStory.exe → Hook API (connect/WSAConnect)
                  → 攔截並修改目標 IP/Port
                  → connect("127.0.0.1:7575") → 私人伺服器
```

### 常見 Hook 點
| 函數 | 說明 |
|------|------|
| `connect()` | Winsock 連線函數 |
| `WSAConnect()` | Winsock SSL 連線函數 |
| `HttpSendRequest()` | HTTP 請求（部分客戶端） |
| `CClientSocket::Connect()` | 客戶端內部連線函數 |

### 登入器工具
| 工具 | 說明 |
|------|------|
| **NGS Bypass** | 繞過反作弊系統 + 封包攔截 |
| **MapleStory Packet Hack** | 經典封包攔截工具 |
| **自訂 DLL Injection** | 自己寫 DLL 注入 |

## 🔧 客戶端 IP 修改詳解 (HxD)

### 目標
將客戶端連接的目標從官方 IP 改為私人伺服器 IP

### 修改步驟
```
1. 開啟 MapleStory.exe（建議先備份）
2. Ctrl+F → 選擇 "Text string"
3. 搜尋 "127.0.0.1" 或 "203.78" 或 "nexon"
4. 找到後替換為你的伺服器 IP（如 "192.168.1.100"）
5. 確認長度一致（IP 字串長度要相同）
6. 存檔
```

### IP 搜尋技巧
```
搜尋字串時注意：
- 有些版本用 hex 表示 IP（如 7F 00 00 01 = 127.0.0.1）
- 某些字串可能被 Split 存放（如 "127" + ".0.0.1"）
- 需確認字串完整性，否則可能造成記憶體崩潰
```

### 推薦搜尋模式
| 搜尋目標 | 說明 |
|---------|------|
| `127.0.0.1` | 標準 localhost |
| `203.78` | 韓國 Nexon IP 前綴 |
| `66.235` | 部分國際版前綴 |
| `login.maplestory.nexon.com` | 域名形式（罕見） |

## 🔐 AES 金鑰與 MapleCrypto

### v83 加密架構
```
┌─────────────────────────────────────────────┐
│  MapleAESOFB (加密引擎)                      │
│  - bKey[32]: 256-bit AES 金鑰                │
│  - ivSend[4]: 發送 IV (客戶端視角)           │
│  - ivRecv[4]: 接收 IV                        │
│  - localize: 地區版本標誌                      │
└─────────────────────────────────────────────┘
```

### MapleCrypto 混淆演算法
```python
# MapleCrypto 頭部混淆（概念）
def maple_encrypt_header(opcode, iv):
    # 對 opcode 進行混淆，防止靜態分析
    verify = (opcode ^ iv[0]) + (iv[1] << 8)
    verify = (verify << 3) | (verify >> 5)  # 位旋轉
    return verify & 0xFFFF
```

### IV 協商流程
```
1. 客戶端生成隨機 SendIV (4 bytes)
2. 客戶端生成隨機 RecvIV (4 bytes)
3. 封包加密時使用：
   - C→S: 用 SendIV 初始化 AES
   - S→C: 用 RecvIV 初始化 AES
4. 伺服器收到 Hello Packet 後，用相同 IV 初始化加密引擎
```

### Hello Packet 完整格式
```
偏移  長度  內容
0x00  1     版本字串長度 (n)
0x01  n     版本字串 (如 "83")
n+1   4     SendIV (客戶端指定)
n+5   4     RecvIV (客戶端指定)
```

### 驗證 MapleCrypto 是否正確
```
徵兆：客戶端能連線但立刻斷線
原因：IV 或 AES 金鑰不同步

解決：檢查 MapleAESOFB.java 的：
  - KEY 常數
  - IV 初始化順序
  - 加密/解密方向
```

- 多層 VM 保護

## 🆕 Hendi's 4G Client - 啟用 4GB 記憶體 (RaGEZONE, 2023-2024)

### 什麼是 4G Flag？
Windows 32-bit 進程預設只能使用 2GB 記憶體。透過修改 PE header 的 "Large Address Space Aware" (LAA) 旗標，可以讓 MapleStory.exe 使用最多 4GB 記憶體。

### Hex 編輯位置
```
位置: 0000013E
預設值: 0f (預設)
修改值: 2f (啟用 4G)
```

### 為什麼這很重要？
- HD 客戶端需要更多記憶體
- 避免記憶體不足崩潰
- 穩定性提升

### 與 Ezorsia v2 搭配使用
```
1. 下載 Hendi's 4G Client (已預設啟用 4G flag)
2. 安裝後替換原 MapleStory.exe
3. 搭配 Ezorsia v2 dinput8.dll
4. 享受 HD + 穩定的遊戲體驗
```

### 手動 Hex 編輯驗證步驟
```
1. HxD 開啟 MapleStory.exe
2. Ctrl+G → 前往 0000013E
3. 確認目前值為 0f
4. 改為 2f
5. 存檔
```

## 🆕 Login Redirector 兩種方案對比 (RaGEZONE, Pixeltris)

### Pixeltris 提出的 127.0.0.1 專用方案

這個方案專門針對 127.0.0.1，不需要完整 Hook：
```
1. Hook connect() 函數
2. 檢查目標地址是否為 127.0.0.1
3. 如果是，直接連線（不改變）
4. 如果不是，覆寫為 127.0.0.1
```

### WinSock Hook 核心實作概念

```cpp
// 概念：Hook WSAConnect 實現 IP 重定向
typedef int (WSAAPI *WSAConnect_t)(SOCKET, const sockaddr*, int, LPWSABUF, LPWSABUF, LPWSABUF, LPQOS);
WSAConnect_t Original_WSAConnect;

int WSAAPI Hook_WSAConnect(SOCKET s, const sockaddr* name, int namelen, 
                            LPWSABUF lpCallerData, LPWSABUF lpCalleeData, 
                            LPQOS lpSQOS, LPQOS lpGQOS) {
    // 檢查是否為 MapleStory 連線
    if (name->sa_family == AF_INET) {
        sockaddr_in* addr = (sockaddr_in*)name;
        // 將目標 IP 改為 127.0.0.1
        addr->sin_addr.s_addr = inet_addr("127.0.0.1");
        addr->sin_port = htons(7575);
    }
    return Original_WSAConnect(s, name, namelen, lpCallerData, lpCalleeData, lpSQOS, lpGQOS);
}
```

- 使用普遍支援的 `cm.` 函數
- 避免使用特定音源專有的 Package 路徑
- 避免 `importPackage`（JDK 7+ 已廢棄）
- 測試時在目標音源上驗證

## 🆕 v83 MapleCrypto 完整解密流程

### CInPacket 封包接收完整流程

```cpp
void __thiscall CClientSocket::ProcessPacket(CClientSocket *this) {
    while (true) {
        BYTE header[4];
        recv(socket, header, 4);
        
        // MapleCrypto 解混淆
        int packetLen = MapleCrypto::decryptLength(header, ivRecv);
        int opcode = MapleCrypto::decryptOpcode(header, ivRecv);
        
        byte *buffer = new byte[packetLen];
        recv(socket, buffer, packetLen);
        
        // AES-OFB + Shanda 解密
        MapleCrypto::decrypt(buffer, packetLen, ivRecv);
        
        CInPacket packet;
        packet.nType = opcode;
        packet.buffer = buffer;
        
        switch (opcode) {
            case 0x00: CLogin::OnPacket(&packet); break;
            case 0x01: CLogin::OnLoginPassword(&packet); break;
        }
    }
}
```

### MapleCrypto 解密長度演算法

```java
public static int decryptLength(byte[] header, int iv) {
    int length = (header[2] & 0xFF) | ((header[3] << 8) & 0xFF00);
    length ^= (iv >> 16) & 0xFFFF;
    return length;
}
```

```cpp
int WSAAPI Hook_WSAConnect(SOCKET s, const sockaddr* name, int namelen, 
                            LPWSABUF lpCallerData, LPWSABUF lpCalleeData, 
                            LPQOS lpSQOS, LPQOS lpGQOS) {
    // 檢查是否為 MapleStory 連線
    if (name->sa_family == AF_INET) {
        sockaddr_in* addr = (sockaddr_in*)name;
        // 將目標 IP 改為 127.0.0.1
        addr->sin_addr.s_addr = inet_addr("127.0.0.1");

    }

}
```

```
    ↓

    ↓
    若在路由器後 → Port Forward: 7575 (login), 8484 (game)
```

**錯誤流程**：
```
❌ 直接用伺服器的 WZ.XML 覆蓋客戶端 WZ
```
