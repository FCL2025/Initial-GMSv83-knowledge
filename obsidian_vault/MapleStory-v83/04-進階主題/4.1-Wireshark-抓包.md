## 🛠️ Wireshark 基本操作

### 安裝
1. 下載：https://www.wireshark.org/
2. 安裝時選擇 WinPcap（Windows）

1. 選擇網路介面卡（Loopback 適配器或實際網卡）

3. 執行遊戲動作
4. 點擊 **Stop**

若要監控 localhost 流量，需要安裝 **Rawcap** 或 **npcap**：
```
npcap: https://npcap.com/
rawcap: https://www.netresec.com/?page=RawCap
```

## 🔍 過濾條件

### 基本過濾

| 過濾條件 | 說明 |
|----------|------|
| `tcp` | 只看 TCP 封包 |
| `tcp.port == 7575` |只看 port 7575（登入伺服器） |
| `tcp.port == 8484` | 只看 port 8484（遊戲伺服器） |
| `ip.addr == 127.0.0.1` |只看本地流量 |
| `tcp.port == 7575 && ip.addr == 127.0.0.1` | 本地登入流量 |

### 組合過濾

```
tcp.port == 7575 && ip.addr == 127.0.0.1
tcp.port == 8484 && ip.addr == 127.0.0.1
```

## 🔐 v83 封包加密機制深度分析

### MapleStory v83 加密流程

v83 版本使用 **AES-OFB 加密**配合 MapleStory 自定義加密：

```
明文封包：[Opcode: 2 bytes][Data: variable]
    ↓
MapleCrypto (頭部混淆 + IV 更新)
    ↓
AES-OFB 加密 (256-bit key)
    ↓
密文封包 → 傳輸
```

### 關鍵加密元件

| 元件 | 說明 |
|------|------|
| `MapleCrypto` | 會話金鑰混淆頭部（XOR + 位移） |
| `AES-OFB` | 256-bit 金鑰，Output Feedback 模式 |
| `SendIV/RecvIV` | 初始向量，連線時隨機生成並在 Hello Packet 中傳遞 |
| `Shanda` | 額外自定義加密層（某些 v83 客戶端有） |

### v83 封包加密特性
- **頭部 4 bytes**：長度(2) + 混淆Opcode校驗(2)
- **AES 金鑰**：256-bit (32 bytes)，靜態或動態協商
- **IV 協商**：在 Hello Packet（第一個未加密封包）中交換
- **第一個封包**：純文字，未加密

### v83 頭部格式
```
[Length: 2 bytes][Encrypted Opcode+Data: variable]
```
頭部包含：長度 + Opcode (經 MapleCrypto 混淆) + Checksum

## 🤝 登入握手流程（Hello Packet）詳解

### v83 完整登入流程

```
1. C → S: Hello Packet (未加密)
   - MAPLE_VERSION (如 "83")
   - 客戶端發送 SendIV / RecvIV (各4 bytes)
   - 請求登入

2. S → C: 握手回應 (未加密)
   - 伺服器確認 IV
   - 之後所有封包開始加密

3. C → S: 登入請求 (Opcode 0x05) - 已加密
   [Opcode 0x05][帳號][密碼][版本]

4. S → C: 登入回應 (Opcode 0x06) - 已加密
   [Opcode 0x06][結果][錯誤碼][伺服器列表]

5. C → S: 選擇伺服器 (Opcode 0x09) - 已加密

6. S → C: 進入遊戲 (Opcode 0x0B) - 已加密
```

### Hello Packet 結構（客戶端 → 伺服器）
```
[Version length: 1][Version: n bytes]
[SendIV: 4 bytes]    // 客戶端生成的 IV
[RecvIV: 4 bytes]    // 希望伺服器使用的 IV
```
在 Java 實現中：
```java
// MapleAESOFB.java v83
public MapleAESOFB(byte[] key, boolean localize) {
    this.bKey = new byte[32]; // 256-bit key
    this.ivSend = new byte[4];
    this.ivRecv = new byte[4];
    // IV 由 Hello Packet 中的隨機值初始化
}
```

## 🛠️ Maple_Pshark 使用教學 (v83 專用)

GitHub: https://github.com/obstriker/Maple_Pshark

### 功能
1. **攔截 Recv/Send 封包** - 掛鉤加密函數，完整解密
2. **修改封包** - 動態修改封包內容，無需重新加密
3. **注入封包** - 發送自訂封包到客戶端或伺服器
4. **規則過濾** - 基於規則的封包處理

### 使用方式
```
1. 執行 Maple_Pshark.exe
2. 選擇 localhost IP（127.0.0.1）
3. 設定監聽端口（8484）
4. 點擊 Start
5. 啟動 MapleStory 客戶端
6. 在 Maple_Pshark 中查看捕獲的封包
```

### Maple_Pshark 截圖預覽
- 可即時查看 Recv/Send 封包
- 支援 hex 和明文兩種檢視模式
- 支援斷點（breakpoint）功能

## 🛠️ 抓包分析工具比較

| 工具 | 優點 | 缺點 |
|------|------|------|
| **Wireshark** | 功能完整，支援所有協定 | 需處理加密 |
| **Maple_Pshark** | v83 專用，自動解密，可修改/注入 | 僅支援 v83 |
| **Charles Proxy** | 支援 HTTP，解密方便 | 不支援原生 TCP |
| **MaplePacketPuller** | Python 封包分析器 | 需自行處理加密 |
| **MaplePE** | 封包編輯器，可發送自訂封包 | 介面較舊 |

### 推薦工具鏈 (v83)
```
1. Maple_Pshark (主要攔截工具)
   - GitHub: obstriker/Maple_Pshark
   - 功能：攔截 Recv/Send 封包，修改，注入，規則過濾
   - 支援 v83 加密

2. Wireshark + npcap (輔助分析)
   - 監控 TCP 層級流量
   - 確認是否有異常封包

3. MaplePE (封包編輯器)
   - 解析封包結構
   - 發送自訂封包
```

## 💡 範例：分析登入封包（v83）

### 步驟 1：過濾流量
```
tcp.port == 7575 && ip.addr == 127.0.0.1
```

### 步驟 2：捕獲 Hello Packet

**第一個封包（客戶端→伺服器）- 未加密：**
```
xx xx           // 版本長度
"83"            // 版本號
[SendIV: 4 bytes]
[RecvIV: 4 bytes]
```

**第二個封包（伺服器→客戶端）- 未加密：**
```
00              // 握手成功標誌
```

### 步驟 3：捕獲加密封包

**客戶端 → 伺服器 (Opcode 0x05)**
```
05 00           // Opcode: 登入請求（經 MapleCrypto 混淆）
[帳號字串: null-terminated]
[密碼字串: null-terminated]
[版本校驗]
```

**伺服器 → 客戶端 (Opcode 0x06)**
```
06 00           // Opcode: 登入回應
[結果代碼: 1 byte]
  00 = 成功
  05 = 帳號或密碼錯誤
  13 = 帳號被鎖定
[錯誤訊息: 如果失敗]
```

**伺服器 → 客戶端 (Opcode 0x08)**
```
08 00           // Opcode: 伺服器列表
[伺服器數量: 1 byte]
  [伺服器名稱: string]
  [IP: string]
  [Port: 2 bytes (little-endian)]
  ...重複...
```

### 步驟 4：常見問題排查

| 問題 | 可能原因 | 解決方式 |
|------|---------|---------|
| 無法登入 | IP 修改錯誤 | 用 HxD 檢查 127.0.0.1 是否正確 |
| 連線超時 | 防火牆阻擋 | 開放 7575/8484 端口 |
| 版本錯誤 | 客戶端版本不符 | 確認 MAPLE_VERSION 常數 |
| 加密失敗 | AES IV 不同步 | 確認 MapleAESOFB IV 與客戶端一致 |
| Hello Packet 失敗 | 金鑰版本不匹配 | 檢查 MapleCrypto 版本 |

## 🔧 本地流量截取方案

### 方案 1：Maple_Pshark（推薦 v83）
直接監聽加密函數，自動解密，無需處理 localhost 問題

### 方案 2：Rawcap + Wireshark
```batch
rawcap -p 127.0.0.1 output.pcap
```

### 方案 3：Wireshark + npcap
安裝 npcap 後選擇 `Adapter for loopback traffic`

### 方案 4：修改客戶端連接外部 IP
使用 HxD 將 `127.0.0.1` 替換為實際 IP（如 `192.168.1.100`），然後監控該 IP 的流量

### IP 修改步驟 (HxD)
```
1. 開啟 MapleStory.exe
2. Ctrl+F 搜尋 "127.0.0.1"
3. 替換為目標 IP（如 192.168.1.100）
4. 注意長度一致！（7 bytes）
5. 存檔
```
