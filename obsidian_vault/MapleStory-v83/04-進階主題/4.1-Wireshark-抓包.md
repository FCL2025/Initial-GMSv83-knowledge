# 4.1 Wireshark 抓包教學

## 🎯 目的

分析 MapleStory 網路封包，理解遊戲通訊協定的運作方式

## 🛠️ Wireshark 基本操作

### 安裝
1. 下載：https://www.wireshark.org/
2. 安裝時選擇 WinPcap（Windows）

### 開始捕獲

1. 選擇網路介面卡（Loopback 適配器或實際網卡）
2. 點擊 **Start**
3. 執行遊戲動作
4. 點點擊 **Stop**

### Windows Loopback 設置
若要監控 localhost 流量，需要安裝 **Rawcap** 或 **npcap**：
```
npcap: https://npcap.com/
rawcap: https://www.netresec.com/?page=RawCap
```

## 🔍 過濾條件

### 基本過濾

| 過濾條件 | 說明 |
|----------|------|
| `tcp` | 只看 TCP 封包 |
| `tcp.port == 7575` |只看 port 7575 |
| `ip.addr == 192.168.1.1` |只看特定 IP |
| `tcp.port == 8484` | 客戶端端口 |

### 組合過濾

```
tcp.port == 7575 && ip.addr == 127.0.0.1
tcp.port == 8484 && ip.addr == 127.0.0.1
```

## 📊 分析封包

### 查看內容

1. 點擊一個封包
2. 查看 **Packet Details**（封包詳情）
3. 查看 **Packet Bytes**（封包位元組）

### 找出 Opcode

1. 找到 TCP Stream（追蹤 TCP 流）
2. 分析封包結構
3. 記錄 Opcode 和資料格式

## 🔐 v83 封包加密機制深度分析

### MapleStory v83 加密流程

v83 版本使用 **AES 加密**配合 MapleStory 自定義加密：

```
明文封包：[Opcode: 2 bytes][Data: variable]
    ↓
頭部加密 (MapleCrypto)
    ↓
傳輸層加密 (AES)
    ↓
密文封包
```

### 關鍵加密元件

| 元件 | 說明 |
|------|------|
| `MapleCrypto` | 單向會話金鑰，用於頭部混淆 |
| `AES` | 128-bit AES-ECB，用於內容加密 |
| `SendIV/RecvIV` | 初始向量，每個方向不同 |

### v83 頭部格式
```
[Encrypted Header: 4 bytes][Encrypted Data: variable]
```
頭部包含：長度(2) + Opcode(2) + Checksum

### 登入階段（未加密）
```
第一個封包（客戶端→伺服器）：純文字
[版本資訊][帳號][密碼]
```

### 遊戲階段（加密）
```
COutPacket::Encode4(length)
COutPacket::Encode2(opcode)
COutPacket::EncodeBuffer(data)
```

## 🛠️ 抓包分析工具比較

| 工具 | 優點 | 缺點 |
|------|------|------|
| **Wireshark** | 功能完整，支援所有協定 | 需處理加密 |
| **Charles Proxy** | 支援 HTTP，解密方便 | 不支援原生 TCP |
| **Fiddler** | Windows 友好 | 同上 |
| **netstat** | 命令列快速查看連接狀態 | 無法分析內容 |
| **Rawcap** | 可截取 localhost 流量 | 需後續分析 |

### 推薦工具鏈 (v83)
```
1. Rawcap (截取 localhost)
   或
   Wireshark + npcap (直接監控)

2. Maple_Pshark (v83 專用攔截工具)
   - GitHub: obstriker/Maple_Pshark
   - 功能：攔截 Recv/Send 封包
   - 可修改和注入封包
   - 支援 v83 加密

3. MaplePE (封包編輯器)
   - GitHub: zhyonc/MaplePE
   - 解析封包結構
   - 發送自訂封包
```

## 💡 範例：分析登入封包

### 步驟 1：過濾流量
```
tcp.port == 7575 && ip.addr == 127.0.0.1
```

### 步驟 2：捕獲登入流程

**客戶端 → 伺服器 (Opcode 0x05)**
```
05 00                        // Opcode: 登入請求
[帳號字串: null-terminated]
[密碼字串: null-terminated]
[客戶端版本]
```

**伺服器 → 客戶端 (Opcode 0x06)**
```
06 00                        // Opcode: 登入回應
[結果代碼: 1 byte]
[錯誤訊息: 如果失敗]
```

**伺服器 → 客戶端 (Opcode 0x08)**
```
08 00                        // Opcode: 伺服器列表
[伺服器數量: 1 byte]
  [伺服器名稱: string]
  [IP: string]
  [Port: 2 bytes]
  ...重複...
```

### 步驟 3：分析常見問題

| 問題 | 可能原因 | 解決方式 |
|------|---------|---------|
| 無法登入 | IP 修改錯誤 | 檢查 HxD 修改 |
| 連線超時 | 防火牆阻擋 | 開放 7575/8484 |
| 版本錯誤 | 客戶端版本不符 | 確認伺服器版本設定 |
| 加密失敗 | AES 金鑰不同步 | 重啟客戶端和伺服器 |

## 🔧 本地流量截取方案

### 方案 1：Rawcap (推薦)
```batch
rawcap -p 127.0.0.1 output.pcap
```

### 方案 2：Wireshark + npcap
安裝 npcap 後選擇 `Adapter for lookback traffic` 或實際網卡

### 方案 3：修改客戶端連接外部 IP
使用 HxD 將 `127.0.0.1` 替換為實際 IP（如 `192.168.1.100`），然後監控該 IP 的流量

### IP 修改步驟 (HxD)
```
1. 開啟 MapleStory.exe
2. Ctrl+F 搜尋 "127.0.0.1"
3. 替換為目標 IP（如 192.168.1.100）
4. 注意長度一致！
5. 存檔
```

## 📚 Maple_Pshark 使用教學 (v83 專用)

GitHub: https://github.com/obstriker/Maple_Pshark

### 功能
1. **攔截 Recv 封包** - 伺服器發送到客戶端的封包
2. **修改封包** - 動態修改封包內容
3. **注入封包** - 發送自訂封包到客戶端或伺服器
4. **規則過濾** - 基於規則的封包處理

### 使用方式
```
1. 執行 Maple_Pshark.exe
2. 選擇 localhost IP（127.0.0.1）
3. 設定監聽端口（8484）
4. 點擊 Start
5. 啟動 MapleStory 客戶端
6. 在 Maple_Pshark 中查看捕獲的封包
```
