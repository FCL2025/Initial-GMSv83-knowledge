# 5.6 WzImg-MCP-Server — AI 輔助 WZ 分析工具 (2026-03-05 新增)

## 概述
WzImg-MCP-Server 是一個基於 **Model Context Protocol (MCP)** 的伺服器，提供 74 個工具讓 AI 助手直接與 MapleStory IMG 檔案互動。

**GitHub**: https://github.com/lastbattle/WzImg-MCP-Server

## 核心功能
- **74 tools across 10 categories** 供 AI 助手互動 MapleStory IMG 檔案
- AI 可以直接瀏覽提取的 IMG 數據、檢查屬性、搜尋資源、導出圖片/音頻、進行結構化編輯

## 工具類別總覽

| 類別 | 工具數 | 功能 |
|------|--------|------|
| FileTools | 7 | 資料來源初始化、目錄掃描 |
| NavigationTools | 6 | 導航、列出類別/圖片 |
| PropertyTools | 11 | 屬性讀取、搜尋、批次取得 |
| ImageTools | 9 | Canvas、動畫幀讀取 |
| AudioTools | 4 | 聲音元數據和數據讀取 |
| ExportTools | 6 | JSON/XML/PNG/MP3 導出 |
| AnalysisTools | 6 | 統計、UOL 損壞檢測、屬性比較 |
| ModifyTools | 14 | 屬性修改、圖片替換、導入導出 |
| BatchTools | 4 | 批量導出、批量搜尋 |
| LifecycleTools | 6 | 圖片解析/卸載/預加載 |

## 使用範例

### Claude Desktop 配置
```json
{
  "mcpServers": {
    "wzimg": {
      "command": "dotnet",
      "args": ["run", "--project", "E:/path/to/WzImg-MCP-Server/WzImgMCP.csproj"],
      "env": {
        "WZIMGMCP_DATA_PATH": "D:\\Extract\\v83"
      }
    }
  }
}
```

### 實用工具範例

```csharp
// 搜尋屬性
search_by_name(pattern: "*lvmax*", category: "Quest")
search_by_value(value: 100, type: "Int")

// 導出
export_png(path: "Character/00002000.img/info/icon")
export_to_json(path: "Npc/9000000.img")

// 批量操作
batch_export_images(category: "Map", outputPath: "D:\\Export\\Maps")
batch_search(pattern: "*skill*", categories: ["Skill", "Character"])
```

## 與 HaRepacker 的比較

| 特性 | HaRepacker | WzImg-MCP-Server |
|------|-----------|------------------|

| 互動方式 | 手動操作 | AI 指令 |

## AI 輔助 WZ 分析工作流程

```
Step 1: 設定 WzImg-MCP-Server
  → 確認資料路徑指向 v83 IMG 目錄
  → 啟動 MCP 伺服器

Step 2: AI 分析 WZ 結構
  → 搜尋目標屬性（如 lvmax）
  → 比較不同版本的數值

Step 3: 批量導出
  → 導出需要修改的 IMG 檔案
  → 或直接進行修改

Step 4: 驗證
  → 確認修改後的 WZ 能正確載入
```

## 🆕 NX Format — WZ 的高效能替代方案 (2026-02 新增)

**來源**: RaGEZONE - https://forum.ragezone.com/threads/wz-and-nx-file-questions.1260605/

### NX vs WZ 比較
| 特性 | WZ | NX |
|------|-----|-----|
| 效能 | 較慢 | ✅ 更快 |
| 檔案大小 | 較大 | ✅ 更小 |
| 工具支援 | HaRepacker 直接支援 | 需要轉換 |
| 主流程度 | 廣泛使用 | 官方伺服器偏好 |

### 使用方式
- 使用 HaRepacker 開啟 .wz 檔案
- 可轉換為 .nx 格式
- 詳見 RaGEZONE NX 格式說明帖：
  - https://forum.ragezone.com/threads/the-nx-format.891992/
  - https://forum.ragezone.com/threads/converted-nx-files-for-maplestory.891937/

### NX 轉換注意事項
- NX 是二進制格式，比 XML 更緊湊
- 轉換需要完整重新導出
- 並非所有工具都支援 NX

## 🆕 MaplePacketPuller — Python 封包結構分析 (2026-03 更新)

**GitHub**: https://github.com/Bratah123/MaplePacketPuller

| 特色 | 說明 |
|------|------|
| Python 分析工具 | 跨平台 |
| 批量封包分析 | 支援 .pcap 檔案 |
| 自動識別 opcode | 輸出結構化結果 |
| 輸出 Decode 函數 | 可直接用於伺服器端實作 |

### SpiritIDAPlugin 支援環境
| 環境 | 支援狀態 |
|------|---------|
| Python 3.8.5 | ✅ 測試通過 |
| Python 3.6.12 & 3.8.6 | ✅ 測試通過 |
| IDA Pro 7.0 (32-bit) | ✅ 測試通過 |
| IDA Pro 7.5 | ✅ 測試通過 |
| IDA Pro 6.8 | ✅ 測試通過 |
| Python 2.7 | ❌ 不相容 (os.scandir) |

### 完整工作流程
```
Step 1: IDA Pro 反編譯
  → 在 IDA 中找到 CField::OnPacket 或 CLogin::OnPacket
  → 產生 pseudocode

Step 2: 複製 pseudocode
  → 在 Functions/ 目錄建立 .txt 檔案
  → 貼上 IDA 產生的 C-style pseudocode

Step 3: Python 分析
  → cd MaplePacketPuller/IDA\ Maple\ Script/src/main/python
  → python main.py
  → 輸入分析類型（完整/標準）

Step 4: 輸出結果
  → 封包結構自動寫入 FuncOutput/ 目錄
  → 控制台顯示 Decode1/Decode2/Decode4/DecodeStr 等欄位
```
