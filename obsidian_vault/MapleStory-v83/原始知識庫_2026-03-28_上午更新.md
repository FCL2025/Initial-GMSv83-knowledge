### localhost.exe IP 修改流程
```
1. 使用 STREDIT 開啟 localhost.exe
2. 選擇 "No" (v83 非 v118+)
3. 選擇 "Yes" (IP 搜尋模式)
4. 點擊 "Force to a IP"
5. 輸入目標 IP (如 127.0.0.1 或 VPS IP)
6. 保存
```

## 🔍 抓包分析 — 加密機制詳解

### MapleStory v83 封包加密三層架構

```
┌─────────────────────────────────────────────────────────────┐
│  1️⃣ MapleCrypto (頭部混淆)                                   │
│     - 4 bytes header: [長度(2) + 混淆 opcode(2)]            │
│     - 防止靜態分析 opcode                                   │
│                                                            │
│  2️⃣ Shanda Encryption (IOBufferManipulator)                │
│     - 函數: IOBufferManipulator.Encrypt(pBuff)             │
│     - AES 之前應用                                          │
│     - v118+ 已移除                                          │
│                                                            │
│  3️⃣ AES-OFB (主加密)                                      │
│     - 256-bit key, 4-byte IV                               │
│     - 版本決定的靜態密鑰                                    │
└─────────────────────────────────────────────────────────────┘

發送流程: 明文 → Shanda Encrypt → AES-OFB → MapleCrypto header → 網路
```

### Hello Packet (第一個封包 — 純文字)

**客戶端 → 伺服器**:
```
偏移    長度   內容
0x00    2      封包總長度 (little endian)
0x02    2      Maple 版本號 (0x0053 = 83)
0x04    n      版本字串 ("83")
n+4     4      SendIV (客戶端發送 IV)
n+8     4      RecvIV (客戶端接收 IV)
n+12    2      地區標誌 (MAPLE_LOCALE)
n+14    2      Unknown
```

**伺服器 → 客戶端**:
```
偏移    長度   內容
0x00    2      封包總長度
0x02    2      Maple 版本號
0x04    n      版本字串
n+4     4      RecvIV (伺服器回應)
n+8     4      SendIV (伺服器回應)
n+12    2      地區標誌
n+14    2      Unknown
```

### 重要原則
> **第一個發送/接收的封包永遠是純文字（未加密）**

## 📋 Opcode 結構 — v83

### 封包格式
```
[Header (4 bytes)][Opcode (2 bytes)][Data (變長)]
```

### Header 格式
- 2 bytes: 封包長度 (little endian)
- 2 bytes: 混淆後的 opcode

### 主要 Opcode (v83)

| Opcode | 名稱 | 方向 |
|--------|------|------|
| 0x01 | LOGIN_REQUEST | C→S |
| 0x02 | LOGIN_RESPONSE | S→C |
| 0x03 | CHECK_PASSWORD | C→S |
| 0x04 | CHECK_PASSWORD_RESULT | S→C |
| 0x0B | CHANGE_MAP | C→S |
| 0x0C | CHANGE_MAP_RESULT | S→C |
| 0x2E | SPAWN_PORTAL | S→C |

### 加密 Key 協商
- 兩個 Key: 一個用於加密發送封包，一個用於解密接收封包
- Key 在 Hello Packet 中隨機生成並傳遞

## 🤖 NPC 腳本編寫 — 完整指南

### 腳本存放位置
```
source/scripts/npc/
```

### 基礎模板 (status 方式)
```javascript
var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1 || mode == 0) {
        cm.dispose();
        return;
    }
    
    if (mode == 1) {
        status++;
    } else {
        status--;
    }
    
    switch(status) {
        case 0:
            cm.sendNext("你好！歡迎來到...");
            break;
        case 1:
            cm.sendSimple("請選擇:\n #b#L0#購買道具#l\n #L1#出售道具#l");
            break;
        case 2:
            if (selection == 0) {
                // 購買邏輯
            }
            cm.dispose();
            break;
    }
}
```

### 常用 cm 方法
| 方法 | 說明 |
|------|------|
| cm.sendNext(text) | 發送下一頁對話 |
| cm.sendOk(text) | 發送確定對話 |
| cm.sendSimple(text) | 發送選擇對話 |
| cm.sendYesNo(text) | 發送 Yes/No 對話 |
| cm.sendBackNext(text) | 發送返回/下一頁 |
| cm.sendBackOk(text) | 發送返回/確定 |
| cm.dispose() | 結束對話 |
| cm.getPlayer() | 獲取玩家對象 |
| cm.getNpc() | 獲取 NPC ID |

### 條件判斷範例
```javascript
// 等級檢查
if (cm.getPlayer().getLevel() >= 50) {
    cm.sendOk("你已達到 50 級！");
} else {
    cm.sendOk("你需要 50 級才能使用此功能。");
}

// 地圖檢查
if (cm.getPlayer().getMapId() == 100000000) {
    cm.sendOk("你在射手村！");
}

// 道具檢查
if (cm.haveItem(4000000, 1)) {
    cm.sendOk("你有一把木劍！");
} else {
    cm.sendOk("你沒有木劍！");
}
```

### 給予/扣除道具
```javascript
// 給予道具
cm.gainItem(4000000, 1);  // 給予 1 把木劍

// 扣除道具
cm.gainItem(4000000, -1); // 扣除 1 把木劍
```

### 開啟商店
```javascript
function start() {
    cm.openShop(123);  // 商店 ID
    cm.dispose();
}
```
