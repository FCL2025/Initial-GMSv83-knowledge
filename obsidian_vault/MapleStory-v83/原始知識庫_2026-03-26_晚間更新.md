### 1.2 Hex 編輯器 IP 修改步驟（HxD 詳細流程）

**目標：** 將客戶端連線目標從 `127.0.0.1` 改為你的伺服器 IP

**步驟：**
1. 用 HxD 開啟客戶端 `MapleStory.exe`（建議先備份）
2. `Ctrl+F` → 選擇 "Text (UTF-8)" → 輸入 `127.0.0.1`
3. 會找到**三個連續出現**的位置
4. 將第一個 `127` 前面，游標放在 `1` 前面
5. 輸入你的 IP 位址（覆蓋現有內容）
6. 對其他兩個位置重複同樣操作
7. 儲存

**注意：**
- 備份原始檔案！
- IP 長度可能不同（如 `192.168.1.100` vs `203.0.113.1`）
- 某些客戶端可能將 IP 分散在不同位置

**Cosmic-client 官方說明：**
```
Cosmic-client 說明：客戶端有時啟動困難。如果 server logs 顯示
"Client connected (...)" 但客戶端沒有啟動或顯示錯誤，
請嘗試快速連續雙擊客戶端。
```

```
2. File > Open new file... → 選擇 MapleStory.exe
3. 出現三個 IP 欄位（Configured IPs），全部改為你的 IP
4. Ctrl+S 儲存
```

### 1.5 Themida 脫殼流程（OreansUnvirtualizer + OllyDbg）

**環境需求：**
- Windows 7 x86 SP1（建議 VM 虛擬機）
- OllyDbg v1.10 + 插件
- OreansUnvirtualizer 插件

**流程：**
1. OllyDbg 載入加殼客戶端
2. 插件選單 → OreansUnvirtualizer → Unvirtualize
3. 等待自動找到入口點（EP）
4. Dump Process 或 Fix IAT
5. 儲存為新檔案（unpacked.exe）

**⚠️ 注意：** Themida 是商業加殼器，完整脫殼可能涉及法律問題。建議使用 MapleEzorsia-v2 代替。

### 1.6 LAN 多人連線錯誤 0x80004005 (-2147467259)

**症狀：** 同一網路中其他電腦無法連線到伺服器

**原因：** 防火牆或權限問題

**解決方案：**

```batch
# 測試用：關閉 Windows 防火牆（所有設定檔）
netsh advfirewall set allprofiles state off
```

**確認伺服器監聽設定：**
- 確認 listen 為 `0.0.0.0`（而非 `127.0.0.1`）
- 檢查 `ServerConstants.java` 或對應設定檔

**使用 Hamachi 的替代方案：**
1. 安裝 Hamachi
2. 建立虛擬區域網路
3. 使用 Hamachi IP（而非 public IP）
4. 雙方都加入同一 Hamachi 網路

**功能列表：**

2. 修改封包內容後再轉發

4. 基於規則的封包操作/過濾

**使用場景：**

- 測試自定義 Opcode 處理
- 排查登入/連線問題

### 2.3 MaplePacketPuller + SpiritIDAPlugin 工作流

**完整工作流：**

```
IDA Pro 反編譯客戶端
    ↓
SpiritIDAPlugin 格式化 pseudocode
    ↓
複製到 .txt 檔案
    ↓
MaplePacketPuller Python 分析
    ↓
輸出 packet structure
```

**SpiritIDAPlugin：** IDA Pro 插件，直接將反編譯出來的 pseudocode 格式化為適合伺服器端 encode 的形式。

**MaplePacketPuller 功能：**
- 讀取 IDA 產生的 .txt 偽代碼
- 分析 Decode 函數
- 輸出 packet structure
- 搜尋 InHeader opcodes

**⚠️ 限制：**
- while() 迴圈處理不完全準確（可用 `GET_ALL_DECODES = true` 提高準確度）
- 不顯示 switch case（可手動添加）
- 需要 Python 3.6+

### 2.4 IDA Pro + Wireshark 組合分析流程

**第一步：Wireshark 捕獲流量**

過濾條件：
```

```

**第二步：定位封包對應的 IDA 函數**

在 IDA 中搜尋關鍵字：
```
Alt+T → 搜尋 "LOGIN" 或 "username"
```

**第三步：分析 Decode 函數**

使用 `CInPacket::Decode*` 系列函數讀取封包欄位。

## 三、Opcode 結構

### 3.1 CInPacket Decode 函數 IDA 完整對照表

| IDA 函數 | Java 實現（伺服器發送）| 大小 | 範例 |
|---------|---------------------|------|------|
| `Decode1` | `mplew.write(1)` | 1 byte | `type`, `direction` |
| `Decode2` | `mplew.writeShort()` | 2 bytes | `opcode`, `mapId` |
| `Decode4` | `mplew.writeInt()` | 4 bytes | `objectId`, `damage` |
| `DecodeBuffer(x, 8)` | `mplew.writeLong()` | 8 bytes | `timestamp` |
| `DecodeBuffer(x, 13)` | `mplew.writeAsciiString(str, 13)` | 13 bytes | 固定長度字串 |
| `DecodeStr` | `mplew.writeMapleAsciiString()` | 可變 | `name`, `message` |

### 3.2 IDA 分析Opcode 關鍵函數

| IDA 函數 | 處理範圍 |
|---------|---------|
| `CClientSocket::ProcessPacket` | 總入口，所有進來流量 |
| `CLogin::OnPacket` | 登入相關封包（opcode < 0x100）|
| `CWvsContext::OnPacket` | 客戶端上下文封包 |
| `CField::OnPacket` | 遊戲地圖相關封包（opcode >= 0x100）|
| `CInPacket::Decode1/2/4` | 讀取 byte/short/int |
| `CInPacket::DecodeStr` | 讀取字串 |
| `CInPacket::DecodeBuffer` | 讀取原始 bytes |

### 3.3 v83 登入結果碼（LOGIN_STATUS state）

| 結果碼 | 意義 | 說明 |
|--------|------|------|
| 0 | 登入成功 | 包含 accountId |
| 5 | 帳號或密碼錯誤 | 最常見的登入失敗原因 |
| 7 | 帳號已停用 | 帳號被-ban 或停權 |
| 13 | 刪除中（冷靜期）| 角色正在刪除中，無法登入 |
| 23 | 登入太頻繁 | 短時間內嘗試次數過多 |

**除錯提示：** 如果 Wireshark 看到 state=5，先確認：
- 密碼是否正確
- 資料庫 accounts 表是否正確
- 密碼是否使用正確的加密方式儲存

### 4.2 HaRepacker WZ 編輯黃金法則

**⚠️ 核心原則：**

1. **客戶端 WZ 使用 GMS (old) 加密**（v83 適用）
2. **禁止刪除/重新導入節點**（會造成數據丟失）
3. 正確流程：**編輯客戶端 WZ → Private Server... 導出 XML → 覆蓋伺服器 wz 目錄**
4. **千萬不要**將伺服器 XML 直接匯入客戶端 WZ！

**編輯流程：**
```
1. HaRepacker-resurrected 打開客戶端 WZ（GMS old 加密）
2. 找到要編輯的節點，修改值
3. 點 "Apply Changed Value"
4. File > Save
5. Tools > Export Selection > XML > Private server
6. 匯出到伺服器 wz/ 目錄
7. 重啟伺服器
```

### 5.4 動態配方系統腳本源

```javascript
/*
    動態配方系統 - 無需 SQL
    適合：道具製作、材料兌換
*/

var recipes = [
    {
        name: "楓之葉地圖",
        materials: [[4000000, 10]],  // [物品ID, 數量]
        result: [[2300000, 1]],
        meso: 1000
    },
    {
        name: "神秘強化石",
        materials: [[4000000, 50], [4000001, 20]],
        result: [[2340000, 1]],
        meso: 5000
    }
];

var status = 0;
var selectedRecipe = -1;

function start() {
    var selStr = "請選擇配方：#b\r\n";
    for (var i = 0; i < recipes.length; i++) {
        selStr += "#L" + i + "#" + recipes[i].name + "#l\r\n";
    }
    cm.sendSimple(selStr);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
        return;
    }
    if (mode == 0) {
        status--;
    } else {
        status++;
    }

    if (status == 0) {
        selectedRecipe = selection;
        var recipe = recipes[selection];
        var matStr = "材料需求：\r\n";
        for (var i = 0; i < recipe.materials.length; i++) {
            var m = recipe.materials[i];
            matStr += "#t" + m[0] + "# x" + m[1] + "\r\n";
        }
        matStr += "金幣：" + recipe.meso + "\r\n";
        matStr += "確定製作？";
        cm.sendYesNo(matStr);
    } else if (status == 1) {
        var recipe = recipes[selectedRecipe];
        
        // 檢查金幣
        if (cm.getMeso() < recipe.meso) {
            cm.sendOk("金幣不足！需要 " + recipe.meso + " 金幣。");
            cm.dispose();
            return;
        }
        
        // 檢查材料
        for (var i = 0; i < recipe.materials.length; i++) {
            var m = recipe.materials[i];
            if (!cm.haveItem(m[0], m[1])) {
                cm.sendOk("材料不足：#t" + m[0] + "#");
                cm.dispose();
                return;
            }
        }
        
        // 扣除
        cm.gainMeso(-recipe.meso);
        for (var i = 0; i < recipe.materials.length; i++) {
            var m = recipe.materials[i];
            cm.gainItem(m[0], -m[1]);
        }
        
        // 給予成品
        for (var i = 0; i < recipe.result.length; i++) {
            var r = recipe.result[i];
            cm.gainItem(r[0], r[1]);
        }
        
        cm.sendOk("製作成功！");
        cm.dispose();
    }
}
```
