## 🆕 MapleStory v83 加密機制最新研究 (maplelib/Go, maplesecrets 2026)

### 完整加密流程 (v83)

MapleStory 使用**兩組金鑰**：
1. **Send Key** — 客戶端加密、伺服器解密
2. **Recv Key** — 伺服器加密、客戶端解密

兩組金鑰在連線時透過 Hello Packet 協商（明文傳輸），之後所有通訊皆加密。

### 三層加密架構 (v83)

```
明文封包
    ↓
Step 1: MapleCrypto (版本特定 XOR + 移位)
    ↓
Step 2: AES-OFB (主要對稱加密)
    ↓
Step 3: Shanda 加密 (早期版本有，v83 仍有殘留)
    ↓
網路傳輸
```

**v83 移除 Shanda 的版本**：`MapleCustomEncryption` 取代了 Shanda。

### 第一個封包 (Hello Packet) 重要原則

> "The first packet sent / received will always be in plain-text (not encrypted)"

第一個封包是**明文**，用於交換 AES IV，之後所有封包皆加密。

### maplelib (Go) 加密實現要點

**來源**: https://pkg.go.dev/github.com/defaultmagi/maplelib

```go
// 兩組金鑰
type MapleCrypto struct {
    sendKey []byte  // 客戶端發送/伺服器接收
    recvKey []byte  // 伺服器發送/客戶端接收
    sendIV  []byte
    recvIV  []byte
}

// 握手流程
func (c *MapleCrypto) HelloPacket() []byte {
    // 回傳版本號 + 初始 IV（明文）
}
```

### AES Key 和 IV 協商流程

```
1. 客戶端連線 → 發送版本號 (明文)
2. 伺服器驗證版本 → 回傳 serverIV (明文)
3. 客戶端生成 clientIV → 回傳 (明文)
4. 之後所有封包：
   - 發送時：明文 → MapleCrypto → AES-OFB → 密文
   - 接收時：密文 → AES-OFB → MapleCrypto → 明文
```

### MapleCrypto 混淆原理

```java
// Opcode 混淆公式（v83）
int verify = (opcode ^ iv[0]) + (iv[1] << 8);
verify = (verify << 3) | (verify >> 13);  // rotate left 3 bits
verify = verify & 0xFFFF;
```

## 🆕 MapleStory v83 封包Opcode結構速查 (mapleref.fandom.com 2026)

**URL**: https://mapleref.fandom.com/wiki/Packets

### 封包格式 (v83)
```
[Header: 4 bytes][Opcode: 2 bytes][Data: 可變]
```

- Header 包含封包長度和加密後的 opcode
- Opcode 為 **2 bytes** (little endian)
- `0x` 前綴表示十六進位

### 關鍵Opcode速查 (v83)

| Opcode | 名稱 | 方向 | 說明 |
|--------|------|------|------|
| `0x01` | LOGIN_REQUEST | C→S | 登入請求 |
| `0x02` | LOGIN_RESPONSE | S→C | 登入回應 |
| `0x0B` | CHANGE_MAP | C→S | 換地圖 |
| `0x0C` | CHANGE_MAP_RESULT | S→C | 換地圖結果 |
| `0x1C` | INVENTORY_OPERATION | S→C | 物品操作 |
| `0x2E` | SPAWN_PORTAL | S→C | 生成傳送門 |
| `0x37` | MOVE_PLAYER | C→S | 玩家移動 |
| `0x38` | CLOSE_RANGE_ATTACK | C→S | 近程攻擊 |
| `0x39` | RANGED_ATTACK | C→S | 遠程攻擊 |
| `0x3A` | MAGIC_ATTACK | C→S | 魔法攻擊 |
| `0x4F` | NPC_TALK | S→C | NPC 對話 |
| `0x50` | NPC_TALK_MORE | C→S | NPC 繼續對話 |
| `0x6A` | MONSTER_SPAWN | S→C | 怪物生成 |

> ⚠️ 具體 Opcode 值可能因私服而異，需使用工具實際抓包確認。

## 🆕 MapleStory v83 NPC 腳本編寫速查 (RaGEZONE 2026)

### 腳本位置
```
scripts/npc/      # 一般 NPC (檔名 = NPC ID)
scripts/quest/    # 任務 NPC
scripts/event/    # 事件腳本
```

### 核心模板 (Cosmic/HeavenMS)

```javascript
var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1 || (mode == 0 && status == 0)) {
        cm.dispose();
        return;
    }
    
    if (mode == 1) {
        status++;
    } else {
        status--;
    }
    
    switch(status) {
        case 0:
            cm.sendNext("你好！");
            break;
        case 1:
            cm.sendSimple("選擇：\n #b#L0#購買#l\n #L1#離開#l");
            break;
        case 2:
            if (selection == 0) {
                cm.openShop(1);
            }
            cm.dispose();
            break;
    }
}
```

| 函數 | 用途 |

### Dynamic Shop NPC (避免重啟伺服器)

```javascript
var items = [
    [2000002, 1000, 100],   // [物品ID, 價格, 數量]
    [2000003, 500, 50]
];

function start() {
    var shop = cm.openShop(1);
    for (var i = 0; i < items.length; i++) {
        shop.addItem(items[i][0], items[i][1], items[i][2]);
    }
    shop.send();
    cm.dispose();
}
```

### NPC 腳本常見錯誤

| 錯誤 | 原因 | 解決 |
|------|------|------|
| `cm is not defined` | 腳本不在正確目錄 | 確認路徑 |
| 無法關閉對話 | `dispose()` 位置錯誤 | 移到正確分支 |
| 商店不開 | `shopId` 不存在 | 確認資料庫 |
