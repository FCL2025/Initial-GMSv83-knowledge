## 一、登入器 / 客戶端修改

### 1.1 最新客戶端修改流程（Cosmic 模式）

Cosmic 是 2021 年啟動的 v83 私人伺服器專案，由 Ponk 維護，以 Java 21 為基礎：

**客戶端安裝流程：**
1. 下載 [Cosmic-client](https://github.com/P0nk/Cosmic-client) 的 client files
2. 使用 [HaRepacker-resurrected](https://github.com/lastbattle/Harepacker-resurrected) 管理 WZ 檔案
3. 編輯時使用加密模式：`GMS (old)`
4. 用 UI 直接修改 WZ 節點（避免 copy/paste 節點，可能造成資料流失）
5. 儲存後覆寫客戶端資料夾的 WZ
6. 最後用 "Private Server" 匯出選項將 XML 匯入伺服器端的 WZ.xml

**重要規則：**
- **永遠從客戶端 WZ 匯出到伺服器 XML，不要反向操作**
- 不這麼做會導致客戶端與伺服器資料不同步

### 1.2 localhost.exe / 登入器修改（傳統方式）

當需要修改客戶端連線 IP 時：

1. 用 **Neo Hex Editor** 打開 `localhost.exe`
2. 搜尋字串 `127.0.0.1`，找到所有 IP 位置
3. 替換為目標 IP（需保持位元組長度一致）
4. 儲存並退出

**Windows 8/10+ 相容性問題處理：**
- `lolwut.exe` + Fraysa's method 可解決相容性問題
- localhost.exe 屬性需設定：
  - 相容模式：Windows 7
  - 解析度：640 x 480
  - 取消「高 DPI 停用」
  - 以系統管理員身分執行

### 1.3 客戶端必做修改（v83 私人伺服器通用）

從 MapleStory 安裝資料夾刪除：
- `HShield\` 資料夾
- `ASPLauncher.exe`
- `MapleStory.exe`
- `patcher.exe`

用 `localhost.exe` 或自訂登入器取代原本執行檔。

### 1.4 MapleSolaxiaV2 / HeavenMS 的客戶端同步原則

MapleSolaxiaV2 README 特別強調：

> **DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate some kind of bugs afterwards.**

正確流程：
1. 用 HaRepacker 開啟客戶端 WZ，用 UI 修改
2. 儲存客戶端 WZ
3. 再用 HaRepacker 的 "Private Server" 匯出功能，匯出 XML 到伺服器的 wz.xml 目錄

## 三、Opcode 結構

### 3.1 v83 Packet Header 結構

根據 MapleRef Wiki 和多個開源專案驗證：

**Header 格式（共 4 bytes）：**
```
[Header] = 4 bytes
  - 包含 Send IV + Packet Length + Version
```

**對客戶端發送的封包：**
- Version = `-(version + 1)`（例如 v83 = -84）
- 這是 MapleStory 用來驗證封包來源的機制

**Opcode 格式：**
- Opcode 通常為 **2 bytes**
- 位置在 Header 之后

### 3.2 封包加密機制

MapleStory 使用兩層加密：

1. **MapleCustomEncryption（第一層）**
   - 在 AES 之前套用
   - 涉及 `shuffle` 和 `XOR` 操作
   - 程式碼可在各開源伺服器（OdinMS → HeavenMS → Cosmic）中找到

2. **MapleAESOFB（第二層）**
   - AES-128 OFB 模式
   - 使用 IV（初始向量）進行加密
   - Key 是固定的，根據版本不同（如 v83 = MAPLE_TYPE 8）

### 3.3 登入流程封包（重要！）

**第一個封包永遠是明文（plain-text）**，這是調試的關鍵切入點。

典型登入流程：
```
客戶端 → 伺服器：LOGIN_START (0x01)        [明文]
伺服器 → 客戶端：加密 handshake 要求
客戶端 → 伺服器：交握封包（啟用加密）
之後所有封包：均為加密狀態
```

### 3.4 C++ / Rust 實作參考

- **gmh5225/maplestory-v83MaplestoryCPP**：完整 C++ v83 私人伺服器，使用 POCO C++  Libraries、AES library (Brian Gladman)、MySQL (Oracle)
- **neeerp/RustMS**：Rust 實作，包含 `crypt` crate 提供完整的 MapleStory 加密實作
- **defaultmagi/maplelib (Go)**：提供 `EncryptNoShanda` 函數

### 3.5 IDA 分析Opcode（進階）

使用 IDA Pro 分析客戶端 binary 可找出：
- 各 Opcode 的硬編碼位置
- 加密函數的位址
- Send/Receive handler

RaGEZONE 有完整指南：[Getting Packet Structures & Opcodes Using IDA](https://forum.ragezone.com/threads/getting-packet-structures-opcodes-using-ida.792436/)

### 4.2 WZ 檔案結構

v83 的 WZ 是加密的 binary 格式，內含：
- **Sprite 圖像資源**（客戶端使用）
- **XML 資料**（從 .wz 匯出為 XML 後供伺服器讀取）
- **音效資源**

### 4.3 WZ 修改工作流程（標準流程）

```
1. 用 HaRepacker-resurrected 開啟客戶端 WZ（如 Item.wz）
2. 找到要修改的節點（例：某装备的属性）
3. 直接在 UI 中修改數值
4. 儲存（自動加密）
5. 用 "Private Server" 匯出 XML
6. 將 XML 覆寫到伺服器的 wz/ 目錄
7. 重啟伺服器
```

### 4.4 常見 WZ 修改項目

| WZ 檔案 | 修改內容 |
|---------|---------|
| `Character.wz` | 角色外觀、臉型、髮型 |
| `Item.wz` | 物品屬性、ID、名稱 |
| `Mob.wz` | 怪物屬性、血量、掉落 |
| `Skill.wz` | 技能效果、等級、MP消耗 |
| `Map.wz` | 地圖物件、傳送點 |
| `Quest.wz` | 任務條件、獎勵 |
| `Npc.wz` | NPC 腳本、對話內容 |
| `Sound.whz` | 背景音樂 |

### 4.5 重要 WZ 修改警告（來自 MapleSolaxiaV2 README）

> 編輯客戶端 WZ 時，**不要刪除/重新匯入節點**，只修改現有節點的數值。
> 刪除節點可能導致客戶端崩潰或資料丟失。

### 4.6 MobBookUpdate 工具（特殊案例）

MapleSolaxiaV2 提供一個特殊工具用於更新怪物書：
- 從資料庫讀取當前掉落資料
- 自動更新 `MonsterBook.img.xml`
- 需用 HaRepacker 特殊處理

## 五、NPC 腳本編寫

### 5.1 NPC 腳本基礎結構（JavaScript / Java）

v83 私人伺服器的 NPC 腳本通常有兩種語言：
- **Java**（多數基於 OdinMS/HeavenMS）：`source/scripts/npc/`
- **JavaScript**（Cosmic）：`scripts/npc/`

### 5.2 典型 NPC 腳本範例（Java 風格）

```javascript
// 簡單對話型 NPC
var status = -1;

function start() {
    cm.sendNext("你好冒險者！歡迎來到弓箭手村。");
}

function action(mode, type, selection) {
    if (mode === 1) {
        status++;
    } else {
        status--;
    }
    
    if (status === 0) {
        cm.sendNextPrev("你想做什麼？");
    } else if (status === 1) {
        cm.sendYesNo("要購買 goods 嗎？");
    } else if (status === 2) {
        if (mode === 1) {
            cm.dispose();
        }
    }
}
```

### 5.3 NPC 腳本核心函數

| 函數 | 用途 |
|------|------|
| `cm.sendNext(text)` | 顯示對話框，點擊後繼續 |
| `cm.sendPrev(text)` | 顯示對話框，回上一頁 |
| `cm.sendYesNo(text)` | 是/否 選擇對話框 |
| `cm.sendAcceptDecline(text)` | 接受/拒絕 對話框 |
| `cm.sendSimple(text)` | 簡單選項對話 |
| `cm.sendGetNumber(text, def, min, max)` | 數字輸入對話 |
| `cm.sendGetText(text)` | 文字輸入對話 |
| `cm.gainItem(id, count)` | 給予物品 |
| `cm.gainMeso(amount)` | 給予金幣 |
| `cm.getPlayer()` | 獲取玩家物件 |
| `cm.startQuest(id)` / `cm.completeQuest(id)` | 任務操作 |
| `cm.dispose()` | 關閉對話 |

### 5.4 動態商店 NPC（進階）

RaGEZONE 有完整教學：動態商店腳本不依賴 SQL，可以即時定義商品：

```javascript
function start() {
    var shop = cm.getPlayer().getShop();
    
    // 定義商品
    var items = [
        { id: 1002000, price: 1000, quantity: 1 },  // 蘋果
        { id: 2000000, price: 500, quantity: 100 }, // MP藥水
    ];
    
    // 註冊商店
    cm.openShop(1, items);
    cm.dispose();
}
```

### 5.5 任務系統整合

NPC 腳本常見任務流程：

```javascript
// 檢查任務開始條件
if (cm.getQuestStatus(questId) === 0) {  // 未開始
    cm.startQuest(questId);
    cm.gainItem(questItemId, 1);
} else if (/* 任務完成條件 */) {
    cm.completeQuest(questId);
    cm.gainMeso(10000);
    cm.gainItem(rewardId, 1);
}
```

### 5.6 常見腳本錯誤

1. **忘記 `cm.dispose()`**：對話框無法關閉，卡死玩家
2. **status 計數錯誤**：mode = -1（按ESC）時要遞減 status
3. **物品ID不存在**：導致客戶端崩潰
4. **任務條件未滿足就complete**：伺服器驗證會失敗
