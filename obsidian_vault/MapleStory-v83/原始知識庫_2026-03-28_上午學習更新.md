### 1.2 Ezorsia V2 Admin 提權移除（重要發現，2026-03-28）

**來源**: https://github.com/444Ro666/MapleEzorsia-v2/wiki

**問題**：編輯 Admin elevation（提權）後，客戶端仍然要求管理員權限

**根本原因**：當執行檔命名為預設名稱 `MapleStory.exe` 時，Windows UAC 會繞過自訂的 `asInvoker` 設定

**解決方案**：
1. 使用 Hendi's Hex Editor 工具包編輯 `asInvoker` 部分
2. **將執行檔改名為 `MapleStory2.exe` 或其他名稱**
3. 這樣 Windows 就會正確應用 `asInvoker` 設定，不再要求管理員權限

### 1.3 4G Flag (Large Address Space Aware) 設置

**目的**：允許客戶端使用最高 4GB 記憶體，而非預設的 2GB

**修改步驟**：
```
1. 使用 Hex Editor 打開 MapleStory.exe
2. 找到 PE Header 中的特定偏移: 0x0000013E
3. 預設值: 0x0F → 修改為: 0x2F
4. 保存並關閉
```

**作用**：
- 開啟後客戶端可以使用最大 4GB 虛擬記憶體
- 對於大型 WZ 文件（如 HD 客戶端）非常有用
- 避免記憶體不足導致的崩潰

## 2. 抓包分析 — 工具完整解析

### 2.1 MapleShark — C# 封包分析工具（經典）

**GitHub**: https://github.com/diamondo25/MapleShark

**原理**：
- 使用 **SharpPcap**（基於 WinPcap）進行網路層抓包
- 擷取 TCP 流量中的 MapleStory 封包

**腳本引擎（S# 語法）**：
```csharp
// S# 腳本 — 定義封包結構
int AddInt(string name)      // 4 bytes signed int
ushort AddUShort(string name) // 2 bytes unsigned short
string AddPaddedString(string name, int length)
void StartNode(string name)  // 開啟子節點
void EndNode(bool expand)     // 結束子節點
int Remaining()               // 返回剩餘未處理位元組
```

**IDA IDC Script**：https://gist.github.com/diamondo25/be95345a2875ab4342cd
- 用於 IDA Pro 自動化 Opcode 命名

### 2.2 MaplePacketPuller — Python 封包結構分析器

**GitHub**: https://github.com/Bratah123/MaplePacketPuller

**工作流程**：
```
1. IDA Pro 反編譯客戶端 DLL/EXE
2. 複製 C-pseudocode 到 .txt 檔案
3. MaplePacketPuller 分析 pseudocode
4. 輸出封包結構到 FuncOutput/ 目錄
```

**依賴工具**：
- IDA Pro 7.0+（32-bit）
- Python 3.6+（使用 os.scandir，不相容 Python 2.7）

**SpiritIDAPlugin**（配套插件）：https://github.com/Bratah123/SpiritIDAPlugin
- 在 IDA 中直接提取 Opcode 和封包結構

### 2.3 MaplePE — 32-bit MapleStory 封包編輯器（2025 新工具）

**GitHub**: https://github.com/zhyonc/MaplePE

**功能**：
| 功能 | 說明 |
|------|------|
| 封包解析 | 格式化封包結構為列表 |
| 自訂封包發送 | 編輯並發送封包到客戶端 |
| 目標進程提取 | 從目標進程提取 InPacket/OutPacket |
| Opcode 過濾 | 過濾特定 Opcode |
| 程式碼生成 | 從封包結構生成簡單程式碼 |

**技術原理**：
- MaplePE.exe 是 UDP 伺服器，接收封包資訊
- Packet.dll 使用 `CreateRemoteThread + LoadLibraryW` 注入目標進程
- Hook 封包讀寫函數以檢測封包結構
- 使用 UDP client 發送封包資訊

**發送封包前提條件**：
> "Before sending a packet via MaplePE.exe, the server must respond with at least one packet to ensure that the CClientSocket instance can be correctly specified."

伺服器必須至少回應過一個封包，才能正確指定 CClientSocket instance。

**BB+ 版本崩潰問題**：
> "Since some versions after BB add hook-detection in CClientSocket::SendPacket"

**解決方案**：Hook `MakeBufferList` 而非 `SendPacket`，並用 Cheat Engine 將檢測位址填為 `90` (NOP)。

## 3. Opcode 結構 — 詳細解析

### 3.1 Opcode 基礎知識

**來源**: MapleRef Fandom — https://mapleref.fandom.com/wiki/Operation_Code

**定義**：
> "An Operation Code (aliases: Opcode, Header) is the first bytes in a packet to help identify which packet is received or sent in MapleStory."

**封包格式（v83）**：
- **Header**：4 bytes（版本 IV + 長度 + 版本號）
- **Opcode**：2 bytes（用於識別封包類型）
- **Payload**：剩餘 bytes

**版本 Header**：
- 發送到客戶端：`-(version + 1)` 格式
- 發送到伺服器：純版本號

### 3.2 MapleStory 封包加密（v83）

**三層加密**：
| 層次 | 演算法 | 用途 |
|------|--------|------|
| 第1層 | MapleAES (OFP 模式) | 防止明文傳輸 |
| 第2層 | Shanda 加密 | Nexon 自訂 |
| 第3層 | 單向 Hash | 完整性驗證 |

**登入流程**：
```
客戶端 → 發送版本號 (plaintext)
伺服器 → 回傳正確版本 + IV
客戶端 → 發送登入封包 (AES 加密)
伺服器 → 驗證並回應
```

**重要發現**：
> "The first packet sent / received will always be in plain-text (not encrypted)."

### 3.3 v83 Opcode 獲取方法

**方法1：IDA Pro + MaplePacketPuller**
```
1. IDA Pro 開啟 v83 DLL/EXE
2. 找到封包處理函數
3. 擷取 C-pseudocode
4. MaplePacketPuller 分析輸出結構
```

**方法2：Wireshark + 自訂解碼器**
```
1. Wireshark 擷取 TCP 流量（連接 localhost:8484）
2. 過濾條件：tcp.port == 8484
3. 寫入 Lua 插件解密 MapleAES
4. 提取原始 Opcode
```

**方法3：MaplePE 工具**
```
1. 注入 Packet.dll 到 MapleStory.exe
2. MaplePE.exe 接收封包並解析
3. 自動識別常見 Opcode
```

### 3.4 v83 Opcode 地點

**客戶端（發送/接收）**：
- `SendPacket` 函數 — 發送封包到伺服器
- `OnPacket` / `ProcessPacket` — 處理接收封包
- 函數表（Function Table）中找到偏移

**伺服器端（發送/接收）**：
- `PacketHandler` 類 — 處理客戶端封包
- `MapleClient.sendPacket()` — 發送封包到客戶端
- Opcode enum 定義檔案

### 3.5 MapleRef Fandom Opcode 參考

**URL**: https://mapleref.fandom.com/wiki/Operation_Code

| 版本 | Opcode 格式 | 備註 |
|------|------------|------|
| 早期版本 | 1 byte | 已廢棄 |
| v83 | 2 bytes | 標準格式 |

**Opcode 查找技巧**：
1. 在 MapleRef 搜尋特定 Opcode（如 `0x0D`）
2. 查看該 Opcode 對應的封包名稱
3. 在伺服器源碼中搜尋 enum 定義
4. 交叉比對客戶端函數名稱

## 4. WZ 修改 — 完整指南

### 4.1 WZ 檔案結構

**WZ 格式**：MapleStory 專用的加密序列化格式
- **wz.xml**：定義 WZ 檔案路徑和映射
- **.wz 檔案**：加密的遊戲資源（圖片、音效、數據）

**主要 WZ 檔案**：
| 檔案 | 內容 |
|------|------|
| `Map.wz` | 地圖資訊和背景 |
| `Character.wz` | 角色外觀（髮型、臉型、性別） |
| `Item.wz` | 道具和裝備資料 |
| `Mob.wz` | 怪物資料 |
| `Skill.wz` | 技能資料 |
| `Quest.wz` | 任務資料 |
| `Npc.wz` | NPC 資料 |
| `Sound.wz` | 音樂和音效 |
| `UI.wz` | UI 介面元素 |

### 4.2 WZ 編輯器

| 工具 | 說明 |
|------|------|
| **HaRepacker 4.2.4** | 經典 WZ 編輯器，僅支援 v83~v140 |
| **HaRepacker-resurrected** | 現代版本，支援 64-bit + Lua + v230+ |

**HaRepacker-resurrected** (GitHub): https://github.com/lastbattle/Harepacker-resurrected

### 4.3 Ezorsia V2 WZ 編輯建議（重要）

> "I suggest doing all your edits in your own EzorsiaV2_UI.wz or MapleEzorsiaV2wzfiles.img (if using .IMG file) file to avoid WZ conflicts with the base game, and to ensure you can keep track of them."

**建議**：將所有自訂 WZ 編輯放在 `EzorsiaV2_UI.wz` 或 `MapleEzorsiaV2wzfiles.img` 中，避免與原廠 WZ 衝突。

### 4.4 WZ 修改類型

**物品修改（Item.wz）**：
- 修改道具屬性（攻擊力、防禦力）
- 添加新道具
- 修改道具圖示

**地圖修改（Map.wz）**：
- 修改地圖背景
- 調整物件位置
- 新增隱藏地圖

**NPC 修改（Npc.wz）**：
- 修改 NPC 名稱
- 調整 NPC 位置
- 設定 NPC TYPE（0=普通、1=商店）

**技能修改（Skill.wz）**：
- 修改技能等級上限
- 調整技能冷卻時間
- 修改技能效果

### 4.5 WZ 衝突解決

**問題**：多個修改模組同時修改同一個 WZ 檔案

**解決方案**：
1. 分割修改到不同的 WZ 檔案
2. Ezorsia V2 用戶使用專用 `EzorsiaV2_UI.wz`
3. 使用 wz.xml 定義優先級

### 5.3 status 變量機制（傳統寫法）

```javascript
var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1 || mode == 0) {
        cm.dispose();
        return;
    }
    
    if (mode == 1) {
        status++;
    } else {
        status--;
    }
    
    switch(status) {
        case 0:
            cm.sendNext("你好！");
            break;
        case 1:
            cm.sendSimple("選擇:\n #b#L0#購買#l\n #L1#出售#l");
            break;
    }
}
```

### 5.4 ES6 寫法 vs 傳統寫法

| 特性 | 舊寫法 (JDK 6) | ES6 (Nashorn) |
|------|--------------|--------------|
| 引入類別 | `importPackage(Packages.xxx)` | `const X = Java.type("...")` |
| 除錯輸出 | - | `print("...")` |
| 作用域 | `var` | `const` / `let` |
| 函數 | 傳統 | 箭頭函數 `=>` |
| 字串 | 傳統 | 模板字串 `` `Hello ${name}` `` |
