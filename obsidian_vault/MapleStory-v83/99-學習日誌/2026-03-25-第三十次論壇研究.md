## 2. 抓包分析

### 🆕 Maple_Pshark - v83 專用封包 Logger（最新功能）

**GitHub**: https://github.com/obstriker/Maple_Pshark

**功能**：
1. **攔截 Recv 訊息** - 攔截客戶端接收的封包
2. **修改訊息** - 即時修改封包內容
3. **注入封包** - 注入 Recv 或 Send 封包
4. **Rule-based 過濾** - 基於規則的封包過濾和操作

**使用場景**：
- 分析 MapleRoyals 等私人伺服器的封包格式
- 測試新功能時的封包發送
- 開發封包編輯工具

**技術原理**：
```
Hook 到加密函數 → 截獲明文封包 → 解析 Opcode → 修改/注入
```

### 🆕 botnored v83 封包類別結構（重要基礎知識）

**來源**: https://botnored.weebly.com/blog/maplestory-classes-v83

**重要原則**：
> **第一個發送/接收的封包永遠是純文字（未加密）**
> MapleStory 封包透過 TCP Socket 傳輸

**客戶端主要類別**：
| 類別 | 功能 |
|------|------|
| `CWvsContext` | 世界狀態管理（角色資料、伺服器列表） |
| `CClientSocket` | 網路連接管理（Send/Recv） |
| `CField` | 當前地圖/遊戲場景處理 |
| `CLogin` | 登入畫面處理 |

**封包流向**：
```
CClientSocket::ProcessPacket()
    ↓
根據 Header 分發
    ↓
CField::OnPacket() 或 CLogin::OnPacket()
    ↓
處理具體 Opcode
```

### 🆕 MapleRef Wiki - Packet 結構總覽（Send vs Recv 區分）

**來源**: https://mapleref.fandom.com/wiki/Packets

**核心概念**：
> "Actions that are defined by an identifier called an Operation Code (opcode for short), which is the first 2 bytes of the packet (a short), though in earlier versions it used to be the first byte (a byte)."

**Opcode 存放位置**：
> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

**Send vs Recv 方向對照**：
| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **Send** | 伺服器 → 客戶端 | Server Opcode Set |
| **Recv** | 客戶端 → 伺服器 | Client Opcode Set |

**重要**：同一個數字的 Opcode 可能代表完全不同的含義，視方向而定！

### 🆕 v83 登入結果碼（Login Status Code）

| 結果碼 | 意義 |
|--------|------|
| 0 | 登入成功 |
| 5 | 帳號或密碼錯誤 |
| 7 | 帳號已停用 |
| 13 | 刪除中（冷靜期） |
| 23 | 登入太頻繁（防機器人延遲） |

**來源**: https://mapleref.fandom.com/wiki/Operation_Code

**重要原則**：
> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |

**重要**：同一個 Opcode 數字可能代表不同含義，視方向而定！

## 4. WZ 修改

### 🆕 WZ 編輯黃金法則（MapleSolaxiaV2 README 強調）

> **DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs.**

**正確流程**：
```
1. HaRepacker 4.2.4 開啟 GMS (old) 加密的客戶端 WZ
2. 使用 UI 直接修改數值（禁止刪除/重新導入節點！）
3. 儲存 WZ
4. 在 HaRepacker 中選擇 "Private Server..." 導出
5. 將導出的 XML 覆蓋到伺服器的 WZ.XML 目錄
```

**重要原因**：
- 客戶端和伺服器的 WZ 結構有差異
- 直接用伺服器 XML 覆蓋會導致客戶端崩潰
- 客戶端 WZ 使用 GMS (old) 加密，伺服器使用 XML 明文

### 🆕 NX 格式 - WZ 的高效替代方案（2026-02 新討論）

**來源**: https://forum.ragezone.com/threads/wz-and-nx-file-questions.1260605/

**WzToNx.exe 使用方式**：
```
將 .wz 檔案拖曳到 WzToNx.exe 上
自動在同一目錄生成 .nx 檔案
```

**NX 格式優點**：
| 優點 | 說明 |
|------|------|
| 檔案更小 | 比 WZ 節省 30-50% 空間 |
| 載入速度更快 | 二進位格式解析更快 |
| 效能更好 | 減少 CPU 和記憶體使用 |
| Git 友好 | 差異更容易追蹤 |

**v83 支援狀態**：
| 音源 | 支援 NX |
|------|--------|
| ✅ Cosmic | 支援（開發中） |
| ⚠️ HeavenMS | 可能需要額外設定 |
| ❌ 傳統 WZ 音源 | 不支援 |

### 🆕 動態 NPC 商店腳本源 (v83, RaGEZONE 2016)
```javascript
var status = 0;
var items = Array(
    Array(1302000, 10000, 1),
    Array(1302001, 20000, 1),
    Array(1302002, 30000, 1)
);

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        if (mode == 1)
            status++;
        else
            status--;
        
        if (status == 0) {
            var msg = "歡迎光臨！\r\n請選擇商品：\r\n";
            for (var i = 0; i < items.length; i++) {
                msg += "#L" + i + "##i" + items[i][0] + "# - " + items[i][1] + " 金幣#l\r\n";
            }
            cm.sendSimple(msg);
        } else if (status == 1) {
            var item = items[selection];
            if (cm.getMeso() >= item[1]) {
                if (cm.canHold(item[0])) {
                    cm.gainMeso(-item[1]);
                    cm.gainItem(item[0], item[2]);
                    cm.sendOk("購買成功！");
                } else {
                    cm.sendOk("背包空間不足！");
                }
            } else {
                cm.sendOk("金幣不足！");
            }
            cm.dispose();
        }
    }
}
```
