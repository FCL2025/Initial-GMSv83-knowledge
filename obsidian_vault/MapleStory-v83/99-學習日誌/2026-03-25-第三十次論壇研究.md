**RustMS 加密實現亮點**：

```rust
// MapleAES - 負責封包加密的核心結構
```

### 🆕 BlackCipher / NGS Anticheat Bypass (2025-11)

**狀態**：Cloudflare blocked，無法直接獲取內容

**已知資訊**：
- MapleStory 使用 Nexon Game Security (NGS, 原名 BlackCipher)
- 透過心跳（heartbeat）機制驗證客戶端完整性
- **繞過方法**：使用假的 `NGClient.dll` 進行 **side-loading**
- 假的 DLL 攔截伺服器的 heartbeat 請求，模擬合法的安全客戶端回應
- 不需要記憶體修補或移除完整性檢查

**技術特點**：
- 模擬而非破壞安全客戶端的行為
- 可用於封包分析、記憶體修改等各種客戶端研究
- 完全繞過 anticheat，不再擔心客戶端偵測

### 🆕 localhost.exe / DLL Hook 兩種方案對比

| 方案 | 原理 | 適用場景 | 複雜度 |
|------|------|---------|--------|
| **localhost.exe** | HxD 修改 IP 位址 | 簡單 LAN 連線 | ⭐ 簡單 |
| **DLL Hook** | 注入 DLL 攔截 connect() | 需要更多控制 | ⭐⭐ 中等 |
| **Ezorsia v2** | MapleEzorsia v2 DLL | HD 客戶端支援 | ⭐⭐ 中等 |

**localhost.exe 修改流程**：
```
1. HxD 開啟 localhost.exe
2. Ctrl+F 搜尋 "127.0.0.1"（會找到三處）
3. 將三處都替換為目標 IP（注意長度一致性！）
4. 存檔
```

### 🆕 封包加密三層架構 (v83)

```
┌─────────────────────────────────────────────────────┐
│ Layer 1: MapleCrypto (頭部混淆)                     │
│   - 4 bytes header，含混淆 opcode + length           │
│   - 防止靜態分析 opcode                             │
│                                                        │
│ Layer 2: Shanda Encryption (對稱加密)                │
│   - v83 仍使用，已於 v118+ 移除                     │
│                                                        │
│ Layer 3: AES-OFB (主加密)                           │
│   - 256-bit key, 4-byte IV                          │
│   - 版本決定的靜態金鑰                              │
└─────────────────────────────────────────────────────┘
```

**第一個封包原則**：
> **第一個發送/接收的封包永遠是純文字（未加密）**

### 🆕 MapleRef Wiki Send/Recv Opcode 核心概念 (2026)

**來源**: https://mapleref.fandom.com/wiki/Operation_Code

**重要原則**：
> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

**重要**：同一個 Opcode 數字可能代表不同含義，視方向而定！

### 🆕 MapleCrypto Header 加密/解密公式 (v83 核心)

**加密端**：
```java
public byte[] getPacketHeader(int opcode, int length, int iv) {
    int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
    verify = (verify << 3) | (verify >>> 5);
    verify &= 0xFFFF;
    return new byte[] {
        (byte) (verify & 0xFF),
        (byte) ((verify >>> 8) & 0xFF),
        (byte) (length & 0xFF),
        (byte) ((length >>> 8) & 0xFF)
    };
}
```

**解密端（反向）**：
```java
public int retrieveOpcode(byte[] header, int iv) {
    int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
    verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
    return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
}
```

| 音源 | 解析器特點 | 注意事項 |

| HeavenMS | JDK 6 時代遺留 | 有 `SavedLocationType` 問題 |

### 相容性檢查清單

```javascript
// ✅ 相容寫法（所有音源通用）

var MapleInventory = Packages.server.MapleInventory;

// ❌ 不相容寫法（JDK 7+ 不支援）
importPackage(Packages.client);
importPackage(Packages.server);
```
