## 三、Opcode 結構 — 最新研究成果

### 3.1 MapleRef Wiki Send/Recv Opcode 核心概念

**來源**: https://mapleref.fandom.com/wiki/Operation_Code

> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

**重要**：同一個 Opcode 數字可能代表不同含義，視方向而定！

### 3.2 v83 登入結果碼（Login Status Code）

| 結果碼 | 意義 |
|--------|------|
| 0 | 登入成功 |
| 5 | 帳號或密碼錯誤 |
| 7 | 帳號已停用 |
| 13 | 刪除中（冷靜期） |
| 23 | 登入太頻繁（防機器人延遲） |

### 3.3 CClientSocket::ProcessPacket 封包分發流程 (RaGEZONE 2019)

**來源**: https://forum.ragezone.com/threads/how-does-one-find-a-structure-of-a-packet-via-ida.1162654/

**所有客戶端接收封包的入口點**：
```
CClientSocket::ProcessPacket()
    ↓
根據 MapleCrypto header 解密
    ↓
提取 opcode (2 bytes, little endian)
    ↓
switch(opcode) 分發
    ↓
CLogin::OnPacket()    // 登入畫面
    或
CField::OnPacket()    // 遊戲中
```

### 3.4 IDA 中的 Decode 函數對照表

| C++ 函數 (IDA) | Java 實現 (伺服器端) | 讀取大小 |
|---------------|---------------------|---------|
| `Decode1` | `mplew.write(1)` | 1 byte |
| `Decode2` | `mplew.writeShort(1)` | 2 bytes |
| `Decode4` | `mplew.writeInt(1)` | 4 bytes |
| `Decode8` | `mplew.writeLong(1)` | 8 bytes |
| `DecodeBuffer(x, n)` | `mplew.write(/* n bytes */)` | n bytes |
| `DecodeStr` | `mplew.writeMapleAsciiString()` | 可變 |

### 4.2 HaRepacker 4.2.3 WZ 編輯核心原則

**來源**: https://github.com/jonnylin13/Maple83

> "DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs."

**WZ 編輯黃金法則**：
```
1. HaRepacker 4.2.4 開啟 GMS (old) 加密的客戶端 WZ
2. 使用 UI 直接修改數值（禁止刪除/重新導入節點！）
3. 儲存 WZ
4. 在 HaRepacker 中選擇 "Private Server..." 導出
5. 將導出的 XML 覆蓋到伺服器的 WZ.XML 目錄
```

### 4.3 正確的客戶端 vs 伺服器 WZ 流程

```
客戶端 WZ 修改流程：
1. HaRepacker 開啟客戶端的 Item.wz (GMS old 加密)
2. 找到目標物品，直接修改數值
3. 儲存客戶端 WZ

伺服器 WZ XML 流程（獨立）：
1. HaRepacker 開啟伺服器的 Item.wz (GMS old 加密)
2. 同一位置修改數值
3. 導出為 XML 覆蓋伺服器目錄

⚠️ 兩者不能混用！
客戶端 WZ 修改 ≠ 伺服器 XML 修改
```

### 4.4 Quest 等級限制移除 (LvMax Removal)

**來源**: https://forum.ragezone.com/threads/maplestory-v83-method-to-disable-quest-level-restrictions.1250107/

**方法一：HaRepacker 圖形化操作**
```
1. HaRepacker 開啟 Quest.wz
2. Tools → Export Selection → XML → Classic
   （❗選 New 會使用新版加密，與私人伺服器不相容！）

4. 搜尋 lvmax 屬性並刪除

```

```
搜尋：lvmax="\d+"\s*
取代：（留空）
```

| 錯誤 | 原因 | 解決方案 |

### 5.1 NPC 腳本源間相容性問題 (RaGEZONE 2026-03)

**來源**: https://forum.ragezone.com/threads/kinoko-npc-scripts.1261538/

**問題**：不同音源（Cosmic、Kinoko、HeavenMS）的 NPC 腳本解析器版本存在差異

| 音源 | 解析器特點 | 注意事項 |
|------|-----------|---------|
| Cosmic | 最新，支援 JDK 7+ | 移除了 `importPackage` |
| Kinoko | 混合版本 | 需注意相容性 |
| HeavenMS | JDK 6 時代遺留 | 有 `SavedLocationType` 問題 |

### 5.2 NPC腳本源間相容性最佳實踐

**✅ 相容寫法（所有音源通用）**：
```javascript
var MapleCharacter = Packages.client.MapleCharacter;
var MapleInventory = Packages.server.MapleInventory;
```

**❌ 不相容寫法（JDK 7+ 不支援）**：
```javascript
importPackage(Packages.client);
importPackage(Packages.server);
```

### 5.3 Quest NPC 腳本要點 (v83)

**來源**: https://forum.ragezone.com/threads/how-to-script-quest-npcs-v83.1063962/

**核心問題**：
- 如何讓 NPC 腳本同時處理普通對話和任務劇情？
- 如何移除任務燈泡，讓 NPC 像普通 NPC 一樣運作？

**解決方案**：
- 透過 `src → scripts → npc` 處理，而非 `quest` 腳本
- Quest NPC 需要發送可腳本化的 MigrateIn 封包
- 使用 `CLogin::OnPacket` 或 `CField::OnPacket` 處理劇情

### 5.4 動態 NPC 商店腳本完整範例 (v83)

```javascript
/*
 * 動態 NPC 商店 - 不需要 SQL
 * NPC ID: 9010009
 */
var status = 0;
var items = Array(
    Array(1302000, 10000, 1),   // [物品ID, 價格, 庫存]
    Array(1302001, 20000, 1),
    Array(1302002, 30000, 1),
    Array(1302003, 50000, 1)
);

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        
        if (mode == 1)
            status++;
        else
            status--;
        
        if (status == 0) {
            var msg = "歡迎光臨！\r\n請選擇商品：\r\n";
            for (var i = 0; i < items.length; i++) {
                msg += "#L" + i + "##i" + items[i][0] + "# - " + items[i][1] + " 金幣#l\r\n";
            }
            cm.sendSimple(msg);
        } else if (status == 1) {
            var item = items[selection];
            if (cm.getMeso() >= item[1]) {
                if (cm.canHold(item[0])) {
                    cm.gainMeso(-item[1]);
                    cm.gainItem(item[0], item[2]);
                    cm.sendOk("購買成功！");
                } else {
                    cm.sendOk("背包空間不足！");
                }
            } else {
                cm.sendOk("金幣不足！");
            }
            cm.dispose();
        }
    }
}
```
