## 📚 新學知識

### 1. 登入器/客戶端修改 (Login Client Modification)

#### 最新資源
- **MapleSolaxiaV2** (ronancpl): 經典 v83 伺服器，需要使用 `localhost.exe` 替換原始客戶端的 ASPLauncher.exe 和 MapleStory.exe
- **Cosmic** (P0nk): 目前最活躍的 GMS v83 伺服器模擬器，基於 HeavenMS，2021年啟動

#### IP 修改方法
客戶端的 `localhost.exe` 需要用 Hex 編輯器修改 IP 位址：
1. 使用 Neo Hex Editor 或任何十六進位編輯器
2. 搜尋 "127.0.0.1" 字串
3. 替換為目標伺服器 IP
4. 注意：通常有 3 處需要修改

#### 客戶端必備步驟
```
1. 安裝 MapleStory 到指定資料夾 (如 C:\Nexon\MapleStory)
2. 刪除：HShield/、ASPLauncher.exe、MapleStory.exe、patcher.exe
3. 用 localhost.exe 替換
4. 用修改過的 WZ 檔案覆蓋原版
```

#### 重要警告
- ⚠️ **不要**將伺服器的 XML 直接匯入客戶端 WZ！這會導致 Bug！
- ✅ 正確流程：編輯客戶端 WZ → 匯出為「Private Server」格式 → 再匯入伺服器 XML

### 2. 抓包分析 (Packet Capture Analysis)

#### 關鍵發現
- v83 的 Opcode 已經被廣泛映射，不需要從頭追蹤
- 幾乎所有 v83 的 recv/opcode 都已有文檔

#### 工具推薦
- **Wireshark**: 通用網路抓包工具
- **MapleSEA WZSearcher**: MSEA 專用抓包工具
- **IDA Pro**: 用於分析客戶端二進位碼取得 Opcode 結構

#### Opcode 分析技巧
使用 IDA 分析 `CWvsContext__OnPacket` 函數：
1. 找到 `SendPacket` 相關函數
2. 分析封包頭部結構
3. 提取Opcode值

#### 參考資源
- [Getting packet structures and opcodes with IDA](https://forum.ragezone.com/f922/getting-packet-structures-opcodes-ida-872876/) - 經典教程
- [Packet Editing for Maplestory](https://forum.ragezone.com/threads/packet-editing-for-maplestory.1212354/) - RaGEZONE 2023

#### 重要概念
```
封包結構通常為：
[Header][Opcode][Data...]
 - Header: 長度信息
 - Opcode: 2-4 bytes 操作碼
 - Data: 實際數據
```

### 3. Opcode 結構

#### v83 Opcode 特性
- v83 使用 AES 加密 + 自定義加密
- 每個版本 Opcode 可能不同
- v83 客戶端已被高度修改，工作流程與其他版本不同

#### 推薦分析流程
1. 使用 Wireshark 抓取登入過程封包
2. 嘗試找出加密前的模式
3. 使用 IDA 分析客戶端加密邏輯
4. 對比伺服器端代碼

#### 現有開源專案
- **Maple83** (jonnylin13): GitHub 上最早的 v83 模擬器之一
- **HeavenMS**: 最完整的 v83 模擬器，2019年停止維護
- **Cosmic**: HeavenMS 的 fork，目前最活躍

### 4. WZ 修改

#### 工具
- **HaRepacker-resurrected** (lastbattle): 現代 WZ 編輯器，支援 GMS (old) 加密
- **HaRepacker 4.2.4**: 經典版本

#### 加密格式
v83 使用 `GMS (old)` 加密格式

#### WZ 編輯流程
```
客戶端 WZ 修改流程：
1. 用 HaRepacker 打開 .wz 檔案 (加密: GMS old)
2. 在 UI 中進行修改 (不要直接刪除/重新匯入節點！)
3. 保存修改到客戶端資料夾
4. 使用「Private Server」匯出選項導出 XML
5. 將 XML 覆蓋到伺服器的 wz 目錄

⚠️ 警告：永遠從客戶端 WZ 匯出到伺服器 XML，不要反向操作！
```

#### 重要 WZ 檔案
- `String.wz`: 字串、NPC 名稱、物品描述
- `Skill.wz`: 技能數據
- `Item.wz`: 物品屬性
- `Mob.wz`: 怪物數據
- `Map.wz`: 地圖數據

#### WZ 編輯風險
- 刪除/重新匯入節點可能導致數據丟失
- 只使用 UI 進行修改
- 定期備份原始檔案

### 5. NPC 腳本編寫

#### 腳本語言
v83 伺服器使用 **JavaScript** 編寫 NPC 腳本

#### 腳本存放位置
`source/scripts/npc/` 目錄下，每個 NPC 一個 .js 檔案

#### 基礎範例
```javascript
/* 
 * NPC ID: 9010000 (測試NPC)
 * 位置: 自由市場
 */

var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        
        if (mode == 1)
            status++;
        else
            status--;
        
        if (status == 0) {
            cm.sendNext("你好！歡迎來到我的商店！");
        } else if (status == 1) {
            cm.sendSimple("你想做什麼？\r\n#b#L0#購買物品#l\r\n#L1#出售物品#l\r\n#L2#離開#l");
        } else if (status == 2) {
            if (selection == 0) {
                // 打開商店
                cm.openShop(1);
                cm.dispose();
            } else if (selection == 1) {
                cm.sendOK("抱歉，收購服務暫時關閉。");
                cm.dispose();
            } else {
                cm.dispose();
            }
        }
    }
}
```

#### 常用函數
| 函數 | 功能 |
|------|------|
| `cm.sendNext(text)` | 發送下一個對話框 |
| `cm.sendSimple(text)` | 發送選項對話框 |
| `cm.openShop(id)` | 打開商店 |
| `cm.gainItem(id, qty)` | 給予物品 |
| `cm.getPlayer()` | 獲取玩家對象 |
| `cm.dispose()` | 關閉對話 |

#### 常見錯誤
- `importPackage` 在 JDK 7+ 不再支援，應移除
- `SavedLocationType` 未定義錯誤需要更新腳本
- NPC 腳本必須放在正確目錄，否則伺服器日誌會顯示 "NPC xxx.js is not coding yet"
