## 🆕 v83 加密架構總結

### 三層加密結構
```
明文 → Shanda 加密 (對稱) → AES-OFB → MapleCrypto header → 網路
```

### 重要原則
> **第一個封包永遠是純文字 (Hello Packet)**

### Hello Packet 結構 (v83)
```
偏移    長度   內容
0x00    2      封包總長度
0x02    2      Maple 版本號 (0x0053 = 83)
0x04    n      版本字串
n+4     4      SendIV
n+8     4      RecvIV
n+12    2      地區標誌
n+14    2      Unknown
```

## 🆕 v83 登入流程封包序列 (10 步)

```
1. C→S Hello (明文) - 版本、IV 協商
2. S→C Hello (明文) - 版本確認、回應 IV
3. C→S LOGIN_PASSWORD (加密) - Opcode 0x01
4. S→C LOGIN_STATUS (加密) - Opcode 0x00
5. C→S SERVERLIST_REQUEST (加密) - Opcode 0x06
6. S→C SERVERLIST (加密) - Opcode 0x0C
7. C→S CHARLIST_REQUEST (加密) - Opcode 0x07
8. S→C CHARLIST (加密) - Opcode 0x03
9. C→S CHAR_SELECT (加密) - Opcode 0x08
10. C→S PLAYER_DROP (加密) - Opcode 0x0B
```

### 登入結果碼
| 結果碼 | 意義 |
|--------|------|
| 0 | 登入成功 |
| 5 | 帳號或密碼錯誤 |
| 7 | 帳號已停用 |
| 23 | 登入太頻繁 |

### IMG 檔案系統工作流
- **Hot-swap for .img files**：直接拖放替換遊戲數據
- **WZ-to-IMG 提取**：將 .wz 檔案提取為目錄式佈局
- **Git 友好**：提取後易於 diff 和追蹤變更

### AI 地圖編輯器
- **對話式編輯**：從一次性 prompt 改為迭代逐步編輯
- **OpenCode CLI 支援**：可使用 Claude Code 和 Codex

### v83 WZ 編輯黃金法則
> **DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs.**

**正確流程**：
```
1. HaRepacker 打開 GMS (old) 加密的客戶端 WZ
2. 使用 UI 直接修改數值（禁止刪除/重新導入！）
3. 儲存 WZ → "Private Server..." 導出
4. 將導出的 XML 覆蓋到伺服器的 WZ.XML 目錄
```
