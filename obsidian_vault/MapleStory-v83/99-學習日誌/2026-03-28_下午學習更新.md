## 📌 新知識 1：IDA + MaplePacketPuller 封包結構分析工具鏈

### 背景
當需要分析 MapleStory v83 客戶端的封包結構時，手動追蹤非常耗時。MaplePacketPuller + SpiritIDAPlugin 提供了一套完整的解決方案。

### 工具鏈架構
```
IDA Pro (產生虛擬碼)
    ↓ 複製到 .txt
MaplePacketPuller (Python CLI 分析)
    ↓ 輸出結構
封包結構文字檔 (FuncOutput/)
```

### SpiritIDAPlugin（IDA 插件）
**GitHub**: https://github.com/Bratah123/SpiritIDAPlugin

| 項目 | 內容 |
|------|------|
| 語言 | IDAPython |
| 支援 IDA 版本 | Pro 7.0（7.5 不相容）|
| 支援 Python | 2.7（IDAPython 原生）|
| 安裝方式 | 複製到 IDA plugins 目錄 |
| 功能 | 右鍵選單分析封包結構 |

**主要功能：**
- 分析任何函數的封包結構
- 自動搜尋所有 OutPacket Header
- 自動搜尋 InPacket 結構
- 將函數寫入 C:\SpiritIDA\ 目錄

**安裝步驟：**
```
1. 將插件目錄內容複製到 IDA plugins/
2. 修改 init.py 中的路徑常數指向 SpiritMS.png
3. 啟動 IDA → 輸出視窗顯示 "Successfully Loaded"
4. 右鍵點擊函數 → 存取工具
```

### MaplePacketPuller（Python CLI）
**GitHub**: https://github.com/Bratah123/MaplePacketPuller

| 項目 | 內容 |
|------|------|
| 語言 | Python 3.6+（不相容 2.7）|
| 輸入格式 | IDA pseudocode .txt |
| 輸出格式 | 封包結構文字檔 |

**使用方式：**
```
1. 在 IDA 中對目標函數產生 C pseudocode
2. 複製貼上到 Functions/ 目錄下的 .txt
3. 執行 python main.py
4. 選擇分析類型和輸入檔名
5. 結果輸出至 FuncOutput/ 目錄
```

**已知限制：**
- while() 迴圈處理不完全（建议启用 GET_ALL_DECODES=true）
- 不顯示 switch case（需用戶自行添加）
- 需預先對 decode 函數命名

### IDA Decode 函數對照表（v83）
| IDA 函數 | 伺服器對應 | 說明 |
|----------|---------|------|
| `Decode1` | mplew.write | 1 byte |
| `Decode2` | mplew.writeShort | 2 bytes |
| `Decode4` | mplew.writeInt | 4 bytes |
| `DecodeStr` | mplew.writeMapleAsciiString | 可變長度字串 |
| `DecodeBuffer(x, 8)` | mplew.writeLong | 8 bytes |
| `DecodeBuffer(x, 13)` | mplew.writeAsciiString | 固定13字節ASCII |

## 📌 新知識 2：MapleSolaxiaV2 完整架設流程（精華整理）

**GitHub**: https://github.com/skhzhang/MapleSolaxiaV2
**客戶端及工具**: https://drive.google.com/drive/folders/0BzDsHSr-0V4MYVJ0TWIxd05hYUk

### 完整架設流程（Windows 環境）

#### Step 1: 安裝必要工具
```
- WampServer2.0i.exe → MySQL 資料庫
- hamachi.msi → 網路隧道（可選）
- mysql-query-browser-1.1.20-win.msi → SQL 管理工具
- jdk-7u79-windows-x64.exe → Java 7
- netbeans-8.0.2-javase-windows.exe → 開發環境
```

#### Step 2: 安裝 JDK Unlimited Policy
覆蓋以下路徑的 JAR 檔案：
```
C:\Program Files\Java\jre7\lib\security\
C:\Program Files\Java\jdk1.7.0_01\jre\lib\security\
```

#### Step 3: 資料庫設定
```
1. 開啟 MySQL Query Browser
2. 連線：localhost:3306, username: root
3. 依序執行 sql/ 目錄下的 SQL 腳本：
   - db_database.sql
   - db_drops.sql
   - (可選) db_shopupdate.sql
4. 手動在 accounts 資料表建立第一個帳號
```

#### Step 4: 客戶端設定（重要）
```
1. 用 ManagerMsv83.exe 安裝 MapleStory
2. 刪除：HShield/、ASPLauncher.exe、MapleStory.exe、patcher.exe
3. 用 localhost.exe 取代
4. 用 client_wz 資料夾的 WZ 檔案覆蓋原版
5. 如伺服器 IP 非 127.0.0.1 → 用 Hex Editor 修改 localhost.exe
```

#### Step 5: IP 修改（Hex Editor 方式）
```
工具：Neo Hex Editor (free-hex-editor-neo.exe)
1. 用十六進位編輯器開啟 localhost.exe
2. 搜尋 "127.0.0.1" 字串
3. 在 3 個地址替換為目標 IP
4. 保存並退出
```

### WZ 修改流程（HaRepacker 同步）
```
1. 用 HaRepacker 4.2.4 開啟客戶端 WZ（加密：GMS old）
2. 透過 UI 修改內容
3. 保存 WZ
4. 重新匯出 XML 到伺服器端的 WZ.XML（"Private Server..." 選項）
5. 覆蓋伺服器 WZ.XML

⚠️ 不要使用伺服器端的 XML 重新匯入客戶端 WZ，會造成 Bug！
⚠️ 同步是最重要的步驟，否則客戶端與伺服器會不同步
```

## 📌 新知識 3：v83 Opcode 結構分析要點

### 封包格式
```
[Header 4 bytes][Opcode 2 bytes][Data]
```

### 關鍵原則
1. **第一個封包是明文**：Hello Packet 不加密，用於交換 IV
2. **Opcode 為 2 bytes**：v83 從早期 1 byte 升級
3. **v83 已完整映射**：大多數情況不需要重新追蹤 Opcode
4. **客戶端修改才需要 IDA**：如去除 UI 限制、修改操作

### IDA 分析流程
```
1. 找到 SendPacket / RecvPacket 函數
2. 確認 packet header（例：0x1C = InventoryOperation）
3. 追蹤 Decode 函數調用鏈
4. 記錄每個欄位的長度和類型
```
