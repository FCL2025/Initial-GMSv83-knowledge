## 2. 抓包分析 (Packet Capture Analysis)

### 工具
- **MaplePacketPuller** (Bratah123) - Python-based 封包結構分析器
- **Maple_Pshark** - Mapleroyals 封包攔截器，專用於 v83
- **Wireshark** - 通用網路封包分析

### 封包結構 (v83)
- **Header**: 4 bytes (包含 send IV, packet length, version)
- **Opcode**: 2 bytes (short)
- **首個封包為明文傳輸 (未加密)**

### 封包加密機制
- 使用 IV (Initialization Vector) 進行加密
- MapleStory 版本影響加密參數

### 抓包分析論壇資源
- RaGEZONE: Packet Opcode Analysis 討論串
- 需使用 IDA 分析封包結構與 Opcode

## 3. Opcode 結構 (Opcode Structure)

### v83 Opcode 特性
- Opcode 為 2 bytes (short)
- 最大可用 Opcode 數: 65,536 (2^16)
- 早期版本 Opcode 為 1 byte (僅 255 個可用)

### Header 結構
```
[4 bytes header]
  - Send/Recv IV
  - Packet length (不含 header)
  - Version (傳送至客戶端為 -(version + 1))

[2 bytes opcode]
```

### Opcode 分析工具
- IDA Pro - 用於靜態分析客戶端
- MaplePacketPuller - 自動分析封包結構

### 重要 Opcode 用途
- `COutPacket::Encode2` - 編碼 2 bytes 資料
- 移動封包包含位置與速度資訊

## 4. WZ 修改 (WZ Modification)

### WZ 檔案功能
| 檔案 | 功能 |
|------|------|
| Base.wz | 武器、技能、音效等資訊 |
| Character.wz | 角色相關Sprite和裝備 |
| Effect.wz | 傷害數字、墓碑、升級特效 |
| Etc.wz | 任務、BUFF、詛咒、角色創建 |
| Item.wz | 物品圖片 |
| Map.wz | 地圖視覺 |
| Mob.wz | 怪物屬性(速度、HP、MP、技能) |
| Npc.wz | NPC 資料 |
| Quest.wz | 任務資料 |
| Reactor.wz | 可破壞物件(箱子、植物、門) |
| Skill.wz | 技能資料 |
| Sound.wz | 音效和歌曲 |
| String.wz | 伺服器端資訊(不可編輯) |
| TamingMob.wz | 野生怪物 |

### WZ 編輯工具
- **HaRepacker** (建議 4.2.4) - 主流 WZ 編輯器
- **deMSwZ** - 批量修改工具
- **Neo Hex Editor** - Hex 編輯

### 編輯流程 (客戶端 ↔ 伺服器)
1. 用 HaRepacker 開啟客戶端 WZ (encryption: GMS old)
2. 進行修改並保存
3. 用 HaRepacker 的 "Private Server..." 導出 XML
4. 覆蓋伺服器端的 WZ.XML 檔案

### Mob.wz 常用修改
- `bodyAttack` → 0 = GodMode (完全無傷)
- `acc` → 0/1 = 命中率調整
- `speed` → 126 = 怪物速度
- `jump` → 移除 = 無跳躍
- `damagedByMob` → 0 = 怪物免疫

### Map.wz 重要概念
- **Foothold**: 怪物/角色站立點
- `lt`/`rb`: 攻擊範圍 (left/top, right/bottom)
- 傳送門 Spawn Point 設定

### 背景參數 (Background)
- `bS`: 背景樣式名稱
- `front`: 前景顯示優先級
- `ani`: 動畫切換 (-1 ~ 1)
- `type`: 顯示類型 (0-20)
- `a`: 亮度 (0-255)
- `f`: 水平翻轉 (0/1)

## 5. NPC 腳本編寫 (NPC Scripting)

### 腳本語言
- NPC 腳本基於 **JavaScript** (部分類似 Java)
- 所有腳本 extends 其基礎類型 (NpcScript, Reactors, etc.)

### 基本 API 函數
```javascript
// 對話
cm.sendOk("訊息");        // 顯示訊息框
cm.sendYesNo("問題?");     // 是/否對話框
cm.sendNext("下一步");      // 下一頁對話

// 狀態/選項
cm.getPlayer().getLevel();  // 取得玩家等級
cm.getMap();                // 取得當前地圖

// 給予/移除
cm.gainItem(itemId, quantity);  // 給予物品
cm.gainMeso(amount);            // 給予金錢
cm.gainNX(amount);              // 給予NX

// 技能
cm.teachSkill(skillId, level, maxLevel);  // 給予技能

// 屬性
cm.getStr();   cm.getDex();   cm.getInt();   cm.getLuk();
cm.getHp();    cm.getMp();

// 任務
cm.startQuest(questId);   // 開始任務
cm.completeQuest(questId); // 完成任務
```

### NPC 腳本存放位置
- HeavenMS: `scripts/npc/`
- 檔案名稱對應 NPC ID

### 腳本範例結構
```javascript
var status = 0;
function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 1)
            status++;
        else
            status--;
        
        if (status == 0) {
            cm.sendNext("你好，冒險者！");
        } else if (status == 1) {
            cm.sendNext("這是我的商店。");
        } else {
            cm.dispose();
        }
    }
}
```

### 商店腳本
- 動態商店可根據玩家等級/職業調整商品
- NPC ID 決定商店功能

### GM 命令
- 存放在 `source/src/client/command/`
- 分類: player/admin/gm commands
