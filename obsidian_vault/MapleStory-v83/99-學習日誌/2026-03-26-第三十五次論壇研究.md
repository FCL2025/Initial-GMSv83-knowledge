## 🆕 MapleStory 封包加密核心概念 (maplelib/golang, 2026)

**來源**: https://pkg.go.dev/github.com/defaultmagi/maplelib

### 雙金鑰機制
> MapleStory uses two keys: one is used to encrypt sent packets and the other is used to decrypt received packets.

- 兩個金鑰隨機生成，客戶端連線時透過 Hello Packet 交換
- 這是 MapleStory 加密的核心設計

### AES-OFB 雙向加密
- 客戶端和伺服器各維護一個 SendIV 和 RecvIV
- 每個封包加密後 IV 都會遞增

### MapleCrypto 頭部混淆
```
加密端：
  verify = (opcode ^ iv[0]) + (iv[1] << 8)
  verify = (verify << 3) | (verify >> 5)  // rotate left 3 bits
  輸出 2 bytes 混淆 opcode + 2 bytes length
```

### MapleStory 各版本加密變化

| 版本 | 加密方式 | 備註 |
|------|---------|------|
| v83~v117 | Shanda + AES-OFB | ✅ 仍在使用 |
| v118+ | 僅 AES-OFB | Shanda 已移除 |

### MapleCrypto Handshake 流程
```
1. C→S: Hello Packet (plaintext) - 版本、SendIV、RecvIV
2. S→C: Hello Packet (plaintext) - 確認版本、回應 IV
3. 之後所有封包使用 MapleCrypto + AES-OFB 加密
```

## 🆕 NPC 腳本編寫學習要點

### 腳本核心概念
- 使用 JavaScript 編寫
- 狀態機追蹤對話進度
- cm (NPCConversationManager) 物件提供所有方法

### 狀態機基本模板
```javascript
var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        if (mode == 1) status++;
        else status--;
        
        if (status == 0) {
            cm.sendNext("第一頁對話");
        } else if (status == 1) {
            cm.sendOk("完成");
            cm.dispose();
        }
    }
}
```

### 常用 cm 方法速查
| 方法 | 功能 |
|------|------|
| cm.sendNext(text) | 下一頁 |
| cm.sendSimple(text) | 選項選單 |
| cm.sendYesNo(text) | 是/否 |
| cm.sendOk(text) | 確定 |
| cm.dispose() | 關閉對話 |
| cm.gainItem(id, qty) | 給予物品 |
| cm.gainMeso(qty) | 給予/收取金幣 |
| cm.warp(mapId) | 傳送 |
| cm.openShop(shopId) | 開商店 |
