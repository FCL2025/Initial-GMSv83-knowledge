## 🆕 客戶端 Hex 編輯 - IP 修改完整流程 (maplesecrets 2013 + RaGEZONE 2023)

**來源**：
- maplesecrets.blogspot.com: http://maplesecrets.blogspot.com/2013/04/how-to-hex-edit-clients-easily.html
- RaGEZONE: https://forum.ragezone.com/threads/solved-cant-find-ip-in-client-hex-editing-ap-message-popup.943982/
- RaGEZONE: https://forum.ragezone.com/threads/help-some-one-to-change-localhost-ip.1218181/

### 核心原理
v83 客戶端將伺服器 IP 位址以明文或簡單混淆形式儲存在二進位文件中。透過 Hex 編輯器直接替換位址字節，實現本地或遠端連接。

### 完整步驟 (HxD 為例)

**Step 1：準備工具**
| 工具 | 用途 |
|------|------|
| HxD | Hex 編輯器，下載：https://mh-nexus.de/en/downloads.php |
| v83 客戶端 | 原始 .exe 文件 |
| 備份 | 必ず備份原文件 |

**Step 2：搜尋 IP 位址**
```
1. 用 HxD 打開 MapleStory.exe
2. 按 Ctrl+G 前往... 選擇「起始位置」標籤
3. 選擇「從檔案開頭」(From beginning of file)
4. 按 Ctrl+F 開啟搜尋對話框
5. 選擇「Hex 值」(Hex-values)
6. 輸入目標 IP 的 Hex 表示，例如：
   - 127.0.0.1 → 31 37 32 2E 30 2E 30 2E 31 00
   - 192.168.1.100 → 43 33 32 2E 31 36 38 2E 31 2E 31 30 30 00
7. 按「搜尋全部」(Find All)
```

**Step 3：理解 Hex 字節表示**
| IP 位址 | Hex 表示 | 說明 |
|---------|---------|------|
| `127.0.0.1` | `31 37 32 2E 30 2E 30 2E 31 00` | localhost，結尾 null terminator |
| `192.168.1.1` | `43 33 32 2E 31 36 38 2E 31 2E 31 00` | 常見 LAN IP |
| 實際 IP | 各自計算 | 使用線上工具轉換 |

**IP 字節計算方法**：
```
1. 把 IP 分成四段：192 . 168 . 1 . 100
2. 每段轉為 Hex：
   - 192 → C0
   - 168 → A8
   - 1 → 01
   - 100 → 64
3. 反向排列（Little Endian）：
   - C0 A8 01 64
4. 加上點號的 Hex 值 2E：
   - C0 2E A8 2E 01 2E 64
5. 加上結尾 null：00
6. 完整：43 33 32 2E 31 36 38 2E 31 2E 31 30 30 00
```

**Step 4：替換 IP**
```
1. 找到 IP 位址後，選中要替換的部分
2. 確認長度一致：
   - 127.0.0.1 = 9 bytes (含結尾 00)
   - 替換為 192.168.1.100 = 14 bytes
   
   ⚠️ 若長度不同，需要使用「填充」(Padding)
   → 較長的 IP：直接替換，但可能影響後面數據
   → 較短的 IP：用 00 填充至相同長度
```

**Step 5：驗證**
```
1. 儲存修改後的文件
2. 嘗試連線測試
3. 若失敗，檢查：
   - IP 是否正確替換
   - 是否有多處 IP 需修改（通常有 3 處）
   - 版本號是否匹配
```

### v83 客戶端 IP 位置參考

| 版本 | IP 位置 (大約) | 說明 |
|------|--------------|------|
| v83 標準版 | 0x0040xxxx | 需實際搜尋 |
| v83 localhost | 127.0.0.1 | 預設為本地 |

### 常見錯誤與解決

| 錯誤 | 原因 | 解決方案 |
|------|------|---------|
| 客戶端無響應 | IP 長度不匹配 | 保持相同長度或重新計算偏移 |
| AP message popup | 版本不匹配 | 檢查 MapleVersion 或 ServerIP |
| 無法連線 | 防火牆 | 確認 7575/8484 埠口開放 |

## 🆕 WZ 編輯黃金法則 (RaGEZONE MapleSolaxiaV2 資源包)

**重要提醒**：

> **「千萬不要使用伺服器的 XML 來重新導入到客戶端的 WZ，這會產生 BUG！」**

### 正確流程 vs 錯誤流程

**❌ 錯誤流程**：
```
1. 從客戶端 WZ 導出 XML
2. 在伺服器端修改 XML
3. 將修改後的 XML 導回客戶端 WZ
→ 結果：客戶端崩潰或顯示異常
```

**✅ 正確流程**：
```
1. HaRepacker 直接打開「客戶端 WZ」
2. 使用 HaRepacker UI 直接修改數值
3. 儲存客戶端 WZ
4. 從「客戶端 WZ」導出 XML
5. 將導出的 XML 覆蓋到「伺服器 WZ.XML」目錄
```

### WZ 編輯的雙向性

| 方向 | 目的 | 方法 |
|------|------|------|
| 客戶端 WZ 編輯 | 修改客戶端顯示/行為 | HaRepacker 直接修改 |
| 伺服器 WZ.XML 編輯 | 修改伺服器資料/掉落 | 導出 XML 後修改 |

**兩者需保持一致**，否則客戶端和伺服器之間會有數據不匹配。

## 🆕 抓包分析實戰 - Hello Packet 解讀

### Hello Packet 為什麼是明文？

MapleStory 使用「加密握手」流程：
- 第一個封包用於客戶端和伺服器**協商 IV（初始化向量）**
- 雙方需要用相同的 IV 初始化 AES-OFB 加密器
- 若加密，雙方都無法解密對方的第一個訊息

### v83 Hello Packet 結構 (C→S)

| 偏移 | 長度 | 內容 |
|------|------|------|
| 0x00 | 2 | 封包總長度 (little endian) |
| 0x02 | 2 | Maple 版本號 (0x0053 = 83) |
| 0x04 | n | 版本字串 (如 "83") |
| n+4 | 4 | SendIV (客戶端指定的發送 IV) |
| n+8 | 4 | RecvIV (客戶端指定的接收 IV) |
| n+12 | 2 | 地區標誌 (MAPLE_LOCALE) |
| n+14 | 2 | Unknown |

### Wireshark 分析 Hello Packet

```
過濾表達式：
tcp.port == 7575 || tcp.port == 8484

Follow TCP Stream：
- 查看 C→S 和 S→C 的明文交換
- 確認版本號 0x0053 (83)
- 確認 IV 交換成功
```
