## 🆕 登入器/客戶端修改 - HxD Hex 編輯詳細流程

**來源**: Cosmic-client README + HeavenMS

### 目標
將客戶端從連接官方伺服器改為連接到私人伺服器。

### 修改步驟

1. **下載 HxD**: https://mh-nexus.de/en/hxd/

2. **複製客戶端exe**（保留原始檔案）

3. **用 HxD 開啟複製品**

4. **搜尋 IP 位址**:
   - `Ctrl+F`
   - 選擇 "Text-string"
   - 輸入 `127.0.0.1`

5. **找到三個連續出現的位置**:
   ```
   127.0.0.1
   127.0.0.1
   127.0.0.1
   ```
   （這是 localhost 的三個綁定點）

6. **逐一修改**:
   - 將第一個 `127` 替換為目標 IP
   - 對其他兩個重複同樣操作
   - 點擊儲存

### 遠端伺服器連線
若要連接到非本機伺服器：
- 將 `127.0.0.1` 替換為伺服器 IP 或網域
- 確認防火牆開放 7575 和 8484 埠口

### ⚠️ 客戶端被標記為病毒的問題
```
Windows Security 會自動刪除修改過的客戶端exe
原因：客戶端被修改用於本地播放和其他自訂功能

解決方案：
在 Windows Security → Virus & threat protection → Exclusions
新增下載資料夾和安裝目錄為排除項目
```

## 🆕 v83 封包加密三層架構 - 完整整理

### 加密流程圖
```
發送端 (C→S):
明文資料
  → AES-OFB 加密 (使用版本 Key + IV)
  → Shanda 加密 (對稱加密)
  → MapleCrypto 頭部混淆 (4-byte header)
  → 網路傳輸

接收端 (S→C):
網路接收
  → MapleCrypto 解混淆 (提取 opcode + length)
  → Shanda 解密
  → AES-OFB 解密
  → 明文資料
```

### 重要原則
> **第一個發送/接收的封包永遠是純文字（未加密）**

這個未加密的封包就是 **Hello Packet**，用於協商加密參數。

### MapleCrypto 頭部混淆演算法
```python
# 加密端：對 opcode 進行混淆
def maple_encrypt_header(opcode, iv):
    verify = (opcode ^ iv[0]) + (iv[1] << 8)
    verify = (verify << 3) | (verify >> 5)  # 位旋轉 3 位
    return verify & 0xFFFF

# 解密端：反向操作
def maple_decrypt_header(encrypted, iv):
    verify = encrypted
    verify = (verify >> 3) | ((verify << 5) & 0xFFFF)  # 反向旋轉
    return (verify - (iv[1] << 8)) ^ iv[0]
```

## 🆕 Hello Packet 結構

### 客戶端 → 伺服器 (C→S) Hello
| 偏移 | 長度 | 內容 |
|------|------|------|
| 0x00 | 2 | 封包總長度 (little endian) |
| 0x02 | 2 | Maple 版本號 (如 0x0053 = 83) |
| 0x04 | n | 版本字串 (如 "83") |
| n+4 | 4 | SendIV (客戶端指定的發送 IV) |
| n+8 | 4 | RecvIV (客戶端指定的接收 IV) |
| n+12 | 2 | 地區標誌 (MAPLE_LOCALE) |
| n+14 | 2 | Unknown |

### 伺服器 → 客戶端 (S→C) Hello
| 偏移 | 長度 | 內容 |
|------|------|------|
| 0x00 | 2 | 封包總長度 |
| 0x02 | 2 | Maple 版本號 |
| 0x04 | n | 版本字串 |
| n+4 | 4 | RecvIV (伺服器回應) |
| n+8 | 4 | SendIV (伺服器回應) |
| n+12 | 2 | 地區標誌 |
| n+14 | 2 | Unknown |

## 🆕 WZ 編輯黃金法則

> DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs.

**正確流程**：
1. HaRepacker 4.2.4 開啟 **GMS (old)** 加密的客戶端 WZ
2. **使用 UI 直接修改數值**（禁止刪除/重新導入節點！）
3. 儲存 WZ
4. 在 HaRepacker 中選擇 **"Private Server..."** 導出
5. 將導出的 XML 覆蓋到伺服器的 WZ.XML 目錄
