### 2. MapleRef Wiki - Opcode 結構最新資源

**來源**: https://mapleref.fandom.com/wiki/Operation_Code
https://mapleref.fandom.com/wiki/Packets

**核心概念**：
> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

**重要**：同一個 Opcode 數字可能代表不同含義，視方向而定！

### 3. v83 封包加密三層架構實戰

**三層加密流程圖**：
```
┌─────────────────────────────────────────────────────┐
│ Layer 1: MapleCrypto (頭部混淆)                      │
│   - 4 bytes header (混淆 opcode + length)           │
│                                                        │
│ Layer 2: Shanda Encryption (對稱加密)                │
│   - v83 仍使用，已於 v118+ 移除                      │
│                                                        │
│ Layer 3: AES-OFB (主加密)                            │
│   - 256-bit key, 4-byte IV                          │
│   - 版本決定的靜態金鑰                               │
└─────────────────────────────────────────────────────┘
```

**排查解密失敗的檢查清單**：
| 檢查項目 | 正確值 | 症狀 |
|---------|--------|------|
| AES Key | 版本決定的 32-byte key | 解密出乱码 |
| IV | Hello Packet 協商的 IV | opcode 不匹配 |
| Shanda Key | 版本特定的對稱金鑰 | 資料順序錯誤 |
| 版本號 | 0x0053 (83) | Hello Packet 失敗 |

### 4. v83 WZ 編輯最佳實踐

**WZ 編輯黃金法則** (Cosmic README)：
> **DO NOT 直接刪除/重新導入 WZ 節點，可能造成數據丟失！**

**正確工作流程**：
```
1. HaRepacker 打開 WZ（encryption: GMS (old)）
2. 使用 UI 直接修改數值
3. 儲存
4. 使用 "Private Server..." 導出選項
5. 覆蓋伺服器的 WZ XML 目錄
```

**client WZ vs server WZ XML**：
- **Client WZ**：玩家看到的遊戲資源（圖片、道具數據）
- **Server WZ XML**：伺服器用於邏輯判斷的資料（可讀格式）
- 永遠從 client WZ 匯出到 server XML，不要反向操作

### 6. NPC腳本源間相容性最新討論 (RaGEZONE, 2026-03)

**來源**: https://forum.ragezone.com/threads/kinoko-npc-scripts.1261538/

**問題**：不同音源（Cosmic、Kinoko、HeavenMS）的腳本解析器版本存在差異

| 音源 | 解析器特點 | 注意事項 |
|------|-----------|---------|
| Cosmic | 最新，支援 JDK 7+ | 移除了 `importPackage` |
| Kinoko | 混合版本 | 需注意相容性 |
| HeavenMS | JDK 6 時代遺留 | 有 `SavedLocationType` 問題 |

**解決方案**：
- 使用普遍支援的 `cm.` 函數
- 避免使用特定音源專有的 Package 路徑
- 移除 `importPackage`（JDK 7+ 已廢棄）
- 先在本地測試再部署

### 7. NPC腳本源間相容性檢查清單

```
5. 執行 SQL 初始化資料庫
6. 設定 ServerConstants.java 中的 IP 和埠口
7. HxD 修改客戶端 IP（三個 127.0.0.1 全部替換）
8. 啟動客戶端和伺服器
```

## 📋 本次學習要點

### 登入器/客戶端修改
- **HxD IP 修改**：搜尋 `127.0.0.1`，三個位置全部替換為目標 IP
- **LAN 連線問題**：錯誤 `-2147467259` (0x80004005) 通常是防火牆問題
- **客戶端被標記為病毒**：修改過的客戶端會被 Windows Defender 偵測，需加入排除清單

### 抓包分析
- **Maple_Pshark** (GitHub) 仍是 v83 專用封包 Logger 的首選
- **三層加密結構**：MapleCrypto → Shanda → AES-OFB
- **第一個封包是純文字**：Hello Packet 用於協商 IV

### Opcode結構
- **v83 opcode 幾乎全部已文檔化**
- **MapleRef Wiki** 提供完整的封包結構參考
- **Send vs Recv**：同一 Opcode 數字可能代表不同含義

### WZ修改
- **HaRepacker 加密選擇**：`GMS (old)` 適用於 v83
- **WZ 編輯黃金法則**：使用 UI 直接修改數值，不要刪除/重新導入節點
- **lvmax Regex 移除**：`lvmax="\d+"\s*` → 留空
- **Quest.wz** 是移除任務等級限制的目標檔案

### NPC腳本編寫
- **Cosmic** 使用標準 JavaScript NPC 腳本（Nashorn 引擎）
- **cm 物件**仍是所有腳本操作的核心介面
- **跨音源相容性**：避免 `importPackage`，使用 `Packages.xxx` 完整路徑
- **status 機制**：`mode=1` 時 `status++`，`mode=0` 時 `status--`
