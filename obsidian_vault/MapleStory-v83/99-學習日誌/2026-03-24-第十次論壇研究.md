### 1. Cosmic-Client 官方客戶端設定指南 (P0nk, 2025-2026)

**GitHub**：https://github.com/P0nk/Cosmic-client

Cosmic 是目前最活躍的 v83 模擬器，其客戶端設定有完整官方文件：

#### 必備檔案
| 檔案 | 說明 |
|------|------|
| `MapleGlobal-v83-setup.exe` | 原廠安裝程式（2008年代） |
| `HeavenMS-localhost-WINDOW.exe` | 修改過的本地客戶端 |
| `cosmic-wz/` | 自訂 WZ 檔案套件 |

#### 設定步驟（官方流程）
```
1. 執行 MapleGlobal-v83-setup.exe（預設 C:\Nexon\MapleStory）
2. 刪除 HShield 目錄（整個）
3. 刪除：ASPLauncher.exe、MapleStory.exe、Patcher.exe
4. 複製 cosmic-wz 所有 WZ 檔案覆蓋安裝目錄
5. 複製 localhost.exe 到安裝目錄
6. 雙擊 localhost.exe 啟動
```

#### ⚠️ 病毒誤判問題
```
Windows Security 會誤判修改過的客戶端為病毒
解決：在「病毒與威脅防護設定」中將下載和安裝目錄加入排除清單
```

#### HxD IP 修改（官方說明）
```
1. 用 HxD 開啟客戶端 exe（先複製一份備份）
2. Ctrl+F 搜尋 "127.0.0.1"
3. 會找到三個相連的 IP 位址
4. 分別覆蓋為目標 IP 後儲存
```

#### Cosmic 特殊 WZ 需求說明
```
Cosmic 需要自訂 WZ 文件，原因：
- 基於 HeavenMS，有大量 WZ 修改（文字、裝備等级等）
- 無法和原廠 WZ 完全相容
未來目標：完全移除自訂 WZ
建議：使用 github.com/P0nk/Cosmic-client 提供的 cosmic-wz
```

#### 客戶端疑難排解
| 問題 | 解決方案 |
|------|---------|
| 客戶端無法啟動 | 狂按雙擊客戶端，有機會成功開啟 |
| 無法連線到登入伺服器 | 確認伺服器已啟動 |
| Windows 刪除客戶端exe | 將目錄加入排除清單 |
| 版本不匹配 | 確認 Cosmic 和 Cosmic-client 的 release 版本一致 |

### 3. CMS (中國瑪奇) 登入封包三層解密實戰案例 (RaGEZONE, 2018)

**來源**：https://forum.ragezone.com/threads/help-remove-encrypt-decrypt-in-client-cms-chinese-maplestory.1155998/

這是 CMS v83 的登入封包實際解密案例：

#### 原始封包（C→S）
```
64 A4 9F 07 FD 4D 6C 49 4A 3D 96 FE C8 44 85 E4 79 FE EE E3 C7 3E 9B BF 42 3B 97 B2 4E 36 F8 DB F4 8C 0D D9
```

#### 第一步：AES-OFB 解密
```
解密後：71 1A 5B 6A 7E B9 C9 EA 15 17 7E E4 C0 B4 64 6D 1E E7 8A 13 C9 B0 8C 09 4C 7F F5 6F F8 A9 2C 5D 03 2D D4 D5
```

#### 第二步：Shanda Decryption
```
解密後：E2 FB 40 7E 67 08 B6 77 F4 CC 0E 3E 5D 72 D1 9A 2C 7C F6 44 16 6A 7F 2F ED 39 C8 DE 96 F7 95 04 59 88 C9 6F
```

#### 問題分析
```
opcode 為 E2 FB → 0xFBE2
但正確的 LOGIN_PASSWORD opcode 應為 0x0001
→ 原因：AES Key 版本不匹配，或 IV 錯誤
→ 重要教訓：解密出 opcode 不對，優先檢查 AES Key 版本！
```

#### 三層加密流程
```
發送端：明文 → AES-OFB 加密 → Shanda 加密 → 網路傳輸
接收端：網路接收 → Shanda 解密 → AES-OFB 解密 → 明文
```

### 4. 快速傳送 NPC 腳本源 (Z41N, RaGEZONE 2020)

**來源**：https://forum.ragezone.com/threads/fast-travel-npc-v83.1191556/

```javascript
var status = 0;

function start() {
    cm.sendYesNo("嘿，我是 Sunny！你想傳送到 Ludibrium 嗎？");
}

function action(mode, type, selection) {
    status++;
    if (mode != 1) {
        if(mode == 0 && type != 1)
            status -= 2;
        else if(type == 1 || (mode == -1 && type != 1)) {
            if(mode == 0) {
                cm.sendOk("嗯... 看來你還有事？");
                cm.dispose();
                return;
            }
        }
    }
    if (status == 1) {
        if (cm.haveItem(4310000))
            cm.sendNext("#b檢測到快速傳送票#k。");
        else
            cm.sendNext("當然可以！");
    } else if (status == 2) {
        cm.warp(220000000);
        cm.dispose();
    }
}
```

**關鍵 API**：`cm.haveItem(id)`、`cm.warp(mapId)`

### 5. sendAcceptDecline / sendGetNumber 完整行為表

**來源**：https://forum.ragezone.com/threads/add-on-learning-npcs-start-to-finish.643364/

#### sendAcceptDecline (Type = 12)
| Mode 值 | 意義 |
|---------|------|
| `mode == -1` | 關閉對話框 |
| `mode == 1` | 點擊「接受」(Accept) |
| `mode == 0` | 點擊「拒絕」(Decline) |

#### sendGetNumber (Type = 3)
| Mode 值 | 意義 |
|---------|------|
| `mode == 0` | 玩家輸入數字後點擊確定 |
