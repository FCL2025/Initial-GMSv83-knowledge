## 研究成果摘要

### 1. 登入器/客戶端修改

#### MapleEzorsia v2 - DLL注入式客戶端修改（最簡方案）
- **GitHub**: https://github.com/444Ro666/MapleEzorsia-v2
- **特點**：放入 dinput8.dll 即可，無需 Hex 編輯
- **支援 HD 解析度 + localhost 自動導向**
- **不破壞原廠 exe**

#### Kimmy's v83 localhost 客戶端集合
- **來源**: https://forum.ragezone.com/threads/maplestory-client-localhost-archive.1101897/
- 已整理過的 v83 localhost 客戶端，可直接搭配私服使用

#### v83 LAN 多人遊戲 - 錯誤排查
- **問題**：PC2 連線時出現 -2147467259 錯誤
- **原因**：客戶端 IP 指向 127.0.0.1，需改為 PC1 的 LAN IP（如 192.168.1.x）
- **解決**：`netsh advfirewall firewall add rule name="MapleStory" dir=in action=allow protocol=tcp localport=8484`

#### fonts folder 問題
- 客戶端無文字顯示時，檢查是否缺少 `fonts/` folder
- 從正常客戶端複製即可修復

### 3. Opcode結構

#### MapleRef Wiki - Send vs Receive Opcode 核心概念
- **來源**: https://mapleref.fandom.com/wiki/Operation_Code
- S→C (Send)：伺服器發送，客戶端接收
- C→S (Recv)：客戶端發送，伺服器接收
- **同一 Opcode 數字可能代表不同含義，視方向而定**

#### v83 登入結果碼（Login Status Code）
| 結果碼 | 意義 |
|--------|------|
| 0 | 登入成功 |
| 5 | 帳號或密碼錯誤 |
| 7 | 帳號已停用 |
| 13 | 刪除中（冷靜期） |
| 23 | 登入太頻繁（防機器人延遲） |

#### Packet Opcode Analysis (RaGEZONE, 2024-12)
- **來源**: https://forum.ragezone.com/threads/packet-opcode-analysis.1238027/
- v83 Opcode 幾乎全部被映射，不需要額外追蹤
- 唯一需要的反向工程是修改客戶端時

### 4. WZ 修改

#### WZ to NX 轉換工具
- **工具**: WzToNx.exe（拖曳 .wz 到工具上自動轉換）
- **優點**：檔案更小、載入更快、Git 友好
- **v83 支援**：Cosmic 支援，其他音源可能不支援

#### 跨版本物品 Import 必備屬性檢查
- **來源**: https://forum.ragezone.com/threads/maplestory-v83-newer-items-import.1256083/
- `info/lvlm`：等級上限（需 ≤ v83 支援範圍）
- `info/STR/DEX/INT/LUK`：屬性需求
- `info/tuc`：可強化次數
- `info/cursed/only`：詛咒標誌和職業限制

#### Community WZ Repack for v83
- **來源**: https://forum.ragezone.com/threads/community-wz-img-repack-for-v83.1221835/

#### Quest 等級限制移除（LvMax Removal）
