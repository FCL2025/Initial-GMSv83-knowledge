### 2. 抓包分析

**v83 三層加密架構**:
```
明文 → IOBufferManipulator.Encrypt() (Shanda) → AES-OFB → MapleCrypto header → 網路
```

**重要原則**: 第一個封包永遠是明文（Hello Packet）

**工具鏈**:
- Maple_Pshark: 即時 Hook 截獲明文
- MaplePacketPuller + SpiritIDAPlugin: 離線分析 IDA pseudocode
- Wireshark: TCP 監控 (tcp.port == 7575 || tcp.port == 8484)
- MaplePE: 封包編輯 + 發送

**RustMS (Rust 實現)**:
- 2026 年 3 月重啟，支援加密協商
- 完全 Rust，編譯後無需 JVM
- 適合學習現代密碼學應用

### 3. Opcode 結構

**封包處理函數 (v83 Rev 3.00)**:
| 函數 | 位址 | 說明 |
|------|------|------|

|---------|----------------------|---------|

| `CInPacket::Decode4` | `writeInt` | 4 bytes |
| `CInPacket::DecodeBuffer` | `writeLong` | 8 bytes |

### 4. WZ 修改

**v83 WZ 編輯黃金法則**:
> DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs.

**正確流程**:
1. HaRepacker 4.2.4 開啟 GMS (old) 加密的客戶端 WZ
2. 使用 UI 直接修改數值（禁止刪除/重新導入節點！）
3. 儲存 WZ
4. "Private Server..." 導出為 XML
5. 覆蓋到伺服器的 WZ.XML 目錄

**HaRepacker-resurrected v8.0.0 (2026-03-05)**:
- WZ-to-IMG 提取
- AI 地圖編輯器（Turn-based chat）
- MapSimulator Player MVP
- 64-bit 打包/提取

**WzImg-MCP-Server**: 74 tools，供 AI 助手直接互動 MapleStory IMG/WZ

**NX 格式**: WZ 的高效能替代方案（Cosmic 支援）

### 5. NPC 腳本編寫

**主流音源相容性**:
| 寫法 | Cosmic | Kinoko | HeavenMS |
|------|--------|--------|----------|
| 傳統 status 方式 | ✅ | ✅ | ✅ |
| Kinoko 新型（無需 status） | ⚠️ | ✅ | ⚠️ |
| ECMAScript 6 寫法 | ✅ | ✅ | ✅ |

**NPC TYPE 值**:
| Type 值 | 客戶端行為 | 腳本需求 |
|---------|-----------|---------|
| `0` | 點擊後發送 NPC_TALK 封包 | 需要 `.js` 腳本 |
| `1` | 點擊後直接開啟商店介面 | 可有可無 |

**跨音源相容最佳實踐**:
- ✅ 使用普遍支援的 `cm.` 函數
- ❌ 避免 `importPackage`（JDK 7+ 已廢棄）
- ✅ 使用 `var` 宣告而非 Java imports
- ✅ 先在本地測試再部署
