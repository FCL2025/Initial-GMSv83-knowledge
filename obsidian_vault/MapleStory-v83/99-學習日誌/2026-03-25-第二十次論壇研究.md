## 1. 登入器 / 客戶端修改

### 🆕 Ezorsia v83 HD Client 資源更新 (RaGEZONE, 2023)
**來源**: https://forum.ragezone.com/threads/release-v83-major-update-to-ezorsia-v2-the-hd-client-patch-is-now-a-standalone-hd-dll-client-localhost.1220746/

**重點資訊**：
- Ezorsia v2 HD Client 更新發布
- 獨立 HD DLL 客戶端（可直接替換）
- 內建 CRC bypass
- Hendi 客戶端僅經 Themida Strip，無其他修改
- Hendi 客戶端需使用 VirtualProtect 使記憶體可寫

**結論**：
```
Ezorsia v2 適合一般使用
Hendi 客戶端適合進階修改研究（乾淨的 Themida unvirtualized）
```

### 🆕 v83 UPX Packing 討論 (RaGEZONE, 2020)
**來源**: https://forum.ragezone.com/threads/v83-upx-packing-guide.1175132/

**v83 與 v90 Themida 差異**：
- v90: Hendi 完全 devirtualized Themida 並清理保護
- v83: Themida VM 仍存在，程式碼處於虛擬化狀態
- v83 Themida 在 VM 內部被 alteration/bypass
- v90 客戶端更適合研究和修改

**結論**：
```
若要深入研究客戶端，建議從 v90 入手
v83 已被高度修改，Themida 保護複雜
```

### 🆕 v83 Client Hack 位址 (RaGEZONE, 2019)
**來源**: https://forum.ragezone.com/threads/v83-client-hack-addresses.1163507/

**HD Client 缺失功能**：
- Text Spam
- UFJ (Universal Flight Jump)
- Damage Cap Removal
- Unlimited Jump

**修改方法**：
- 需使用 IDA/Cheat Engine 分析記憶體位址
- VirtualProtect 改記憶體屬性後可直接修改

### 🆕 解決 Client Bug (RaGEZONE, 2022)
**來源**: https://forum.ragezone.com/threads/fix-client-bugs.1205112/

**修復內容**：
1. **圖形 bug（Zakum 等）**：HeapAlloc 返回 0 導致
   - Hook HeapAlloc 修復
2. **舊版 Themida 在 Windows 10 相容問題**
   - Themida 使用 Raw Address 而非 Virtual Address 取得 EAT
   - 導致 Windows 10 上崩潰

### 🆕 Localhost Workshop 新聞 (RaGEZONE, 2025-12)
**來源**: https://forum.ragezone.com/threads/localhost-workshop.1202021/

**重要聲明**：
> "I can only make localhosts for versions <= GMS v111 (or equivalents for other maple regions - ca. 2012). This is because VMProtect - and later on newer Themida VM models - cannot be unvirtualized as perfectly as the old CISC-2."

**對 v83 的影響**：
- v83 使用 Themida 保護，無法完美 unvirtualize
- 只能製作版本 ≤ GMS v111 的 localhost

### 🆕 v83 分辨率修改位址 (RaGEZONE, 2019)
**來源**: https://forum.ragezone.com/threads/all-addresses-for-v83-resolution-change.1161938/

**重要發現**：
> "you can't protect a v83 client for obvious reasons (one of which it still contains the Themida VM table)"

v83 客戶端保護限制：
- Themida VM table 仍然存在
- 任何人都可以分析

## 2. Opcode 結構分析

### 🆕 MapleRef Wiki 封包結構 (2026)
**來源**: https://mapleref.fandom.com/wiki/Packets

**核心概念**：
> "Packet encryption in MapleStory is a combination of AES and MapleStory's own encryption. Packet headers are 4 bytes long, and are generated with the use of the send IV, packet length (without header) and the version of MapleStory."

**v83 封包加密特點**：
| 特性 | 說明 |
|------|------|
| 加密方式 | AES + MapleStory 專屬加密 |
| Header 長度 | 4 bytes |
| Opcode 長度 | 通常 2 bytes |
| 版本號 | -(version + 1) 發送，否則明文 |

### 🆕 Maple_Pshark v83 封包 Logger (GitHub, 2026)
**來源**: https://github.com/obstriker/Maple_Pshark

**功能**：
1. 攔截 Recv 訊息
2. 修改訊息
3. 注入封包（Send 或 Recv）
4. 規則式封包過濾

**原理**：
- Hook 加密函數，直接截獲明文
- 不需要處理 AES/Shanda 加密層

### 🆕 MapleStory 加密討論 (RaGEZONE, 2016)
**來源**: https://forum.ragezone.com/threads/maple-story-encription.1103482/

**v83 加密流程**：
> "You have to get the send and recv crypto from the server. The first packet it sends is the handshake and won't be encrypted at all."

> "Well, in newer versions, Shanda was removed (MapleCustomEncryption), but you're doing v83. If this is a server-side end, the 'crypt' is doing an AES crypt..."

**重要**：v83 仍使用 Shanda 加密（v118+ 移除）

### 🆕 Packet Editing 學習建議 (RaGEZONE, 2023)
**來源**: https://forum.ragezone.com/threads/packet-editing-for-maplestory.1212354/

**學習建議**：

### 🆕 NPC 腳本基本結構 (RaGEZONE, 2011)
**來源**: https://forum.ragezone.com/threads/add-on-npc-scripting.704641/

**基本範例**：
```javascript
function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && status == 0) {
            cm.dispose();
            return;
        }
        if (mode == 1)
            status++;
        else
            status--;

        if (status == 0) {
            cm.sendYesNo("Would you like a free sword to help you in your battles?");
        } else if (status == 1) {
            cm.gainItem(1302000, 1);
            cm.sendOk("Here you go. May the gods of Maplestory be with you.");
            cm.dispose();
        }
    }
}
```
