## 1. 登入器 / 客戶端修改

### 登入器原理（Login Redirector）

登入器是一個 DLL 或外部程式，攔截客戶端的網路連線，將其導向私人伺服器。

**原理架構**：
```
原始流程：
  MapleStory.exe → connect("203.78.xx.xx:7575") → Nexon 官方伺服器

使用登入器：
  MapleStory.exe → Hook API (connect/WSAConnect)
                  → 攔截並修改目標 IP/Port
                  → connect("127.0.0.1:7575") → 私人伺服器
```

**常見 Hook 點**：
| 函數 | 說明 |
|------|------|
| `connect()` | Winsock 連線函數 |
| `WSAConnect()` | Winsock SSL 連線函數 |
| `HttpSendRequest()` | HTTP 請求（部分客戶端） |
| `CClientSocket::Connect()` | 客戶端內部連線函數 |

### 兩種 IP 修改方案對比

| 方案 | 難度 | 穩定性 | HD 支援 | LAN 支援 | 推薦度 |
|------|------|--------|---------|---------|--------|
| HxD Hex 編輯 | ⭐ | ⭐⭐⭐⭐ | 需額外 | ✅ 直接 | ⭐⭐⭐ |
| Ezorsia v2 DLL | ⭐⭐ | ⭐⭐⭐⭐⭐ | 內建 | ⚠️ 需設定 | ⭐⭐⭐⭐⭐ |
| 第三方 localhost | ⭐ | ⭐⭐⭐ | 各版本不同 | ✅ 直接 | ⭐⭐⭐ |

### localhost.exe IP 修改標準流程

```
Step 1: 使用 HxD 開啟 localhost.exe
Step 2: Ctrl+F → 選擇 "Text-string" → 搜尋 "127.0.0.1"
        → 會找到三個連續出現的 "127.0.0.1"
Step 3: 將三處 "127.0.0.1" 都替換為目標 IP
        → 存檔
```

⚠️ **IP 長度一致性原則**：127.0.0.1 是 9 字元，替換後的 IP 長度必須相同！

### LAN 多人連線常見錯誤 (-2147467259 = 0x80004005)

| 問題 | 解決方案 |
|------|---------|
| 防火牆阻擋 | `netsh advfirewall firewall add rule name="MapleStory" dir=in action=allow protocol=tcp localport=7575,8484` |
| 伺服器 listen 127.0.0.1 | 設定為 0.0.0.0 |
| 客戶端 IP 未指向主機 | HxD 修改為 PC1 的 LAN IP |

```
        python main.py

```

### 核心原則
> **第一個發送/接收的封包永遠是純文字（未加密）**

封包加密三層架構：
```
Layer 1: MapleCrypto (頭部混淆)     — 4-byte header
Layer 2: Shanda Encryption          — 對稱加密，v118+ 已移除
Layer 3: AES-OFB                   — 256-bit key，版本決定的靜態金鑰
```

### v83 登入結果碼
| 結果碼 | 意義 |
|--------|------|
| 0 | 登入成功 |
| 5 | 帳號或密碼錯誤 |
| 7 | 帳號已停用 |
| 13 | 刪除中（冷靜期） |
| 23 | 登入太頻繁（防機器人延遲） |

## 3. Opcode 結構

### v83 Opcode 現況 (RaGEZONE, 2024-12)
- ✅ 幾乎全部被映射，不需要額外追蹤封包
- ✅ 大多數 opcode 表已在開源專案（Cosmic、HeavenMS）中完整提供
- ⚠️ 唯一需要的反向工程是**修改客戶端時**

### Send vs Recv Opcode 區分
| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

> 同一個 Opcode 數字可能代表不同含義，視方向而定！

### MapleCrypto Header 加密/解密公式

**加密端**：
```java
int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
verify = (verify << 3) | (verify >>> 5);
verify &= 0xFFFF;
```

**解密端（反向）**：
```java
verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
```

### IDA 中 Decode 函數對照
| IDA 函數 | Java 實現 | 讀取大小 |
|---------|---------|---------|
| `Decode1` | `mplew.write(1)` | 1 byte |
| `Decode2` | `mplew.writeShort(1)` | 2 bytes |
| `Decode4` | `mplew.writeInt(1)` | 4 bytes |
| `DecodeStr` | `mplew.writeMapleAsciiString()` | 可變 |
| `DecodeBuffer(x, 8)` | `mplew.writeLong` | 8 bytes |

| HeavenMS | 完整 | ⚠️ 停止維護 |

## 5. NPC 腳本編寫

### 基本模板
```javascript
var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        if (mode == 1)
            status++;
        else
            status--;
        // 根據 status 處理不同對話
    }
}
```

### 常用 cm 函數速查
| 函數 | 功能 |
|------|------|
| `cm.sendNext(text)` | 下一個對話框 |
| `cm.sendSimple(text)` | 選擇選單 |
| `cm.sendYesNo(text)` | 是/否對話 |
| `cm.gainItem(id, qty)` | 給予物品 |
| `cm.gainMeso(qty)` | 給予/收取楓幣 |
| `cm.openShop(id)` | 開啟商店 |
| `cm.warp(mapId)` | 傳送地圖 |
| `cm.changeJob(jobId)` | 轉職 |
| `cm.dispose()` | 關閉對話 |
| `cm.haveItem(itemId, count)` | 是否持有物品 |
| `cm.getPlayer().getName()` | 取得玩家名稱 |

### 對話格式標籤速查
| 標籤 | 功能 |
|------|------|
| `#b` | 藍色文字 |
| `#r` | 紅色文字 |
| `#h#` | 玩家名稱 |
| `#i[itemid]#` | 物品圖片 |
| `#L0#...#l` | 選項標記 |
| `#c[itemid]#` | 玩家持有數量 |
| `\r\n` | 換行 |

### 音源相容性問題（JDK 7+）
| 問題 | 解決方案 |
|------|---------|
| `importPackage is not defined` | 移除 importPackage，使用完整路徑 |
| `SavedLocationType is not defined` | 檢查音源版本相容性 |
| 不同音源解析器差異 | 使用普遍 `cm.` 函數，避免特定音源專有函數 |

### NPC腳本源間相容性檢查清單
- ✅ 無 `importPackage`（JDK 7+ 已廢棄）
- ✅ 使用 `Packages.xxx` 完整路徑
- ✅ 使用 `cm.` 而非直接類別名
- ✅ 正確的 `status` 狀態機邏輯
