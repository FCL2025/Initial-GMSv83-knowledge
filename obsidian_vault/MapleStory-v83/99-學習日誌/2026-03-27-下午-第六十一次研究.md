## 📌 研究重點 1: MapleEzorsia V2 HD 客戶端 (2026-03 最新)

**GitHub**: https://github.com/444Ro666/MapleEzorsia-v2

### Ezorsia V2 核心原理

Ezorsia V2 是一個 **dinput8.dll**，透過以下技術實現客戶端修改：

1. **記憶體補丁 (Code Caves)**: 在記憶體中新增程式碼區塊
2. **Function Replacement**: 替換原廠函數
3. **String Pool Hook**: 載入自訂 WZ 文件

### 與 Hendi's 4G Client 比較

| 特性 | Ezorsia V2 | Hendi's 4G Client |
|------|------------|-------------------|
| 4G 記憶體 | ✅ 需手動開啟 | ✅ 預設 |
| Admin 移除 | ✅ 需手動修改 | ✅ 預設 |
| HD 解析度 | ✅ 原生支援 | ❌ 需配合 Ezorsia |
| WZ 衝突 | ❌ 無衝突 | ⚠️ 可能有衝突 |

### 安裝流程（最新 2026-03）
```
1. 下載 dinput8.dll 從 GitHub Releases
2. 將 dinput8.dll 放入 v83 遊戲目錄
3. 執行遊戲 → 自動生成 config.ini
4. 編輯 config.ini:
   Width=1280
   Height=720
5. 若需要 4G：需使用 Hex Editor 修改 PE header
   位址: 0000013E
   值: 0f → 2f (開啟 Large Address Space Aware)
```

### 4G Flag 修改步驟（需 Hendi's Hex Editor）
```
1. 下載 Hendi's Hex Editor 工具
2. 打開 MapleStory.exe
3. 找到 PE Header 中的 DWORDS ptr
4. 修改值: 0f → 2f
5. 保存並關閉
```

### Ezorsia V2 安全性注意
- DLL 使用 Windows API Hook，涉及：
  - `VirtualAlloc`, `VirtualProtect`
  - `CreateFileW`, `WriteFile`
  - `CreateProcessW`
- ⚠️ 下載第三方編譯版本需谨慎，建議從 GitHub Releases 或自行編譯

### 第三方 DLL 載入功能
Ezorsia V2 支援最多 3 個第三方 DLL：
```ini
CustomDLL1=mydll.dll
CustomDLL2=
CustomDLL3=
```
這些 DLL 會在 Ezorsia V2 之後載入，可用於額外修改。

## 📌 研究重點 3: MapleRef Wiki Send/Recv Opcode 核心 (2026-03)

**URL**: https://mapleref.fandom.com/wiki/Operation_Code

### 核心原則
> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

### Opcode 方向對照表

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送，客戶端接收 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送，伺服器接收 | Client Opcode Set |

### v83 常用 Opcode 對照

| Opcode | S→C 名稱 | C→S 名稱 |
|--------|---------|---------|
| 0x00 | LOGIN_STATUS | - |
| 0x01 | - | LOGIN_PASSWORD |
| 0x03 | CHARLIST | - |
| 0x06 | - | SERVERLIST_REQUEST |
| 0x07 | - | CHARLIST_REQUEST |
| 0x08 | - | CHAR_SELECT |
| 0x0B | - | PLAYER_DROP |

### MapleRef 使用方式
```
1. 前往 https://mapleref.fandom.com/wiki/Operation_Code
2. 選擇目標版本 (v83)
3. 查看 Send (S→C) 或 Recv (C→S) 封包結構
4. 配合 Wireshark + Maple_Pshark 抓包分析
```

## 📌 研究重點 4: v83 NPC 腳本編寫動態 (2026-03)

### NPC TYPE 值含義（RaGEZONE, 2026-03）
**來源**: https://forum.ragezone.com/threads/npc-type.1262188/

| Type 值 | 名稱 | 客戶端行為 | 腳本需求 |
|---------|------|-----------|---------|
| `0` | 普通 NPC | 點擊後發送 NPC_TALK 封包 | 需要 `.js` 腳本 |
| `1` | 商店 NPC | 點擊後直接開啟商店介面 | 需要/不需要 `.js` 腳本 |
| `2~6` | 特殊 NPC | 拍賣、保險箱等 | 不需要腳本 |

在 HaRepacker 中查看：`NPC.wz` → `XXX.img` → `info` → `type`

### NPC 腳本源相容性問題
**來源**: https://forum.ragezone.com/threads/kinoko-npc-scripts.1261538/

| 問題 | 解決方案 |
|------|---------|
| 不同音源腳本解析器版本不同 | 使用普遍支援的 `cm.` 函數 |
| 較舊腳本不相容 | 避免特定音源專有 Package 路徑 |
| `importPackage` 廢棄 | 使用 `var` 而非 `import` |

### Dynamic NPC Shop Script (v83)
**來源**: https://forum.ragezone.com/threads/dynamic-npc-shop-script-v83.1089979/

使用 JavaScript 動態定義商店，避免每次修改都要重啟伺服器。

```javascript
var items = Array(
    Array(4000000, 1000, 1), // [物品ID, 價格, 數量]
    Array(4000001, 2000, 5)
);

function start() {
    var shop = cm.openShop(1); // 商店ID
    for (var i = 0; i < items.length; i++) {
        shop.addItem(items[i][0], items[i][1], items[i][2]);
    }
    shop.send();
}
```

### NPC 腳本最佳實踐（2026-03 更新）
- ✅ 使用普遍支援的 `cm.` 函數
- ✅ 使用 `status` 變數追蹤狀態
- ✅ `mode == 1` 表示下一步/確認，`mode == 0` 表示取消/返回
- ❌ 避免使用特定音源專有的 Package 路徑
- ❌ 避免 `importPackage`（JDK 7+ 已廢棄）
