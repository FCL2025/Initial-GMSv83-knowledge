## 🆕 學習重點一：v83 Skill.wz 跨職業編輯深入解析 (Reddit 2024)

### 問題核心

**來源**: https://www.reddit.com/r/mapleservers/comments/19cz6eu/v83_skillwz_editing/

將某職業技能直接複製到新手技能（如 beginner.img）時，提示缺少檔案：

### 原因分析

| 原因 | 說明 |
|------|------|
| 技能相依性 | 技能需要完整的所有相依節點結構 |
| 職業別限定 | Skill.wz 中每個職業有獨立目錄，複製過去會缺少綁定 |
| 動畫依賴 | 技能依賴 Character.wz 中的動畫定義 |

### 正確流程

```
錯誤做法：
直接在 HaRepacker 中複製 1302000.img → 粘貼到 beginner.img
→ 提示缺少相依檔案

正確做法：
1. 先確認目標職業的 Skill.wz 目錄結構完整
2. 確保 Character.wz 中有對應的動畫定義
3. 使用 Dependency Checker 檢查完整性
4. 只複製可獨立存在的技能（如被動技能）
```

### Skill.wz 目錄結構對照

| 目錄 | 職業 |
|------|------|
| `0.img` | 初心者 |
| `1.img` | 劍士 |
| `2.img` | 法師 |
| `3.img` | 弓箭手 |
| `4.img` | 盜賊 |
| `100.img` | 騎士團 |
| `800.img` | 冒險家 |

### Skill.wz 必備屬性

| 屬性路徑 | 說明 | 重要性 |
|---------|------|--------|
| `info/skillType` | 技能類型（主動/被動） | ✅ 必備 |
| `info/masterLevel` | 技能上限等級 | ✅ 必備 |
| `info/attackProp` | 攻擊加成百分比 | ⚠️ 主動技能必備 |
| `info/elemAttr` | 屬性攻擊 | ⚠️ 法師技能必備 |
| `effect` | 技能特效 | ✅ 必備 |

## 🆕 學習重點二：v83 WZ 編輯安全最佳實踐 (RaGEZONE 2026)

### 編輯前必做檢查清單

```
□ 備份原始 WZ 檔案
□ 確認使用正確的加密格式 (GMS old for v83)
□ 不要刪除節點，只修改數值
□ 每次修改只改一個項目
□ 測試後再繼續
□ 記錄所有修改項目
□ 建立修改日誌
```

### 編輯後必做檢查清單

```
□ 客戶端是否正常啟動
□ 目標功能是否生效
□ 是否有異常錯誤
□ 遊戲是否穩定
□ 確認無崩潰風險後再部署
```

### 防止數據丟失的核心原則

**重要發現** (RaGEZONE/Jonnylin13):
> Once the MonsterBook.img does not hold client specific data in it's node contents, importing the XML causes no harm at all. However, try not to remove/reimport nodes from WZ files, as it may cause data losses. Use the HaRepacker's UI instead to make the changes.

| 操作 | 風險等級 | 建議 |
|------|---------|------|
| 刪除 WZ 節點後重新導入 | ❌ 高風險 | 避免 |
| 使用 UI 直接修改數值 | ✅ 安全 | 推薦 |
| 導出 XML 修改後重新導入 | ⚠️ 中風險 | 僅必要時使用 |
| 替換整個 WZ 檔案 | ⚠️ 中風險 | 僅完整替換時使用 |

## 🆕 學習重點三：Cosmic/P0nk 客戶端完整架設流程 (2026)

**來源**: https://github.com/P0nk/Cosmic-client

### 標準架設六步驟

```
Step 1: 安裝階段
  ↓
  下載 MapleGlobal-v83-setup.exe 並執行
  安裝到 C:\Nexon\MapleStory 或自選目錄

Step 2: 清理階段（重要！）
  ↓
  刪除 HShield/ 資料夾（整個）
  刪除 ASPLnchr.exe
  刪除 MapleStory.exe
  刪除 Patcher.exe

Step 3: WZ 替換階段
  ↓
  複製 cosmic-wz 中的所有 WZ 檔案
  覆蓋到安裝目錄

Step 4: 客戶端複製階段
  ↓
  複製 HeavenMS-localhost-WINDOW.exe
  到安裝目錄

Step 5: IP 設定階段
  ↓
  使用 HxD 修改 localhost.exe 中的 IP（如需非 127.0.0.1）

Step 6: 啟動階段
  ↓
  先啟動伺服器
  然後執行 localhost.exe
```

### 常見問題：客戶端被標記為病毒

```
問題：Windows Security 將 localhost.exe 標記為威脅並刪除

原因：修改過的 MapleStory 客戶端會被識別為潛在風險

解決方案：
1. Windows Security → Virus & threat protection
2. 管理設定 → 新增排除項目
3. 加入下載資料夾和安裝目錄
```

### 疑難解答

**Q: 客戶端啟動後沒有任何反應？**
```
→ 嘗試快速連續點擊 localhost.exe 多次
→ 檢查是否是 64-bit 問題（需使用 32-bit 版本）
→ 確認 HShield 完全刪除
→ 確認 fonts folder 存在且有讀取權限
```

**Q: "We are unable to connect to the login server"？**
```
1. 確認 Cosmic/HeavenMS 伺服器已啟動並顯示 "online"
2. 確認 localhost.exe 中的 IP 正確
3. 確認 Port 正確（預設 7575）
4. 檢查防火牆
```

**來源**: https://forum.ragezone.com/threads/maple-patcher.1170344/

MaplePatcher 是一個現代化的 MapleStory 客戶端修補/更新系統：

| 功能 | 說明 |
|------|------|
| CRC32 完整性檢查 | 對比本地檔案與遠端 HTTP 主機的 CRC32，確保一致性 |

```

```

## 🆕 學習重點五：跨版本物品 Import 失敗的完整排查流程 (RaGEZONE 2025-11)

**來源**: https://forum.ragezone.com/threads/maplestory-v83-newer-items-import.1256083/

### 問題症狀
- 物品可以生成（可在背包看到）
- 但**無法裝備**（灰色無法點擊）

### 根本原因

| 原因 | 說明 |
|------|------|
| `.img` 結構不完整 | 較新版本的物品 `.img` 包含 v83 不支援的屬性節點 |
| `Character.wz` 缺少定義 | 裝備需要在 Character.wz 中有對應的造型定義 |
| 新屬性不存在 | v83 客戶端無法識別某些 v100+ 才引入的屬性 |
| 屬性值不合規範 | `lvlm`、`STR`、`DEX` 等屬性在 v83 有不同範圍限制 |

### 解決方案流程圖

```
1. 從同版本提取（推薦）
   ↓
   直接從 v83 WZ 提取所需物品
   ✅ 最安全，完全相容

2. 若必須跨版本
   ↓
   a. 提取完整的 .img 文件
   b. 移除 v83 不支援的屬性節點
   c. 確認 lvlm、STR、DEX 在合理範圍內
   d. 在 Character.wz 新增對應的裝備造型定義
   e. 在 String.wz 新增物品名稱
   ↓
3. 驗證
   ↓
   在客戶端測試物品是否正常顯示
   確認物品可正常裝備
```

### v83 裝備必備屬性檢查清單

| 屬性路徑 | 說明 | v83 限制 |
|---------|------|---------|
| `info/lvlm` | 等級上限 | ≤ 200 |
| `info/STR` | 力量需求 | 0-999 |
| `info/DEX` | 敏捷需求 | 0-999 |
| `info/INT` | 智力需求 | 0-999 |
| `info/LUK` | 幸運需求 | 0-999 |
| `info/tuc` | 可強化次數 | 0-127 |
| `info/cursed` | 詛咒標誌 | 0 或 1 |
| `info/only` | 職業限制 | 有效職業代碼 |

## 🆕 學習重點六：Nexon 字體（Fonts）問題修復 (2026 新發現)

### 問題徵兆
- 客戶端啟動後文字無法顯示
- 只有背景圖片和框架
- UI 文字全部變成方塊或空白

### 根本原因
- 缺少 `fonts` folder
- 字體檔案未正確複製
- 某些改過的客戶端移除了 fonts

### 解決方案

```
Step 1: 找到正常的客戶端
  - 從備份或其他正常運作的客戶端複製

Step 2: 複製 fonts folder
  - 將 fonts folder 粘貼到目標客戶端目錄
  - 確認路徑：MapleStory/fonts/

Step 3: 檢查權限
  - 確認 fonts folder 有讀取權限
  - 嘗試以管理員身份執行客戶端

Step 4: 若問題持續
  - 嘗試使用完整客戶端重新安裝
  - 或從 MapleSolaxiaV2 資源包取得完整 fonts
```

## 🆕 學習重點七：Nexus.NX 格式工具 (RaGEZONE 2026-02)

**來源**: https://forum.ragezone.com/threads/wz-and-nx-file-questions.1260605/

### WZ to NX 轉換工具

**下載**: `WzToNx.exe`

**使用方法**：
```
將 .wz 檔案拖曳到 WzToNx.exe 上
自動在同一目錄生成 .nx 檔案
```

### NX 格式優點

| 優點 | 說明 |
|------|------|
| 檔案更小 | 比 WZ 節省 30-50% 空間 |
| 載入速度更快 | 二進位格式解析更快 |
| 效能更好 | 減少 CPU 和記憶體使用 |
| Git 友好 | 差異更容易追蹤 |

### v83 支援狀態

```
並非所有 v83 模擬器都支援 NX 格式：
- ✅ Cosmos：支援 NX（開發中）
- ⚠️ HeavenMS：可能需要額外設定
- ❌ 傳統 WZ 音源：不支援

使用前請確認你的音源是否支援！
```
