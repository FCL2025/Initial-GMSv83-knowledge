## 🆕 新發現 1: Maple_Pshark v83 專用封包 Logger (GitHub, 持續更新)

**GitHub**: https://github.com/obstriker/Maple_Pshark

### 核心功能
這是專為 **MapleRoyals v83 私人伺服器** 設計的封包截獲工具，可直接攔截並解密往來封包。

### 功能一覽
| 功能 | 說明 |
|------|------|
| 攔截 Recv 封包 | 截獲伺服器→客戶端的封包 |
| 攔截 Send 封包 | 截獲客戶端→伺服器的封包 |
| 修改封包 | 可在傳輸過程中修改封包內容 |
| 注入封包 | 可向任意方向注入新封包 |
| 規則過濾 | 基於規則的封包過濾/操縱 |

### 技術原理
```
MapleStory.exe (客戶端)
       ↓ Hook 加密函數
Maple_Pshark.exe (截獲點)
       ↓
localhost.exe 或 私服伺服器
```
通過 **Hook 加密函數**，Maple_Pshark 可以在 AES/Shanda 解密之前截獲明文封包。

### 使用場景
- 分析私服自訂 Opcode 和封包結構
- 測試客戶端對特定封包的反應
- 開發新功能時快速驗證封包格式
- 排查客戶端-伺服器通訊問題

## 🆕 新發現 3: MapleRef Wiki - Opcode 結構速查 (2026 新版)

**來源**: https://mapleref.fandom.com/wiki/Operation_Code
**來源**: https://mapleref.fandom.com/wiki/Packets

### 重要原則
> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

### Send vs Receive Opcode 區分

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

**重要**：同一個 Opcode 數字可能代表不同含義，視方向而定！

### v83 Opcode 結構關鍵點
- Opcode 是 **2 bytes** (little endian)
- 例如：Opcode `0x0001` 在封包中顯示為 `01 00`
- 封包頭部為 4 bytes (包含混淆後的 opcode + length)

## 🆕 新發現 4: botnored - MapleStory v83 封包類別結構

**來源**: https://botnored.weebly.com/blog/maplestory-classes-v83

### 重要原則
> **第一個發送/接收的封包永遠是純文字（未加密）**
> MapleStory 封包透過 TCP Socket 傳輸

### 客戶端主要類別
| 類別 | 功能 |
|------|------|
| `CWvsContext` | 世界狀態管理（角色資料、伺服器列表） |
| `CClientSocket` | 網路連接管理（Send/Recv） |
| `CField` | 當前地圖/遊戲場景處理 |
| `CLogin` | 登入畫面處理 |

### 封包流向
```
CClientSocket::ProcessPacket()
    ↓
根據 Header 分發
    ↓
CField::OnPacket() 或 CLogin::OnPacket()
    ↓
處理具體 Opcode
```

## 🆕 新發現 9: v83 商場封包 Dupe 分析 (PDFCOFFEE)

**來源**: https://pdfcoffee.com/v83-dupe-pdf-free.html

### Opcode 0x6F (SHOP_BUY) 結構
```
6F 00 20 00 FF FF FF 80 00
```
- `6F 00`: Opcode (little endian = 0x006F)
- `20 00`: 商店ID (0x0020 = 32)
- `FF FF FF 80`: 物品索引 + 最大 Mesos 標誌 (0x80000000)
- `00`: 結束標記

> ⚠️ 僅供學習研究用途，請勿用於作弊
