### 3) Opcode結構 ✅
| 方向 | 說明 | 重要原則 |
|------|------|---------|
| **S→C (Send)** | 伺服器發送，客戶端接收 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送，伺服器接收 | Client Opcode Set |

**⚠️ 同一Opcode數字在Send/Recv方向代表完全不同含義！**

### 4) WZ修改 ✅
**黃金法則**：
> DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs.

**正確流程**：
1. HaRepacker 開啟 GMS (old) 加密的客戶端 WZ
2. 使用 UI 直接修改數值（禁止刪除/重新導入節點！）
3. 儲存 → "Private Server..." 導出為 XML
4. 覆蓋到伺服器的 WZ.XML 目錄
