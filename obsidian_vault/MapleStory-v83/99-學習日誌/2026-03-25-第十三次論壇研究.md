## 🆕 新發現 2: Ezorsia v2 獨立 DLL 客戶端詳解 (RaGEZONE Nov 2023, 持續更新)

### 突破性特點
**GitHub**: https://github.com/444Ro666/MapleEzorsia-v2

> "this is the first ever standalone dll client ever released (at least for v83)"
> — RaGEZONE, 2023-11

### Ezorsia v2 工作原理
```
┌─────────────────────────────────────────────┐
│  Ezorsia v2 DLL (dinput8.dll)              │
│                                             │
│  1. Hook dinput8.dll (第二早加載的 DLL)     │
│  2. 修改記憶體中的客戶端位址                 │
│  3. 啟用 HD 解析度                          │
│  4. 自動載入 Ezorsia WZ edits               │
│  5. 自動連接到設定的伺服器 IP               │
└─────────────────────────────────────────────┘
```

### Ezorsia v2 完整安裝步驟
```
1. 安裝預設 v83 客戶端（不要刪除任何檔案）
2. 下載 Ezorsia v2 DLL（從 releases 頁面）
3. 將 dinput8.dll 放到遊戲目錄
4. 編輯 config.ini 設定伺服器 IP
5. 啟動遊戲
```

### config.ini 關鍵設定
```ini
[Connection]
serverIP=127.0.0.1
serverPort=8484

[Display]
width=1280
height=720
windowMode=true

[Advanced]
skipIntro=true
```

### Hendi 4G 客戶端 Hex 編輯（可選）
```
位置: 0000013E
原始值: 0f
修改為: 2f
目的: 啟用 "Large Address Space Aware" (LAA) 旗標
效果: 客戶端可用到 4GB 記憶體，而非僅 2GB
```

## 🆕 新發現 3: MapleClientEditTemplate 開源編輯框架

**GitHub**: https://github.com/oung/MapleClientEditTemplate

### 核心功能
| 功能 | 說明 |
|------|------|
| **WinSock Hook** | 攔截網路連線，實現 IP 導向 |
| **CreateMutex Hook** | 防止多開檢測 |
| **IJL15 Proxy Injection** | 自動注入 dinput8.dll |
| **GenericLauncher** | 啟動 MapleStory 並注入 LEN.dll |

### commonconfig.h 主要設定
```cpp
#define MAPLE_INJECT_USE_IJL    // 啟用 IJL 自動注入
#define MAPLE_INSTAJECT         // 立即注入（無需等待客戶端解包）
#define ENABLE_WINKSOCK_HOOK    // 啟用 WinSock Hook（v83 預設需啟用）
#define ENABLE_IP_REDIRECT      // 啟用 IP 導向
#define REDIRECT_IP "127.0.0.1"  // 目標 IP
#define REDIRECT_PORT 7575      // 目標 Port
```

### 編譯步驟（x86）
```
1. git clone https://github.com/oung/MapleClientEditTemplate
2. Visual Studio 2019+ 打開 .sln
3. 確認專案設定為 x86（不是 x64）
4. 複製 detours.lib 到 Debug/Release 資料夾
5. 編譯輸出 DLL 命名為 ijl15.dll 或 LEN.dll
```

## 🆕 新發現 4: MapleRef Wiki Opcode 結構 (mapleref.fandom.com)

**來源**: https://mapleref.fandom.com/wiki/Operation_Code

### Opcode 核心概念
> "Actions that are defined by an identifier called an Operation Code (opcode for short), which is the first 2 bytes of the packet (a short)"

### Send vs Receive Opcode 區分
| 方向 | 說明 |
|------|------|
| **S→C (Send)** | 伺服器發送，客戶端接收 |
| **C→S (Recv)** | 客戶端發送，伺服器接收 |

**重要**：同一個 Opcode 數字可能代表不同含義，視方向而定

### MapleRef Packets 結構
**來源**: https://mapleref.fandom.com/wiki/Packets
- 完整的封包結構文檔
- 涵蓋各版本 Opcode 對應關係
- 可配合 Wireshark 使用

## 🆕 新發現 6: v83 Hex Edit 客戶端修改原理 (RaGEZONE 2010)

**來源**: https://forum.ragezone.com/threads/how-to-hex-edit-clients-easily.703014/

### 測試環境
- v83 MapleStory localhost
- HxD Hex Editor

### IP 修改步驟
```
1. 開啟 MapleStory.exe 或 localhost.exe
2. Ctrl+F → Text String → 搜尋 "127.0.0.1"
3. 替換為目標 IP（注意長度一致）
4. 存檔
```

### 常見問題
- IP 字串可能 Split 存放（分多段）
- 需確認長度一致否則記憶體崩潰
- 部分版本用 Hex 表示 IP (7F 00 00 01 = 127.0.0.1)

## 🆕 新發現 7: v83 WZ 編輯實戰技巧 (maplesecrets, 2014)

**來源**: http://maplesecrets.blogspot.com/2014/05/tutorial-wz-editing-for-maplestory-d.html

### Mob.wz 常用編輯

#### Miss Godmode (怪物必打不中)
```
1. Open deMSwZ
2. Open Mob.wz
3. Untick all boxes except acc
4. Tick acc, set value = 0
5. Process All → Save
```

#### Jesus Mode (怪物只打1血)
```
1. Open deMSwZ
2. Open Mob.wz
3. Untick all except MADamage, PADamage, acc
4. Set those values = 1
5. Process All → Save
```

#### No Jump (移除跳躍怪)
```
HaRepacker → Mob.wz → [怪物ID] → Remove "jump" → Save
```

### Item.wz 透明道具
```
HaRepacker → Item.wz → [物品] → Delete iconRaw → Save
```

## 🆕 新發現 9: MapleStory v83 封包類別結構 (botnored)

**來源**: https://botnored.weebly.com/blog/maplestory-classes-v83

### 重要原則
> **第一個發送/接收的封包永遠是純文字（未加密）**
> MapleStory 封包透過 TCP Socket 傳輸

### 客戶端主要類別
| 類別 | 功能 |
|------|------|
| `CWvsContext` | 世界狀態管理（角色資料、伺服器列表） |
| `CClientSocket` | 網路連接管理（Send/Recv） |
| `CField` | 當前地圖/遊戲場景處理 |
| `CLogin` | 登入畫面處理 |

### 封包流向
```
CClientSocket::ProcessPacket()
    ↓
根據 Header 分發
    ↓
CField::OnPacket() 或 CLogin::OnPacket()
    ↓
處理具體 Opcode
```

## 🆕 新發現 10: v83 商場 Dupe 封包分析

**來源**: https://pdfcoffee.com/v83-dupe-pdf-free.html

### Opcode 0x6F (SHOP_BUY) 結構
```
6F 00 20 00 FF FF FF 80 00
```
- `6F 00`: Opcode (little endian)
- `20 00`: 商店ID
- `FF FF FF 80`: 物品索引 + 最大 Mesos 標誌
- `00`: 結束標記

> ⚠️ 僅供學習研究用途
