## 📊 2. Opcode 結構 - 深化學習

### MapleRef Wiki 封包結構參考
- **URL**: https://mapleref.fandom.com/wiki/Packets
- **Opcode Wiki**: https://mapleref.fandom.com/wiki/Operation_Code
- **重要原則**：Opcode 是封包開頭用於識別類型的唯一識別符（2 bytes little endian）
- **伺服器端管理**：通常放在枚舉（enum）中方便管理

### MapleStory IDC Script - IDA Opcode 自動化命名
- **GitHub Gist**: https://gist.github.com/diamondo25/be95345a2875ab4342cd
- **功能**：自動化 IDA 中的 Opcode 命名腳本
- **搭配**：Wireshark 抓包分析，可大幅加速封包結構對照

### v83 夫妻訊息 Opcode
| Opcode | 名稱 | 方向 | 結構 |
|--------|------|------|------|
| 0x79 | ON_COUPLE_MESSAGE | S→C | `byte, string, byte, string` |
| 0x88 | COUPLE_MESSAGE | C→S | 同上 |

## 📊 3. WZ 修改 - 完整流程

### HaRepacker WZ 導出流程 (RaGEZONE, 2023)
- **論壇帖**: https://forum.ragezone.com/threads/community-wz-img-repack-for-v83.1221835/
- **導出步驟**：
  1. 用 HaRepacker 打開 WZ 檔案
  2. 選擇要導出的節點
  3. 點擊 Export → 選擇 XML (Private Server) 格式
  4. 用文字編輯器修改
  5. 重新導入 WZ
- **加密格式**：v83 WZ 需選擇 `GMS (old)`
- **注意**：不要直接刪除/重新導入節點，可能造成數據丟失

### WZ 編輯器使用技巧
| 編輯項目 | WZ 檔案 | 說明 |
|---------|---------|------|
| NPC 外觀 | NPC.wz | 0001337.img 為模板 |
| 地圖物件 | Map.wz | 地形、出生點 |
| 怪物資料 | Mob.wz | type=M 為怪物 |
| 物品屬性 | Item.wz | 武器/防具/消耗品 |

### NPC WZ 修改具體步驟
```
1. 複製 0001337.img 作為模板
2. 修改 frame 和 origin 設定
3. 調整尺寸
4. 將修改後的 img 放入 NPC.WZ
5. 在伺服器端編寫對應的 .js 腳本
```

## 📊 7. MapleStory v83 全10步登入封包序列

```
1. C→S Hello Packet (plaintext)
   - 告知版本、SendIV、RecvIV

2. S→C Hello Packet (plaintext)
   - 伺服器確認版本、回應 IV

3. C→S LOGIN_PASSWORD (encrypted)
   Opcode: 0x01
   Data: 帳號 + 密碼 (AES-OFB 加密)

4. S→C LOGIN_STATUS (encrypted)
   Opcode: 0x00
   Data: 結果碼 + 帳號資訊

5. C→S SERVERLIST_REQUEST (encrypted)
   Opcode: 0x06

6. S→C SERVERLIST (encrypted)
   Opcode: 0x0C

7. C→S CHARLIST_REQUEST (encrypted)
   Opcode: 0x07

8. S→C CHARLIST (encrypted)
   Opcode: 0x03

9. C→S CHAR_SELECT (encrypted)
   Opcode: 0x08

10. C→S PLAYER_DROP (encrypted)
    Opcode: 0x0B
```

### 登入結果碼 (Login Status Code)
| 結果碼 | 意義 |
|--------|------|
| 0 | 登入成功 |
| 5 | 帳號或密碼錯誤 |
| 7 | 帳號已停用 |
| 13 | 刪除中（冷靜期） |
| 23 | 登入太頻繁（防機器人延遲） |

## 📊 9. sendGetNumber / sendGetText / sendAcceptDecline 完整行為表

### sendAcceptDecline (Type = 12)
| Mode 值 | 意義 |
|---------|------|
| `mode == -1` | 玩家關閉對話框 |
| `mode == 1` | 玩家點擊「接受」(Accept) |
| `mode == 0` | 玩家點擊「拒絕」(Decline) |

### sendGetNumber (Type = 3)
```javascript
cm.sendGetNumber("請輸入你要購買的數量：", 1, 1, 99);
// 參數：提示文字、最小值（1）、最大值（99）、預設值
```

### sendGetText
```javascript
cm.sendGetText("請輸入你的角色暱稱：");
// 玩家輸入的文字透過 action 的 selection 參數傳回
```

```

3. 移除 HShield 目錄（整個刪除）
```
