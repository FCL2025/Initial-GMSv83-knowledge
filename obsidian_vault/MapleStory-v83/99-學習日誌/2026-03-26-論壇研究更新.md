### 4. MapleStory v83 三層加密實戰
- **加密流程**：
  ```
  客戶端發送：原始封包 → Shanda 加密 → AES 加密 → 傳送
  伺服器接收：AES 解密 → Shanda 解密 → 處理
  ```
- **關鍵參考**：
  - maplelib (Go): https://pkg.go.dev/github.com/defaultmagi/maplelib
  - MapleCrypto AES-256-ECB 實現
  - RustMS (Rust): https://github.com/neeerp/RustMS
- **登入封包分析**：
  - 輸入帳號 "aaaa" / 密碼 "111111"
  - AES 解密後仍需 Shanda 解密
  - Opcode 應為 0x0001 (LOGIN_PASSWORD)

## Opcode 結構要點

### v83 Opcode 特性
- **Opcode 長度**：2 bytes (little endian)
- **格式**：`[封包頭部][Opcode (2 bytes)][資料...]`
- **頭部混淆**：MapleCrypto 頭部（4 bytes）

### 頭部混淆原理
```
原始 opcode: 0x0001
IV: [0x7A, 0x52, ...]

Step 1: XOR + SHIFT
  verify = (opcode ^ iv[0]) + (iv[1] << 8)

Step 2: 位旋轉 (rotate left 3 bits)

Step 3: 取低位 16 bits
```

### 雙層加密架構
```
客戶端發送：原始封包 → Shanda 加密 → AES 加密 → 傳送
伺服器接收：AES 解密 → Shanda 解密 → 處理
```

## NPC 腳本要點

### 基本模板
```javascript
var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        if (mode == 1)
            status++;
        else
            status--;
        
        if (status == 0) {
            cm.sendNext("文字");
        } else if (status == 1) {
            // 下一個訊息
        }
    }
}
```

### 常用方法
- `cm.sendNext()` - 顯示「下一步」
- `cm.sendYesNo()` - Yes/No 選擇
- `cm.sendSimple()` - 選單
- `cm.dispose()` - 結束腳本

### 腳本源間相容性
- Cosmic、Kinoko、HeavenMS 的腳本解析器不同
- 較舊腳本在新解析器可能不相容
- 建議使用通用的腳本格式
