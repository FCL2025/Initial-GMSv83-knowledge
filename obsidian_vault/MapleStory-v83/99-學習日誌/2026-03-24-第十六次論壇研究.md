## 🆕 MapleStory v83 登入封包三層加密實戰（RaGEZONE 2018）

**來源**：https://forum.ragezone.com/threads/help-remove-encrypt-decrypt-in-client-cms-chinese-maplestory.1155998/

### 實戰案例：中國 MapleStory (CMS) v83 登入封包解密

**觀察到的客戶端原始發送資料（加密後）**：
```
64 A4 9F 07 FD 4D 6C 49 4A 3D 96 FE C8 44 85 E4 79 FE EE E3 C7 3E 9B BF 42 3B 97 B2 4E 36 F8 DB F4 8C 0D D9
```

**第一步：AES-OFB 解密**（使用版本決定的 Key 和 IV）
```
結果：71 1A 5B 6A 7E B9 C9 EA 15 17 7E E4 C0 B4 64 6D 1E E7 8A 13 C9 B0 8C 09 4C 7F F5 6F F8 A9 2C 5D 03 2D D4 D5
```

**第二步：Shanda 對稱解密**
```
結果：E2 FB 40 7E 67 08 B6 77 F4 CC 0E 3E 5D 72 D1 9A 2C 7C F6 44 16 6A 7F 2F ED 39 C8 DE 96 F7 95 04 59 88 C9 6F
```

**Opcode 識別**：
- 解密後第一個 2 bytes：`E2 FB`（Big Endian）= `0xFBE2`
- 預期 LOGIN_PASSWORD opcode 應為 `0x01 00`
- **差異原因**：版本不匹配（AES Key/IV 與伺服器不一致）

### 關鍵學習點
| 觀察 | 意義 |
|------|------|
| 加密後 `64 A4 9F...` | AES 加密後的密文 |
| `71 1A 5B...` | AES 解密成功（看起來仍像隨機） |
| `E2 FB 40 7E...` | Shanda 解密後可識別 opcode |
| `E2 FB` ≠ `01 00` | 版本/Key 不匹配 |

### v83 三層加密流程圖
```
明文資料
    ↓
[Layer 3] AES-OFB 加密（版本靜態 Key + IV）
    ↓
[Layer 2] Shanda 對稱加密
    ↓
[Layer 1] MapleCrypto 頭部混淆（4 bytes header）
    ↓
網路傳輸
```

## 🆕 MapleStory 封包加密/解密核心概念（maplelib, Go 語言）

**來源**：https://pkg.go.dev/github.com/defaultmagi/maplelib

### EncryptNoShanda（純 AES，不含 Shanda）
```go
// 只使用 AES 的加密方式
// 注意：陣列前面必須預留 4 bytes 空間給加密後的 Header
EncryptNoShanda(data []byte, key []byte, iv []byte) []byte
```

### MapleCrypto 頭部
- 每個封包前面有 **4 bytes** 的加密頭部
- 頭部包含：混淆後的 opcode + 封包長度
- 頭部使用 **IV（初始化向量）** 動態生成

### MapleStory 封包定義
```go
// Packet 是已解密的 MapleStory 封包
type Packet struct {
    Opcode uint16
    Data   []byte
}
```

### 封包加密後的格式
```
[4-byte MapleCrypto Header][加密內容...]
         ↓
   包含 length + 混淆 opcode
```

## 🆕 MapleStory v83 封包類別結構（botnored, 2016）

**來源**：https://botnored.weebly.com/blog/maplestory-classes-v83

### 核心類別

| 類別 | 功能 |
|------|------|
| `CWvsContext` | 世界狀態管理（角色資料、伺服器列表、玩家狀態） |
| `CClientSocket` | 網路連接管理（TCP 連接、封包發送/接收） |
| `CField` | 當前地圖/遊戲場景處理（遊戲中狀態） |
| `CLogin` | 登入畫面處理（登入狀態） |

### CClientSocket 封包處理流程
```
接收 TCP 資料
    ↓
CClientSocket::ProcessPacket()
    ↓
解析 MapleCrypto Header (4 bytes)
    ↓
解密並提取 opcode (2 bytes)
    ↓
剩餘為 payload
    ↓
根據 opcode 分發到 CField 或 CLogin
```

### CLogin 類別（登入畫面）
- 處理所有登入相關的封包
- 收到伺服器回應後更新 UI
- 負責切換到遊戲畫面

### CField 類別（遊戲中）
- 處理遊戲內所有互動封包
- NPC 對話、戰鬥、移動、道具操作等

### CWvsContext 類別
- 儲存玩家狀態
- 管理角色資料結構
- 處理伺服器發送的狀態更新封包

## 🆕 MapleStory v83 登入器/客戶端修改完整流程（RaGEZONE 2015）

**來源**：https://forum.ragezone.com/threads/source-how-to-create-your-own-v83-maplestory-private-server.759889/

### 客戶端修改步驟（官方流程）

**Step 1: 安裝客戶端**
- 使用 `ManagerMsv83.exe` 安裝 MapleStory 到 `C:\Nexon\MapleStory`

**Step 2: 清理必要檔案**
```
移除 HShield 資料夾（ anti-cheat）
刪除 ASPLauncher.exe
刪除 MapleStory.exe
刪除 Patcher.exe
```

**Step 3: 安裝 localhost.exe**
- 複製 `localhost.exe` 到客戶端目錄
- 這是修改過的啟動器，會連接到本地/私人伺服器

**Step 4: 修改 WZ 檔案**
- 用修改過的 WZ（來自伺服器資源包）覆蓋原版
- 這些 WZ 包含：正確的伺服器 IP、地圖資料、NPC 資料

**Step 5: Hex 修改 IP（如需要）**
- 如果使用非 127.0.0.1 的 IP
- 用 HxD/Neo Hex Editor 開啟 `localhost.exe`
- 搜尋 `127.0.0.1` 並替換為目標 IP
- 通常有 3 處需要同時修改

### v83 localhost.exe IP 修改位置
```
HxD 搜尋 "127.0.0.1"
找到後逐一替換為目標 IP（如 192.168.1.100）
通常出現在：
- 登入伺服器 IP
- 遊戲伺服器 IP
- 保留字段
```
