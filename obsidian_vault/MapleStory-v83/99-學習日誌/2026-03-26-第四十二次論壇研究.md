## 🆕 MaplePacketPuller + SpiritIDAPlugin 完整工作流程 (2026-03)

**GitHub**:

- SpiritIDAPlugin: https://github.com/Bratah123/SpiritIDAPlugin

### SpiritIDAPlugin 功能
SpiritIDAPlugin 是 MaplePacketPuller 的 IDA Pro 插件版，可直接在 IDA 中提取封包結構。

### 支援環境
| 環境 | 支援狀態 |
|------|---------|
| Python 3.8.5 | ✅ 測試通過 |
| Python 3.6.12 / 3.8.6 | ✅ 測試通過 |
| IDA Pro 7.0 (32-bit) | ✅ 測試通過 |
| IDA Pro 7.5 | ✅ 測試通過 |
| IDA Pro 6.8 | ✅ 測試通過 |
| Python 2.7 | ❌ 不相容 (os.scandir) |

### 完整工作流程
```
Step 1: IDA Pro 反編譯
  → 在 IDA 中找到 CField::OnPacket 或 CLogin::OnPacket
  → 產生 pseudocode (.p64 或 .txt)

Step 2: 複製 pseudocode
  → 在 Functions/ 目錄建立 .txt 檔案
  → 貼上 IDA 產生的 C-style pseudocode

Step 3: Python 分析
  → cd MaplePacketPuller/IDA\ Maple\ Script/src/main/python
  → python main.py
  → 輸入分析類型（完整/標準）
  → 輸入要分析的檔案名稱

Step 4: 輸出結果
  → 封包結構自動寫入 FuncOutput/ 目錄
  → 控制台顯示 Decode1/Decode2/Decode4/DecodeStr 等欄位
```

### SpiritIDAPlugin 自動化亮點
- 直接在 IDA 中格式化 pseudocode 為伺服器 encode 友好格式
- 支援自訂準確度選項
- 自動搜尋 InHeader opcode
- ⚠️ 限制：while() 迴圈處理不完善（建議設 GET_ALL_DECODES=true）

### MaplePacketPuller 產出範例
```
輸入 (IDA pseudocode):
int __thiscall sub_ABC1230(CInPacket *this)
{
  CInPacket::Decode4(this);   // opcode
  CInPacket::DecodeStr(this); // username
  CInPacket::DecodeStr(this); // password
  ...
}

輸出 (封包結構):
LOGIN_PASSWORD (Opcode: 0x01)
  - int opcode
  - string username
  - string password
```

|------|------|

## 🆕 MapleStory v83 登入流程Opcode 序列 (最新研究)

### 完整 10 步登入封包序列
```
1. C→S Hello Packet (plaintext - 無 opcode header)
   - 告知版本、SendIV、RecvIV
   - 為純文字，Wireshark 直接可見

2. S→C Hello Packet (plaintext)
   - 伺服器確認版本、回應 IV

3. C→S LOGIN_PASSWORD (encrypted)
   Opcode: 0x01
   Data: 帳號字串 + 密碼字串 (AES-OFB 加密)

4. S→C LOGIN_STATUS (encrypted)
   Opcode: 0x00
   Data: 結果碼 + 帳號資訊

5. C→S SERVERLIST_REQUEST (encrypted)
   Opcode: 0x06
   - 請求世界伺服器列表

6. S→C SERVERLIST (encrypted)
   Opcode: 0x0C
   Data: 世界伺服器名稱、負載、人數上限

7. C→S CHARLIST_REQUEST (encrypted)
   Opcode: 0x07
   - 請求指定世界的角色列表

8. S→C CHARLIST (encrypted)
   Opcode: 0x03
   Data: 角色資料陣列（名稱、等級、職業、外觀）

9. C→S CHAR_SELECT (encrypted)
   Opcode: 0x08
   - 選擇角色

10. C→S PLAYER_DROP (encrypted)
    Opcode: 0x0B
    - 進入遊戲世界
```

### v83 登入結果碼（Login Status Code）

| 結果碼 | 意義 |
|--------|------|
| 0 | 登入成功 |
| 5 | 帳號或密碼錯誤 |
| 7 | 帳號已停用 |
| 13 | 刪除中（冷靜期） |
| 23 | 登入太頻繁（防機器人延遲） |
| 其他 | 未知錯誤 |

## 🆕 MapleRef Wiki - Opcode/Packet 結構參考 (2026-03)

**URL**: https://mapleref.fandom.com/wiki/Operation_Code
**URL**: https://mapleref.fandom.com/wiki/Packets

### 核心概念
> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

**重要**：同一個 Opcode 數字可能代表不同含義，視方向而定！

### MapleRef 使用方式
```
1. 前往 https://mapleref.fandom.com/wiki/Packets
2. 選擇目標版本 (v83)
3. 查看 Send (S→C) 或 Recv (C→S) 封包結構
4. 點擊具體 Opcode 查看封包結構
5. 配合 Wireshark 抓包分析
```

## 🆕 v83 WZ 修改核心原則 (2026-03 最新)

### WZ 編輯最重要原則
> **千萬不要用伺服器端的 XML 重新導入客戶端 WZ，會產生 bug！**

### 正確 WZ 修改流程
```
1. HaRepacker 4.2.4 開啟 GMS (old) 加密的客戶端 WZ
2. 使用 UI 直接修改數值
   （❗不要刪除/重新導入節點！）
3. 儲存 WZ（覆蓋客戶端目錄）
4. 在 HaRepacker 中選擇 "Private Server..." 導出
5. 將導出的 XML 覆蓋到伺服器的 wz/ 目錄
```

### 為什麼不能混用客戶端和伺服器 WZ？
| 問題 | 原因 |
|------|------|
| 客戶端崩潰 | 客戶端 WZ 可能包含 sprite 數據，伺服器 XML 沒有 |
| 顯示異常 | 客戶端 WZ 和伺服器 XML 的加密不同 |
| 版本衝突 | 不同音源的 WZ 版本號不同 |

### lvmax 等級限制移除方法
```
方法1：HaRepacker 圖形化操作
  1. HaRepacker 開啟 Quest.wz
  2. Tools → Export Selection → XML → Classic
  3. 用文字編輯器開啟匯出的 XML
  4. 搜尋 lvmax 屬性並刪除
  5. 重新匯入

方法2：Notepad++ Regex 批量移除
  搜尋：lvmax="\d+"\s*
  取代：（留空）
```

## 🆕 v83 NPC 腳本資源總整理 (2026-03)

### NPC 腳本核心概念

#### Status 狀態機制
```javascript
status = -1   // 對話結束
status = 0    // 第一個訊息
status = 1    // 第二個訊息（按 Next/Yes/No 後）
status++      // 每次按「下一步」遞增
```

#### Mode 參數含義
```javascript
mode == -1   // 關閉對話框（按 X 或結束）
mode == 0    // 按「返回」/「否」
mode == 1    // 按「下一步」/「是」
```

#### NPCConversationManager 常用方法
```javascript
// 對話發送
cm.sendNext("文字");           // 顯示「下一步」按鈕
cm.sendYesNo("文字");          // 顯示「是/否」按鈕
cm.sendSimple("文字\\r\\n#b#L0#選項0#l\\r\\n#L1#選項1#l");
cm.sendStyle("文字", styleId);  // 髮型/染色選擇
cm.sendGetNumber("文字", defaultNum, min, max);
cm.sendGetText("文字");
cm.dispose();                   // 結束腳本，關閉對話框

// 玩家狀態
cm.haveItem(2000005, 10);       // 檢查物品數量
cm.gainItem(2000005, 10);       // 給予物品
cm.getPlayer().getLevel();      // 獲取玩家等級
cm.warp(100000000);             // 傳送地圖
```
