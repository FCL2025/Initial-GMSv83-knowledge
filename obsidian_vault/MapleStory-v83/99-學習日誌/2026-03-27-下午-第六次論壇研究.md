### 2. 抓包分析 — 新資源

#### MapleProxy — 2026 新發現的 v83 專用代理工具
- **GitHub**：https://github.com/tylerhasman/MapleProxy
- **使用方式**：
  ```
  java -jar MapleProxy.jar
  → 提示輸入目標登入伺服器 IP
  → localhost.exe [目標IP] [代理端口, 預設 8480]
  ```
- **核心功能**：
  - 封包攔截（Send/Recv）
  - 封包修改動態重發
  - 封包重放測試
  - Opcode 過濾
  - 登入流程模擬
- **技術棧**：Lombok + Netty 4.0.15-Final + Eclipse
- **與 Maple_Pshark 對比**：
  | 工具 | MapleProxy | Maple_Pshark |
  |------|-----------|--------------|
  | 平台 | Java (JVM) | 獨立工具 |
  | 封包修改 | ✅ | ✅ |
  | Hook 加密函數 | ❌ | ✅ |
  | 易用性 | 中等 | 簡單 |

#### Maple_Pshark — v83 專用封包 Logger（持續活躍）

  2. 修改封包內容
  3. 注入自訂封包

### 5. NPC腳本編寫 — 新發現

#### Dynamic NPC Shop Script（RaGEZONE, 2016）
- **來源**：https://forum.ragezone.com/threads/dynamic-npc-shop-script-v83.1089979/
- **核心優點**：動態定義商店，無需修改資料庫 + 重啟伺服器
- **範例**：使用 `cm.openShop(shopId)` + `shop.addItem()` 動態添加商品

#### NPC TYPE 值含義解析（RaGEZONE, 2026-03 新發現）
| Type 值 | 名稱 | 客戶端行為 | 腳本需求 |
|---------|------|-----------|---------|
| `0` | 普通 NPC | 點擊後發送 NPC_TALK | ✅ 需要 .js |
| `1` | 商店 NPC | 直接開啟商店 | ⚠️ 可有可無 |
| `2` | 拍賣 NPC | 開啟拍賣介面 | ❌ 不需要 |
| `3` | 保險箱 NPC | 開啟保險箱 | ❌ 不需要 |
| `4~6` | 特殊 NPC | 各類特殊功能 | ❌ 不需要 |

#### IntransigentMS — ECMAScript 6 NPC 腳本源碼（GitHub）
- **GitHub**：https://github.com/NoetherEmmy/intransigentms-scripts
- **特點**：使用 Nashorn 引擎 + ECMAScript 6
- **範例**：
  ```javascript
  const MapleCharacter = Java.type("net.sf.odinms.client.MapleCharacter");
  print("debug message");  // 輸出到伺服器 console
  ```

#### NPC 腳本源相容性問題（RaGEZONE, 2026-03 新發現）
- 不同音源（Cosmic、Kinoko、HeavenMS）的 NPC 腳本解析器版本不同
- `importPackage` 在 JDK 7+ 已廢棄
- **最佳實踐**：
  - 使用 `cm.` 函數作為主要介面
  - 避免 `importPackage`
  - 先在本地測試再部署
