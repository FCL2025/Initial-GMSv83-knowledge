## 📌 客戶端修改：v83 Client Addresses (2024年8月更新)

### 重要參考資源

### Hex Edit 客戶端位址一覽

| 修改項目 | 類型 | 位址/修改方式 |
|---------|------|--------------|
| **Speed Cap** | Hex Edit | 速度上限修改 (250 speed = `FA 00 00 00`) |
| **Damage Cap** | Double | 傷害上限修改 |
| **BattleShip Rope Climb** | 記憶體修改 | Credit Rulax |
| **Chain Lightning / Piercing Arrow** | No damage fall off | `75C004` → `JMP SHORT 0075C006` |

### 速度修改計算方式
```
目標速度值 → 轉為 hex (little endian)
例如：250 speed → 0xFA → `FA 00 00 00`
```

### Themida 脫殼重點
- v83 客戶端普遍使用 **Themida** 保護殼
- `hendi` 的客戶端是 Themida 脫殼版本，適合修改
- 原版 HMS 客戶端內建 CRC bypass
- Themida 脫殼後需要 `VirtualProtect` 才能修改記憶體

- 優點：不需修改 SQL 資料庫表格
- 重點：透過 NPC 腳本動態生成商店介面

### 腳本結構示例
```javascript
var status = 0;
var items = [1302000, 1322005, 1442009]; // 物品ID列表
var prices = [1000, 5000, 10000]; // 價格列表

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && status == 2) {
            cm.dispose();
        } else {
            if (mode == 1)
                status++;
            else
                status--;
            if (status == 0) {
                var selStr = "歡迎來到我的商店！\r\n";
                for (var i = 0; i < items.length; i++) {
                    selStr += "#L" + i + "# #z" + items[i] + "# - " + prices[i] + " 金幣#l\r\n";
                }
                cm.sendSimple(selStr);
            } else if (status == 1) {
                var selectedItem = items[selection];
                var price = prices[selection];
                if (cm.getMeso() >= price) {
                    cm.gainMeso(-price);
                    cm.gainItem(selectedItem, 1);
                    cm.sendOk("交易完成！");
                } else {
                    cm.sendOk("金幣不足！");
                }
                cm.dispose();
            }
        }
    }
}
```

## 📌 封包加密機制深入分析

### 雙層加密結構
```
客戶端發送封包：
Raw Packet → Shanda Encrypt → AES Encrypt → Network

伺服器接收封包：
Network → AES Decrypt → Shanda Decrypt → Process
```

### 第一個封包特性
- **第一個封包為明文**（無加密）
- 用於握手和金鑰交換
- 之後所有封包使用動態生成的金鑰

### AES 金鑰管理
- MapleStory 使用 **AES-128** 加密
- 金鑰在連線時動態協商
- 客戶端和伺服器各自維護發送/接收金鑰

### Shanda 加密
- Nexon 早期使用的對稱加密
- 可透過逆向工程客戶端取得演算法
- 使用固定查表方式

### Wireshark 分析技巧
```
過濾式：
tcp.port == 8484  (MapleStory 預設連接埠)

解密密文：
1. 找到第一個明文封包（金鑰交換）
2. 提取 AES 金鑰
3. 使用 Wireshark Lua 插件解密後續封包
```
