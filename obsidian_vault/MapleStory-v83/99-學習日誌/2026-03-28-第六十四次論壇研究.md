### 🆕 MapleEzorsia V2 Wiki 開發指南深度解析

**來源**: https://github.com/444Ro666/MapleEzorsia-v2/wiki/v83%E2%80%90Client%E2%80%90Setup%E2%80%90and%E2%80%90Development%E2%80%90Guide

#### 核心設置要點
- 使用 Hendi's v83 客戶端（移除 Admin 提權 + 4G Flag）
- Ezorsia V2 DLL 直接 inject，無需刪除原廠檔案
- 最佳解析度：1280x720 + Magpie 全螢幕投射
- 4G Flag：PE Header 偏移 0x0000013E，0x0F → 0x2F

#### Admin 提權移除步驟（重要）
- 使用 Hendi's Hex Editor 工具包
- 編輯 `asInvoker` 部分（避免客戶端要求管理員權限）
- 編輯後需將執行檔改名為 `MapleStory2` 或其他名稱
- 原因：原名會繞過 UAC 提權請求

#### Magpie 全螢幕投射設定
- GitHub: https://github.com/Blinue/Magpie
- 選擇 MapleStory 視窗
- 啟用 "Auto scale when in foreground"
- 優勢：無拉伸、保持 16:9 比例、無損畫質

#### Ezorsia V2 UI.wz 說明
- 截至 2023/11/21，Ezorsia V2 使用獨立 WZ（EzorsiaV2_UI.wz）
- 不再需要手動替換 UI.wz
- 自動載入自訂登入畫面框架

### 🆕 Ezorsia V2 config.ini 完整設定參考

```ini
; 解析度設定
Width=1280
Height=720

; 登入畫面設定
EzorsiaV2WzIncluded=true
CustomLoginFrame=true

; 可選：指定第三方 DLL
CustomDLL1=
CustomDLL2=
CustomDLL3=
```

#### 第三方 DLL 安全建議
- 只使用可信來源的 DLL
- 建議自行編譯或審計源代碼
- Ezorsia V2 的 Windows API Hook 功能很強大，惡意 DLL 可造成嚴重後果

## 本次研究關鍵發現

### 1. 登入器/客戶端修改
- Ezorsia V2 + Hendi's Client 是 2026 年 3 月最佳組合
- 4G Flag 設置：PE Header 偏移 0x0000013E，0x0F → 0x2F
- Admin 提權移除後需改名執行檔

### 2. 抓包分析
- Maple_Pshark 是 v83 即時截獲首選工具
- 第一個 Hello Packet 永遠是明文
- Wireshark + Maple_Pshark + MapleRef Wiki 是完整工具鏈

### 3. Opcode 結構
- v83 Opcode 幾乎全部被映射
- 開源專案 Cosmic/HeavenMS 提供完整 Opcode 表
- IDA 分析只在修改客戶端時需要

### 4. WZ 修改
- GMS (old) 加密格式是 v83 標準
- 客戶端 WZ → 導出 XML → 伺服器 wz.xml
- WzImg-MCP-Server 提供 AI 輔助分析

### 5. NPC 腳本編寫
- 使用 cm. 作為主要介面
- 避免 importPackage（JDK 7+ 已廢棄）
- IntransigentMS 使用 ECMAScript 6 寫法
