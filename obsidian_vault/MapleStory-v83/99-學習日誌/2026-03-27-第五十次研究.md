## 一、AugurMS — AI Game Master 驅動的 MapleStory v83 私服

### 1.1 專案概覽

**GitHub**: https://github.com/themrzmaster/augurms

AugurMS 是一個基於 **Cosmic**（v83）並由 AI 驅動 Game Master 的 MapleStory 私人伺服器專案。

### 1.2 技術架構

```
augurms/
├── server/        # Java 遊戲伺服器 (Cosmic v83, Docker, Fly.io)
├── dashboard/      # Next.js 15 admin panel + APIs
├── launcher/      # Electron 桌面啟動器 (auto-update, Windows)
├── client/        # Patched MapleStory v83 client + custom WZ files
└── mcp-server/   # Standalone MCP tools server (external integrations)
```

### 1.3 AI Game Master 實作

- 使用 **Claude Code SDK** (`@anthropic-ai/claude-agent-sdk`)
- 約 **35 個 MCP tools**，用於讀取/修改遊戲狀態（drops, shops, events, spawns, config, WZ files）
- 透過 SSE 串流回應（thinking, tool calls, text）
- 可自主管理事件、調整經濟、引導玩家
- Dashboard UI 位於 `/gamemaster`，含 preset prompts、session history、live tool execution logs

### 1.4 部署架構

| 元件 | 平台 | 規格 |
|------|------|------|
| Game Server | Fly.io | 2GB RAM, 2 CPU |
| Dashboard | Fly.io | 512MB |
| MySQL | Fly.io | v8.4 + Liquibase migrations |
| DNS/CDN | Cloudflare | augurms.com |
| Client Files | GitHub Releases | Launcher 下載宿主 |

### 1.5 啟動器實作亮點

- Electron app for Windows（NSIS installer via GitHub Releases）
- Manifest-based auto-update：檢查 SHA256 hash，下載變更檔案
- 伺服器狀態顯示、News feed（來自 GM 活動）

### 1.6 對於學習者的價值

| 方向 | 啟發 |
|------|------|
| AI + 遊戲結合 | 展示 AI 驅動 GM 的可能性 |

### 3.2 客戶端修改必檢查清單

- [ ] 確認 HxD 修改了**三個** 127.0.0.1 位置
- [ ] 確認 IP 長度一致
- [ ] 確認 Windows 8/10 使用 lolwut.exe 輔助啟動

- [ ] 確認防火牆開啟 `7575` 和 `8484` 埠口
- [ ] 若錯誤碼 `0x80004005` → 防火牆/權限問題
- [ ] 若錯誤碼 `-2147467259` → 同上

## 四、WZ 和 NX 檔案格式最新討論 (RaGEZONE, Feb 2026)

**來源**: https://forum.ragezone.com/threads/wz-and-nx-file-questions.1260605/

### 4.1 WZ vs NX 格式比較

| 特性 | WZ | NX |
|------|----|----|
| 格式 | 專有加密格式 | 更高效的替代格式 |
| 效能 | 較慢 | 效能更好 |
| 工具支援 | HaRepacker 等 | 需要不同工具 |
| 用途 | 客戶端主要格式 | 私服優化常用 |

### 4.2 NX 格式優勢

> "It's a better performing alternative to wz/xml files"

NX 格式作為 WZ/XML 的高效替代方案，在效能上有顯著優勢：
- 載入速度更快
- 記憶體佔用更低
- 適合高流量私服

## 五、v83 客戶端修改最新討論 (RaGEZONE, Feb 2026)

**來源**: https://forum.ragezone.com/threads/client-changing.1260376/

### 5.1 問題背景
> "hi! how can i change clients files? write to memory and etc? v83 cosmic"

### 5.2 客戶端修改方法對比

| 方法 | 難度 | 風險 | 適用場景 |
|------|------|------|---------|
| Hex 編輯 | 低 | 低 | IP 修改、簡單字串 |
| IDA + 記憶體寫入 | 高 | 中 | 動態修改、函數 Hook |
| DLL 注入 | 高 | 中 | 進階功能 |
| Themida 脫殼 + 修改 | 很高 | 高 | 完整去除保護 |

### 5.3 v83 客戶端修改基本原理

v83 客戶端的網路連線邏輯：
```
localhost.exe 啟動
  → 讀取內建 IP 位址（預設 127.0.0.1）
  → 連線到 MapleStory 伺服器 (port 8484)
  → 驗證版本號 (0x0053 = 83)
  → 交換 AES IV
  → 進入遊戲
```

修改 IP 的本質就是用 HxD 找到並修改這些 IP 字串。

## 六、Kinoko NPC Scripts 最新討論 (RaGEZONE, ~4週前)

**來源**: https://forum.ragezone.com/threads/kinoko-npc-scripts.1261538/

### 6.1 討論背景
> "How do you usually create missing NPC scripts? Do you base them on an existing reference from somewhere (like an official script dump / another emulator), or do you just write them from scratch following Kinoko's script patterns?"

### 6.2 建立缺失 NPC 腳本的方法

| 方法 | 說明 |
|------|------|
| 官方腳本 Dump | 參考官方伺服器的 NPC 腳本 |
| 其他模擬器腳本 | Cosmic, HeavenMS 等開源專案 |
| 從零開始 | 按照 Kinoko 的腳本模式編寫 |

### 6.3 Kinoko 與其他音源的差異

| 差異點 | 說明 |
|--------|------|
| `status` 用法 | Kinoko 不需要 `status` 來確定如何反應 |
| 舊版腳本 | 使用 `status` 的腳本在 Kinoko 中不相容 |
| 解決方案 | 在 source 中 include 舊版 script reader |

### 6.4 Kinoko 音源特點

**問題核心**：
> "The difference between Kinoko and the aforementioned sources is that previously, 'status' was used to determine how the NPC would react. In Kinoko and other sources, this is not necessary."

**解決方案**：
1. 包含較舊版本的 script reader 以讀取其他來源的腳本
2. 或使用 Kinoko 友好的腳本格式重新編寫

### 6.5 測試 Job Advance 腳本時的發現

- 測試伺服器時，進行 job advance（轉職）到某個職業
- NPC 腳本需要正確處理職業轉換
- 需要確保 `scripts/npc/` 中有對應的 NPC 腳本

## 七、v83 Client Boss HP Bar 封包最新實作 (RaGEZONE, Feb 2026)

**來源**: https://forum.ragezone.com/threads/v83-client-boss-percentage-bar.1260584/

### 7.1 Boss HP Bar 封包實作

```java
/**
 * 顯示怪物 HP 百分比條
 * @param oid 怪物物件 ID
 * @param remhppercentage 剩餘 HP 百分比
 */
public static byte[] showMonsterHP(int oid, int remhppercentage) {
    final MaplePacketLittleEndianWriter mplew = new MaplePacketLittleEndianWriter();
    mplew.writeShort(SendOpcode.SHOW_MONSTER_HP.getValue());
    mplew.writeInt(oid);
    mplew.write(remhppercentage);
    return mplew.getPacket();
}

/**
 * 顯示 Boss HP 條（帶顏色標籤）
 */
public static byte[] showBossHP(int oid, int currHP, int maxHP, byte tagColor, byte tagBgColor) {
    final MaplePacketLittleEndianWriter mplew = new MaplePacketLittleEndianWriter();
    mplew.writeShort(SendOpcode.FIELD_EFFECT.getValue());
    mplew.write(5);  // type = 5 為 Boss HP bar
    mplew.writeInt(oid);
    mplew.writeInt(currHP);
    mplew.writeInt(maxHP);
    mplew.write(tagColor);
    mplew.write(tagBgColor);
    return mplew.getPacket();
}
```

### 7.2 封包結構分析

| Opcode | 名稱 | 說明 |
|--------|------|------|
| `0x3E` (FIELD_EFFECT) | 場景效果 | type=5 時為 Boss HP bar |
| `0x4C` (SHOW_MONSTER_HP) | 顯示怪物 HP | 普通怪物 HP 百分比 |

### 7.3 FIELD_EFFECT type=5 封包格式
```
[3E 00] [05] [OID] [currHP] [maxHP] [tagColor] [tagBgColor]
```
