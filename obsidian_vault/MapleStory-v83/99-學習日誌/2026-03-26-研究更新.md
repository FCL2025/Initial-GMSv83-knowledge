### 1. 登入器/客戶端修改

#### localhost.exe IP 修改流程（HeavenMS README 2026）
根據 HeavenMS 官方 README，localhost.exe 修改流程：

1. 使用 Neo Hex Editor（或 free-hex-editor-neo.exe）
2. 開啟 `localhost.exe` 進行編輯
3. 搜尋 "Type: String" "127.0.0.1"
4. 在 3 個位址替換為目標 IP
5. 儲存變更並離開編輯器

**重要提醒**：
- localhost.exe 會被防毒軟體標記為潛在惡意軟體（因 reverse engineering 技術）
- 如遇到相容性問題（Windows 8/10），可使用 `lolwut.exe` 相容模式
- 建議設定：Windows 7 相容模式、640x480 解析度、管理員權限執行

### 2. 抓包分析

#### Maple_Pshark v83 專用封包 Logger（2026 更新）
GitHub: https://github.com/obstriker/Maple_Pshark

**功能特色**：
1. 攔截接收（Recv）訊息
2. 修改訊息內容
3. 注入封包（發送/接收）
4. 基於規則的封包操作/過濾

**使用方式**：
```
1. 啟動 MapleStory localhost 客戶端
2. 啟動 Maple_Pshark
3. 觀察封包交換
4. 自動解密並解析 opcode
```

**技術原理**：
- Hook 到加密函數
- 直接擷取客戶端與伺服器之間的訊息
- 適用於私人伺服器環境

#### v83 封包編碼流程（RaGEZONE 2016 研究）
```
客戶端發送流程：
明文資料
  → IOBufferManipulator.Encrypt()  (Shanda 加密)
  → AES-OFB 加密
  → MapleCrypto 頭部混淆
  → 網路傳輸

客戶端接收流程：
網路接收
  → MapleCrypto 解混淆
  → AES-OFB 解密
  → IOBufferManipulator.Decrypt()  (Shanda 解密)
  → 處理
```

**重要發現**：
- 第一個發送/接收的封包永遠是明文（Hello Packet）
- 這是客戶端與伺服器交換 IV 的時機
- 之後所有封包都經過雙重加密

### 5. NPC 腳本編寫

#### NPC 腳本位置
通常位於：`source/scripts/npc/`
每個 NPC 一個 `.js` 檔案，檔名為 NPC ID

#### 基本模板結構
```javascript
var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        if (mode == 1)
            status++;
        else
            status--;
        
        // 根據 status 處理不同對話
        if (status == 0) {
            cm.sendNext("你好！");
        } else if (status == 1) {
            cm.sendNext("再見！");
        }
    }
}
```

#### 動態 NPC 商店腳本（RaGEZONE 2016）
優勢：
- 不需要重啟伺服器即可更新商店
- 不需要修改 SQL 資料庫
- 透過修改 JavaScript 檔案即可變更商品

## 學習心得

### 關於客戶端修改
v83 客戶端修改已經非常成熟，有多種方案可選：
1. **傳統 Hex 編輯**：修改 localhost.exe IP（最簡單）
2. **DLL 注入**：使用 Ezorsia v2 dinput8.dll（最方便）
3. **完整客戶端**：使用 HeavenClient（最靈活）

### 關於封包分析
- Maple_Pshark 是專為 v83 設計的工具，適合快速分析
- 對於深層分析，仍需使用 Wireshark + 腳本
- v83 的加密相對簡單（已被完全破解）

### 關於 Opcode
v83 Opcode 已經高度文檔化，開源專案如 HeavenMS 和 Cosmic 提供了完整的映射表。對於自定義伺服器，只需修改對應的處理函數即可。

### 關於 WZ 修改
HaRepacker-resurrected 是最推薦的工具，支援 GMS old 加密格式。對於常見修改（怪物屬性、物品數值、地圖設定），使用 WZ 編輯器即可完成。

### 關於 NPC 腳本
JavaScript 腳本格式已經標準化，不同模擬器之間的語法略有差異。建議參考 HeavenMS 的腳本源碼作為範例。
