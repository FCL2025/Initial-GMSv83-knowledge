## 核心技術摘要

### 1. 登入器/客戶端修改 (2.7 指南)
```
三大方案：
├── HxD Hex 編輯（⭐ 簡單）
├── Ezorsia v2 DLL（⭐⭐⭐⭐⭐ 最佳）
└── MapleClientEditTemplate（⭐⭐ 通用框架）

Ezorsia v2 特點：
- dinput8.dll 形式，無需修改原 exe
- 內建 HD 解析度支援
- 無 WZ 衝突（生成獨立 WZ）
- 可載入 3 個自訂 DLL
```

### 2. 抓包分析 (4.1 指南)
```
工具鏈：
├── Wireshark（TCP 層流量）
├── Maple_Pshark（即時 Hook，明文截獲）
└── MaplePacketPuller（離線 .pcap 分析）

三層加密：
1. MapleCrypto 頭部混淆（4-byte header）
2. Shanda 對稱加密（IOBufferManipulator）
3. AES-OFB（主加密，v83 仍在使用）
```

### 3. Opcode結構 (4.2 指南)
```
Send vs Recv 區分：
├── S→C (Send)：伺服器發送到客戶端
└── C→S (Recv)：客戶端發送到伺服器

v83 現況：
- ✅ Opcode 幾乎全部被映射
- ✅ Cosmic/HeavenMS 開源專案完整提供
- ⚠️ IDA 分析價值：僅在修改客戶端時需要
```

### 4. WZ 修改 (2.6 + 4.5 指南)
```
⚠️ 黃金法則：
"DO NOT USE the server's XMLs for reimporting into the client's WZ"

正確流程：
1. HaRepacker 開啟 GMS (old) 加密的 client WZ
2. UI 直接修改數值（禁止刪除/重新導入！）
3. 儲存 client WZ
4. 選擇 "Private Server..." 導出 XML
5. 覆蓋到伺服器的 wz.xml 目錄
```

### 5. NPC 腳本編寫 (4.3 指南)
```
狀態機：
status = 0 → sendNext
status = 1 → 回應後遞增
status++ (mode=1) 或 status-- (mode=0)

✅ 相容寫法：
- 使用 cm. 函數（通用）
- 避免 importPackage（JDK 7+ 廢棄）
- status 正確遞增
- 每分支 dispose()

❌ 不相容寫法：
importPackage(Packages.client); // JDK 7+ 錯誤
```
