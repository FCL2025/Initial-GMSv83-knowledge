## 🆕 1. 登入器/客戶端修改

### dinput8.dll 自動化方案（MapleEzorsia v2 - 2023）

**GitHub**: https://github.com/444Ro666/MapleEzorsia-v2

**為什麼是 dinput8.dll？**
- Windows 會自動加載 `dinput8.dll` 到所有使用 DirectInput 的遊戲
- MapleStory 使用 DirectInput 進行鍵盤/滑鼠輸入
- 因此 `dinput8.dll` 不需要 EasyHook 或 SetWindowsHookEx
- 遊戲啟動時自動加載，無需注入器

**完整功能表**：
| 功能 | 說明 |
|------|------|
| HD Resolution | 任意視窗和 Canvas 解析度 |
| localhost redirect | 自動將伺服器 IP 導向 127.0.0.1 |
| No Patcher | 跳過官方 Patch 過程 |
| Anti-HShield | 部分繞過 HackShield 保護 |
| FPS Unlock | 解除幀率限制 |

### Hex 編輯 v83 核心原則

**三層修改方法**：

| 方法 | 工具 | 難度 |
|------|------|------|
| 動態記憶體修改 | Cheat Engine + OllyDbg | 中 |
| 直接二進制修改 | Hex 編輯器 (HxD) | 低 |
| 記憶體寫回磁片 | OllyDbg 複製到執行檔 | 高 |

**localhost.exe Hex 編輯**：
```
1. HxD 開啟 localhost.exe（先備份！）
2. Ctrl+F → 搜尋 "127.0.0.1"
3. 會找到三處幾乎相連的 IP 位址
4. 將第一個 IP 改為目標 IP
5. 確保 IP 長度一致（不足用 \x00 填充）
6. 儲存
```

## 🆕 3. Opcode 結構

### MapleCrypto + Shanda + AES 三層加密架構

```
【發送端 C→S】
明文封包 [opcode][payload]
    ↓
1. AES-OFB 加密 (版本 Key + 當前 IV)
    ↓
2. Shanda (IOBufferManipulator) 加密
    ↓
3. MapleCrypto Header (4 bytes: 混淆 opcode + length)
    ↓
網路傳輸
```

### CInPacket 解碼函數對照

| C++ (IDA) | Java (伺服器) | 大小 |
|-----------|--------------|------|
| `Decode1` | `mplew.write(1)` | 1 byte |
| `Decode2` | `mplew.writeShort()` | 2 bytes |
| `Decode4` | `mplew.writeInt()` | 4 bytes |
| `Decode8` | `mplew.writeLong()` | 8 bytes |
| `DecodeBuffer(x, n)` | `mplew.write(n bytes)` | n bytes |
| `DecodeStr` | `mplew.writeMapleAsciiString()` | 可變 |

## 🆕 5. NPC 腳本編寫

### NPCConversationManager 核心方法

| 方法 | 按鈕 | 用途 |
|------|------|------|
| `cm.sendNext(text)` | 下一步 | 多頁對話 |
| `cm.sendNextPrev(text)` | 上一步+下一步 | 來回對話 |
| `cm.sendPrev(text)` | 返回 | 返回上一頁 |
| `cm.sendOk(text)` | 確定 | 簡單確認 |
| `cm.sendYesNo(text)` | 是/否 | 二元選擇 |
| `cm.sendSimple(text)` | 選項清單 | 選單對話 |
| `cm.sendGetNumber(text, default, min, max)` | 數字輸入 | 數量選擇 |
| `cm.sendGetText(text)` | 文字輸入 | 玩家輸入 |
| `cm.dispose()` | - | 關閉對話 |

### 狀態機 (status) 運作方式

| status | 意義 |
|--------|------|
| `-1` | 初始調用 start() |
| `0` | 第一個訊息 |
| `1` | 玩家回應後 |
| `status++` | 每次按下一步遞增 |

### if 判斷實務範例

**等級判斷**：
```javascript
var playerLv = player.getLevel();
if(playerLv >= 120){
    str += "哇 120級 四轉大神欸！";
} else if(playerLv >= 100){
    str += "100級 快四轉了！";
}
```

**楓幣判斷**：
```javascript
var meso = cm.getMeso();
if(meso >= 1000000){
    str += "好多錢!!";
} else {
    cm.gainMeso(5000); // 贈送
}
```

**道具判斷**：
```javascript
if(cm.haveItem(4000000, 10)){ // 藍寶殼 10個
    cm.gainItem(4000000, -10); // 回收
}
```
