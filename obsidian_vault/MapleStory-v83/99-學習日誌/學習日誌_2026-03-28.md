### 2. 抓包分析工具更新

**新增工具發現：**

| 工具 | 類型 | 特色 |
|------|------|------|
| **mapleSniffer** | 通用截獲 | 開源可審計，跨版本研究 |
| **MapleProxy** | 代理工具 | 支援封包重放和登入模擬 |
| **maplelib (Go)** | 加密庫 | 完整的 MapleStory 加密實現參考 |

**完整工具鏈（2026-03 最新）：**
```
Wireshark (TCP連線確認) → Maple_Pshark (即時Hook明文) → MaplePacketPuller (離線分析) → MapleRef Wiki (Opcode對照)
```

**maplelib (Go) 加密概念：**
- 四個 IV 機制：客戶端/伺服器各有自己的 SendIV 和 RecvIV
- 第一個封包永遠純文字（版本協商）
- v83 使用 Shanda + AES-OFB 雙層加密

### 3. Opcode結構最新狀態

- **v83 Opcode 完整性**：幾乎全部被映射，開源專案中 Cosmic 最完整
- **MaplePE 封包發送前提**：伺服器需先回應至少一個封包，才能正確指定 CClientSocket instance
- **MapleRef Wiki**：Send (S→C) 和 Recv (C→S) Opcode 是**完全不同**的集合
- **BB+ 版本注意**：CClientSocket::SendPacket 有 Hook 檢測，需 Hook `MakeBufferList` 而非直接 SendPacket

### 4. WZ編輯最新技術

- **MapleEdge v83 私服**：展示 Custom WZ Edits 實戰，包含 Item Level System、Arcane Gear Shop
- **v83 新版本物品導入技術（Reskin 法）**：
  1. HaRepacker 開啟高版本 WZ（如 v176 Character.wz）
  2. Export 目標物品為 XML
  3. 在 v83 中選擇閒置物品，用文本編輯器覆蓋內容
  4. Import 回 v83，測試裝備

- **WZ vs NX**：NX 效能更好、檔案更小，但工具支援不及 WZ 廣泛

- **WzImg-MCP-Server**：AI 輔助 WZ 分析工具，74 tools across 10 categories

### 5. NPC腳本編寫最新相容性

- **Nashorn ECMAScript 6 完全相容**：Cosmic 和 IntransigentMS 最活躍
- **Java.type() 完全取代 importPackage**：JDK 7+ 標準
- **相容性最終確認**：
  - ✅ `cm.` 通用函數
  - ✅ `Java.type()` (Nashorn ES6)
  - ❌ `importPackage` (已廢棄)

- **配方系統腳本**：動態、無需 SQL，可版本控制
