## 🆕 知識庫核心重點回顧

### 1. 登入器/客戶端修改

**最佳實踐 (2026)**:
- Ezorsia V2 + Hendi's 4G Client 組合
- 4G Flag 修改：PE Header 偏移 0x13E，0x0F → 0x2F
- STREDIT 修改 localhost.exe IP
- Magpie 全螢幕投射解決方案

**LAN 多人連線關鍵**:
- 錯誤碼 0x80004005 = 防火牆問題
- config.yaml HOST 需設為 LAN IP（非 127.0.0.1）
- 防火牆開啟 TCP 7575, 8484

### 2. 抓包分析

**完整工具鏈**:
```
Wireshark (TCP 確認) → Maple_Pshark (Hook 明文) → MaplePacketPuller (結構分析) → MapleRef Wiki (Opcode 對照)
```

**核心原則**:
> 第一個發送/接收的封包永遠是純文字（未加密）

**三層加密架構**:
1. MapleCrypto（頭部混淆）
2. Shanda Encryption
3. AES-OFB（主加密）

### 3. Opcode結構

**v83 Opcode 現況**:
- ✅ 幾乎全部被映射
- ✅ 開源專案（Cosmic、HeavenMS）已完整提供
- ⚠️ 唯一需要的反向工程是修改客戶端時

**Send vs Recv**:
- S→C (Send)：伺服器發送，客戶端接收
- C→S (Recv)：客戶端發送，伺服器接收
- 同一個 Opcode 數字可能代表完全不同含義

### 4. WZ修改

**WZ 編輯黃金法則**:
> DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs.

**正確流程**:
1. HaRepacker → GMS (old) → 開啟 WZ
2. UI 直接修改數值（禁止刪除/重新匯入！）
3. 存檔
4. Export → Private Server XML
5. 覆蓋伺服器 wz 目錄

### 5. NPC腳本編寫

**相容性核心原則**:
- ✅ 無 `importPackage`（JDK 7+ 已廢棄）
- ✅ 使用 `cm.` 函數
- ✅ 正確的 `status` 狀態機
