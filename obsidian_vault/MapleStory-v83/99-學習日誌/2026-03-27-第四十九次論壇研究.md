## 📡 Opcode 結構分析

### MapleRef Wiki - 封包結構參考

**URL**: https://mapleref.fandom.com/wiki/Packets

### Opcode 核心概念
> "Actions that are defined by an identifier called an Operation Code (opcode for short), which is the first 2 bytes of the packet (a short)"

### Send vs Receive Opcode 區分
| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

**重要**：同一個 Opcode 數字可能代表不同含義，視方向而定！

### v83 封包加密三層架構

```
┌─────────────────────────────────────────────────────┐
│ Layer 1: MapleCrypto (頭部混淆)                      │
│   - 4 bytes header (含混淆 opcode + length)         │
│   - 防止靜態分析 opcode                            │
│                                                        │
│ Layer 2: Shanda Encryption (對稱加密)               │
│   - v83 仍在使用                                    │
│   - 對稱式加密，無解密困難                          │
│   - 已於 v118+ 移除                                │
│                                                        │
│ Layer 3: AES-OFB (主加密)                          │
│   - v83 主要加密方式                                │
│   - 256-bit key, 4-byte IV                         │
│   - 版本決定的靜態金鑰                             │
└─────────────────────────────────────────────────────┘
```

### MapleCrypto 頭部混淆公式

```java
// 加密端 (getPacketHeader)
int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
verify = (verify << 3) | (verify >>> 5);  // rotate left 3 bits
verify &= 0xFFFF;
return new byte[] {
    (byte) (verify & 0xFF),
    (byte) ((verify >>> 8) & 0xFF),
    (byte) (length & 0xFF),
    (byte) ((length >>> 8) & 0xFF)
};

// 解密端 (retrieveOpcode)
int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
```

### IDA 分析 Opcode 步驟

```
Step 1: 找到 SendPacket / RecvPacket 函數
Step 2: 分析封包加密邏輯 (MapleCrypto)
Step 3: 提取 Opcode 表
Step 4: 對比伺服器端代碼
```

### IDA 中 Decode 函數對照

| IDA 函數名 | MaplePacketWriter 方法 | 讀取大小 |
|-----------|----------------------|---------|
| `Decode1` | `mplew.write(1)` | 1 byte |
| `Decode2` | `mplew.writeShort(1)` | 2 bytes |
| `Decode4` | `mplew.writeInt(1)` | 4 bytes |
| `Decode8` | `mplew.writeLong(1)` | 8 bytes |
| `DecodeStr` | `mplew.writeMapleAsciiString()` | 可變 |
| `DecodeBuffer(x, n)` | `mplew.write(/* n bytes */)` | n bytes |

### ✅ 相容寫法（所有音源通用）
```javascript
// JavaScript 基本寫法 - 適用所有音源
var status = 0;

function start() {
    cm.sendNext("你好！");
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
        return;
    }
    if (mode == 1) status++;
    else status--;
    
    if (status == 0) {
        cm.sendSimple("選擇：\r\n#b#L0#選項1#l\r\n#L1#選項2#l");
    } else if (status == 1) {
        if (selection == 0) {
            cm.gainItem(1302000, 1);
            cm.sendOk("獲得物品！");
        }
        cm.dispose();
    }
}
```

### ❌ 不相容寫法（JDK 7+ 已廢棄）
```javascript
// ❌ 不相容 (JDK 7+ 已廢棄)
importPackage(Packages.client);
importPackage(Packages.server);
```

### NPC對話框類型完整對照表

| Type 值 | 對話框類型 | 使用方法 | 結束行為 |
|---------|-----------|---------|---------|
| 0 | 標準對話 | `sendNext`, `sendOk`, `sendPrev` | 需手動 dispose |
| 1 | Yes/No | `sendYesNo` | 需手動 dispose |
| 2 | 選項選單 | `sendSimple` | 需手動 dispose |
| 3 | 數字輸入 | `sendGetNumber` | selection = 輸入值 |
| 4 | 自動關閉 | - | 自動 dispose |
| 12 | 接受/拒絕 | `sendAcceptDecline` | 需手動 dispose |

### NPC 腳本狀態機流程
```
status = -1 → 初始調用 start()
  ↓
status = 0 → 第一次 sendNext/sendSimple
  ↓
status = 1 → 玩家回應後
  ↓
status = 2 → 第二次回應後
  ↓
... 依此類推
```

## 🎮 WZ 編輯

### HaRepacker 導出 WZ 為 XML 步驟
```
1. 用 HaRepacker 打開 WZ 檔案
2. 選擇要導出的節點
3. 點擊 Export → 選擇 XML (Private Server) 格式
4. 導出後可使用文字編輯器修改
5. 重新導入 WZ
```

**注意**：
- v83 WZ 加密格式需選擇 `GMS (old)`
- 不要直接刪除/重新導入節點，可能造成數據丟失
- 建議使用 HaRepacker UI 直接編輯數值

### WZ 編輯黃金法則
> DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs.

**正確流程**：
1. HaRepacker 4.2.4 開啟 **GMS (old)** 加密的客戶端 WZ
2. **使用 UI 直接修改數值**（禁止刪除/重新導入節點！）
3. 儲存 WZ
4. 在 HaRepacker 中選擇 **"Private Server..."** 導出
5. 將導出的 XML 覆蓋到伺服器的 WZ.XML 目錄

### Quest 等級限制移除（LvMax Removal）Notepad++ Regex
```
尋找：lvmax="\d+"\s*
取代：（留空）
```

### NPC Tags WZ 編輯位置
在 HaRepacker 中：`NPC.wz` → `NpcId.img` → `info` → `tags`
- 這是 NPC 名稱下的額外描述文字
- 翻譯時需同步修改 `String.wz` 中對應的條目
