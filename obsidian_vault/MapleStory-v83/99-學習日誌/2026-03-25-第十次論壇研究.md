## 🆕 研究重點一：Cosmic v83 模擬器最新架構分析

### Cosmic v83 完整架設流程 (2026-03 最新)

**來源**：https://github.com/P0nk/Cosmic

Cosmic 是目前最活躍的 v83 模擬器，由 P0nk 維護，基於 Java 21。以下是完整架設流程：

### 資料庫架設（MySQL 8+）

**步驟**：
1. 下載並安裝 MySQL Community Server 8+
2. 使用 HeidiSQL 客戶端連線
3. 建立新資料庫 schema：`cosmic`
4. 設定 root 密碼（記錄下來，後續需填入 config.yaml）

**config.yaml 資料庫設定**：
```yaml
DB_HOST: localhost
DB_PORT: 3306
DB_NAME: cosmic
DB_USER: root
DB_PASS: <your_password>
```

### 伺服器架設（Java 21）

**步驟**：
```bash
# 1. Clone 專案
git clone https://github.com/P0nk/Cosmic

# 2. 使用 IntelliJ IDEA 打開專案
# 3. 找到 config.yaml，設定 DB_PASS

# 4. 啟動伺服器
# main method: net.server.Server

# 5. 成功啟動的標誌
# "Cosmic is now online" 出現在 console
```

### Docker 支援（可選）

Cosmic 原生支援 Docker：
```bash
docker compose up
```

### 客戶端安裝（Cosmic-client）

**來源**：https://github.com/P0nk/Cosmic-client

```
1. 下載 MapleGlobal-v83-setup.exe 並執行安裝
2. 刪除 HShield/ 資料夾（整個）
3. 刪除 ASPLnchr.exe
4. 刪除 MapleStory.exe
5. 刪除 Patcher.exe
6. 複製 cosmic-wz 中的 WZ 檔案到安裝目錄
7. 複製 HeavenMS-localhost-WINDOW.exe 到安裝目錄
```

### 預設登入帳號
- 使用者名稱：`admin`
- 密碼：`admin`
- PIN：`0000`
- PIC：`000000`

### 支援自動註冊（預設開啟）
直接輸入新的帳號密碼即可自動建立帳號。

### WZ 檔案處理（HaRepacker-resurrected）

Cosmic 需要特殊的 WZ 格式處理：

| 工具 | 用途 |
|------|------|
| HaRepacker-resurrected | 編輯 .wz 檔案 |
| 加密格式 | `GMS (old)` |

**編輯 WZ 的工作流程**：
```
1. HaRepacker 打開 .wz 檔案
2. 使用 node hierarchy 進行修改（複製/貼上節點可能不穩定）
3. 儲存修改後的 .wz
4. 重新導出為 XML（Private Server 格式）
5. 複製到伺服器的 wz 目錄
```

**⚠️ 重要原則**：
- 永遠從客戶端 .wz 導出 XML 到伺服器
- 不要反向操作（伺服器 XML → 客戶端 .wz）
- 否則可能造成奇怪的行為

## 🆕 研究重點二：NPC Type 含義解析 (RaGEZONE, 2026-03)

**來源**：https://forum.ragezone.com/threads/npc-type.1262188/

### NPC Type 完整對照表

NPC Type 定義了 NPC 的行為和對話類型：

| Type 值 | 名稱 | 說明 |
|---------|------|------|
| 0 | 普通 NPC | 標準對話，可觸發腳本 |
| 1 | 商店 NPC | 開啟商店介面 |
| 2 | 任務 NPC | 顯示任務選項 |
| 3 | 資訊 NPC | 顯示公告或資訊 |
| 4 | 傳送 NPC | 帶有傳送腳本 |
| 5 | 寵物 NPC | 寵物相關功能 |
| 6 | 禮物 NPC | 發放禮物/獎勵 |
| 7 | 銀行 NPC | 倉庫/銀行功能 |
| 8 | 結婚 NPC | 婚禮相關功能 |

### 如何在 WZ 中修改 NPC Type

使用 HaRepacker：
1. 打開 `Npc.wz`
2. 找到目標 NPC 的 `info` 節點
3. 找到 `type` 欄位
4. 修改為目標值
5. 儲存並導出

## 🆕 研究重點四：MapleStory v83 LAN 多人連線錯誤排查 (RaGEZONE, 2026-03)

**來源**：https://forum.ragezone.com/threads/need-help-v83-lan-server-works-on-pc1-but-pc2-gets-error-2147467259.1262187/

### 錯誤碼 -2147467259 含義
這是 Windows 的 `DISP_E_EXCEPTION` 錯誤，通常表示：
- 網路連線失敗
- 防火牆阻擋
- IP/Port 設定錯誤

### 完整排查清單

| 檢查項目 | PC1 (主機) | PC2 (客戶端) | 狀態 |
|---------|-----------|-------------|------|
| 防火牆關閉/開放 7575 | ✅ | ✅ | 必填 |
| IP 修改正確 | ✅ (127.0.0.1) | ✅ (主機IP) | 必填 |
| 同一網段 | N/A | ✅ | 必填 |
| 伺服器正常運行 | ✅ | N/A | 必填 |
| 客戶端版本 v83 | ✅ | ✅ | 必填 |
| localhost.exe 存在 | ✅ | ✅ | 必填 |

### 解決方案

**方案一：手動修改 IP**
```bash
# 使用 HxD 修改 localhost.exe 中的 IP
# 1. 開啟 localhost.exe
# 2. Ctrl+F → "Text string" → "127.0.0.1"
# 3. 替換為主機 IP（如 192.168.1.100）
```

**方案二：使用 Ezorsia v2 DLL**
```bash
# 將 dinput8.dll 放入遊戲目錄
# 在 config.ini 中設定 ServerIP=主機IP
```

**方案三：檢查 Windows Security**
```bash
# 如果 localhost.exe 被標記為威脅
# Windows Security → Virus & threat protection
# 管理設定 → 新增排除項目 → 加入下載資料夾
```

## 🆕 研究重點七：Wireshark 抓包實戰技巧 (2026 新增)

### 過濾器組合技巧

```wireshark
# 只看 MapleStory 相關流量
tcp.port == 7575 || tcp.port == 8484

# 只看客戶端發送
tcp.srcport == xxxx && tcp.dstport == 7575

# 只看加密封包（長度異常）
tcp.len > 20 && tcp.len < 100

# 排除 Hello Packet（明文）
tcp.len > 10

# Follow TCP Stream 分析完整會話
```

### 識別封包類型技巧

| 徵兆 | 封包類型 |
|------|---------|
| 長度固定 (每次 32 bytes) | MOVE_PLAYER (0x14) |
| 包含可讀字串 | CHAT (聊天) |
| 第一個 byte 為 0x00 | 伺服器回應封包 |
| 封包包含 00 00 00 00 | PING/心跳封包 |
| 封包較大 (>100 bytes) | CHARLIST (角色列表) |

### 顏色規則設定

```
紅色 (異常):    tcp.len > 2000      (大型封包)
綠色 (客戶端):  ip.src == 127.0.0.1 (本地發送)
藍色 (伺服器):  ip.dst == 127.0.0.1 (本地接收)
黃色 (登入):    tcp.port == 7575    (登入端口)
```

**來源**：https://forum.ragezone.com/threads/kinoko-npc-scripts.1261538/

### NPC 腳本源間相容性問題

| 問題 | 原因 | 解決方案 |
|------|------|---------|
| `importPackage` 錯誤 | JDK 7+ 廢棄 | 移除 importPackage |
| `SavedLocationType` 未定義 | 變數未定義 | 使用相容格式 |
| 不同音源解析器 | 版本差異 | 使用普遍的 `cm.` 函數 |
| 函數不存在 | 音源專有函數 | 避免使用 |

### 相容性檢查清單

| 檢查項目 | 狀態 |
|---------|------|
| 無 `importPackage` | ✅ 確認 |
| 使用完整 Package 路徑 | ✅ 確認 |
| 使用 `cm.` 而非直接類別名 | ✅ 確認 |
| `SavedLocationType` 已處理 | ✅ 確認 |
| 狀態機邏輯正確 | ✅ 確認 |

### 不同音源的 NPC 腳本特點

| 音源 | 解析器特點 | 注意事項 |
|------|-----------|---------|
| Cosmic | 最新 JDK 21 | 最嚴格，無 importPackage |
| Kinoko | 混合版本 | 需注意相容性 |
| HeavenMS | JDK 6 時代 | 有 SavedLocationType 問題 |
