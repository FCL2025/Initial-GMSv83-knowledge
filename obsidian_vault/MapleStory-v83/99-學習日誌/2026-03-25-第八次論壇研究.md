#### 🛠️ localhost.exe IP 修改流程 (HeavenMS, 2019)

GitHub Issue: https://github.com/ronancpl/HeavenMS/issues/437

**標準 Hex Edit 流程**：
1. 使用 HxD 開啟 localhost.exe
2. Ctrl+F → Text-string → 搜尋 "127.0.0.1"
3. 會找到 **三個連續出現** 的 "127.0.0.1"
4. 將三處都替換為目標 IP
5. 存檔

**常見問題**：
- 搜尋不到 "127.0.0.1" → 可能 IP 是 split 形式存放（如 "127" + ".0.0.1"）
- 需確認長度一致（用相同長度的 IP）

#### localhost.exe Hex Edit 常見問題

**Q: 為什麼要修改三處 "127.0.0.1"？**
A: 客戶端通常在多處使用 IP 字串（登入伺服器、地圖伺服器、商城伺服器等），需全部修改。

**Q: IP 長度不同怎麼辦？**
A: 盡量使用相同長度的 IP。例如：
- `127.0.0.1` (9字元) → `192.168.1.100` (13字元) ✗
- `127.0.0.1` (9字元) → `127.0.0.1` (9字元) ✓
- 解決方案：使用數字 IP（如 `7F000001` = 127.0.0.1）

### 📡 Opcode 結構

#### MapleRef Wiki 關鍵資訊

**Opcode 位元組歷史**：
- 早期版本：Opcode 為 1 byte（最大 255）
- v83+：Opcode 改為 2 bytes（最大 65536）
- Packet header 為 4 bytes（長度 + 混淆 opcode）

**封包頭部結構**：
```
偏移    長度   內容
0x00    2      MapleCrypto 混淆的 verify
0x02    2      封包長度（不含 header）
```

#### Opcode 搜尋 IDA 分析技巧

```
1. 搜尋 StringPool 中的字串（如 "opcode"、"Packet"）
2. 找到 `push opcode` 指令的交叉引用
3. AOB 搜尋已知 opcode bytes
4. 使用 IDC Script 自動化命名（diamondo25's IDC Script）
```

#### sendAcceptDecline 行為（Type = 12）

| Mode 值 | 意義 |
|---------|------|
| `mode == -1` | 玩家關閉對話框 |
| `mode == 1` | 玩家點擊「接受」(Accept) |
| `mode == 0` | 玩家點擊「拒絕」(Decline) |

#### 職業進階腳本（完整版本）

```javascript
var status = 0;
var jobName = "";
var jobId = 0;

function start() {
    cm.sendNext("恭喜 #b10級#k！請選擇二轉職業：\r\n"
        + "#b#L0#火與毒法師#l\r\n"
        + "#L1#冰與雷法師#l\r\n"
        + "#L2#僧侶#l");
}

function action(mode, type, selection) {
    if (mode != 1) {
        if (mode == 0) cm.sendOk("等你想好了再來。");
        cm.dispose();
        return;
    }
    status++;
    if (status == 1) {
        if (selection == 0) { jobName = "火與毒法師"; jobId = 210; }
        else if (selection == 1) { jobName = "冰與雷法師"; jobId = 220; }
        else { jobName = "僧侶"; jobId = 230; }
        cm.sendYesNo("確定選擇 #b" + jobName + "#k？");
    } else if (status == 2) {
        cm.gainJob(jobId);
        cm.sendOk("成功！你是 #r" + jobName + "#k 了！");
        cm.dispose();
    }
}
```
