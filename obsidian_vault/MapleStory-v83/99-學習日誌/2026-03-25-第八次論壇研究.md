## 🆕 本次新發現

### 1. MapleRef Wiki Packet 結構深度更新 (2026)

**來源**: https://mapleref.fandom.com/wiki/Packets
**Opcode Wiki**: https://mapleref.fandom.com/wiki/Operation_Code

**核心概念更新**：
- Packet headers 為 4 bytes 長
- Header 由 `send IV`、`packet length` (不含 header)、`version` 產生
- Opcode 通常為 **2 bytes** (little endian)
- Send/Recv Opcode 是**兩套獨立集合**：
  - **S→C (Send)**: 伺服器發送到客戶端
  - **C→S (Recv)**: 客戶端發送到伺服器

### 8. botnored v83 封包類別結構重點

**來源**: https://botnored.weebly.com/blog/maplestory-classes-v83

**重要原則**：
> **第一個發送/接收的封包永遠是純文字（未加密）**
> MapleStory 封包透過 TCP Socket 傳輸

**客戶端主要類別**：
| 類別 | 功能 |
|------|------|
| `CWvsContext` | 世界狀態管理（角色資料、伺服器列表） |
| `CClientSocket` | 網路連接管理（Send/Recv） |
| `CField` | 當前地圖/遊戲場景處理 |
| `CLogin` | 登入畫面處理 |

### 9. MapleStory v83 三層加密架構

```
Layer 1: MapleCrypto (頭部混淆)
  - 4 bytes header (混淆 opcode + length)
  - 防止靜態分析 opcode

Layer 2: Shanda Encryption (對稱加密)
  - v83 仍使用，已於 v118+ 移除

Layer 3: AES-OFB (主加密)
  - 256-bit key, 4-byte IV
  - 版本決定的靜態金鑰
```

### 10. v83 商場封包 Dupe 分析（學習用）

**來源**: https://pdfcoffee.com/v83-dupe-pdf-free.html

Opcode `0x6F` (SHOP_BUY) 結構：
```
6F 00 20 00 FF FF FF 80 00
```
- `6F 00`: Opcode (little endian = 0x006F)
- `20 00`: 商店ID (0x0020 = 32)
- `FF FF FF 80`: 物品索引 + 最大 Mesos 標誌 (0x80000000)
- `00`: 結束標記

## 📋 本次學習要點

### 登入器/客戶端修改
- **MapleRef Wiki**：最完整的 Packet/Opcode 結構參考
- **Ezorsia v2**：一體化 DLL 客戶端，自動完成 HD + localhost 設定
- **TimelessMS Beta V2**：2025-10 仍在開放，代表社群活躍
- **version v83 的特點**：高度修改過的客戶端，不反映一般工作流程

### 抓包分析
- **第一個封包是明文**：Hello Packet 是分析的關鍵起點
- **CClientSocket::ProcessPacket**：所有接收封包的總入口
- **MapleRef 使用**：選擇版本 → 查看 Send/Recv Opcode → 了解封包結構

### Opcode結構
- **Send vs Recv 是兩套獨立集合**：同一數字在不同方向代表不同含義
- **Opcode 為 2 bytes (little endian)**：如 0x01 在封包中顯示為 `01 00`
- **v83 Opcode 幾乎全部被映射**：無需額外追蹤，開源專案已有完整表

### WZ修改
- **WZ to NX 轉換**：使用 WzToNx.exe，檔案更小、效能更好
- **HaRepacker Export/Import**：完整 WZ 編輯工作流
- **fonts folder**：文字缺失時檢查此目錄
- **正確流程**：先編輯客戶端 WZ → 導出為 Private Server XML → 覆蓋到音源

### NPC腳本編寫
- **JavaScript 仍是主流**
- **cm 物件核心**：狀態機和遊戲狀態操作
- **ImportPackage 已廢棄**：JDK 7+ 需使用完整 Package 路徑
- **跨音源相容性**：Cosmic、Kinoko、HeavenMS 解析器版本不同
