## 2. Opcode結構 (Opcode Structure)

### 2.1 MapleStory封包基本結構
- **封包標頭(Packet Header)為4位元組**
- 包含：send IV + 封包長度(不含標頭) + 版本號
- 版本號發送到客戶端時為 -(version + 1)
- **Opcode通常為2位元組(Short類型)**
- 最大Opcode數量：65,536

### 2.2 封包加密流程
- v83使用**雙層加密**：Shanda + AES
- MapleCrypto頭部加密/解密公式（已知）
- 登入握手流程：Hello Packet → AES-IV協商

### 2.3 Opcode分析工具
| 工具 | 語言 | 用途 |
|------|------|------|
| MaplePacketPuller | Python | 封包結構分析 |
| Maple_Pshark | - | v83專用封包日誌攔截器 |
| MapleShark | C# + SharpPcap | 網路層封包抓取 |
| MaplePE | 32-bit | 封包編輯器，可解析並發送自訂封包 |

### 2.4 IDA Opcode分析方法
- 使用IDA分析客戶端 binary
- 找 `Decode` 函數對應 MaplePacketWriter
- SendOps/RecvOps 表在伺服器原始碼中
- 建議從 https://forum.ragezone.com/threads/getting-packet-structures-opcodes-using-ida.792436/ 開始

## 3. WZ修改 (WZ Editing)

### 3.1 HaRepacker使用要點
- **使用HaRepacker 4.2.4**（或HaRepacker-resurrected）
- v83 WZ文件需要選擇**"GMS (old)"**加密模式
- v117及以上版本需要使用BMS加密

### 3.2 客戶端WZ編輯流程（Cosmic官方）
```
1. 使用 HaRepacker-resurrected 編輯器，"GMS (old)" 加密
2. 打開要編輯的 .wz 文件
3. 使用UI進行修改（不要用複製/粘貼節點，可能不穩定）
4. 保存修改後的WZ到客戶端文件夾
5. 最後使用 "Private Server..." 導出選項
6. 將導出的XML覆蓋服務器的WZ.XML文件
```

### 3.3 WZ修改重要警告
- **千萬不要用服務器的XML直接import回客戶端WZ，會造成bug**
- 不要移除/重新導入WZ節點，可能導致數據丟失
- 只能用UI進行修改

### 3.4 MobBookUpdate示例（MapleSolaxiaV2）
- 用於更新怪物的怪物書掉落數據
- 流程：NetBeans build → 複製MonsterBook.img.xml到dist → java -jar MobBookUpdate.jar → 導出XML → HaRepacker覆蓋String.wz節點

### 3.5 Skill.wz編輯
- 需要Assembly/低級程式設計基礎
- 複雜度高，建議最後學習

## 4. NPC腳本編寫 (NPC Scripting)

### 4.1 腳本存放位置
- HeavenMS: `source/scripts/npc/`
- 檔案名對應NPC ID
- 使用模板（Template）加速開發

### 4.2 NPC腳本基本結構
```
importPackage(Packages.client);

var status = 0;

function start() {
    cm.sendNext("你好！");
}

function action(mode, type, selection) {
    if (mode == 1) {
        status++;
    } else {
        status--;
    }
    
    if (status == 1) {
        cm.sendNext("見到你很高興！");
    } else {
        cm.dispose();
    }
}
```

### 4.3 動態商店腳本（Dynamic NPC Shop）
- 不需要每次修改SQL表格並重啟伺服器
- 通過Java代碼擴展支援動態腳本
- 參考：https://forum.ragezone.com/threads/dynamic-npc-shop-script-v83.1089979/

### 4.4 NPC腳本源間相容性
- 主要版本：Cosmic、HeavenMS、Kinoko
- 核心API略有不同
- 對話框Type對照：0=Simple，1=YesNo，4=Input，5=AcceptDecline等
- 建議參考目標伺服器的現有腳本

### 4.5 常見GM NPC腳本命令
```
#gm - 開啟GM商店
#zakum - 生成Boss
@commands - 顯示所有可用命令
@hide - 隱身模式
```

## 5. 伺服器架設 (Server Setup)

### 5.1 MapleSolaxiaV2 完整架設流程
**伺服器文件**: https://github.com/ronancpl/MapleSolaxiaV2
**客戶端文件**: https://drive.google.com/drive/folders/0BzDsHSr-0V4MYVJ0TWIxd05hYUk

**環境需求**：
- JDK 7（注意是7不是8）
- NetBeans 8.0.2
- WampServer 2.0i
- MySQL Workbench 5.2.39
- Hamachi（可選，遠程連線用）

**JCE無限強度策略**：
```bash
覆蓋 C:\Program Files\Java\jdk1.7.0_01\jre\lib\security\ 下的
local_policy.jar 和 US_export_policy.jar
```

**SQL初始化**：
1. db_database.sql - 創建資料庫
2. db_drops.sql - 怪物掉落數據
3. 可選：db_shopupdate.sql - 商店更新

**configuration.ini 配置**：
- localhost = 127.0.0.1（僅本機）
- Hamachi IP = 遠程多人

### 5.2 HeavenMS 架設流程
**伺服器文件**: https://github.com/ronancpl/HeavenMS
**客戶端和工具**: Google Drive（同上）

**特色功能**：
- HPQ/KPQ/LPQ/LMPQ/OPQ/EllinPQ/PiratePQ/MagatiaPQ/HorntailPQ
- Mu Lung Dojo
- GuildPQ + Guild queue
- 結婚系統
- 現金商城功能
- 自動GM NPC和名人堂
- 增強的AP/buff/自動pot系統

### 5.3 Cosmic 現代架設流程（2026年最新）
**特點**：
- Java 21（Amazon Corretto推薦）
- MySQL 8+ 或 MariaDB
- IntelliJ IDEA
- Docker Desktop 可選
- 自動帳號創建（輸入使用者名和密碼即可）
- 預設管理員帳號：admin / admin / 0000 / 000000

**config.yaml 配置**：
- 只需修改 DB_PASS 為MySQL root密碼
- WZ路徑通過系統屬性配置：wz-path

**啟動命令**：
```bash
./mvnw.cmd clean package    # 建構
java -Dwz-path=/path/to/wz -jar target/cosmic.jar  # 運行
docker compose up           # Docker方式
```

### 5.4 端口配置
| 端口 | 用途 |
|------|------|
| 7575-7575+N | 遊戲頻道（默認1個） |
| 8484 | 登入伺服器 |
| 80 | Web伺服器（可選） |

### 5.5 端口轉發 (Port Forwarding)
1. 訪問路由器 default gateway（ipconfig查詢）
2. 創建自定義服務，打開 TCP/UDP 7575-7575+N 和 8484
3. Windows防火牆創建入站/出站規則，允許 MapleStory.exe
