## 🔐 登入器 IP 修改 — STREDIT 詳細流程

### STREDIT v83 修改步驟

```
1. File → Open → 選擇 localhost.exe
2. 詢問「是否 v118+」→ 選「No」（v83 適用）
3. 詢問「IP 搜尋模式」→ 選「Yes」
4. 點擊「Force to a IP」按鈕
5. 輸入目標 IP（如 127.0.0.1 或 VPS IP）
6. 保存
```

### Hex Editor 直接修改 IP

- `127.0.0.1` → Hex: `31 37 32 2E 30 2E 30 2E 31`
- `.` (IP 分隔符) → Hex: `2E`
- 若目標 IP 比 `127.0.0.1` 長，剩餘空間填 `00`

### Wireshark 過濾表達式

```
# 登入埠口
tcp.port == 7575

# 遊戲埠口
tcp.port == 8484

# 組合過濾
tcp.port == 7575 || tcp.port == 8484

# 只看客戶端發送
tcp.srcport == xxxx && tcp.dstport == 7575

# Follow TCP Stream
# 右鍵點擊封包 → Follow → TCP Stream

# 只看明文 Hello Packet
tcp.payload[0:1] == 53  # 版本號 0x53 = 83
```

### 識別封包類型的技巧

| 徵兆 | 封包類型 |
|------|---------|
| 長度固定 (如每次 32 bytes) | 可能是 MOVE_PLAYER (0x14) |
| 包含可讀字串 | 可能是 CHAT (聊天) |
| 第一個 byte 為 0x00 | 伺服器回應封包 |
| 封包包含 00 00 00 00 | 可能是 PING/心跳封包 |

## 🧩 Opcode 結構 — 核心概念

### MapleCrypto 頭部混淆演算法

```java
// 加密端
int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
verify = (verify << 3) | (verify >>> 5);
verify &= 0xFFFF;
return new byte[] {
    (byte) (verify & 0xFF),
    (byte) ((verify >>> 8) & 0xFF),
    (byte) (length & 0xFF),
    (byte) ((length >>> 8) & 0xFF)
};

// 解密端
int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
```

### 封包加密三層架構

```
┌─────────────────────────────────────────────────────┐
│ Layer 1: MapleCrypto (頭部混淆)                      │
│   - 4 bytes header (混淆 opcode + length)           │
│                                                        │
│ Layer 2: Shanda Encryption (對稱加密)                │
│   - v83 仍在使用，v118+ 已移除                       │
│                                                        │
│ Layer 3: AES-OFB (主加密)                            │
│   - v83 主要加密方式                                 │
│   - 256-bit key, 4-byte IV                           │
└─────────────────────────────────────────────────────┘

發送端: 明文 → Shanda → AES-OFB → MapleCrypto header → 網路
接收端: 網路 → MapleCrypto → AES-OFB → Shanda → 明文
```

### v83 AES Key 參考 (CMS)

```java
// AES UserKey
0x13, 0x08, 0x06, 0xB4, 0x1B, 0x0F, 0x33, 0x52

// IV Transformation Key
0xC65053F2

// 客戶端版本
0x23 (35 decimal)
```

### Send vs Recv Opcode 區分

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

## 📝 NPC 腳本編寫 — 完整指南

### 腳本位置

| 類型 | 路徑 |
|------|------|
| 一般 NPC | `scripts/npc/` |
| Quest NPC | `scripts/quest/` |
| 事件腳本 | `scripts/event/` |

檔案名稱 = NPC ID（如 `9000000.js`）

### 傳統 status 寫法（Cosmic, HeavenMS）

```javascript
var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1 || mode == 0) {
        cm.dispose();
        return;
    }
    
    if (mode == 1) {
        status++;
    } else {
        status--;
    }
    
    switch(status) {
        case 0:
            cm.sendNext("你好！");
            break;
        case 1:
            cm.sendSimple("選擇:\n #b#L0#購買#l\n #L1#出售#l");
            break;
        case 2:
            if (selection == 0) {
                // 購買邏輯
            }
            cm.dispose();
            break;
    }
}
```

### 動態 NPC 商店腳本 (RaGEZONE 2016)

**URL**: https://forum.ragezone.com/threads/dynamic-npc-shop-script-v83.1089979/

**概念**：使用 JavaScript 動態生成商店，而非傳統的 SQL table 方式。

```javascript
var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
        return;
    }
    
    if (mode == 0) {
        status--;
    } else {
        status++;
    }
    
    if (status == 0) {
        cm.sendSimple("歡迎來到我的商店！\n #b#L0#查看商品#l\n #L1#離開#l");
    } else if (status == 1) {
        if (selection == 0) {
            // 動態生成商品列表
            var shopItems = [
                { itemId: 1302000, price: 1000, quantity: 1 },  // 短劍
                { itemId: 1312000, price: 2000, quantity: 1 },  // 雙手劍
            ];
            // 顯示商店介面
            cm.openShop(9999999);  // 自訂商店 ID
            cm.dispose();
        } else {
            cm.dispose();
        }
    }
}
```

### NPC 腳本核心函數

| 函數 | 說明 |
|------|------|
| `cm.sendNext(msg)` | 發送下一個對話（單選）|
| `cm.sendNextPrev(msg)` | 發送帶有上一頁/下一頁按鈕 |
| `cm.sendSimple(msg)` | 發送簡單選擇列表 |
| `cm.sendYesNo(msg)` | 發送 Yes/No 選擇 |
| `cm.sendAcceptDecline(msg)` | 發送接受/拒絕選擇 |
| `cm.sendGetNumber(msg, def, min, max)` | 獲取數字輸入 |
| `cm.sendGetText(msg)` | 獲取文字輸入 |
| `cm.dispose()` | 結束對話 |
| `cm.openShop(id)` | 開啟商店 |
| `cm.gainItem(id, amount)` | 給予物品 |
| `cm.getMeso()` | 獲取金幣 |
| `cm.gainMeso(amount)` | 增加/減少金幣 |
| `cm.getPlayer()` | 獲取玩家物件 |

### NPC 商店 WZ 編輯

使用 HaRepacker 修改 `Shop.wz`：
1. 開啟 `Shop.wz`
2. 找到目標商店 ID
3. 修改 `shopType`（1 = 普通商店，2 = 購買商店等）
4. 修改 `itemList`（商品清單）
5. 保存

使用 HaRepacker 修改 `Quest.wz`：
1. 開啟 `Quest.wz`
2. 找到目標任務 ID（如 `2000.img`）
3. 修改 `questInfo`（任務說明）
4. 修改 `act`（完成條件）
5. 保存
