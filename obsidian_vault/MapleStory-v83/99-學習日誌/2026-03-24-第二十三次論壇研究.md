## 📌 抓包分析

### MapleStory 封包加密核心 (mapleref.fandom.com)

**加密演算法**: AES-OFB (Advanced Encryption Standard using Output Feed Buffer)

**兩層加密架構**:
```
客戶端發送流程：
1. 原始封包: [Opcode][Data...]
2. Shanda 解密/加密 (對稱式)
3. AES 解密 (CBC 模式)
4. 傳送

伺服器接收流程：
1. 接收加密封包
2. AES 解密
3. Shanda 解密
4. 處理
```

### MapleCrypto 頭部混淆

**重要原則**: 第一個發送/接收的封包永遠是純文字（未加密）

**封包頭部格式** (4 bytes):
```
偏移  長度   內容
0x00  4      MapleCrypto 混淆頭部
      (包含長度 + 混淆 opcode)
```

**v83 Opcode 格式**: Opcode 為 2 bytes (little endian)

## 📌 Opcode 結構

### MapleCrypto Header 加密/解密公式

**加密端（發送）**:
```java
public byte[] getPacketHeader(int opcode, int length, int iv) {
    int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
    verify = (verify << 3) | (verify >>> 5);
    verify &= 0xFFFF;
    return new byte[] {
        (byte) (verify & 0xFF),
        (byte) ((verify >>> 8) & 0xFF),
        (byte) (length & 0xFF),
        (byte) ((length >>> 8) & 0xFF)
    };
}
```

**解密端（接收）**:
```java
public int retrieveOpcode(byte[] header, int iv) {
    int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
    verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
    return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
}
```

### v83 AES Key 參考 (CMS/Chinese MS v83)

```java
// AES UserKey (first column per row)
0x13, 0x08, 0x06, 0xB4, 0x1B, 0x0F, 0x33, 0x52

// IV Transformation Key
0xC65053F2

// 客戶端版本
0x23 (35 decimal)
```

### IDA 中 Decode 函數對照表

| IDA函數 | MaplePacketWriter方法 | 說明 |
|--------|---------------------|------|
| `Decode1` | `mplew.write` | 1 byte |
| `Decode2` | `mplew.writeShort` | 2 bytes |

| `DecodeStr` | `mplew.writeMapleAsciiString` | 字串 |
| `DecodeBuffer(x, 8)` | `mplew.writeLong` | 8 bytes |

### 完整登入流程封包序列 (10步)

```
1. C→S Hello Packet (plaintext)
2. S→C Hello Packet (plaintext)
3. C→S LOGIN_PASSWORD (encrypted) - Opcode 0x01
4. S→C LOGIN_STATUS (encrypted) - Opcode 0x00
5. C→S SERVERLIST_REQUEST (encrypted) - Opcode 0x06
6. S→C SERVERLIST (encrypted) - Opcode 0x0C
7. C→S CHARLIST_REQUEST (encrypted) - Opcode 0x07
8. S→C CHARLIST (encrypted) - Opcode 0x03
9. C→S CHAR_SELECT (encrypted) - Opcode 0x08
10. C→S PLAYER_DROP (encrypted) - Opcode 0x0B
```

### CClientSocket::ProcessPacket 封包分發流程

```
CClientSocket::ProcessPacket()
    ↓ 根據 MapleCrypto header 解密
    ↓ 提取 opcode (2 bytes, little endian)
    ↓ switch(opcode) 分發
    → CLogin::OnPacket()    // 登入畫面
    → CField::OnPacket()     // 遊戲中
    → CShop::OnPacket()      // 商場
```

## 📌 NPC 腳本編寫

### 狀態機核心 (Status State Machine)

```javascript
status = -1 → 初始調用 start()
    ↓
status = 0 → 第一次 sendNext/sendSimple
    ↓
status = 1 → 玩家回應後
    ↓
... 依此類推
```

### Mode 參數含義

| Mode 值 | 意義 |
|---------|------|
| 1 | 點擊"下一頁"或選項 |
| 0 | 點擊"返回" |
| -1 | 關閉對話框 (ESC/X) |

### NPC 對話框類型

| Type 值 | 對話框類型 | 使用方法 |
|---------|-----------|---------|
| 0 | 標準對話 | `sendNext`, `sendOk`, `sendPrev` |
| 1 | Yes/No | `sendYesNo` |
| 2 | 選項選單 | `sendSimple` |
| 3 | 數字輸入 | `sendGetNumber` |
| 12 | 接受/拒絕 | `sendAcceptDecline` |

### NPC 對話格式標籤

| 標籤 | 功能 |
|------|------|
| `#b` | 藍色文字 |
| `#r` | 紅色文字 |
| `#g` | 綠色文字 |
| `#l` | 選項標記 |
| `#i[itemid]` | 物品圖片 |
| `#h#` | 玩家名稱 |
| `#c[itemid]#` | 持有物品數量 |
| `#t[itemid]` | 物品名稱 |
