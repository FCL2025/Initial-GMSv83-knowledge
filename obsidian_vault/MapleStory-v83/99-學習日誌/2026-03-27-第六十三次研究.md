### 3. Opcode 結構 — 三層加密架構

v83 封包加密流程：
```
明文資料 → AES-OFB 加密 → MapleCrypto 混淆 → 網路傳輸
```

關鍵特性：
- **Header**: 4 bytes = [2 bytes 封包長度][2 bytes 混淆後 opcode]
- **Opcode 混淆**: 使用版本號和 IV 進行混淆
- **Shanda 加密**: v83 仍保留中國盛大公司的自定義加密
- **第一次封包**: 明文（用於交換 IV）

客戶端封包處理函數（v83 Rev 3.00）：
- `CClientSocket::ProcessPacket` — 封包分發入口
- `CLogin::OnPacket` (0x0051E062) — 登入相關 opcode
- `CField::OnPacket` (0x004D7212) — 遊戲中所有其餘 opcode

IDA 分析資源: https://forum.ragezone.com/threads/getting-packet-structures-opcodes-using-ida.792436/

#### WZ 檔案功能速查
| 檔案 | 功能 |
|------|------|
| `Mob.wz` | 怪物速度、HP、EXP、動畫、技能、傷害計算 |
| `Item.wz` | 物品/裝備數據和圖片 |
| `Character.wz` | 角色相關 sprite 和裝備 |
| `Map.wz` | 地圖視覺、foothold、背景 |
| `Skill.wz` | 技能數據 |
| `Sound.wz` | 音效和 BGM |

#### Mob.wz 編輯重點
- `bodyAttack=0` → GodMode (怪物無法攻擊)
- `acc=0` → Miss GodMode
- `speed=126` → 怪物極速移動
- 移除 `jump` → 怪物無法跳躍（利於 Vac）
- `attackAfter=65535` → 無限遠程攻擊範圍

#### Map.wz 背景變量
- `bS`: 背景資源名稱（如 toyCastle）
- `ani`: 動畫切換（-1~1）
- `type`: 顯示效果類型（0~20+，注意 4/6 可能導致崩潰）
- `a`: 亮度（0=全黑，255=全亮）

#### NPC TYPE 值完整對照

#### Kinoko vs 傳統寫法對比
| 寫法 | 狀態管理 | 適用音源 |
|------|---------|---------|
| 傳統 status 方式 | 手動 status++ | Cosmic, HeavenMS |
| Kinoko 新型 | 無需 status | Kinoko |

#### IntransigentMS — ECMAScript 6 NPC 腳本
- **GitHub**: https://github.com/NoetherEmpy/intransigentms-scripts
- 使用 Nashorn 引擎 + ECMAScript 6 標準
- `Java.type()` 取代廢棄的 `importPackage`
- 現代寫法，可直接移植到 HeavenMS / Cosmic

#### Dynamic NPC Shop Script — 動態商店
- **來源**: RaGEZONE Dynamic NPC Shop Script (v83)
- 優點: 修改商店無需重啟伺服器，直接改 .js 即可
- 適用場景: 頻繁更新商品的商店

#### NPC 腳本相容性重點
- ✅ 使用普遍支援的 `cm.` 函數
- ❌ 避免 `importPackage`（JDK 7+ 已廢棄）
- ❌ 避免特定音源專有 Package 路徑
- ✅ 優先使用 `var` 而非 Java 類型聲明
- ✅ 先本地測試再部署
