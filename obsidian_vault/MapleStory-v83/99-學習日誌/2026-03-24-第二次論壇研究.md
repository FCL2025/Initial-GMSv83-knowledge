### 2. 抓包分析 - MapleCrypto 加密詳解

#### v83 三層加密架構
```
Layer 1: MapleCrypto (頭部混淆) - 防止靜態分析
Layer 2: Shanda Encryption (早期版本，已於 v118+ 移除)
Layer 3: AES-OFB (主加密) - 256-bit key
```

#### Hello Packet 重要原則
> **第一個發送的封包永遠是純文字（未加密）**

這是分析 v83 封包的最關鍵起點！

#### MapleCrypto 頭部混淆演算法
```python
# 混淆
verify = (opcode ^ iv[0]) + (iv[1] << 8)
verify = (verify << 3) | (verify >> 5)

# 解混淆
verify = (verify >> 3) | ((verify << 5) & 0xFFFF)
opcode = (verify - (iv[1] << 8)) ^ iv[0]
```

### 3. Opcode 結構 - v83 MapleCrypto 實作

#### 重要 Opcode 參考
| Opcode | 名稱 | 說明 |
|--------|------|------|
| 0x00 | LOGIN_STATUS | 登入結果 |
| 0x01 | LOGIN_PASSWORD | 登入請求 |
| 0x06 | SERVERLIST_REQUEST | 請求伺服器列表 |
| 0x07 | CHARLIST_REQUEST | 請求角色列表 |
| 0x0B | PLAYER_DROP | 進入遊戲 |

#### IDA 字串搜尋技巧
- 搜尋 `opcode` 相關字串
- AOB (Array of Bytes) 搜尋可找到 opcode 表

#### 開源 Opcode 映射
- **Cosmic**: https://github.com/P0nk/Cosmic （最完整）
- 包含完整的 Java 實現，可對照學習

### 5. NPC 腳本編寫 - 進階

#### 動態 NPC 商店
- 不需要 SQL 資料庫
- 每次修改無需重啟伺服器
- 使用 JavaScript 陣列定義商品

#### NPC 腳本狀態機
```
mode=1  → 玩家點擊"下一頁"或選項
mode=0  → 玩家點擊"返回"
mode=-1 → 玩家關閉對話框
```

#### 常見錯誤
- `importPackage` 在 JDK 7+ 廢棄，應使用完整路徑
- 忘記 `cm.dispose()` 導致對話框無法關閉
