**GitHub**：https://github.com/obstriker/Maple_Pshark

**功能**：

#### 🆕 MaplePacketPuller - Python 封包分析器 (GitHub)
**GitHub**：https://github.com/Bratah123/MaplePacketPuller

**功能**：
- Python 撰寫，跨平台
- 支援多種 MapleStory 版本
- 自動解析封包結構
- 可自訂腳本過濾特定 Opcode

#### 🆕 第一個封包特性（重要原則）
> **第一個發送/接收的封包永遠是純文字（未加密）**

**用途**：
- 協商 MapleCrypto 參數 (IV)
- TCP 協定傳輸
- Hello Packet 是分析的起點

#### Wireshark 過濾表達式
```
tcp.port == 7575 || tcp.port == 8484
```

#### 識別加密封包的徵兆
- 封包長度固定（如每次都是 48 bytes）
- 內容看起來像隨機位元組
- 相同操作產生的封包每次都不同（因為 IV 變化）

### 3. Opcode 結構

#### 🆕 MapleRef Wiki Opcode / Packet 結構（最完整參考）
**Packet 結構**：https://mapleref.fandom.com/wiki/Packets
**Opcode 列表**：https://mapleref.fandom.com/wiki/Operation_Code

**內容涵蓋**：

- 各版本 Opcode 對應關係
- 可配合 Wireshark 使用

#### 🆕 IDA 分析 Opcode 進階技巧 (RaGEZONE)
**RaGEZONE**：https://forum.ragezone.com/threads/getting-packet-structures-opcodes-using-ida.792436/

**分析關鍵點**：
1. 找 `CClientSocket::ProcessPacket` 函數
2. 客戶端接收封包流向：`CClientSocket::ProcessPacket → CField::OnPacket / CLogin::OnPacket`
3. 這就是為什麼有 opcode 清單就能輕鬆找到客戶端接收封包的處理函數

**RaGEZONE**：https://forum.ragezone.com/threads/how-does-one-find-a-structure-of-a-packet-via-ida.1162654/

**AOB (Array of Bytes) 搜尋**：
- 在 IDA Hex View 搜尋已知 opcode 的 bytes 模式
- 可找到 opcode 表的記憶體位置

#### 🆕 重要 Opcode 參考 (v83)

| Opcode | 名稱 | 說明 |
|--------|------|------|
| 0x00 | LOGIN_STATUS | 登入結果 |
| 0x01 | LOGIN_PASSWORD | 登入請求 |
| 0x02 | ACCEPT_LOGIN | 同意登入 |
| 0x03 | CHARLIST | 角色列表 |
| 0x04 | CHAR_INFO | 角色資訊 |
| 0x06 | SERVERLIST_REQUEST | 請求伺服器列表 |
| 0x07 | CHARLIST_REQUEST | 請求角色列表 |
| 0x08 | CHAR_SELECT | 選擇角色 |
| 0x0B | PLAYER_DROP | 進入遊戲 |
| 0x14 | MOVE_PLAYER | 移動 |
| 0x15 | CLOSE_RANGE_ATTACK | 近距離攻擊 |
| 0x16 | RANGED_ATTACK | 遠距離攻擊 |
| 0x28 | NPC_TALK | NPC 對話 |
| 0x6A | MOB_APPLY_CONTROL | 怪物生成+HP顯示 |

#### MapleCrypto Header 加密原理
```
封包頭部結構 (4 bytes):
  [2 bytes: 混淆後的長度 + opcode]
  [2 bytes: 實際長度]

加密公式：
  verify = (opcode ^ iv[0]) + (iv[1] << 8)
  verify = (verify << 3) | (verify >>> 5)
  verify &= 0xFFFF
```

### 4. WZ 修改

#### 🆕 WZ vs NX 格式比較 (RaGEZONE, 2026-02)
**來源**：https://forum.ragezone.com/threads/wz-and-nx-file-questions.1260605/

**背景**：
- NX 格式是 WZ 的二進位替代方案
- 比 WZ/xml 性能更好
- 可從 WZ 轉換為 NX
- v83 主要仍使用 WZ 格式

**轉換工具**：
- https://forum.ragezone.com/threads/the-nx-format.891992/
- https://forum.ragezone.com/threads/converted-nx-files-for-maplestory.891937/

#### 🆕 新版物品 Import 到 v83 教學 (RaGEZONE, 2025-11)
**來源**：https://forum.ragezone.com/threads/maplestory-v83-newer-items-import.1256083/

**問題**：
- 嘗試將新版物品 Import 到 v83
- 物品可以生成但無法裝備

**可能原因**：
- 物品的 `.img` 檔案 Import 時結構不完整
- 需要確保 Character.wz 中有對應的裝備定義
- 裝備限制屬性（Level, STR, DEX 等）在 Item.wZ 中設置不正確

#### 🆕 Quest lvmax 限制移除詳細步驟 (RaGEZONE, 2025-07)
**來源**：https://forum.ragezone.com/threads/maplestory-v83-method-to-disable-quest-level-restrictions.1250107/

**方法**：
1. 用 Harepacker 開啟 Quest.wz，encryption: **GMS (old)**
2. Tools → Export selection → XML → **Classic**（選擇 New 會使用新版加密）
3. 用 Notepad++ 開啟 XML，搜尋所有 `lvmax` 屬性並移除
4. 存檔後用 Harepacker 重新導入

#### 🆕 Boss HP 百分比血條實現 (RaGEZONE, 2026-02)
**來源**：https://forum.ragezone.com/threads/v83-client-boss-percentage-bar.1260584/

**原理**：
- 封包 Opcode 0x6A（怪物生成）配合客戶端顯示
- 需要修改 UI.wz 或使用源碼內建的 Mobgage 機制
- MapleRoyals 有完整實現

**編輯客戶端 WZ**：

### 5. NPC 腳本編寫

#### 🆕 NPC 腳本源間相容性問題 (RaGEZONE, 2026-03)
**來源**：https://forum.ragezone.com/threads/kinoko-npc-scripts.1261538/

**問題**：
- 不同音源（Cosmic、Kinoko、HeavenMS）的 NPC 腳本格式存在差異
- 較舊的腳本在新解析器可能不相容
- 原因：腳本解析器版本不同

**解決方案**：
- 在 source 中 include 較舊版本的 script reader
- 或使用相容格式編寫腳本

**最佳實踐**：
- 使用普遍支援的 `cm.` 函數
- 避免使用特定音源專有的 Package 路徑
- 先在本地測試再部署到正式伺服器

#### 🆕 動態 NPC 商店腳本 (RaGEZONE 2016)
**來源**：https://forum.ragezone.com/threads/dynamic-npc-shop-script-v83.1089979/

**特點**：
- 不需要 SQL 資料庫
- 每次修改無需重啟伺服器
- 使用 JavaScript 陣列定義商品

#### NPC 腳本 JavaScript 模板
```javascript
var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        
        if (mode == 1)
            status++;
        else
            status--;
        
        if (status == 0) {
            cm.sendNext("你好！歡迎來到我的商店！");
        } else if (status == 1) {
            cm.sendSimple("你想做什麼？\r\n#b#L0#購買物品#l\r\n#L1#離開#l");
        } else if (status == 2) {
            if (selection == 0) {
                cm.openShop(1);
                cm.dispose();
            } else {
                cm.dispose();
            }
        }
    }
}
```
