## 🆕 學習重點一：MapleSolaxiaV2 完整架設流程更新 (RaGEZONE 2025-2026)

**來源**: https://github.com/skhzhang/MapleSolaxiaV2

### 環境架設摘要

```
Step 1: 安裝軟體環境
  - JDK 7u79 (必須使用 JDK 7，JDK 8 可能相容但非官方推薦)
  - WampServer 2.0i (MySQL 資料庫)
  - Hamachi (可選，用於網路對戰)
  - MySQL Workbench GPL 5.2.39

Step 2: 安裝 JCE 無限強度加密原則
  - jce_policy-7_unlimited.zip
  - 覆蓋 JRE 和 JDK 的 lib/security/ 中的JAR

Step 3: SQL 資料庫設定
  - 執行 db_database.sql (建立 maplesolaxia schema)
  - 執行 db_drops.sql (物品掉落數據)
  - 可選：db_shopupdate.sql (自訂商店)

Step 4: 帳號建立
  - 手動在 accounts 資料表新增一筆記錄
  - 設定 login 和 password

Step 5: IP 設定
  - 編輯 configuration.ini
  - localhost = 127.0.0.1 (單機)
  - 或使用 Hamachi IP / 公開 IP

Step 6: 啟動伺服器
  - NetBeans 開啟專案
  - Clean and Build
  - 執行 launch.bat
```

### 客戶端架設摘要

```
Step 1: 使用 ManagerMsv83.exe 安裝原始客戶端

Step 2: 刪除保護相關檔案
  - 刪除 HShield/ 資料夾
  - 刪除 ASPLauncher.exe
  - 刪除 MapleStory.exe
  - 刪除 Patcher.exe

Step 3: 複製 localhost.exe 到客戶端目錄

Step 4: 使用 HxD 編輯 localhost.exe
  - 搜尋 "127.0.0.1" (Text String 模式)
  - 替換為你的伺服器 IP

Step 5: 複製修改後的 WZ 檔案到客戶端目錄
```

### WZ 編輯重要原則

> DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate some kind of bugs afterwards.

**正確流程**：
```
1. 使用 HaRepacker 4.2.4，打開 GMS (old) 加密的客戶端 WZ
2. 使用 UI 直接修改數值（不要刪除/重新導入節點！）
3. 儲存 WZ
4. 在 HaRepacker 中選擇 "Private Server..." 導出選項
5. 將導出的 XML 覆蓋到伺服器的 WZ.XML 目錄
```

### MobBookUpdate 專案

Ronan C. P. Lana 開發的工具，用於更新所有怪物的掉落數據：

```
1. 用 NetBeans 開啟 MobBookUpdate 專案
2. Build it
3. 複製 MonsterBook.img.xml（從原始 WZ v83）到 dist/ 目錄
4. 在 dist/ 中執行 java -jar MobBookUpdate.jar
5. 輸出會自動更新資料庫
```

## 🆕 學習重點五：v83 NPC 腳本實戰 - Fast Travel NPC 完整範例

**來源**: https://forum.ragezone.com/threads/fast-travel-npc-v83.1191556/

### 完整腳本源

```javascript
/*
 * 作者: Z41N
 * NPC 名稱: Sunny
 * 地圖: 前往 Ludi 的車站 (200000121)
 * 功能: 傳送到 Ludibrium，支援快速傳送票
 */

var status = 0;

function start() {
    cm.sendYesNo("嘿，我是 Sunny！你想傳送到 Ludibrium 嗎？");
}

function action(mode, type, selection) {
    status++;

    // 處理對話框關閉/返回
    if (mode != 1) {
        if(mode == 0 && type != 1)
            status -= 2;
        else if(type == 1 || (mode == -1 && type != 1)) {
            if(mode == 0) {
                cm.sendOk("嗯... 看來你還有事要在這裡處理？");
                cm.dispose();
                return;
            }
        }
    }

    // 狀態機處理
    if (status == 1) {
        if (cm.haveItem(4310000))
            cm.sendNext("#b檢測到快速傳送票#k。\r\n看來你有這張票！");
        else
            cm.sendNext("當然可以！我帶你去 Ludibrium。");
    } else if (status == 2) {
        cm.warp(220000000);  // Ludi
        cm.dispose();
    }
}
```

### 關鍵學習點

| 函數 | 功能 | 範例 |
|------|------|------|
| `cm.sendYesNo` | 顯示是/否對話框 | `cm.sendYesNo("你想傳送嗎？")` |
| `cm.haveItem(itemId)` | 檢查背包是否有物品 | `cm.haveItem(4310000)` |
| `cm.warp(mapId)` | 傳送到指定地圖 | `cm.warp(220000000)` |
| `status` | 狀態機追蹤 | `status++` 遞增 |
| `mode` | 玩家按鈕行為 | `mode=1` 確認，`mode=0` 取消 |
| `cm.dispose()` | 關閉對話框 | 最後必須呼叫 |

### 為什麼需要動態商店？

| 難以做條件判斷 | 可做任何 JavaScript 邏輯 |

### 完整動態商店腳本源

```javascript
/*
 * 動態 NPC 商店 - 不需要 SQL
 * NPC ID: 9010009
 * 功能：玩家可自訂義購買清單
 */
var status = 0;

// 商店商品陣列 [物品ID, 價格, 數量]
var shopItems = Array(
    Array(1302000, 10000, 1),   // 短劍
    Array(1302001, 20000, 1),   // 長劍
    Array(1302002, 30000, 1),   // 雙手劍
    Array(1302003, 50000, 1)    // 斧頭
);

var selectedItem = -1;
var selectedShop = -1;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        
        if (mode == 1)
            status++;
        else
            status--;
        
        if (status == 0) {
            // 顯示商店分類
            var msg = "歡迎光臨！\r\n請選擇商品分類：\r\n";
            msg += "#b#L0#武器#l\r\n";
            msg += "#b#L1#防具#l\r\n";
            msg += "#b#L2#飾品#l";
            cm.sendSimple(msg);
        } else if (status == 1) {
            selectedShop = selection;
            showItems(selection);
        } else if (status == 2) {
            selectedItem = selection;
            var item = getItem(selectedShop, selection);
            cm.sendYesNo("確定要購買 #i" + item[0] + "# #t" + item[0] + "# 嗎？\r\n價格：" + item[1] + " 金幣");
        } else if (status == 3) {
            var item = getItem(selectedShop, selectedItem);
            if (cm.getMesos() >= item[1]) {
                if (cm.canHold(item[0])) {
                    cm.gainMesos(-item[1]);
                    cm.gainItem(item[0], item[2]);
                    cm.sendOk("購買成功！");
                } else {
                    cm.sendOk("背包空間不足！");
                }
            } else {
                cm.sendOk("金幣不足！");
            }
            cm.dispose();
        }
    }
}

function showItems(shopType) {
    // 根據商店類型過濾商品
    var filtered = shopItems.filter(function(item) {
        return Math.floor(item[0] / 10000) == (shopType + 130);  // 130xxx = 武器
    });
    
    var msg = "請選擇商品：\r\n";
    for (var i = 0; i < filtered.length; i++) {
        msg += "#L" + i + "##i" + filtered[i][0] + "# #t" + filtered[i][0] + "# - " + filtered[i][1] + " 金幣#l\r\n";
    }
    cm.sendSimple(msg);
}

function getItem(shopType, itemIndex) {
    var filtered = shopItems.filter(function(item) {
        return Math.floor(item[0] / 10000) == (shopType + 130);
    });
    return filtered[itemIndex];
}
```

## 🆕 學習重點七：Opcode 結構深度解析 - MapleCrypto 三層加密實戰

### 三層加密架構圖

```
┌────────────────────────────────────────────────────────┐
│ Layer 1: MapleCrypto (頭部混淆)                        │
│   Header = 4 bytes                                      │
│   ┌────────┬────────┬────────┬────────┐                │
│   │ byte 0 │ byte 1 │ byte 2 │ byte 3 │                │
│   │  verify[0]  │  verify[1]  │  length   │            │
│   └────────┴────────┴────────┴────────┘                │
│   verify = (opcode ^ iv[0]) + (iv[1] << 8)             │
│   verify = (verify << 3) | (verify >> 13)               │
│                                                        │
│ Layer 2: Shanda Encryption (對稱加密)                  │
│   - v83 及之前版本使用                                 │
│   - 靜態對稱金鑰                                       │
│   - 已於 v118+ 移除                                   │
│                                                        │
│ Layer 3: AES-OFB (流加密)                             │
│   - 256-bit key                                        │
│   - 4-byte IV                                          │
│   - 版本決定的靜態金鑰                                 │
└────────────────────────────────────────────────────────┘
```

### 第一個封包永遠是明文

**為什麼第一個封包是明文？**

```
目的：客戶端和伺服器協商加密金鑰 (IV)

客戶端發送 (明文):
  - 版本號 (2 bytes)
  - 客戶端驗證訊息

伺服器回應 (明文):
  - 同意的版本
  - 加密金鑰 (IV)

之後所有封包:
  → AES-OFB 加密
  → Shanda 加密
  → MapleCrypto 頭部混淆
```

### 封包解密步驟

```
1. 讀取 4-byte header
2. 從 header 提取長度和混淆後的 opcode
3. 根據 IV 反向計算真正的 opcode
4. 根據長度讀取剩餘 payload
5. AES-OFB 解密
6. Shanda 解密
7. 處理明文資料
```

### MapleCrypto 關鍵公式

```java
// 加密端（發送封包）
public byte[] getPacketHeader(int opcode, int length, int iv) {
    int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
    verify = (verify << 3) | (verify >>> 5);
    verify &= 0xFFFF;
    return new byte[] {
        (byte) (verify & 0xFF),
        (byte) ((verify >>> 8) & 0xFF),
        (byte) (length & 0xFF),
        (byte) ((length >>> 8) & 0xFF)
    };
}

// 解密端（接收封包）
public int retrieveOpcode(byte[] header, int iv) {
    int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
    verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
    return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
}
```
