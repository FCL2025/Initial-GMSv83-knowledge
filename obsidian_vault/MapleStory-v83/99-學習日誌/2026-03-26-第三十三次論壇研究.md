## 二、抓包分析

### 🆕 Maple_Pshark 四大核心功能詳解（GitHub, 2026-03 更新）

**GitHub**: https://github.com/obstriker/Maple_Pshark

| 功能 | 說明 |
|------|------|
| `1. Intercepting Recv messages` | 攔截客戶端接收封包，直接從記憶體獲取明文 |
| `2. Modify messages` | 修改封包內容後再交給客戶端處理 |
| `3. Injecting packets` | 向客戶端或伺服器注入自訂封包 |
| `4. Rule based filtering` | 基於規則的封包過濾和操作 |

**技術原理**：
- Hook `MapleCrypto::AESOFBEncrypt` 等加密函數
- 直接在記憶體層截獲**解密前/加密後**的明文
- 不需要知道 AES Key 和 IV，繞過加密層直接獲取明文

**適用場景**：
- 分析未知封包的實際內容
- 測試客戶端對特定封包的反應
- 開發新的封包功能時的快速驗證

### 🆕 MaplePacketPuller + SpiritIDAPlugin 完整工作流程

**GitHub**: 
- MaplePacketPuller: https://github.com/Bratah123/MaplePacketPuller
- SpiritIDAPlugin: https://github.com/Bratah123/SpiritIDAPlugin

**完整工作流程**：
```
Step 1: IDA Pro 反編譯
  → 在 IDA 中找到 CField::OnPacket 或 CLogin::OnPacket
  → 產生 pseudocode (.p64 或 .txt)

Step 2: 複製 pseudocode
  → 在 Functions/ 目錄建立 .txt 檔案
  → 貼上 IDA 產生的 C-style pseudocode

Step 3: Python 分析
  → cd MaplePacketPuller/IDA\ Maple\ Script/src/main/python
  → python main.py
  → 輸入分析類型（完整/標準）
  → 輸入要分析的檔案名稱

Step 4: 輸出結果
  → 封包結構自動寫入 FuncOutput/ 目錄
  → 控制台顯示 Decode1/Decode2/Decode4/DecodeStr 等欄位
```

### 🆕 v83 封包三層加密架構（鞏固理解）

```
┌─────────────────────────────────────────────────────┐
│ Layer 1: MapleCrypto (頭部混淆)                     │
│   - 4 bytes header (混淆 opcode + length)           │
│   - 防止靜態分析 opcode                            │
│                                                        │
│ Layer 2: Shanda Encryption (對稱加密)                │
│   - Nexon 內部名稱：IOBufferManipulator            │
│   - 已於 v118+ 移除                               │
│                                                        │
│ Layer 3: AES-OFB (主加密)                           │
│   - v83 主要加密方式                                │
│   - 256-bit key, 4-byte IV                         │
│   - 版本決定的靜態金鑰                              │
└─────────────────────────────────────────────────────┘

發送端：
明文 → Shanda 加密 → AES-OFB → MapleCrypto header → 網路

接收端：
網路 → MapleCrypto 解混淆 → Shanda 解密 → AES-OFB → 明文
```

**重要原則**：
> **第一個發送/接收的封包永遠是純文字（未加密）**
> MapleStory 封包透過 TCP Socket 傳輸

## 三、Opcode 結構

### 🆕 MapleRef Wiki Send/Recv Opcode 核心概念

**來源**: https://mapleref.fandom.com/wiki/Operation_Code

### Send vs Receive Opcode 區分

> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

**重要**：同一個 Opcode 數字可能代表不同含義，視方向而定！

### 🆕 v83 登入結果碼（Login Status Code）

| 結果碼 | 意義 |
|--------|------|
| 0 | 登入成功 |
| 5 | 帳號或密碼錯誤 |
| 7 | 帳號已停用 |
| 13 | 刪除中（冷靜期） |
| 23 | 登入太頻繁（防機器人延遲） |

### 🆕 CClientSocket::ProcessPacket 封包分發流程

**所有客戶端接收封包的入口點**：
```
CClientSocket::ProcessPacket()
    ↓
根據 MapleCrypto header 解密
    ↓
提取 opcode (2 bytes, little endian)
    ↓
switch(opcode) 分發
    ↓
CLogin::OnPacket()    // 登入畫面
    或
CField::OnPacket()    // 遊戲中
```

### 🆕 MapleCrypto Header 加密/解密公式（v83 核心）

```java
// 加密端（伺服器/客戶端發送）
public byte[] getPacketHeader(int opcode, int length, int iv) {
    int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
    verify = (verify << 3) | (verify >>> 5);
    verify &= 0xFFFF;
    return new byte[] {
        (byte) (verify & 0xFF),
        (byte) ((verify >>> 8) & 0xFF),
        (byte) (length & 0xFF),
        (byte) ((length >>> 8) & 0xFF)
    };
}

// 解密端（接收方）
public int retrieveOpcode(byte[] header, int iv) {
    int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
    verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
    return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
}
```

### 🆕 動態配方系統腳本源（完整展示）

```javascript
/*
 * 動態配方系統 - 玩家提交材料換取成品
 * NPC ID: 9010010
 */
var status = 0;

var recipes = Array(
    {
        name: "楓之葉地圖",
        materials: Array(Array(4000000, 10), Array(4000001, 5)),
        result: Array(Array(2300000, 1)),
        mesoCost: 1000
    },
    {
        name: "傳說武器",
        materials: Array(Array(4003000, 100), Array(4003001, 50)),
        result: Array(Array(1302000, 1)),
        mesoCost: 50000
    }
);

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        if (mode == 1)
            status++;
        else
            status--;

        if (status == 0) {
            var msg = "歡迎來到工藝系統！\r\n請選擇配方：\r\n";
            for (var i = 0; i < recipes.length; i++) {
                msg += "#L" + i + "#" + recipes[i].name + "#l\r\n";
            }
            cm.sendNext(msg);
        } else if (status == 1) {
            var recipe = recipes[selection];
            var canCraft = true;
            var msg = "你需要以下材料：\r\n";
            for (var i = 0; i < recipe.materials.length; i++) {
                var mat = recipe.materials[i];
                var have = cm.getQuantityOfItem(mat[0]);
                var need = mat[1];
                msg += "#i" + mat[0] + "# #t" + mat[0] + "#: " + have + "/" + need + "\r\n";
                if (have < need) canCraft = false;
            }
            if (cm.getMesos() < recipe.mesoCost) canCraft = false;
            msg += "加工費: " + recipe.mesoCost + " 金幣\r\n";
            msg += canCraft ? "#g可以製作#k" : "#r材料不足#k";
            cm.sendYesNo(msg);
        } else if (status == 2) {
            var recipe = recipes[selection];
            for (var i = 0; i < recipe.materials.length; i++) {
                cm.gainItem(recipe.materials[i][0], -recipe.materials[i][1]);
            }
            cm.gainMesos(-recipe.mesoCost);
            for (var i = 0; i < recipe.result.length; i++) {
                cm.gainItem(recipe.result[i][0], recipe.result[i][1]);
            }
            cm.sendOk("製作成功！");
            cm.dispose();
        }
    }
}
```
