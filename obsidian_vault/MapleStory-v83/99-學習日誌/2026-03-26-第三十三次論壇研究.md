## 🆕 客戶端修改：dinput8.dll 自動化方案 vs 傳統 Hex 修改

### MapleEzorsia v2 工作原理（RaGEZONE 2023, GitHub）

**來源**: https://github.com/444Ro666/MapleEzorsia-v2

**核心定位**：
> This DLL modifies addresses in a default, packed, v83 MapleStory client fresh from installation to enable the game to run; as well as changes the game window and canvas resolution to HD.

**為什麼是 dinput8.dll？**
- Windows 會自動加載 `dinput8.dll` 到所有使用 DirectInput 的遊戲
- MapleStory 使用 DirectInput 進行鍵盤/滑鼠輸入
- 因此 `dinput8.dll` 不需要 `EasyHook` 或 `SetWindowsHookEx`
- 遊戲啟動時自動加載，無需注入器

**dinput8.dll 的優點**：
| 方案 | 優點 | 缺點 |
|------|------|------|
| dinput8.dll | 自動加載，無需注入器，不破壞原 exe | 只能修改有 DirectInput 依賴的遊戲 |
| localhost.exe | 簡單直接 | 需搭配 Hex 編輯，版本匹配問題 |
| Hex 直接修改 | 完全控制 | 需脫殼，破壞原檔 |

**MapleEzorsia v2 完整功能表**：
| 功能 | 說明 |
|------|------|
| HD Resolution | 任意視窗和 Canvas 解析度 |
| localhost redirect | 自動將伺服器 IP 導向 127.0.0.1 |
| No Patcher | 跳過官方 Patch 過程 |
| Anti-HShield | 部分繞過 HackShield 保護 |
| FPS Unlock | 解除幀率限制 |

### 傳統 Hex 編輯 localhost.exe（RaGEZONE 2020）

**來源**: https://forum.ragezone.com/threads/how-to-make-a-localhost-redirector.1192131/

**兩種主要方法**：
```
方法 1: localhost.exe (最廣泛使用)
  - 專門的 localhost 重定向程式
  - 替代原始 MapleStory.exe
  - HxD 修改 IP 位址

方法 2: DLL injection (更靈活)
  - 使用 WinSock Hook 攔截連線
  - 可動態修改目標 IP
  - 需要注入器（如 Extreme Injector）
```

**localhost.exe Hex 編輯步驟**：
```
1. 用 HxD 開啟 localhost.exe（先備份！）
2. Ctrl+F → 搜尋類型 "Text-string" → 輸入 "127.0.0.1"
3. 會找到三處幾乎相連的 IP 位址
4. 將第一個 IP 改為目標 IP（如 "192.168.1.100"）
5. 確保 IP 長度一致（不足用 \x00 填充）
6. 儲存
```

## 🆕 MapleStory 封包加密實戰：MapleCrypto + Shanda + AES 三層結構

### 完整加密/解密流程

```
【發送端 C→S】
明文封包 [opcode][payload]
    ↓
1. AES-OFB 加密 (版本 Key + 當前 IV)
    ↓
2. Shanda (IOBufferManipulator) 加密
    ↓
3. MapleCrypto Header (4 bytes: 混淆 opcode + length)
    ↓
網路傳輸

【接收端 S←C】
網路接收
    ↓
1. MapleCrypto Header 解混淆 → 提取 opcode + length
    ↓
2. Shanda 解密
    ↓
3. AES-OFB 解密
    ↓
明文封包 → 處理
```

### MapleCrypto Header 演算法（Java 實現）

```java
// 發送端：加密 Header
public byte[] getPacketHeader(int opcode, int length, int iv) {
    int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
    verify = (verify << 3) | (verify >>> 5);  // Rotate left 3 bits
    verify &= 0xFFFF;
    return new byte[] {
        (byte) (verify & 0xFF),
        (byte) ((verify >>> 8) & 0xFF),
        (byte) (length & 0xFF),
        (byte) ((length >>> 8) & 0xFF)
    };
}

// 接收端：解密 Header
public int retrieveOpcode(byte[] header, int iv) {
    int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
    verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);  // Rotate right 3 bits
    return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
}

// 解密長度
public static int decryptLength(byte[] header, int iv) {
    int length = (header[2] & 0xFF) | ((header[3] << 8) & 0xFF00);
    length ^= (iv >> 16) & 0xFFFF;
    return length;
}
```

### v83 AES Key 和 IV 參考

| 參數 | CMS (中國) | GMS (國際) |
|------|-----------|-----------|
| AES UserKey | `0x13, 0x08, 0x06, 0xB4, 0x1B, 0x0F, 0x33, 0x52` | 版本決定 |
| IV Transform | `0xC65053F2` | 版本決定 |
| 客戶端版本 | `0x23` (35 decimal) | `0x53` (83 decimal) |

### Hello Packet（第一個純文字封包）結構

```
【C→S Hello Packet】
偏移    長度   內容
0x00    2      封包總長度 (little endian)
0x02    2      Maple 版本號 (0x0053 = 83)
0x04    n      版本字串 (如 "83")
n+4     4      SendIV (客戶端指定的發送 IV)
n+8     4      RecvIV (客戶端指定的接收 IV)
n+12    2      地區標誌 (MAPLE_LOCALE)
n+14    2      Unknown

【S→C Hello Packet 回應】
- 回應相同的 RecvIV/SendIV
- 確認版本匹配
- 若失敗，連線中斷
```

```java
// 在 PlayerLoggedInHandler 中發送
```

### HaRepacker 導出 XML 注意事項

**重要**（RaGEZONE, 2023）：
```
Tools → Export Selection → XML → Classic
❗千萬不要選 "New"！
"New" 會使用新版加密格式，與私人伺服器不相容！
```

**WZ 編輯黃金法則**：
```
DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs.

正確流程：
1. HaRepacker 打開 [Client WZ]（GMS old 加密）
2. UI 直接修改數值
3. 儲存
4. Tools → Export → Private Server XML 導出
5. 覆蓋到 [Server WZ.XML] 目錄
```

### 跨版本物品 Import 必備屬性清單

| 屬性 | 說明 | 缺少時症狀 |
|------|------|-----------|
| `lvlm` | 等級下限 | 客戶端崩潰或無法使用 |
| `lvmax` | 等級上限 | 物品可使用但顯示異常 |
| `STR` | 力量需求 | 戰士無法使用 |
| `DEX` | 敏捷需求 | 弓箭手無法使用 |
| `price` | 商店價格 | 無法在商店顯示 |
| `maxSlot` | 最大疊加數 | 無法疊加或疊加上限錯誤 |


### MapleRef Wiki Send/Recv Opcode 核心概念

**來源**: https://mapleref.fandom.com/wiki/Operation_Code

> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

## 🆕 完整 v83 私服架設流程摘要（2026-03 最新）

### 環境需求
| 軟體 | 版本 | 必要性 |
|------|------|--------|
| JDK | 8+ | 必要（Cosmic 需要 Java 21） |
| JCE | 8 | 必要（加密） |
| MySQL | 5.5+ | 必要 |
| v83 客戶端 | - | 必要 |
| HxD / dinput8.dll | - | 修改 IP 用 |

### 架設步驟摘要
```
1. 安裝 JDK 8，設定 JAVA_HOME
2. 安裝 MySQL Server，創建資料庫
3. 下載 HeavenMS/MapleSolaxiaV2/Cosmic 音源
4. 執行 SQL 初始化資料庫
5. 設定伺服器 IP 和埠口
6. 客戶端：MapleEzorsia v2 (dinput8.dll) 或 HxD 修改 IP
7. 啟動客戶端和伺服器
```

### LAN 多人連線必檢查清單
- [ ] 確認兩台 PC 在同一 LAN 網段
- [ ] 確認伺服器 listen 0.0.0.0（而非 127.0.0.1）
- [ ] 確認防火牆開啟 7575 和 8484 埠口
- [ ] 客戶端 IP 需改為伺服器 LAN IP（如 192.168.1.x）
- [ ] 若錯誤碼 0x80004005 → 防火牆/權限問題
