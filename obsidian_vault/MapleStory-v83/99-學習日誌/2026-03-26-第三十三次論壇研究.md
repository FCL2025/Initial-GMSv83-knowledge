### STREDIT 使用步驟 (RaGEZONE, 2023)

**來源**: https://forum.ragezone.com/threads/help-some-one-to-change-localhost-ip.1218181/

```
1. 下載 STREDIT

8. 存檔
```

## 🆕 Hex Editor 客戶端修改完整教學 (maplesecrets, 2013)

**來源**: http://maplesecrets.blogspot.com/2013/04/how-to-hex-edit-clients-easily.html

### Hex 編輯測試環境
- v83 MapleStory localhost
- HxD Hex Editor（推薦）或任何十六進位編輯器

### IP 修改步驟

```
1. 開啟 MapleStory.exe 或 localhost.exe
2. Ctrl+F → 選擇 "Text string"
3. 搜尋 "127.0.0.1"
4. 替換為目標 IP（如 hamachi IP）
5. 存檔
```

### 常見問題

| 問題 | 說明 |
|------|------|
| IP Split 存放 | 字串可能分成多段（如 "127" + ".0.0.1"） |
| 長度不一致 | 會造成記憶體崩潰 |
| Hex 表示法 | 部分版本用 `7F 00 00 01` = `127.0.0.1` |

### IP 搜尋 Hex 模式參考

| IP | Hex 表示 |
|----|---------|
| `127.0.0.1` | `31 37 32 2E 30 2E 30 2E 31` |
| `0.0.0.0` | `30 2E 30 2E 30 2E 30` |

### LAN IP 範例

```
目標：192.168.1.100
Hex: 31 39 32 2E 31 36 38 2E 31 2E 31 30 30

⚠️ 注意長度：192.168.1.100 (16 chars) vs 127.0.0.1 (9 chars)
若目標 IP 更長，需確保有足夠空位或使用 hex 編輯
```

## 🆕 Login Redirector 兩種方案對比 (RaGEZONE, Pixeltris)

**來源**: https://forum.ragezone.com/threads/how-to-make-a-localhost-redirector.1192131/

### 方案一：直接連線（非 Loopback）修改

```
原理：HxD 直接修改 MapleStory.exe 中的 IP 字串
優點：最簡單快速
缺點：IP 長度必須相同；需要重複修改
適用：本地測試（127.0.0.1）
```

### 方案二：DLL 注入覆寫記憶體

```
原理：注入 DLL，Hook connect/WSAConnect，動態修改目標 IP
優點：支援任意 IP；無需修改 exe
缺點：需要開發 DLL
適用：對外提供伺服器
```

### Pixeltris 提出的 127.0.0.1 專用方案

```
1. Hook connect() 函數
2. 檢查目標地址是否為 127.0.0.1
3. 如果是，直接連線（不改變）
4. 如果不是，覆寫為 127.0.0.1
```

### WinSock Hook 核心實作概念

```cpp
// 概念：Hook WSAConnect 實現 IP 重定向
typedef int (WSAAPI *WSAConnect_t)(SOCKET, const sockaddr*, int, LPWSABUF, LPWSABUF, LPWSABUF, LPQOS);
WSAConnect_t Original_WSAConnect;

int WSAAPI Hook_WSAConnect(SOCKET s, const sockaddr* name, int namelen, 
                            LPWSABUF lpCallerData, LPWSABUF lpCalleeData, 
                            LPQOS lpSQOS, LPQOS lpGQOS) {
    if (name->sa_family == AF_INET) {
        sockaddr_in* addr = (sockaddr_in*)name;
        addr->sin_addr.s_addr = inet_addr("127.0.0.1");
        addr->sin_port = htons(7575);
    }
    return Original_WSAConnect(s, name, namelen, lpCallerData, lpCalleeData, lpSQOS, lpGQOS);
}
```

## 🆕 WZ 編輯教學 - 使用 HaRepacker 修改客戶端 (maplesecrets, 2014)

**來源**: http://maplesecrets.blogspot.com/2014/05/tutorial-wz-editing-for-maplestory-d.html

### 怪物範圍修改教學

```
1. 用 Repacker 開啟 mob.wz
2. 找到目標怪物 ID
3. 前往 attack1 > info > range
4. 點擊 lt 並改為 (Ctrl+M) 0,0 → Enter
5. 點擊 rb 並改為 (Ctrl+M) 0,0 → Enter
6. 儲存
```

**效果**：怪物將失去攻擊範圍（Magic Disarm）

### HaRepacker 加密格式選擇

| 格式 | 用途 |
|------|------|
| **GMS (old)** | 客戶端 WZ 讀取 |
| **GMS** | 新版客戶端 |
| **Private Server...** | 導出為 XML 供伺服器使用 |

### WZ 修改注意事項

> ⚠️ **千萬不要用伺服器的 WZ XML 直接覆蓋客戶端 WZ！這會產生 bug！**

**正確流程**：
```
1. HaRepacker 開啟 GMS (old) 加密的客戶端 WZ
2. 直接修改數值（不要刪除節點！）
3. 儲存
4. 導出為 Private Server XML
5. 覆蓋到伺服器的 wz.xml 目錄
```

## 🆕 MapleRef Wiki Opcode 結構速查 (2026)

**來源**: https://mapleref.fandom.com/wiki/Operation_Code

### Opcode 基礎概念

> "Actions that are defined by an identifier called an Operation Code (opcode for short), which is the first 2 bytes of the packet (a short)"

### Send vs Receive Opcode

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

**重要**：同一個 Opcode 數字可能代表不同含義，視方向而定！

### v83 常用 Opcode

| Opcode | 名稱 | 方向 | 說明 |
|--------|------|------|------|
| 0x00 | LOGIN_STATUS | S→C | 登入結果 |
| 0x01 | LOGIN_PASSWORD | C→S | 登入請求 |
| 0x03 | CHARLIST | S→C | 角色列表 |
| 0x06 | SERVERLIST_REQUEST | C→S | 請求伺服器列表 |
| 0x0B | PLAYER_DROP | C→S | 進入遊戲 |
| 0x14 | MOVE_PLAYER | C→S | 移動 |
| 0x6F | SHOP_BUY | C→S | 商場購買 |

### MapleRef Wiki 使用方式

```
1. 前往 https://mapleref.fandom.com/wiki/Operation_Code
2. 選擇目標版本 (v83)
3. 查看 Send (S→C) 或 Recv (C→S) Opcode 表
4. 點擊具體 Opcode 查看封包結構
```

### Maple_Pshark vs MaplePacketPuller 詳細比較

| 功能 | Maple_Pshark | MaplePacketPuller |
|------|-------------|-------------------|
| 運作方式 | 即時截獲 | 離線分析 |
| 平台 | Windows | 跨平台 (Python) |
| 加密處理 | Hook 函數，直接明文 | 需手動解密 |
| 封包注入 | ✅ 支援 | ❌ |

### 完整工作流程

```
Step 1: Wireshark 確認 TCP 連線正常
  tcp.port == 7575 || tcp.port == 8484
  ↓ 確認 Hello Packet 交換

Step 2: Maple_Pshark 即時截獲
  Hook 加密函數
  直接觀察明文封包
  ↓ 找出感興趣的封包

Step 3: MaplePacketPuller 離線分析
  從 .pcap 提取封包
  自動識別 opcode
  輸出 Decode 函數

Step 4: MapleRef Wiki 對照
  https://mapleref.fandom.com/wiki/Operation_Code
  確認封包含義
```
