```
4. 三處 IP 全部修改
5. 儲存
```

## 🆕 localhost.exe Hex 編輯原理 (maplesecrets/RaGEZONE, 2010)

**來源**: 
- https://forum.ragezone.com/threads/how-to-hex-edit-clients-easily.703014/
- http://maplesecrets.blogspot.com/2013/04/how-to-hex-edit-clients-easily.html

### 核心原理
v83 localhost.exe 中，IP 位址以**明文 ASCII** 儲存，因此可用任何 Hex 編輯器直接搜尋和修改。

### 搜尋方式
```
HxD / Neo Hex Editor:
→ 編輯 → 搜尋 → 選擇「文字」(Text)
→ 輸入 "127.0.0.1"
→ 找到後直接覆蓋新 IP
```

### 注意事項
| 問題 | 解決 |
|------|------|
| IP 長度不同 | 用 `\x00` (null byte) 填補差額 |
| 三處 IP 未全改 | 會導致部分連線失敗 |
| IP 格式錯誤 | 客戶端連不上伺服器 |

### 為什麼需要修改 localhost.exe？
```
官方客戶端：連線到 Nexon 伺服器 (210.114.89.xx)
localhost.exe：攔截並重新導向到本機或自訂 IP
```

## 🆕 MapleStory v83 封包加密三層架構 (mapleref Wiki + RaGEZONE)

**來源**: https://mapleref.fandom.com/wiki/Packets

### 封包頭部結構
> "Packet headers are 4 bytes long, and are generated with the use of the send IV, packet length (without header) and the version of MapleStory."

```
偏移  長度   內容
0x00  2      MapleCrypto 混淆 opcode (verify)
0x02  2      封包長度 (encrypted with IV)
```

> "Packet opcodes are usually 2 bytes long."

### 三層加密架構
```
┌─────────────────────────────────────────────────┐
│ Layer 1: MapleCrypto Header (4 bytes)           │
│   - 混淆 opcode + encrypted length              │
│   - 防止靜態分析                                │
│                                                     │
│ Layer 2: Shanda Encryption                        │
│   - Nexon 內部名稱：IOBufferManipulator         │
│   - 在 AES 之前應用                              │
│   - v83 仍在使用，v118+ 已移除                  │
│                                                     │
│ Layer 3: AES-OFB (256-bit)                      │
│   - 版本決定的靜態金鑰                          │
│   - 4-byte IV (每次遞增)                        │
└─────────────────────────────────────────────────┘
```

### AES Key 參考 (v83 CMS)
```java
// AES UserKey (first column per row)
0x13, 0x08, 0x06, 0xB4, 0x1B, 0x0F, 0x33, 0x52
// IV Transformation Key: 0xC65053F2
```

### MapleCrypto 頭部混淆公式
```java
// 加密端
int verify = (opcode ^ iv[0]) + (iv[1] << 8);
verify = (verify << 3) | (verify >>> 5);
verify &= 0xFFFF;

// 解密端
verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
opcode = (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
```

## 🆕 v83 Opcode 狀態總結 (RaGEZONE 2024-12)

**來源**: https://forum.ragezone.com/threads/packet-opcode-analysis.1238027/

### 核心結論
> "v83 的 Opcode 幾乎全部被映射，不需要額外追蹤封包。唯一需要的反向工程是**修改客戶端時**。"

### Send vs Receive Opcode 區分
| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器 → 客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端 → 伺服器 | Client Opcode Set |

### v83 常用 Opcode（Send/Recv 對照）
| Opcode | 名稱 | 方向 |
|--------|------|------|
| 0x00 | LOGIN_STATUS | S→C |
| 0x01 | LOGIN_PASSWORD | C→S |
| 0x03 | CHARLIST | S→C |
| 0x04 | CHAR_INFO | S→C |
| 0x06 | SERVERLIST_REQUEST | C→S |
| 0x07 | CHARLIST_REQUEST | C→S |
| 0x08 | CHAR_SELECT | C→S |
| 0x0B | PLAYER_DROP | C→S |
| 0x14 | MOVE_PLAYER | C→S |
| 0x28 | NPC_TALK | S→C |
| 0x6F | SHOP_BUY | C→S |

## 🆕 WZ 編輯實務 (RaGEZONE + MapleSecrets)

### HaRepacker 設定（v83 WZ 加密）
```
版本：選擇 "GMS (old)" 或 "GMS old"
❗ 選 New 會使用新版加密，客戶端無法讀取！
```

### WZ 編輯黃金法則
> **DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs.**

### 正確編輯流程
```
1. HaRepacker 開啟客戶端 WZ (GMS old)
2. 使用 UI 直接編輯數值（❌ 不要刪除/重新導入節點！）
3. 儲存
4. 若需導出 XML：選擇 "Private Server..." 格式
5. 覆蓋到伺服器 WZ.XML 目錄
```

### Quest 等級限制移除（批量）
```regex
Notepad++ Regex：
尋找：lvmax="\d+"\s*
取代：（留空）
```

### 跨版本物品 Import 必備屬性
- `lvlm`（等級下限）
- `STR`、`DEX` 等基本屬性
- ❌ 缺少這些屬性會導致客戶端崩潰或顯示異常

### 自訂 NPC WZ 流程
```
1. HaRepacker 開啟 NPC.WZ
2. 找到一個現有 NPC（如 9010000）→ 右鍵 → 複製
3. 粘貼 → 命名為新 NPC ID（如 9010010）
4. 修改 Frame 和 Origin 設定
5. 儲存
6. 在伺服器端建立對應的 .js 腳本
```

### 自訂物品 WZ 流程
```
1. HaRepacker 開啟 Item.wz
2. 找到相似物品 → 複製 img.xml
3. 修改屬性（名稱、屬性值、等級等）
4. 新增物品圖片（如需要）
5. 儲存
6. 重啟伺服器讓設定生效
```

### 狀態機核心
```javascript
var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) { cm.dispose(); return; }
    if (mode == 0 && type == 1) { cm.dispose(); return; }
    if (mode == 1) status++;
    else status--;
    // status == 0 → 第一句話
    // status == 1 → 第二句話...
}
```
