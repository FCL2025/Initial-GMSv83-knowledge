### 2.1 MapleSolaxiaV2 完整架設流程（2026-03 最新）

**來源**: https://github.com/skhzhang/MapleSolaxiaV2 + RaGEZONE

**環境需求**：
| 軟體 | 版本 | 必要性 |
|------|------|--------|
| JDK | 7u79 | 必要（必須 JDK 7） |
| JCE | 7 Unlimited | 必要（加密） |
| WampServer | 2.0i | MySQL 資料庫 |
| MySQL Workbench | 5.2.39 GPL | SQL 客戶端 |
| Hamachi | - | 可選（網路對戰） |

**完整架設步驟**：

```
Step 1: 安裝軟體環境
  - JDK 7u79 + JCE Unlimited Policy
  - WampServer 2.0i
  - MySQL Workbench

Step 2: JCE 無限強度加密原則
  - jce_policy-7_unlimited.zip
  - 覆蓋 lib/security/ 中的 JAR

Step 3: SQL 資料庫設定
  - 執行 db_database.sql
  - 執行 db_drops.sql
  - 可選：db_shopupdate.sql

Step 4: 帳號建立
  - 手動在 accounts 資料表新增一筆記錄

Step 5: IP 設定
  - configuration.ini → localhost / WAN IP

Step 6: 啟動伺服器
  - NetBeans 開啟專案
  - Clean and Build
  - 執行 launch.bat
```

### 2.2 WZ 編輯黃金法則

> **DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs.**

**正確流程**：
```
1. HaRepacker 4.2.4 開啟 GMS (old) 加密的客戶端 WZ
2. 使用 UI 直接修改數值（禁止刪除/重新導入節點！）
3. 儲存 WZ
4. HaRepacker → "Private Server..." 導出
5. 將導出的 XML 覆蓋到伺服器的 WZ.XML 目錄
```

### 2.4 MapleRef Wiki Send/Recv Opcode 核心概念

> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

**重要**：同一個 Opcode 數字可能代表不同含義，視方向而定！
