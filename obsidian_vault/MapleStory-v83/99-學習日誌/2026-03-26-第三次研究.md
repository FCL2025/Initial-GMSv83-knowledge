## 🆕 MapleCrypto + Shanda + AES-OFB 加密流程完整解析

### 雙金鑰機制核心原理 (maplelib, Go)
> MapleStory uses two keys: one is used to encrypt sent packets and the other is used to decrypt received packets.

| IV | 擁有者 | 用途 |
|----|--------|------|
| 客戶端 SendIV | 客戶端 | 加密發送到伺服器的封包 |
| 客戶端 RecvIV | 客戶端 | 解密從伺服器收到的封包 |
| 伺服器 SendIV | 伺服器 | 加密發送到客戶端的封包 |
| 伺服器 RecvIV | 伺服器 | 解密從客戶端收到的封包 |

### 完整加密流程 (C→S)
```
明文資料
  → Shanda 對稱加密
  → AES-OFB 加密 (使用版本 Key + SendIV)
  → MapleCrypto 頭部混淆 (4-byte header)
  → 網路傳輸
```

### MapleCrypto 頭部混淆公式
```java
// 加密端 (getPacketHeader)
int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
verify = (verify << 3) | (verify >>> 5);  // rotate left 3 bits
verify &= 0xFFFF;
return new byte[] {
    (byte) (verify & 0xFF),
    (byte) ((verify >>> 8) & 0xFF),
    (byte) (length & 0xFF),
    (byte) ((length >>> 8) & 0xFF)
};

// 解密端 (retrieveOpcode)
int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
```

## 🆕 MapleRef Wiki — Opcode 結構參考 (2026-03)

**來源**: https://mapleref.fandom.com/wiki/Packets

### Opcode 核心概念
> "Actions that are defined by an identifier called an Operation Code (opcode for short), which is the first 2 bytes of the packet (a short)"

### Send vs Receive Opcode 區分
| 方向 | 說明 |
|------|------|
| **S→C (Send)** | 伺服器發送，客戶端接收 |
| **C→S (Recv)** | 客戶端發送，伺服器接收 |

### v83 登入結果碼
| 結果碼 | 意義 |
|--------|------|
| 0 | 登入成功 |
| 5 | 帳號或密碼錯誤 |
| 7 | 帳號已停用 |
| 13 | 刪除中（冷靜期） |
| 23 | 登入太頻繁（防機器人延遲） |
