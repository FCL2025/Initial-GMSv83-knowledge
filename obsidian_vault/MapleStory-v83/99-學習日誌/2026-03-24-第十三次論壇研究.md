### botnored v83 封包類別結構

**網站**: https://botnored.weebly.com/blog/maplestory-classes-v83

**重要原則**：
> **第一個發送/接收的封包永遠是純文字（未加密）**
> MapleStory 封包透過 TCP Socket 傳輸

**客戶端主要類別**：
| 類別 | 功能 |
|------|------|
| `CWvsContext` | 世界狀態管理（角色資料、伺服器列表） |
| `CClientSocket` | 網路連接管理（Send/Recv） |
| `CField` | 當前地圖/遊戲場景處理 |
| `CLogin` | 登入畫面處理 |

**封包流向**：
```
CClientSocket::ProcessPacket()
    ↓
根據 Header 分發
    ↓
CField::OnPacket() 或 CLogin::OnPacket()
    ↓
處理具體 Opcode
```

### 封包加密三層架構（v83）

```
┌─────────────────────────────────────────────────────┐
│ Layer 1: MapleCrypto (頭部混淆)                     │
│   - 4 bytes header (混淆 opcode + length)           │
│   - 防止靜態分析 opcode                             │
│                                                        │
│ Layer 2: Shanda Encryption (對稱加密)                │
│   - 早期版本使用                                     │
│   - 已於 v118+ 移除                                 │
│                                                        │
│ Layer 3: AES-OFB (主加密)                           │
│   - v83 主要加密方式                                │
│   - 256-bit key, 4-byte IV                          │
│   - 版本決定的靜態金鑰                              │
└─────────────────────────────────────────────────────┘
```

## 📌 3. Opcode 結構

### MapleCrypto 封包頭部加密/解密公式

**加密端**：
```java
public byte[] getPacketHeader(int opcode, int length, int iv) {
    int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
    verify = (verify << 3) | (verify >>> 5);
    verify &= 0xFFFF;
    return new byte[] {
        (byte) (verify & 0xFF),
        (byte) ((verify >>> 8) & 0xFF),
        (byte) (length & 0xFF),
        (byte) ((length >>> 8) & 0xFF)
    };
}
```

**解密端**：
```java
public int retrieveOpcode(byte[] header, int iv) {
    int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
    verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
    return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
}
```

### 封包格式（v83）
- **Header**: 4 bytes (MapleCrypto 混淆)
- **Opcode**: 2 bytes (little endian)
- **Data**: 可變長度

### sendAcceptDecline / sendGetNumber / sendGetText 行為表

**sendAcceptDecline (Type = 12)**：
| Mode 值 | 意義 |
|---------|------|
| `mode == -1` | 玩家關閉對話框 |
| `mode == 1` | 玩家點擊「接受」(Accept) |
| `mode == 0` | 玩家點擊「拒絕」(Decline) |

**sendGetNumber (Type = 3)**：
```javascript
cm.sendGetNumber("請輸入數量：", 1, 1, 99);
// 參數：提示文字、預設值、最小值、最大值
```

### 快速傳送 NPC 腳本源碼

```javascript
var status = 0;

function start() {
    cm.sendYesNo("嘿，想傳送到 Ludibrium 嗎？");
}

function action(mode, type, selection) {
    status++;

    if (mode != 1) {
        if(mode == 0 && type != 1)
            status -= 2;
        else if(type == 1 || (mode == -1 && type != 1)) {
            if(mode == 0) {
                cm.sendOk("嗯... 看來你還有事？");
                cm.dispose();
                return;
            }
        }
    }

    if (status == 1) {
        if (cm.haveItem(4310000))
            cm.sendNext("#b檢測到快速傳送票#k。");
        else
            cm.sendNext("當然可以！");
    } else if (status == 2) {
        cm.warp(220000000);
        cm.dispose();
    }
}
```

### 職業進階 NPC 腳本源碼

```javascript
var status = 0;
var jobName = "";
var jobId = 0;

function start() {
    cm.sendNext("恭喜 #b10級#k！請選擇法師類型：\r\n#b#L0#火與毒法師#l\r\n#L1#冰與雷法師#l\r\n#L2#僧侶#l");
}

function action(mode, type, selection) {
    if (mode != 1) {
        if (mode == 0)
            cm.sendOk("好吧，等你想好了再來找我。");
        cm.dispose();
        return;
    }

    status++;

    if (status == 1) {
        if (selection == 0) { jobName = "火與毒法師"; jobId = 210; }
        else if (selection == 1) { jobName = "冰與雷法師"; jobId = 220; }
        else if (selection == 2) { jobName = "僧侶"; jobId = 230; }
        cm.sendYesNo("確定要選擇 #b" + jobName + "#k 嗎？");
    } else if (status == 2) {
        cm.gainJob(jobId);
        cm.dispose();
    }
}
```
