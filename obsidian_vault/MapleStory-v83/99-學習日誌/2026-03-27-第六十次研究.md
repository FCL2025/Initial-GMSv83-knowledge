## 🆕 MapleEzorsia-v2 完整客戶端架設指南 (2026-03 更新)

**GitHub**: https://github.com/444Ro666/MapleEzorsia-v2  
**Wiki**: https://github.com/444Ro666/MapleEzorsia-v2/wiki/v83%E2%80%90Client%E2%80%90Setup%E2%80%90and%E2%80%90Development%E2%80%90Guide

### 核心概念：史上第一個獨立 DLL 客戶端

Ezorsia-v2 是**史上第一個 standalone DLL 客戶端**（2023-11-16 首次發布），可將預設包裝過的 v83 exe 直接改造為 HD + localhost 可用。

**原理**：
```
MAPLESTORY.EXE (包裝/加殼)
    ↓  加載 dinput8.dll (第二早加載的 DLL)
    ↓
Ezorsia-v2.dll (掛鉤 dinput8.dll)
    ├─ 記憶體修改 → 解除包裝限制
    ├─ 記憶體修改 → 啟用 HD 解析度
    ├─ 渲染修改 → 縮放 UI 組件
    └─ 封包鉤子 → 轉發 localhost
```

### 完整架設步驟（2026-03 驗證版）

**前置需求**：
- v83 MapleStory 客戶端（Nexon 安裝版，**不需要刪除任何檔案**）
- Hendi's 4G Client 或 Cosmic 伺服器
- Mega 下載的管理員權限移除 + 4G Flag 啟用版客戶端

**步驟一：4G Flag（LAA）修改（可選，自行驗證用）**
```
工具：HxD Hex Editor
客戶端：預設安裝的 MapleStory.exe

偏移地址：0x0000013E
原始值：0F (預設)
修改為：2F (啟用 Large Address Space Aware)

這會讓客戶端使用 4GB 記憶體而非 2GB
```

**步驟二：管理員權限移除**
```
偏移位置：asInvoker 設置區域
使用 Cosmic 提供的 hex 编辑工具

完成後需將 exe 重新命名為 "MapleStory2" 或其他名稱
（傳聞：預設名稱會繞過管理員權限移除）
```

**步驟三：安裝 Ezorsia-v2 DLL**
```
1. 從 releases 下載編譯好的 dinput8.dll：
   https://github.com/444Ro666/MapleEzorsia-v2/releases

2. 將 dinput8.dll 放入遊戲目錄（與 MapleStory.exe 同目錄）

3. 執行遊戲
```

**步驟四：配置 config.ini（自動生成）**
```

WindowMode=true

[Network]

ServerPort=8484

[Custom]

LoadExternalDLL2=
LoadExternalDLL3=
EzorsiaV2WzIncluded=true
CustomLoginFrame=false
```

## 🆕 MaplePacketPuller — IDA → 伺服器 Encode 轉換工具 (新發現)

**GitHub**: https://github.com/Bratah123/MaplePacketPuller

### 這是什麼？

將 IDA Pro 產生的 C 偽代碼轉換為**伺服器端 encode 格式**的 Python 工具。

### 工作流程

```
IDA Pro (反編譯 client)
    ↓ 產出偽代碼
C pseudocode (.txt)
    ↓
MaplePacketPuller (Python 分析)
    ↓ 產出結構
FuncOutput/*.txt (封包結構)
    ↓
伺服器端 encode 實現
```

### 使用步驟

```bash
# 1. IDA Pro 反編譯 CClientSocket::ProcessPacket
# 2. 複製偽代碼到 Functions/ 目錄

# 3. 執行分析器
cd MaplePacketPuller/IDA\ Maple\ Script/src/main/python
python main.py

# 4. 輸入分析模式：
#    - GET_ALL_DECODES = True (高準確度)
#    - GET_ALL_DECODES = False (預設，美觀)

# 5. 輸入要分析的功能名稱（如 "ProcessPacket"）
```

### 功能特色

| 功能 | 說明 |
|------|------|
| 關鍵詞搜索 | 從 IDA 偽代碼格式化為伺服器友好格式 |
| Switch Case 支援 | 可搜尋 InHeader opcodes |
| 多模式切換 | 準確度 vs 美觀度的取捨 |
| 批量輸出 | 所有封包結構輸出到 FuncOutput/ |

### 限制

- while() 迴圈處理不完善
- 不顯示 switch case（需用戶自行處理）
- Python 3.6+ 限定（使用 os.scandir()）

### SpiritIDAPlugin 配合使用

建議搭配 [SpiritIDAPlugin](https://github.com/Bratah123/SpiritIDAPlugin) 獲得更好的 IDA 體驗。

## 🆕 封包結構：Little Endian + Opcode 實戰細節

### 位元組序：小端序（Little Endian）

> "The packets in MapleStory are in Little Endian Format."
> — botnored, MapleStory Classes v83

**範例**：
```
Opcode: 0x0053 (83)
封包中呈現：53 00 (低位元組在前)

字串 "Hello" (ASCII):
封包中：48 65 6C 6C 6F
```

### v83 Opcode 結構

| 項目 | 說明 |
|------|------|
| 標頭大小 | 2 bytes (short/ushort) |
| 位元組序 | Little Endian |
| 版本號 | 0x0083 (83) |
| 加密 | AES-like ( MapleCrypto / Shanda Crypto) |

### CClientSocket 封包路由

所有客戶端接收封包都經過這個路徑：

```
CClientSocket::ProcessPacket
    ↓ (讀取前 2 bytes = Opcode)
    ↓
CLogin::OnPacket   (if 登入相關)
CField::OnPacket   (if 遊戲內相關)
    ↓
switch(opcode) → case handler
    ↓
Decode1/Decode2/Decode4/DecodeStr
```

### IDA 分析封包結構步驟

```
1. 找到 CClientSocket::ProcessPacket（匯流排函數）

2. 找到 switch(opcode) 表
   → 每個 case 對應一個 opcode handler

3. 在 handler 函數中分析：
   Decode1()  → mplew.write(1)  → 1 byte
   Decode2()  → mplew.writeShort(1) → 2 bytes
   Decode4()  → mplew.writeInt(1)  → 4 bytes
   Decode8()  → mplew.writeLong(1) → 8 bytes
   DecodeStr() → mplew.writeMapleAsciiString() → 可變

4. 對照開源專案的 send/recv opcode 表
```

### 重要 Opcode 速查（v83）

| Opcode (Hex) | 名稱 | 方向 | 說明 |
|-------------|------|------|------|
| 0x00 | LOGIN_STATUS | S→C | 登入狀態 |
| 0x01 | LOGIN_PASSWORD | C→S | 密碼（已加密） |
| 0x02 | ACCEPT_LOGIN | S→C | 接受登入 |
| 0x03 | CHARLIST | S→C | 角色列表 |
| 0x06 | SERVERLIST_REQUEST | C→S | 請求伺服器列表 |
| 0x07 | CHARLIST_REQUEST | C→S | 請求角色列表 |
| 0x08 | CHAR_SELECT | C→S | 選擇角色 |
| 0x0B | PLAYER_DROP | S→C | 玩家進入世界 |
| 0x14 | MOVE_PLAYER | C→S | 移動 |
| 0x37 | CHANGE_CHANNEL | C→S | 換頻道 |
| 0x4C | SPAWN_NPC | S→C | NPC 出現 |

### Hello Packet 格式（v83）

```
偏移  内容           說明
0x00  0E 00        封包長度 (LE)
0x02  23 00        版本號 0x0023 = 35?  (或 83 根據伺服器)
0x04  01 00        類型
0x06  31 46 72 7A...  驗證字串 (16 bytes)
```

## 🆕 NPC 腳本：動態商店與 Quest NPC 實戰

### 動態 NPC 商店腳本（v83, RaGEZONE 2016, 仍有效）

**來源**: https://forum.ragezone.com/threads/dynamic-npc-shop-script-v83.1089979/

**核心概念**：
傳統 SQL-based商店需要重啟伺服器。動態 NPC 商店腳本讓你**即時更新商品**而不需要重啟。

**JavaScript 實現**：
```javascript
var status = 0;
var shopItems = [
    { id: 4001123, price: 1000000, quantity: 1 },
    { id: 4001124, price: 500000, quantity: 5 },
    { id: 2070006, price: 10000, quantity: 999 }
];

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
        return;
    }
    if (mode == 1) status++;
    
    if (status == 0) {
        var selStr = "#f[動態商店]#k\\n\\n請選擇商品：\\n";
        for (var i = 0; i < shopItems.length; i++) {
            var item = shopItems[i];
            selStr += "#L" + i + "##i" + item.id + "# - " + item.price + " 楓幣 x" + item.quantity + "#l\\n";
        }
        cm.sendSimple(selStr);
    } else if (status == 1) {
        var selected = shopItems[selection];
        if (cm.getMeso() >= selected.price) {
            cm.gainMeso(-selected.price);
            cm.gainItem(selected.id, selected.quantity);
            cm.sendOk("購買成功！");
        } else {
            cm.sendOk("楓幣不足！");
        }
        cm.dispose();
    }
}
```

**與 SQL 商店的比較**：

| 特性 | SQL 商店 | 動態 NPC 腳本 |
|------|---------|--------------|
| 修改商品 | 需要 SQL + 重啟 | 直接修改腳本 |
| 即時更新 | ❌ | ✅ |
| 複雜邏輯 | 困難 | 靈活 |
| 效能 | 較好 | 可忽略 |

### Quest NPC 腳本要點

**如何讓 Quest NPC 出現在 "Other" 選項**：
- 需要在 WZ 中設定 NPC 的 `quest` 屬性
- 腳本文件名需對應 NPC ID
- 使用 `cm.hasQuest(questID)` 檢查任務狀態

**基本 Quest NPC 結構**：
```javascript
var status = 0;
var questId = 1000; // 任務 ID

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
        return;
    }
    
    if (cm.hasQuest(questId)) {
        // 已接過任務
        cm.sendNext("感謝你完成了任務！");
    } else if (!cm.hasQuest(questId) && !cm.getQuest(questId).completed()) {
        // 可接新任務
        cm.sendYesNo("想要接受這個任務嗎？");
    }
    
    if (mode == 1) status++;
    else status--;
    
    if (status == 1 && !cm.hasQuest(questId)) {
        cm.startQuest(questId);
        cm.dispose();
    } else {
        cm.dispose();
    }
}
```

| 標籤 | 效果 | 範例 |
|------|------|------|
| `#b` | 藍色文字 | #b選擇裝備#k |
| `#c[itemid]#` | 顯示背包中該物品數量 | #c4001123# |
| `#d` | 紫色文字 | #d屬性加成#k |

## 🆕 登入器修改：完整的 IP 替換流程

### 三種常見 v83 登入器類型

| 類型 | 特徵 | IP 修改難度 |
|------|------|-----------|
| localhost.exe | 預設單機 exe | 簡單（HxD 直接替換） |
| Themida 殼版 | 加殼保護 | 複雜（需先脫殼） |
| HD DLL 版 | dinput8.dll鉤子 | 中等（Ezorsia V2） |

### HxD Hex 編輯 localhost.exe 完整步驟

**工具下載**：https://mh-nexus.de/en/hxd/

```
步驟 1: 備份原始檔案
        → 複製一份 localhost.exe 為 backup.exe

步驟 2: 用 HxD 開啟 localhost.exe

步驟 3: 搜尋 "127.0.0.1"
        → Ctrl+F → 文字編輯 → 輸入 "127.0.0.1"

步驟 4: 替換 IP
        → 目標 IP（如 "192.168.1.100"）
        → 注意：長度需一致！否則可能破壞結構
        
步驟 5: 儲存
        → Ctrl+S → 確認
```

### IP 長度對照表

| IP | 長度 | v83 儲存空間 |
|----|------|------------|
| 127.0.0.1 | 9 字元 | 9 bytes |
| 192.168.1.1 | 12 字元 | 12 bytes |
| 10.0.0.1 | 9 字元 | 9 bytes |
| WAN IP (如 1.2.3.4) | 9 字元 | 9 bytes |

### 使用 Wireshark 驗證修改

```
篩選條件：
tcp.port == 8484 || tcp.port == 7575

觀察：
1. TCP 三向交握是否成功
2. Hello Packet 版本號是否正確（0x0053）
3. 是否有 Error Packet（版本不符等）
```

### Themida 脫殼後的 Hex 編輯

```
流程：
1. 找 CClientSocket::SetHostname 或類似函數
2. IDA 分析找到硬編碼 IP 的位置
3. 在偏移 0x00485C01 附近找到跳轉指令
4. 修改 NOP 或 JNE 目標

常見偏移（v83）：
0x00485C01 → 75 ? (JNE to 00485C39)
```
