### 3.2 MapleCrypto 頭部混淆公式完整版

```java
// 加密端 getPacketHeader
int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
verify = (verify << 3) | (verify >>> 5);
verify &= 0xFFFF;
return new byte[] {
    (byte) (verify & 0xFF),
    (byte) ((verify >>> 8) & 0xFF),
    (byte) (length & 0xFF),
    (byte) ((length >>> 8) & 0xFF)
};

// 解密端 retrieveOpcode
int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
```

### 5.4 動態 NPC 商店腳本 (RaGEZONE, 2016)

**來源**: https://forum.ragezone.com/threads/dynamic-npc-shop-script-v83.1089979/

不使用 SQL，直接在 .js 腳本中定義商店商品：

```javascript
var items = [
    [4001126, 1000, 1],  // itemId, price, quantity
    [4001127, 2000, 5],
    [4001128, 3000, 10]
];

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == 1) status++;
    else {
        cm.dispose();
        return;
    }

    if (status == 0) {
        var shopText = "歡迎光臨我的商店！\r\n";
        for (var i = 0; i < items.length; i++) {
            shopText += "#L" + i + "##z" + items[i][0] + "# - " + items[i][1] + " 楓幣#l\r\n";
        }
        cm.sendSimple(shopText);
    } else if (status == 1) {
        var item = items[selection];
        if (cm.getMesos() >= item[1]) {
            cm.gainMesos(-item[1]);
            cm.gainItem(item[0], item[2]);
            cm.sendOk("購買成功！");
        } else {
            cm.sendOk("楓幣不足！");
        }
        cm.dispose();
    }
}
```

## 8. v83 LAN 跨 PC 連線疑難排解 (RaGEZONE, 2026-03)

**來源**: https://forum.ragezone.com/threads/need-help-v83-lan-server-works-on-pc1-but-pc2-gets-error-2147467259.1262187/

### 錯誤碼分析
| 錯誤碼 | Hex 值 | 意義 | 根本原因 |
|--------|--------|------|---------|
| `-2147467259` | `0x80004005` | Access Denied | **防火牆/權限問題**（非網路問題）|

### 排查流程
```
PC2 連線失敗
    ↓
確認錯誤碼是否為 0x80004005
    ↓ 是
    ↓
確認兩台 PC 在同一 LAN 網段 ✓
    ↓
確認伺服器 listen 0.0.0.0（而非 127.0.0.1）✓
    ↓
檢查 Windows 防火牆
    ↓
    開啟 控制台 → Windows Defender 防火牆 → 允許應用
    ↓
    確認 java.exe 和客戶端 exe 都能通過 ✓
```

### 防火牆快速設定
```cmd
# 開啟 MapleStory 必要連接埠
netsh advfirewall firewall add rule name="MapleStory Login" dir=in action=allow protocol=tcp localport=7575
netsh advfirewall firewall add rule name="MapleStory Game" dir=in action=allow protocol=tcp localport=8484
```

### Wireshark 排查優先順序
```
1. 確認 Hello Packet 是否交換
   → 看不到 → 防火牆阻止或 listen address 問題
   
2. 確認錯誤時機
   → 連線後立刻斷：版本不匹配
   → 登入後斷線：地圖 ID 或 WZ 版本問題
   
3. 確認 Hello Packet 明文
   → 版本號 = 0x0053 (83)？
```

## 9. 完整架設流程終極檢查清單 (2026-03)

### Step 1: 環境準備
```

□ MySQL 5.5+
□ HxD Hex Editor
□ HaRepacker-resurrected
□ IDA Pro 7.0+ (選用)
□ Cheat Engine (選用)
```

### Step 2: 客戶端準備
```
□ 下載 v83 localhost 客戶端（Kimmy's archive）
□ 刪除 HShield/ASP 相關檔案
□ 確認 localhost.exe 在執行
□ 確認埠口 7575 和 8484 未被占用
□ 測試客戶端能否正常啟動
```

### Step 3: 伺服器架設
```
□ Clone Cosmic 或 HeavenMS 源碼
□ Import SQL 資料庫（db_database.sql + db_drops.sql）
□ 修改 configuration.ini 或 ServerConstants.java 的 IP
□ 修改 MapleAESOFB 中的版本 key（如需要）
□ Clean and Build
□ 執行 launch.bat 或 IDE 啟動
```

### Step 4: 連線測試
```
□ 客戶端啟動 → localhost.exe 在執行
□ 嘗試登入 → 查看伺服器日誌
□ 成功進入 → 確認 WZ 版本匹配
□ 若馬上斷線 → 檢查 AES key / IV
□ 若無窮登入 → 確認 accounts 資料表有帳號
```

### Step 5: WZ 編輯（如需要）
```
□ HaRepacker 4.2.4 開啟 GMS (old) WZ

□ 儲存 WZ
□ 導出為 XML
□ 覆蓋到伺服器的 wz.xml 目錄
□ 重啟伺服器
```

### Step 6: NPC 腳本（如需要）
```
□ 在 source/scripts/npc/ 建立 .js 檔案
□ 確認 NPC ID 與 NPC.wz 中的一致
□ 使用 cm. 函數，避免 importPackage
□ 重啟伺服器或使用 @reload script
□ 遊戲內 @npc [npcId] 測試
```
