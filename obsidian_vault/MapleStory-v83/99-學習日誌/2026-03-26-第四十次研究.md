## 三、Opcode 結構 最新發現

### 3.1 MapleCrypto + Shanda + AES-OFB 加密流程完整解析

**來源**: https://pkg.go.dev/github.com/defaultmagi/maplelib

#### 雙金鑰機制核心原理
> MapleStory uses two keys: one is used to encrypt sent packets and the other is used to decrypt received packets.

| 金鑰 | 用途 |
|------|------|
| **客戶端 SendIV** | 加密要發送給伺服器的封包 |
| **客戶端 RecvIV** | 解密從伺服器收到的封包 |
| **伺服器 SendIV** | 加密要發送給客戶端的封包 |
| **伺服器 RecvIV** | 解密從客戶端收到的封包 |

#### 完整加密流程 (C→S)
```
明文資料
  → Shanda 對稱加密
  → AES-OFB 加密 (使用版本 Key + SendIV)
  → MapleCrypto 頭部混淆 (4-byte header)
  → 網路傳輸
```

#### MapleCrypto 頭部混淆公式
```java
// 加密端 (getPacketHeader)
int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
verify = (verify << 3) | (verify >>> 5);  // rotate left 3 bits
verify &= 0xFFFF;
return new byte[] {
    (byte) (verify & 0xFF),
    (byte) ((verify >>> 8) & 0xFF),
    (byte) (length & 0xFF),
    (byte) ((length >>> 8) & 0xFF)
};

// 解密端 (retrieveOpcode)
int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
```

#### MapleCrypto 頭部長度公式
```java
// 加密端
lengthEncrypted = length ^ ((iv >> 16) & 0xFFFF);

// 解密端
length = lengthEncrypted ^ ((iv >> 16) & 0xFFFF);
```

### 3.3 v83 LAN 連線問題 Opcode 視角排查

**RaGEZONE**: https://forum.ragezone.com/threads/need-help-v83-lan-server-works-on-pc1-but-pc2-gets-error-2147467259.1262187/

#### 從 Opcode 角度看連線失敗
```
正常連線流程（Opcode 視角）：
1. C→S: 0x01 (LOGIN_PASSWORD) - 加密封包
2. S→C: 0x00 (LOGIN_STATUS) - 登入結果

連線失敗時：
→ 若完全看不到任何封包 → 防火牆/網路問題
→ 若 Hello Packet 正常但登入封包異常 → 版本/IV 不匹配
→ 若 Hello Packet 就失敗 → 伺服器未啟動或 listen 問題
```

#### 排查優先順序
```
1. 確認 Hello Packet 是否交換（Wireshark）
   tcp.port == 7575
   → 看不到 → 防火牆或 listen address = 127.0.0.1
   
2. 確認版本匹配
   Hello Packet 中的版本號 = 0x0053 (83)
   
3. 確認 IV 同步
   客戶端和伺服器的 SendIV/RecvIV 一致
```

### 5.7 NPC 腳本除錯：常見崩潰原因與解決方案

**RaGEZONE**: https://forum.ragezone.com/threads/need-help.1262639/

#### 問題：NPC 對話後客戶端崩潰

**可能原因**：
1. `cm.dispose()` 在狀態機中途被呼叫
2. `cm.sendNext()` 的文字包含無效的格式標籤
3. 選項 (`#L0#...#l`) 數量與 `selection` 處理不匹配
4. 嘗試讀取不存在的玩家屬性

#### 解決方案

```javascript
// 檢查項 1：確保 dispose() 只在對話結束時呼叫
if (mode == -1 || (mode == 0 && status == lastStatus)) {
    cm.dispose();
    return;
}

// 檢查項 2：確保格式標籤正確配對
// #b 藍色  → 需 #k 重置
// #r 紅色  → 需 #k 重置
// #e 粗體  → 需 #n 重置

// 檢查項 3：選項數量與處理匹配
// 若有 3 個選項，selection 範圍應為 0~2
if (selection < 0 || selection >= options.length) {
    cm.dispose();
    return;
}
```
