### 多人連線 IP 修改完整流程

**來源**：https://forum.ragezone.com/threads/video-guide-how-to-make-a-maplestory-v83-private-server-–-full-setup-tutorial.1243566/

**Hex 編輯 localhost.exe 三步驟**：
```
Step 1: 使用 HxD 開啟 localhost.exe
Step 2: Ctrl+F → 選擇 "Text-string" → 搜尋 "127.0.0.1"
        → 會找到三個連續出現的 "127.0.0.1"
Step 3: 將三處 "127.0.0.1" 都替換為目標 IP
        → 存檔
```

**⚠️ IP 長度一致性原則**：
- 原 IP `127.0.0.1` = 9 bytes
- 替換後長度必須一致，否則用 `\x00` 填充
- 或使用 STREDIT 自動處理長度問題

## 🆕 學習重點三：Opcode 結構

### MapleCrypto Header 加密/解密公式

**加密端（發送）**:
```java
public byte[] getPacketHeader(int opcode, int length, int iv) {
    int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
    verify = (verify << 3) | (verify >>> 5);
    verify &= 0xFFFF;
    return new byte[] {
        (byte) (verify & 0xFF),
        (byte) ((verify >>> 8) & 0xFF),
        (byte) (length & 0xFF),
        (byte) ((length >>> 8) & 0xFF)
    };
}
```

**解密端（接收）**:
```java
public int retrieveOpcode(byte[] header, int iv) {
    int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
    verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
    return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
}
```

### v83 封包加密三層架構

```
┌─────────────────────────────────────────────────────┐
│ Layer 1: MapleCrypto (頭部混淆)                     │
│   - 4 bytes header (混淆 opcode + length)          │
│   - 防止靜態分析 opcode                             │
│                                                        │
│ Layer 2: Shanda Encryption (對稱加密)                │
│   - 早期版本使用                                     │
│   - 已於 v118+ 移除                                 │
│                                                        │
│ Layer 3: AES-OFB (主加密)                           │
│   - v83 主要加密方式                                │
│   - 256-bit key, 4-byte IV                          │
│   - 版本決定的靜態金鑰                              │
└─────────────────────────────────────────────────────┘
```


### HaRepacker 導出 WZ 為 XML

**來源**: https://forum.ragezone.com/threads/community-wz-img-repack-for-v83.1221835/

**步驟**:
```
1. 用 HaRepacker 打開 WZ 檔案
2. 選擇要導出的節點
3. 點擊 Export → 選擇 XML (Private Server) 格式
4. 導出後可使用文字編輯器修改
5. 重新導入 WZ
```

**注意**:
- v83 WZ 加密格式需選擇 `GMS (old)`
- 不要直接刪除/重新導入節點，可能造成數據丟失
- 建議使用 HaRepacker UI 直接編輯數值

### NX 格式 - WZ 的高效替代方案 (RaGEZONE, 2026-02)

**來源**：https://forum.ragezone.com/threads/wz-and-nx-file-questions.1260605/

**背景**：
- NX 格式是 WZ 的二進位替代方案
- 比 WZ/xml 性能更好（載入速度更快、檔案更小）
- 可從 WZ 轉換為 NX
- v83 主要仍使用 WZ 格式，NX 為較新方案

## 🆕 學習重點五：NPC 腳本編寫

### 狀態機核心概念

```javascript
status = -1 → 初始調用 start()
    ↓
status = 0 → 第一次 sendNext/sendSimple
    ↓
status = 1 → 玩家回應後
    ↓
... 依此類推
```

### Mode 參數含義

| Mode 值 | 意義 |
|---------|------|
| 1 | 點擊"下一頁"或選項 |
| 0 | 點擊"返回" |
| -1 | 關閉對話框 (ESC/X) |

### NPC 對話框類型

| Type 值 | 對話框類型 | 使用方法 |
|---------|-----------|---------|
| 0 | 標準對話 | `sendNext`, `sendOk`, `sendPrev` |
| 1 | Yes/No | `sendYesNo` |
| 2 | 選項選單 | `sendSimple` |
| 3 | 數字輸入 | `sendGetNumber` |
| 12 | 接受/拒絕 | `sendAcceptDecline` |

### NPC 對話格式標籤

| 標籤 | 功能 |
|------|------|
| `#b` | 藍色文字 |
| `#r` | 紅色文字 |
| `#g` | 綠色文字 |
| `#l` | 選項標記 |

### 快速傳送 NPC 腳本源

```javascript
var status = 0;

function start() {
    cm.sendYesNo("嘿，你想傳送到 Ludibrium 嗎？");
}

function action(mode, type, selection) {
    status++;
    if (mode != 1) {
        if(mode == 0 && type != 1)
            status -= 2;
        else if(type == 1 || (mode == -1 && type != 1)) {
            if(mode == 0) {
                cm.sendOk("嗯... 看來你還有事要在這裡處理？");
                cm.dispose();
                return;
            }
        }
    }

    if (status == 1) {
        if (cm.haveItem(4310000))
            cm.sendNext("#b檢測到快速傳送票#k。\r\n看來你有這張票！");
        else
            cm.sendNext("當然可以！我帶你去 Ludibrium。");
    } else if (status == 2) {
        cm.warp(220000000);
        cm.dispose();
    }
}
```
