## 📌 MapleSolaxiaV2 完整架設教學 (GitHub)

### 重要資源
- **GitHub**: https://github.com/skhzhang/MapleSolaxiaV2
- **客戶端工具下載**: https://drive.google.com/drive/folders/0BzDsHSr-0V4MYVJ0TWIxd05hYUk

### 環境需求
- JDK/JRE 7 (必須使用 JDK 7，NetBeans 8.0.2)
- WampServer 2.0i
- MySQL Workbench
- Hamachi (可選，用於網路對戰)

### 客戶端修改重點

1. **移除官方保護**：刪除 `HShield` 資料夾、`ASPLauncher.exe`、`MapleStory.exe`、`patcher.exe`

2. **使用 localhost.exe**：用 `localhost.exe` 替換原本的啟動程式

3. **Hex 編輯 IP**：用 Neo Hex Editor 開啟 `localhost.exe`，搜尋 `127.0.0.1` 字串並替換為目標 IP

4. **WZ 檔案置換**：用提供的 client_wz 資料夾覆蓋原本的 WZ 檔案

### ⚠️ 重要警告
> DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate some kind of bugs afterwards.

### WZ 編輯正確流程
```
1. 用 HaRepacker 打開 WZ (encryption: GMS (old))
2. 使用 UI 直接修改客戶端 WZ
3. 儲存客戶端 WZ
4. 從伺服器端 WZ.xml 匯出時使用 "Private Server..." 匯出選項
```

## 📌 WZ 編輯詳細步驟

### 工具
- **HaRepacker 4.2.4** 或 **HaRepacker-resurrected** (推薦)

### 步驟
1. 以系統管理員執行 HaRepacker
2. 選擇加密格式：`GMS (T) (v56+)`
3. File → Open → 選擇 WZ 檔案
4. 展開樹狀結構找到要編輯的內容
5. 雙擊進行修改
6. **使用 UI 直接編輯**，不要刪除/重新匯入節點
7. Save

### 可編輯的 WZ 檔案

| WZ 檔案 | 內容 |
|---------|------|
| Character.wz | 角色動作、裝備圖片 |
| Effect.wz | 傷害數字、墓碑、升級特效 |
| Item.wz | 所有物品圖示 |
| Mob.wz | 所有怪物資料 |
| Npc.wz | 所有 NPC 資料 |
| Skill.wz | 所有職業技能 |
| Sound.wz | 所有音效和音樂 |

### 客戶端 vs 伺服器 WZ
- **客戶端 WZ**：用 HaRepacker 直接修改，用於顯示
- **伺服器 WZ.xml**：用於遊戲邏輯運算
- **兩者必須分開處理**，不可混用

## 📌 Opcode 結構分析

### Opcode 基本特性
- **長度**：v83 使用 **2 bytes** (little endian)
- **封包頭部**：4 bytes MapleCrypto 混淆
- **Opcode 表**：幾乎全部被映射，高度文檔化

### 封包加密機制
```
客戶端發送：
原始封包 → Shanda 加密 → AES 加密 → 傳送

伺服器接收：
接收 → AES 解密 → Shanda 解密 → 處理
```

### 第一個封包（明文）
- MapleStory 連線時，**第一個封包是明文**（未加密）
- 之後所有封包都經過 Shanda + AES 雙重加密

### IDA 分析技巧
- 所有客戶端接收封包都經過 `CClientSocket::ProcessPacket`
- 然後分發到 `CField::OnPacket`、`CLogin::OnPacket` 等
- 找 opcode 表：`push opcode` 或 StringPool 引用

### 常用分析工具
| 工具 | 用途 |
|------|------|
| Wireshark | 通用網路封包分析 |
| Maple_Pshark | v83 專用封包攔截 |
| MaplePacketPuller | Python 封包結構分析 |

## 📌 NPC 腳本編寫

### 基本結構
```javascript
var status = 0;
var selected = 1;
var wui = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && status == 2) {
            cm.sendNext("...");
        }
    }
}
```

### 狀態機制
- **status = -1**：初始化，進入第一個對話框
- **mode = 1**：下一頁/確認
- **mode = 0**：返回/取消
- **type**：對話框類型
- **selection**：玩家選擇的項目索引

### 常用函數
| 函數 | 說明 |
|------|------|
| `cm.sendNext(text)` | 顯示下一頁對話 |
| `cm.sendSimple(text)` | 顯示選項對話 |
| `cm.sendOk(text)` | 顯示確定對話 |
| `cm.dispose()` | 關閉對話框 |
| `cm.haveItem(itemId, quantity)` | 檢查物品數量 |
| `cm.gainItem(itemId, quantity)` | 給予/移除物品 |
| `cm.getPlayer()` | 獲取玩家物件 |
| `cm.getText()` | 獲取玩家輸入文字 |

### 兩種 NPC 類型
1. **status 類 NPC**：處理狀態轉換
2. **非 status 類 NPC**：簡單的單次對話
