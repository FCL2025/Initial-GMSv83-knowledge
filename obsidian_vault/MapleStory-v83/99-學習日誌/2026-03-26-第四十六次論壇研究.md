    if (mode == -1) {
        cm.dispose();
        return;

        cm.dispose();
    }
}

## 🆕 v83 LAN 多人遊戲 - 錯誤排查 (RaGEZONE, 2026-03)

**來源**: https://forum.ragezone.com/threads/need-help-v83-lan-server-works-on-pc1-but-pc2-gets-error-2147467259.1262187/

### 錯誤碼分析
| 錯誤碼 | Hex 值 | 意義 | 根本原因 |
|--------|--------|------|---------|
| `-2147467259` | `0x80004005` | Access Denied | 防火牆/權限問題 |

### Wireshark 排查優先順序
```
1. 確認 Hello Packet 是否交換
   ↓ 能看到 Hello → 確認 AES IV 同步
   ↓ 看不到 Hello → 防火牆阻止或 listen address 問題
   
2. 確認錯誤時機
   → 連線後立刻斷：伺服器或客戶端版本不匹配
   → 登入後斷線：地圖 ID 或 WZ 版本問題
   
3. Wireshark 確認
   tcp.port == 7575 || tcp.port == 8484
   → 看不到 SYN-ACK → 防火牆/網路問題
```

## 🆕 MapleRef Wiki Send/Recv Opcode 核心概念 (2026)

**來源**: https://mapleref.fandom.com/wiki/Operation_Code

### Send vs Receive Opcode 區分

> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

**重要**：同一個 Opcode 數字可能代表不同含義，視方向而定！
