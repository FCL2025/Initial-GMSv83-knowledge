### 1. MapleEzorsia v2 HD DLL - 記憶體注入式客戶端修改

**來源**：https://github.com/izarooni/MapleEzorsia

這是一個革命性的 v83 客戶端修改方案，與傳統 Hex 編輯不同：

**核心技術**：
- 使用 **Detours** (Microsoft 的 API 攔截庫) 進行 DLL 注入
- 修改遊戲記憶體，而非直接修改 .exe
- 不需要脫殼或修改原始二進位檔

**功能特色**：
| 功能 | 說明 |
|------|------|
| 自訂解析度 | 任意調整遊戲視窗和畫布解析度 |
| 視窗置中 | 啟動時自動將遊戲視窗置中 |
| 聊天紀錄擴展 | 從 64 行擴展到 127 行 |
| 去除登入書籍邊框 | 高解析度下自動隱藏登入畫面書本邊框 |
| Fear 效果修復 | 修復 LimitedView fieldType 在任何解析度下的恐懼效果 |
| 最小化按鈕 | 啟用原本被隱藏的最小化按鈕 |
| 跳過開機動畫 | 節省啟動時間 |
| Discord RPC | 顯示 Discord 遊戲狀態 |

**安裝方式**：
```
1. 在 v83 MapleStory 目錄中
2. 將 nmconew.dll 改名為 nmconew2.dll
3. 將 MapleEzorsia.dll (或 config.ini) 放入目錄
4. 執行客戶端即可
```

**config.ini 範例**：
```ini
[general]
width=1280
height=720
discord=false
debug=false
```

**編譯要求**：只能用 Release x86 配置編譯

### 2. MaplePacketPuller + SpiritIDAPlugin 工作流程更新 (2026)

**來源**：https://github.com/Bratah123/MaplePacketPuller + https://github.com/Bratah123/SpiritIDAPlugin

**支援的 IDA 版本**：
| IDA 版本 | 支援狀態 |
|----------|---------|
| IDA Pro 7.0 (32-bit) | ✅ |
| IDA Pro 7.5 | ✅ |
| IDA Pro 6.8 | ✅ |
| Python 2.7 | ❌ 不支援 |

**工作流程**：
```
Step 1: IDA Pro 反編譯
  → 找到 CField::OnPacket 或 CLogin::OnPacket
  → 產生 pseudocode (.p64 或 .txt)

Step 2: 複製 pseudocode
  → 在 Functions/ 目錄建立 .txt 檔案
  → 貼上 IDA 產生的 C-style pseudocode

Step 3: Python 分析
  → cd MaplePacketPuller/IDA\ Maple\ Script/src/main/python
  → python main.py
  → 輸入分析類型（完整/標準）
  → 輸入要分析的檔案名稱

Step 4: 輸出結果
  → 封包結構自動寫入 FuncOutput/ 目錄
  → Decode1/Decode2/Decode4/DecodeStr 等欄位自動識別
```

### 4. Cosmic 模擬器最新動態 (2026-03)

**來源**：https://github.com/P0nk/Cosmic

Cosmic 是目前最活躍的 v83 模擬器項目：

**技術棧**：
- Java 21（推薦 Amazon Corretto）
- MySQL 8+ 資料庫
- Maven + Maven Wrapper 構建
- Docker 支持（docker compose up）

**特點**：
- 基於 HeavenMS，代碼持續維護
- 減少技術債務，模組化設計
- GitHub Actions 自動構建 + 測試
- Semantic Versioning (semver) 版本管理

**客戶端分開維護**：
- 伺服器：https://github.com/P0nk/Cosmic
- 客戶端：https://github.com/P0nk/Cosmic-client

**安裝摘要**：
```
1. 安裝 MySQL 8+，建立資料庫 cosmic
2. Clone Cosmic → IntelliJ IDEA
3. 修改 config.yaml 中的 DB_PASS
4. 執行 net.server.Server.main()
5. 出現 "Cosmic is now online" 表示成功
```

### 6. v83 封包加密三層架構總結

```
┌─────────────────────────────────────────────────────┐
│ Layer 1: MapleCrypto (頭部混淆)                     │
│   - 4 bytes header (混淆 opcode + length)           │
│   - verify = (opcode ^ iv[0]) + (iv[1] << 8)       │
│   - verify = (verify << 3) | (verify >> 5)         │
│   - 防止靜態分析 opcode                             │
│                                                        │
│ Layer 2: Shanda Encryption (對稱加密)                │
│   - 早期版本使用                                     │
│   - 已於 v118+ 移除                                 │
│                                                        │
│ Layer 3: AES-OFB (主加密)                           │
│   - v83 主要加密方式                                │
│   - 256-bit key, 4-byte IV                          │
│   - 版本決定的靜態金鑰                              │
└─────────────────────────────────────────────────────┘
```

### 7. NPC腳本相容性最新討論 (RaGEZONE 2026-03)

**來源**：https://forum.ragezone.com/threads/kinoko-npc-scripts.1261538/

**問題**：不同音源（Cosmic、Kinoko、HeavenMS）的腳本解析器存在差異

**最佳實踐**：
1. ✅ 使用普遍支援的 `cm.` 函數
2. ❌ 避免使用特定音源專有的 Package 路徑
3. ❌ 避免 `importPackage`（JDK 7+ 已廢棄）
4. ✅ 先在本地測試再部署到正式伺服器
