## 一、登入器/客戶端修改

### 1.1 Hex 編輯 IP 修改流程

**目標**：將客戶端 localhost.exe 的連線目標從 `127.0.0.1` 改為實際伺服器 IP

**步驟**：
1. 使用 HxD 或 Neo Hex Editor 開啟 `localhost.exe`
2. `Ctrl+F` 搜尋 `127.0.0.1`
3. 會找到**三個** IP 位址位置
4. 全部修改為目標 IP（確保長度一致）
5. 儲存

**注意**：
- Windows 8/10 可能需要使用 `lolwut.exe` 輔助啟動
- 某些版本需要設定 Windows 7 相容模式

**替代方案 - Ezorsia v2 HD Client**：
- GitHub: https://github.com/444Ro666/MapleEzorsia-v2
- DLL 修改預設客戶端，支援 HD 解析度
- 只需放置 `dinput8.dll` 到客戶端目錄

### 1.2 v83 客戶端資源下載點

| 資源 | 來源 | 說明 |
|------|------|------|
| **MapleSolaxiaV2** | https://github.com/skhzhang/MapleSolaxiaV2 | 完整伺服器+客戶端 |
| **HeavenMS** | https://github.com/ronancpl/HeavenMS | 最完整的 v83 模擬器 |
| **Cosmic** | https://github.com/P0nk/Cosmic | 最活躍的開源 v83 模擬器 |
| **v83 HD Client Guide** | https://github.com/alohakuhanaba/v83-Client-Setup-and-Development-Guide | HD 客戶端架設指南 |

### 1.3 客戶端修改原理

v83 客戶端的網路連線邏輯：
```
localhost.exe 啟動
  → 讀取內建 IP 位址（預設 127.0.0.1）
  → 連線到 MapleStory 伺服器 (port 8484)
  → 驗證版本號 (0x0053 = 83)
  → 交換 AES IV
  → 進入遊戲
```

修改 IP 的本質就是用 HxD 找到並修改這些 IP 字串。

## 三、Opcode 結構

### 3.1 v83 Opcode 現況

**結論**：v83 的 Opcode 幾乎全部被映射，不需要額外追蹤封包

| 狀態 | 說明 |
|------|------|
| ✅ 完整映射 | 大多數 opcode 表已在開源專案中完整提供 |
| ✅ Cosmic | 最活躍的開源專案，Opcode 最完整 |
| ✅ HeavenMS | 完整但已停止維護 |
| ⚠️ 唯一需求 | 修改客戶端時需要 IDA 分析 |

### 3.2 常用 Opcode 對照

**登入相關 (C→S)**：
| Opcode | 名稱 | 說明 |
|--------|------|------|
| 0x01 | LOGIN_PASSWORD | 登入請求 |
| 0x06 | SERVERLIST_REQUEST | 請求伺服器列表 |
| 0x07 | CHARLIST_REQUEST | 請求角色列表 |
| 0x08 | CHAR_SELECT | 選擇角色 |
| 0x0B | PLAYER_DROP | 進入遊戲 |

**遊戲相關 (C→S)**：
| Opcode | 名稱 | 說明 |
|--------|------|------|
| 0x14 | MOVE_PLAYER | 移動 |
| 0x15 | CLOSE_RANGE_ATTACK | 近距離攻擊 |
| 0x16 | RANGED_ATTACK | 遠距離攻擊 |
| 0x28 | NPC_TALK | NPC 對話 |

**伺服器回應 (S→C)**：
| Opcode | 名稱 | 說明 |
|--------|------|------|
| 0x00 | LOGIN_STATUS | 登入結果 |
| 0x03 | CHARLIST | 角色列表 |
| 0x0C | SERVERLIST | 伺服器列表 |

### 3.3 登入結果碼

| 結果碼 | 意義 |
|--------|------|
| 0 | 登入成功 |
| 5 | 帳號或密碼錯誤 |
| 7 | 帳號已停用 |
| 13 | 刪除中（冷靜期） |
| 23 | 登入太頻繁（防機器人延遲） |

### 3.4 IDA 分析Opcode 流程

```
1. 找 CClientSocket::ProcessPacket - 客戶端接收封包的總入口
2. 從這裡根據 header 分發到 CField::OnPacket 或 CLogin::OnPacket
3. switch(opcode) 表就是所有處理函數的入口
```

## 四、WZ 修改

### 4.1 HaRepacker 使用流程

**用途**：編輯 WZ 文件（客戶端）和 XML（伺服器端）

**客戶端 WZ 編輯流程**：
1. HaRepacker 4.2.4 開啟 **GMS (old)** 加密的客戶端 WZ
2. 使用 UI 直接修改數值（禁止刪除/重新導入節點！）
3. 儲存 WZ

**伺服器 WZ XML 導出流程**：
1. HaRepacker 選擇 **"Private Server..."** 導出選項
2. 將導出的 XML 覆蓋到伺服器的 WZ.XML 目錄

### 4.2 WZ 編輯黃金法則

> ⚠️ DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs.

**正確流程**：
```
客戶端 WZ 修改 → 客戶端 WZ 存檔
     ↓
使用 "Private Server..." 導出
     ↓
覆蓋到伺服器的 WZ.XML 目錄
     ↓
不要反向操作（伺服器 XML → 客戶端 WZ）
```

### 4.3 常見 WZ 編輯內容

### 5.2 基本模板

```javascript
var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        if (mode == 1)
            status++;
        else
            status--;

        if (status == 0) {
            cm.sendNext("你好！");
        } else if (status == 1) {
            cm.sendSimple("選擇：\r\n#b#L0#選項1#l\r\n#L1#選項2#l");
        } else if (status == 2) {
            if (selection == 0) {
                cm.gainItem(1302000, 1);
            }
            cm.dispose();
        }
    }
}
```

### 5.6 動態 NPC 商店腳本

```javascript
var status = 0;
var items = Array(
    Array(1302000, 10000, 1),
    Array(1302001, 20000, 1),
    Array(1302002, 30000, 1)
);

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        if (mode == 1) status++;
        else status--;

        if (status == 0) {
            var msg = "歡迎光臨！\r\n請選擇商品：\r\n";
            for (var i = 0; i < items.length; i++) {
                msg += "#L" + i + "##i" + items[i][0] + "# - " + items[i][1] + " 金幣#l\r\n";
            }
            cm.sendSimple(msg);
        } else if (status == 1) {
            var item = items[selection];
            if (cm.getMeso() >= item[1]) {
                if (cm.canHold(item[0])) {
                    cm.gainMeso(-item[1]);
                    cm.gainItem(item[0], item[2]);
                    cm.sendOk("購買成功！");
                } else {
                    cm.sendOk("背包空間不足！");
                }
            } else {
                cm.sendOk("金幣不足！");
            }
            cm.dispose();
        }
    }
}
```

### 6.2 架設步驟

```
1. 安裝 JDK 8 + JCE 8
2. 安裝 MySQL Server 5.5
3. 下載 v83 localhost 客戶端
4. 架設 Java 伺服器（NetBeans / IntelliJ）
5. 執行 SQL 初始化資料庫
6. 設定 ServerConstants.java 中的 IP 和埠口
7. HxD 修改客戶端 IP（三個 127.0.0.1 全部替換）
8. 啟動客戶端和伺服器
```

### 6.3 LAN 多人連線必檢查清單

- [ ] 確認兩台 PC 在同一 LAN 網段
- [ ] 確認伺服器 listen 0.0.0.0（而非 127.0.0.1）
- [ ] 確認防火牆開啟 7575 和 8484 埠口
- [ ] 若錯誤碼 0x80004005 → 防火牆/權限問題
- [ ] 若錯誤碼 -2147467259 → 同上
