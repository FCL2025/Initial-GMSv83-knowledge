### botnored v83 封包類別結構
**來源**: https://botnored.weebly.com/blog/maplestory-classes-v83

**重要原則**：
> **第一個發送/接收的封包永遠是純文字（未加密）**
> MapleStory 封包透過 TCP Socket 傳輸

**客戶端主要類別**：
| 類別 | 功能 |
|------|------|
| `CWvsContext` | 世界狀態管理（角色資料、伺服器列表） |
| `CClientSocket` | 網路連接管理（Send/Recv） |
| `CField` | 當前地圖/遊戲場景處理 |
| `CLogin` | 登入畫面處理 |

### 識別封包類型的技巧

| 徵兆 | 封包類型 |
|------|---------|
| 長度固定 (如每次 32 bytes) | 可能是 MOVE_PLAYER (0x14) |
| 包含可讀字串 | 可能是 CHAT (聊天) |
| 第一個 byte 為 0x00 | 伺服器回應封包 |
| 封包包含 00 00 00 00 | 可能是 PING/心跳封包 |

## 3. Opcode 結構

### MapleRef Wiki Send/Recv Opcode 核心概念
**來源**: https://mapleref.fandom.com/wiki/Operation_Code

> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

### Send vs Receive Opcode 區分
| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

**重要**：同一個 Opcode 數字可能代表不同含義，視方向而定！

### v83 登入結果碼（Login Status Code）

| 結果碼 | 意義 |
|--------|------|
| 0 | 登入成功 |
| 5 | 帳號或密碼錯誤 |
| 7 | 帳號已停用 |
| 13 | 刪除中（冷靜期） |
| 23 | 登入太頻繁（防機器人延遲） |

### RustMS 加密實現亮點
```rust
// MapleAES - 負責封包加密的核心結構
```

### Quest NPC 腳本與 MigrateIn 機制

**問題**：如何讓 Quest NPC 出現在 "Other" 選項中？

**解決方案**：
- 需要發送 `PlayerMigrateInPacket` 封包
- 封包包含：NPC ID 列表 + FileTime
- 這個封包讓客戶端知道哪些 NPC 有自訂腳本

```java
sendPacket(PlayerMigrateInPacket.encode(npcIds, fileTime));
```

### 配方系統 NPC 腳本源 (動態，無需 SQL)

```javascript
/*
 * 動態配方系統 - 玩家提交材料換取成品
 * NPC ID: 9010010
 */
var status = 0;

var recipes = Array(
    {
        name: "楓之葉地圖",
        materials: Array(Array(4000000, 10), Array(4000001, 5)),
        result: Array(Array(2300000, 1)),
        mesoCost: 1000
    }
);

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        if (mode == 1) status++;
        else status--;

        if (status == 0) {
            var msg = "歡迎來到工藝系統！\r\n請選擇配方：\r\n";
            for (var i = 0; i < recipes.length; i++) {
                msg += "#L" + i + "#" + recipes[i].name + "#l\r\n";
            }
            cm.sendNext(msg);
        } else if (status == 1) {
            var recipe = recipes[selection];
            var canCraft = true;
            var msg = "你需要以下材料：\r\n";
            for (var i = 0; i < recipe.materials.length; i++) {
                var mat = recipe.materials[i];
                var have = cm.getQuantityOfItem(mat[0]);
                var need = mat[1];
                msg += "#i" + mat[0] + "# #t" + mat[0] + "#: " + have + "/" + need + "\r\n";
                if (have < need) canCraft = false;
            }
            if (cm.getMesos() < recipe.mesoCost) canCraft = false;
            msg += "加工費: " + recipe.mesoCost + " 金幣\r\n";
            msg += canCraft ? "#g可以製作#k" : "#r材料不足#k";
            cm.sendYesNo(msg);
        } else if (status == 2) {
            var recipe = recipes[selection];
            for (var i = 0; i < recipe.materials.length; i++) {
                cm.gainItem(recipe.materials[i][0], -recipe.materials[i][1]);
            }
            cm.gainMesos(-recipe.mesoCost);
            for (var i = 0; i < recipe.result.length; i++) {
                cm.gainItem(recipe.result[i][0], recipe.result[i][1]);
            }
            cm.sendOk("製作成功！");
            cm.dispose();
        }
    }
}
```

## 學習心得

### 關於 Maple_Pshark
Maple_Pshark 是目前最完整的 v83 專用封包截獲工具，其 Hook 加密函數的方式可以直接在不需要解密金鑰的情況下截獲明文封包。相比 Wireshark 的純 TCP 分析，Maple_Pshark 更適合分析 MapleStory 特有的加密封包。

### 關於 Opcode 文檔化程度
v83 的 Opcode 已經高度文檔化，幾乎不需要額外的逆向工程。開源專案如 Cosmic 和 HeavenMS 提供了完整的 Opcode 映射表，MapleRef Wiki 也收錄了詳細的封包結構說明。

### 關於 RustMS
RustMS 是一個新興的非 JVM 實現方案，雖然目前只支援 Login Server，但其清晰的代碼結構和完整的加密實現值得學習。對於想要深入理解 MapleStory 加密機制的開發者來說，RustMS 的原始碼比 Java 模擬器更容易理解。

### 關於 NPC 腳本相容性
不同音源之間的 NPC 腳本存在相容性問題，主要體現在 `importPackage` 的支援程度。建議使用 Nashorn ECMAScript 6 語法，避免使用已廢棄的 `importPackage`。
