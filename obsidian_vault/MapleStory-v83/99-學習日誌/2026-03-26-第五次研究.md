### 登入器 IP 修改原理
**核心原理**: localhost.exe 中的 IP 以**明文 ASCII** 儲存，可用 Hex 編輯器直接搜尋修改

**三處 IP 位置**：
1. 登入伺服器 IP
2. 頻道伺服器 IP
3. 商城伺服器 IP

**HxD 修改步驟**：
```
1. HxD 開啟 localhost.exe
2. Ctrl+F → 選擇「文字」→ 輸入 "127.0.0.1"
3. 找到後直接用新 IP 覆蓋（長度差異用 \x00 填補）
4. 重複直到全部3處修改完成
5. 儲存
```

> ⚠️ 必須**一字一字輸入覆蓋**，不可使用取代工具，也不可貼上！

| IDA Pro 6.8 | ✅ |
| Python 2.7 | ❌ |

### 完整工作流程
```
Step 1: IDA Pro 反編譯
  → 在 IDA 中找到 CField::OnPacket 或 CLogin::OnPacket
  → 產生 C-style pseudocode (.p64 或 .txt)

Step 2: 複製 pseudocode
  → 在 Functions/ 目錄建立 .txt 檔案
  → 貼上 IDA 產生的 pseudocode

Step 3: Python 分析
  → cd MaplePacketPuller/IDA\ Maple\ Script/src/main/python
  → python main.py
  → 輸入分析類型（完整/標準）
  → 輸入要分析的檔案名稱

Step 4: 輸出結果
  → 封包結構自動寫入 FuncOutput/ 目錄
  → 控制台顯示 Decode1/Decode2/Decode4/DecodeStr 等欄位
```

### 已知限制
- ⚠️ while() 迴圈處理不完善
- ⚠️ 建議設定 GET_ALL_DECODES = true 以提高準確度
- ⚠️ switch cases 無法自動顯示（需用戶自行處理）

## 🆕 Opcode 結構 - MapleRef Wiki 核心發現

### MapleRef Wiki - Operation Code
**URL**: https://mapleref.fandom.com/wiki/Operation_Code

### Send vs Receive Opcode 核心區分
> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

**重要**: 同一個 Opcode 數字可能代表不同含義，視方向而定！

### MapleRef Packets - Packet Header
**URL**: https://mapleref.fandom.com/wiki/Packets

> "Packet headers are 4 bytes long, and are generated with the use of the send IV, packet length (without header) and the version of MapleStory. The version is -(version + 1) when sending a packet to the client, otherwise just the plain version."

### v83 Opcode 現況總結 (RaGEZONE 2024-12)
- ✅ 幾乎全部被映射，不需要額外追蹤封包
- ✅ 大多數 Opcode 表已在開源專案（Cosmic、HeavenMS、MapleSolaxiaV2）中完整提供
- ⚠️ 唯一需要的反向工程是**修改客戶端時**

### NPC腳本源間相容性問題

**問題來源**: 不同音源（Cosmic、Kinoko、HeavenMS）的 NPC 腳本解析器版本不同

| 音源 | 解析器 | importPackage | 注意事項 |
|------|--------|-------------|---------|
| **Cosmic** | Nashorn (JDK 8+) | ❌ 已移除 | 最普遍相容 |
| **Kinoko** | 混合版本 | ⚠️ 警告 | 需注意 |
| **HeavenMS** | JDK 6 時代 | ✅ 存在 | 有 SavedLocationType 問題 |

**解決方案**：
```javascript
// ✅ 所有音源通用
var MapleCharacter = Packages.client.MapleCharacter;

// ❌ JDK 7+ 不支援
importPackage(Packages.client);
```

## 🆕 動態 NPC 商店腳本 (v83, RaGEZONE 2016)

**URL**: https://forum.ragezone.com/threads/dynamic-npc-shop-script-v83.1089979/

### 動態商店 vs SQL 商店

| 方式 | 優點 | 缺點 |
|------|------|------|
| 動態 NPC 腳本 | 即時生效、可版本控制 | 需編寫 JS 腳本 |
| SQL 商店 | 簡單直接 | 需重啟伺服器 |

### 動態 NPC 商店腳本源
```javascript
/*
 * 動態 NPC 商店 - 不需要 SQL
 * NPC ID: 9010009
 */
var status = 0;
var items = Array(
    Array(1302000, 10000, 1),
    Array(1302001, 20000, 1),
    Array(1302002, 30000, 1),
    Array(1302003, 50000, 1)
);

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        
        if (mode == 1)
            status++;
        else
            status--;
        
        if (status == 0) {
            var msg = "歡迎光臨！\r\n請選擇商品：\r\n";
            for (var i = 0; i < items.length; i++) {
                msg += "#L" + i + "##i" + items[i][0] + "# - " + items[i][1] + " 金幣#l\r\n";
            }
            cm.sendSimple(msg);
        } else if (status == 1) {
            var item = items[selection];
            if (cm.getMeso() >= item[1]) {
                if (cm.canHold(item[0])) {
                    cm.gainMeso(-item[1]);
                    cm.gainItem(item[0], item[2]);
                    cm.sendOk("購買成功！");
                } else {
                    cm.sendOk("背包空間不足！");
                }
            } else {
                cm.sendOk("金幣不足！");
            }
            cm.dispose();
        }
    }
}
```
