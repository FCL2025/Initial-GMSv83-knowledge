## 三、Opcode結構

### 3.1 IDA 分析 Opcode 進階技巧（RaGEZONE）

**關鍵函數**：
- `CClientSocket::ProcessPacket` - 客戶端接收封包的總入口
- `CLogin::OnPacket` - 所有登入相關封包處理基類
- `CField::OnPacket` - 遊戲中封包處理

**為什麼知道 Opcode 就能找到處理函數**：
```
CClientSocket::ProcessPacket 內有大 switch 表
  ↓
每個 case 對應一個 opcode
  ↓
case 內直接調用具體的 OnPacket 處理函數
```

**AOB (Array of Bytes) 搜尋**：
- IDA Hex View 搜尋已知 opcode bytes 模式
- 可找到 opcode 表的記憶體位置
- 適用於：已知某個 opcode，想找它的引用位置

### 3.2 MapleCrypto 頭部加密公式

```java
// 加密端
int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
verify = (verify << 3) | (verify >>> 5);
verify &= 0xFFFF;
return new byte[] {
    (byte) (verify & 0xFF),
    (byte) ((verify >>> 8) & 0xFF),
    (byte) (length & 0xFF),
    (byte) ((length >>> 8) & 0xFF)
};

// 解密端
int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
```
