## 🆕 localhost.exe IP 修改三處原則 (RaGEZONE 2010-2015)

### 核心原理
v83 localhost.exe 中，IP 位址以**明文 ASCII** 儲存，可用任何 Hex 編輯器直接搜尋和修改。

### 三處 IP 位址
localhost.exe 中有 **3 處** "127.0.0.1" 需要全部修改：
1. 登入伺服器 IP
2. 頻道伺服器 IP  
3. 商城伺服器 IP

### Hex 編輯步驟 (HxD / Neo Hex Editor)
```
1. 用 HxD 或 Neo Hex Editor 開啟 localhost.exe
2. Ctrl+F 搜尋 "127.0.0.1"（選擇「文字」模式）
3. 找到後，用實際 IP 覆蓋（注意長度對齊）
4. 重複步驟 2-3，直到全部 3 處都修改完成
5. 儲存
```

### 重要提醒 (RaGEZONE 2015)
> "replace the 127.0.0.1, you have to type your ip over it (do not use the replace tool, and dont paste your ip over it)"

### 與 MapleEzorsia v2 比較
| 方案 | 優點 | 缺點 |
|------|------|------|
| localhost.exe Hex | 傳統方法，廣泛支援 | 需要手動找 3 處 IP |
| MapleEzorsia dinput8.dll | 全自動，一個 DLL 全搞定 | 2023 年新方案 |

## 🆕 NPC 腳本核心函數 (HeavenMS NPCConversationManager)

### sendNext / sendPrev / sendSimple 差異
```javascript
cm.sendNext("文字");     // 下一頁按鈕
cm.sendPrev("文字");     // 上一頁按鈕
cm.sendNextPrev("文字"); // 兩者都有
cm.sendSimple("文字");   // 選擇清單（無按鈕）
cm.sendOk("文字");       // 確定按鈕
cm.sendYesNo("文字");    // 是/否按鈕
cm.sendAcceptDecline("文字"); // 接受/拒絕
```

### status 狀態機模式
```javascript
var status = 0;
function start() { status = -1; action(1, 0, 0); }
function action(mode, type, selection) {
    if (mode == 1) { status++; }
    else { status--; }
    
    if (status == 0) {
        cm.sendNext("第一頁內容");
    } else if (status == 1) {
        cm.sendSimple("選擇頁面\n#L0#選項0\n#L1#選項1");
    } else if (status == 2) {
        if (selection == 0) {
            // 處理選項0
        } else if (selection == 1) {
            // 處理選項1
        }
        cm.dispose();
    }
}
```

### NPC 腳本源相容性
不同伺服器引擎的 NPC 腳本：
- **HeavenMS**: JavaScript (Rhino engine)
- **Cosmic**: JavaScript (Rhino engine)
- **Kinoko**: 可能有差異
- **MapleSolaxiaV2**: JavaScript

常見相容性問題：
- `importPackage` 在新版本可能不存在
- `SavedLocationType` 等類別引用問題

## 🆕 Packet 結構基本概念

### Opcode 位置
> "Actions that are handled by packets on MapleStory are defined by an identifier called an Operation Code (opcode for short), which is **the first 2 bytes of the packet (a short)**"

### 封包加密流程
v83 使用**雙層加密**：
1. **Shanda 加密**: 早期遊戲保護
2. **AES 加密**: 版本保護

### 第一個封包特性
> "The first packet sent / received will always be in plain-text (not encrypted)"

### Header 結構
- v83: 4 bytes header + 2 bytes opcode
- 早期版本: 可能只有 2 bytes opcode
