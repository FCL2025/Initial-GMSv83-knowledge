### 2. 抓包分析
- **Maple_Pshark**（v83 專用封包 Logger, GitHub: obstriker/Maple_Pshark）：
  - 專為 v83 設計，Hook 加密函數，直接截獲明文
  - 支援封包修改和注入（Recv/Send 皆可）
  - 規則化封包操作和過濾

- **v83 封包三層加密架構**：
  ```
  Layer 1: MapleCrypto (頭部混淆) - 4-byte header，含 opcode + length
  Layer 2: Shanda Encryption (對稱加密) - IOBufferManipulator
  Layer 3: AES-OFB (256-bit) - 主要加密，版本決定的靜態金鑰
  ```

- **Shanda 加密關鍵發現**（RaGEZONE 2016）：
  - Nexon 內部稱之為 "IOBufferManipulator"
  - 在 AES 之前應用
  - v83 仍在使用，v118+ 已移除
  - 實作：`IOBufferManipulator.Encrypt(pBuff)`

- **v83 AES Key 參考**（CMS/Chinese MS v83）：
  ```java
  // AES UserKey (first column per row)
  0x13, 0x08, 0x06, 0xB4, 0x1B, 0x0F, 0x33, 0x52
  // IV Transformation Key: 0xC65053F2
  // 客戶端版本: 0x23 (35 decimal)
  ```

### 3. Opcode 結構
- **v83 Opcode 狀態**：
  - ✅ 幾乎全部被映射，不需要額外追蹤封包
  - ✅ 大多數 Opcode 表已在開源專案（Cosmic、HeavenMS）中完整提供
  - ⚠️ 唯一需要的反向工程是修改客戶端時

- **CClientSocket::ProcessPacket 封包分發流程**：
  ```
  網路接收 (TCP)
      ↓
  CClientSocket::ProcessPacket()
      ↓  根據 MapleCrypto header 解密
      ↓  提取 opcode (2 bytes, little endian)
      ↓
  switch(opcode)
      ↓
  CLogin::OnPacket()      // 登入畫面狀態
      或
  CField::OnPacket()      // 遊戲中狀態
  ```

### 5. NPC 腳本編寫
- **狀態機核心概念**：
  - `status` 追蹤對話進度（0=第一句, 1=第二句...）
  - `mode == 1`：下一步；`mode == 0`：返回；`mode == -1`：關閉對話框
  - `type` 區分按鈕類型

- **對話發送方法速查**：
  | 方法 | 對話框類型 |
  |------|---------|
  | `cm.sendNext(text)` | 下一個對話框 |
  | `cm.sendSimple(text)` | 選擇選單 |
  | `cm.sendYesNo(text)` | 是/否對話框 |
  | `cm.sendGetNumber(text, default, min, max)` | 數字輸入 |
  | `cm.sendGetText(text)` | 文字輸入框 |
  | `cm.dispose()` | 關閉對話 |

- **NPC腳本源間相容性**（RaGEZONE 2026-03）：
  - 不同音源（Cosmic、Kinoko、HeavenMS）腳本解析器版本存在差異
  - `importPackage` 在 JDK 7+ 已廢棄，應使用完整 Package 路徑
  - 解決：使用普遍支援的 `cm.` 函數，避免特定音源專有函數

- **動態 NPC 商店腳本**（不需 SQL）：
  - 定義物品陣列：`Array(Array(itemId, price, quantity), ...)`
  - 玩家選擇後檢查金幣和背包空間
  - 優點：即時生效、可版本控制、不需重啟伺服器
