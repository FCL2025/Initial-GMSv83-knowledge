## 二、本次最新發現

### 2.1 封包加密架構（最重要發現）

```
客戶端 → 伺服器：
1. Shanda 加密（每次 IV 遞增更新）
2. AES-OFB 加密
3. 發送

伺服器 → 客戶端：
1. AES-OFB 解密
2. Shanda 解密
```

**v83 是最後一個大量使用 Shanda 的版本！**

### 2.2 握手流程

第一個封包是明文，用於交換：
- 伺服端的 Key/IV（伺服器用它加密發送，客戶端用它解密接收）
- 客戶端的 Key/IV（客戶端用它加密發送，伺服器用它解密接收）

### 2.3 封包格式

```
[Length: 4 bytes][Opcode: 2 bytes][Payload: N bytes]
（小端序）
```

### 2.4 IDA Decode 函數對照

| IDA | mplew | 類型 |
|-----|-------|------|
| Decode1 | write() | byte |
| Decode2 | writeShort() | short |
| Decode4 | writeInt() | int |
| DecodeStr | writeMapleAsciiString() | 字串 |
| DecodeBuffer(x,8) | writeLong() | long |

### 2.5 IDA 封包流向

```
CClientSocket::ProcessPacket
→ CField::OnPacket (Opcode >= 0x100)
→ CLogin::OnPacket / CWvsContext::OnPacket (Opcode < 0x100)
→ CInPacket::Decode1/2/4/Str
```
