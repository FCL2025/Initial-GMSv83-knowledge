**maplelib Go 加密庫核心概念 (pkg.go.dev)**
- https://pkg.go.dev/github.com/defaultmagi/maplelib
- MapleStory uses two keys: one is used to encrypt sent packets and the other is used to decrypt received packets.
- 四個 IV 的意義：
  - 客戶端 SendIV：加密發送到伺服器的封包
  - 客戶端 RecvIV：解密從伺服器收到的封包
  - 伺服器 SendIV：加密發送到客戶端的封包
  - 伺服器 RecvIV：解密從客戶端收到的封包

**完整加密流程 (C→S)**
```
明文資料
  → Shanda 對稱加密
  → AES-OFB 加密 (使用版本 Key + SendIV)
  → MapleCrypto 頭部混淆 (4-byte header)
  → 網路傳輸
```

### 3. Opcode結構

**v83 登入流程封包序列（完整 10 步）**
```
1. C→S Hello Packet (plaintext - 無 opcode header)
2. S→C Hello Packet (plaintext)
3. C→S LOGIN_PASSWORD (encrypted) - Opcode: 0x01
4. S→C LOGIN_STATUS (encrypted) - Opcode: 0x00
5. C→S SERVERLIST_REQUEST (encrypted) - Opcode: 0x06
6. S→C SERVERLIST (encrypted) - Opcode: 0x0C
7. C→S CHARLIST_REQUEST (encrypted) - Opcode: 0x07
8. S→C CHARLIST (encrypted) - Opcode: 0x03
9. C→S CHAR_SELECT (encrypted) - Opcode: 0x08
10. C→S PLAYER_DROP (encrypted) - Opcode: 0x0B
```

**v83 登入結果碼（Login Status Code）**
| 結果碼 | 意義 |
|--------|------|
| 0 | 登入成功 |
| 5 | 帳號或密碼錯誤 |
| 7 | 帳號已停用 |
| 13 | 刪除中（冷靜期） |
| 23 | 登入太頻繁（防機器人延遲） |

**v83 Opcode 現況 (RaGEZONE, 2024-12)**
- v83 的 Opcode 幾乎全部被映射，不需要額外追蹤封包
- 大多數 opcode 表已經在開源專案（Cosmic、HeavenMS）中完整提供
- 唯一需要的反向工程是修改客戶端時

**CClientSocket::ProcessPacket 封包分發流程 (RaGEZONE, 2019)**
- 所有客戶端接收的封包都經過 `CClientSocket::ProcessPacket`
- 從這裡根據 header 分發到 `CField::OnPacket` 或 `CLogin::OnPacket`
- switch(opcode) 分發到具體處理函數

### MapleRef Wiki Send/Recv Opcode 核心概念
- S→C (Send)：伺服器發送到客戶端，使用 Server Opcode Set
- C→S (Recv)：客戶端發送到伺服器，使用 Client Opcode Set
