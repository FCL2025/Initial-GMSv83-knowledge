### JCE Policy 無限強度加密

**覆蓋位置**（兩組共 8 個資料夾）：
```
C:\Program Files\Java\jre7\lib\security
C:\Program Files\Java\jre7\lib\ext\security
C:\Program Files\Java\jdk1.7.0_01\jre\lib\security
C:\Program Files\Java\jdk1.7.0_01\jre\lib\ext\security
```

**為什麼需要覆蓋**：
- JDK 7 預設只支援 128-bit 加密

### 已知限制

| 限制 | 說明 | 解決方案 |
|------|------|---------|
| while() 迴圈處理不完善 | 迴圈內的 decode 無法正確識別 | 設定 `GET_ALL_DECODES = true` |

| 函數需在 IDA 中命名 | 否則腳本無法找到目標 | 先命名再執行腳本 |

### 完整工作流程

```
Step 1: IDA Pro 反編譯
  → 在 IDA 中找到 CField::OnPacket 或 CLogin::OnPacket
  → 產生 pseudocode (.p64 或 .txt)

Step 2: 複製 pseudocode
  → 在 Functions/ 目錄建立 .txt 檔案
  → 貼上 IDA 產生的 C-style pseudocode

Step 3: Python 分析
  → cd MaplePacketPuller/IDA\ Maple\ Script/src/main/python
  → python main.py
  → 輸入分析類型（完整/標準）
  → 輸入要分析的檔案名稱

Step 4: 輸出結果
  → 封包結構自動寫入 FuncOutput/ 目錄
  → 控制台顯示 Decode1/Decode2/Decode4/DecodeStr 等欄位
```

### 產出範例

**輸入 (IDA pseudocode)**：
```c
int __thiscall sub_ABC1230(CInPacket *this)
{
  CInPacket::Decode4(this);   // opcode
  CInPacket::DecodeStr(this); // username
  CInPacket::DecodeStr(this); // password
  ...
}
```

**輸出 (封包結構)**：
```
LOGIN_PASSWORD (Opcode: 0x01)
  - int opcode
  - string username
  - string password
```

### SpiritIDAPlugin 亮點

- 直接在 IDA 中格式化 pseudocode 為伺服器 encode 友好格式
- 支援自訂準確度選項
- 自動搜尋 InHeader opcode
- ⚠️ 限制：while() 迴圈處理不完善（建議設 GET_ALL_DECODES=true）

## 📌 MapleStory v83 封包加密實戰案例 - CMS 中國伺服器

### 實際加密數據示例

**客戶端發送原始密文 (C→S)**:
```
64 A4 9F 07 FD 4D 6C 49 4A 3D 96 FE C8 44 85 E4 79 FE EE E3 C7 3E 9B BF 42 3B 97 B2 4E 36 F8 DB F4 8C 0D D9
```

**AES-OFB 解密後**:
```
71 1A 5B 6A 7E B9 C9 EA 15 17 7E E4 C0 B4 64 6D 1E E7 8A 13 C9 B0 8C 09 4C 7F F5 6F F8 A9 2C 5D 03 2D D4 D5
```

**Shanda Decryption 後**:
```
E2 FB 40 7E 67 08 B6 77 F4 CC 0E 3E 5D 72 D1 9A 2C 7C F6 44 16 6A 7F 2F ED 39 C8 DE 96 F7 95 04 59 88 C9 6F
```

### 問題分析
```
解密後第一個 opcode = 0xFBE2 (Big Endian)
預期 LOGIN_PASSWORD opcode = 0x0001

→ 原因：版本不匹配，或 AES Key/IV 設定錯誤
→ 解決：確認客戶端和伺服器使用相同版本的金鑰
```

### v83 AES Key 參考 (CMS/Chinese MS v83)
```java
// AES UserKey (first column per row)
0x13, 0x08, 0x06, 0xB4, 0x1B, 0x0F, 0x33, 0x52

// IV Transformation Key
0xC65053F2

// 客戶端版本
0x23 (35 decimal)
```

### MapleCrypto + Shanda + AES 三層解密流程圖
```
發送端 (C→S):
明文資料
  → AES-OFB 加密 (使用版本 Key + IV)
  → Shanda 加密 (對稱加密)
  → MapleCrypto 頭部混淆 (4-byte header)
  → 網路傳輸

接收端 (S→C):
網路接收
  → MapleCrypto 解混淆 (提取 opcode + length)
  → Shanda 解密
  → AES-OFB 解密
  → 明文資料
```

### 排查解密失敗的檢查清單

| 檢查項目 | 正確值 | 症狀 |
|---------|--------|------|
| AES Key | 版本決定的 32-byte key | 解密出乱码 |
| IV | Hello Packet 協商的 IV | opcode 不匹配 |
| Shanda Key | 版本特定的對稱金鑰 | 資料順序錯誤 |
| 版本號 | 0x0053 (83) | Hello Packet 失敗 |
| Opcode 表 | 需與客戶端匹配 | switch case 走错分支 |

## 📌 CInPacket 封包接收完整流程

```cpp
// 客戶端接收封包 (IDA pseudocode)
void __thiscall CClientSocket::ProcessPacket(CClientSocket *this) {
    while (true) {
        // 讀取 4-byte header
        BYTE header[4];
        recv(socket, header, 4);
        
        // MapleCrypto 解混淆
        int packetLen = MapleCrypto::decryptLength(header, ivRecv);
        int opcode = MapleCrypto::decryptOpcode(header, ivRecv);
        
        // 分配 buffer
        byte *buffer = new byte[packetLen];
        
        // 接收剩餘資料 (packetLen bytes)
        recv(socket, buffer, packetLen);
        
        // AES-OFB + Shanda 解密
        MapleCrypto::decrypt(buffer, packetLen, ivRecv);
        
        // 構造 CInPacket
        CInPacket packet;
        packet.nType = opcode;
        packet.buffer = buffer;
        
        // 分發到處理函數
        switch (opcode) {
            case 0x00: CLogin::OnPacket(&packet); break;
            case 0x01: CLogin::OnLoginPassword(&packet); break;
            // ...
        }
    }
}
```
