### 🆕 maplelib - Go 語言 MapleStory 加密庫 (defaultmagi)

**來源**: https://pkg.go.dev/github.com/defaultmagi/maplelib

**核心概念**：
> MapleStory uses two keys: one is used to encrypt sent packets and the other is used to decrypt received packets.

**四個 IV 的意義**：
| IV | 擁有者 | 用途 |
|----|--------|------|
| 客戶端 SendIV | 客戶端 | 加密發送到伺服器的封包 |
| 客戶端 RecvIV | 客戶端 | 解密從伺服器收到的封包 |
| 伺服器 SendIV | 伺服器 | 加密發送到客戶端的封包 |
| 伺服器 RecvIV | 伺服器 | 解密從客戶端收到的封包 |

### 🆕 MapleRef Wiki Send/Recv Opcode 核心概念

**來源**: https://mapleref.fandom.com/wiki/Operation_Code

> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

**重要**：同一個 Opcode 數字可能代表不同含義，視方向而定！

### 🆕 botnored v83 封包類別結構

**來源**: https://botnored.weebly.com/blog/maplestory-classes-v83

**重要原則**：
> **第一個發送/接收的封包永遠是純文字（未加密）**
> MapleStory 封包透過 TCP Socket 傳輸

**客戶端主要類別**：
| 類別 | 功能 |
|------|------|
| `CWvsContext` | 世界狀態管理（角色資料、伺服器列表） |
| `CClientSocket` | 網路連接管理（Send/Recv） |
| `CField` | 當前地圖/遊戲場景處理 |
| `CLogin` | 登入畫面處理 |

**封包流向**：
```
CClientSocket::ProcessPacket()
    ↓
根據 Header 分發
    ↓
CField::OnPacket() 或 CLogin::OnPacket()
    ↓
處理具體 Opcode
```

## 關鍵學習點

### 1. 登入器/客戶端修改
- **Ezorsia v2** 是 2026 年最新、最簡單的方案
- 只需將 `dinput8.dll` 放入遊戲目錄即可
- 不需要 Hex 編輯，不需要刪除原廠檔案

### 2. 抓包分析
- **Maple_Pshark** 是 v83 專用的即時封包截獲工具
- **MaplePacketPuller** 是離線封包結構分析工具
- **Wireshark** 仍是最通用的 TCP 分析工具

### 3. Opcode 結構
- v83 的 Opcode 幾乎全部被映射
- 同一個 Opcode 數字可能代表不同含義（S→C vs C→S）
- **MapleRef Wiki** 是最佳的Opcode 查詢資源

### 4. WZ 修改
- 永遠使用客戶端 WZ 編輯，不要用伺服器 WZ.XML 覆蓋
- 永遠使用 GMS (old) 加密格式
- 永遠禁止刪除/重新導入節點

### 5. NPC 腳本編寫
- 使用普遍的 `cm.` 函數
- 避免 `importPackage`（JDK 7+ 已廢棄）
- 狀態機邏輯：mode=1 遞增，mode=-1 結束
