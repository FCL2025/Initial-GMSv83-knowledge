**錯誤碼分析**：
| 錯誤碼 | Hex | 意義 |
|--------|-----|------|
| `-2147467259` | `0x80004005` | Unspecified Error（通常是防火牆） |

**排查流程**：
```
1. 確認錯誤時機
   → 連線後立刻斷 → 防火牆/權限問題
   → 登入後斷線 → 版本/WZ 問題

2. 防火牆檢查
   netsh advfirewall set allprofiles state off  (測試用)
   
3. 確認 listen address
   ServerConstants.java → 確認 listen 0.0.0.0 而非 127.0.0.1
   
4. 確認埠口
   7575 (login) 和 8484 (game) 都需要開啟
```

## 🆕 3. Opcode 結構

### CInPacket Decode 函數 IDA 對照表（完整版）

**來源**: https://forum.ragezone.com/threads/how-to-use-ida-in-v83.1107216/

| IDA 函數 | Java 實現 | 大小 | 說明 |
|---------|-----------|------|------|
| `CInPacket::Decode1` | `mplew.write(1)` | 1 byte | 無符號整數 |
| `CInPacket::Decode2` | `mplew.writeShort()` | 2 bytes | 有符號短整數 |
| `CInPacket::Decode4` | `mplew.writeInt()` | 4 bytes | 有符號整數 |
| `CInPacket::DecodeBuffer(x, 8)` | `mplew.writeLong()` | 8 bytes | 長整數 |
| `CInPacket::DecodeBuffer(x, 13)` | `mplew.writeAsciiString(str, 13)` | 13 bytes | 固定長度字串 |
| `CInPacket::DecodeStr` | `mplew.writeMapleAsciiString()` | 可變 | Maple ASCII 字串（長度前綴） |
| `CInPacket::DecodePos` | 特殊 | 特殊 | 座標編碼（浮點數除法） |
| `CInPacket::Decode1` (bool) | `if (CInPacket::Decode1(v4))` | 1 byte | 布林值判斷 |

### 封包處理函數流向（IDA 分析核心）

```
CClientSocket::ProcessPacket()
    ↓
根據 Header 分發（switch 表）
    ↓
CLogin::OnPacket       // 登入畫面封包
    或
CWvsContext::OnPacket  // 世界狀態封包
    或
CField::OnPacket      // 遊戲中封包
    或
CFarm::OnPacket       // 農場封包
```

**IDA 分析技巧**：
1. 在 IDA 中找到 `CClientSocket::ProcessPacket`
2. 查看 switch 表，找到 opcode 對應的處理函數
3. 進入處理函數，查看 Decode 呼叫序列
4. 根據 Decode 類型重建封包結構

### v83 Opcode 映射現況

**重要結論**：
- ✅ v83 Opcode 已完全映射，開源專案提供完整 Opcode 表

##### 怪物無敵 GodMode 各類型

| GodMode 類型 | 修改方式 | 風險 | 說明 |
|-------------|---------|------|------|
| **Full Body GodMode** | `bodyAttack = 0` | A/B（可能被偵測） | 怪物完全無法攻擊 |


## 🆕 5. NPC 腳本編寫

### 動態配方系統腳本源（2026 最新完整版）

```javascript
/*
 * 動態配方系統 - 玩家提交材料換取成品
 * NPC ID: 9010010
 * 完全相容所有音源
 */
var status = 0;

var recipes = Array(
    {
        name: "楓之葉地圖",
        materials: [[4000000, 10], [4000001, 5]],
        result: [[2300000, 1]],
        meso: 1000
    },
    {
        name: "傳說武器",
        materials: [[4003000, 100], [4003001, 50]],
        result: [[1302000, 1]],
        meso: 50000
    }
);

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
        return;
    }
    if (mode == 0 && type == 1) {
        cm.dispose();
        return;
    }
    if (mode == 1) status++;
    else status--;

    if (status == 0) {
        var msg = "歡迎來到工藝系統！\r\n請選擇配方：\r\n";
        for (var i = 0; i < recipes.length; i++) {
            msg += "#L" + i + "#" + recipes[i].name + "#l\r\n";
        }
        cm.sendNext(msg);
    } else if (status == 1) {
        var recipe = recipes[selection];
        var canCraft = true;
        var msg = "你需要以下材料：\r\n";
        for (var i = 0; i < recipe.materials.length; i++) {
            var mat = recipe.materials[i];
            var have = cm.getQuantityOfItem(mat[0]);
            var need = mat[1];
            msg += "#i" + mat[0] + "# #t" + mat[0] + "#: " + have + "/" + need + "\r\n";
            if (have < need) canCraft = false;
        }
        if (cm.getMesos() < recipe.meso) canCraft = false;
        msg += "加工費: " + recipe.meso + " 金幣\r\n";
        msg += canCraft ? "#g可以製作#k" : "#r材料不足#k";
        cm.sendYesNo(msg);
    } else if (status == 2) {
        var recipe = recipes[selection];
        for (var i = 0; i < recipe.materials.length; i++) {
            cm.gainItem(recipe.materials[i][0], -recipe.materials[i][1]);
        }
        cm.gainMesos(-recipe.meso);
        for (var i = 0; i < recipe.result.length; i++) {
            cm.gainItem(recipe.result[i][0], recipe.result[i][1]);
        }
        cm.sendOk("製作成功！");
        cm.dispose();
    }
}
```

|---------|------|

```javascript
if (status == 1 && mode == 0) {
```
