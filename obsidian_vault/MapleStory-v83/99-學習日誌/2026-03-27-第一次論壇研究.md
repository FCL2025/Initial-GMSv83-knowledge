## 🆕 MapleRef Wiki 封包結構重點

**URL**: https://mapleref.fandom.com/wiki/Operation_Code

### Send vs Receive Opcode 核心區分
> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

| 方向 | Opcode 集合 |
|------|------------|
| S→C (Send) | Server Opcode Set |
| C→S (Recv) | Client Opcode Set |

### 重要原則
- **第一個封包永遠是明文**（Hello Packet）
- MapleStory 使用 TCP Socket
- Opcode 是封包前 2 bytes (v83)

## 🆕 maplelib Go 加密庫核心原理

**URL**: https://pkg.go.dev/github.com/defaultmagi/maplelib

### 雙金鑰機制
> MapleStory uses two keys: one is used to encrypt sent packets and the other is used to decrypt received packets.

### 四個 IV 的意義
| IV | 用途 |
|----|------|
| 客戶端 SendIV | 加密發送到伺服器的封包 |
| 客戶端 RecvIV | 解密從伺服器收到的封包 |
| 伺服器 SendIV | 加密發送到客戶端的封包 |
| 伺服器 RecvIV | 解密從客戶端收到的封包 |

### 三層加密架構
```
明文 → Shanda 加密 → AES-OFB 加密 → MapleCrypto 頭部混淆 → 網路
```

### 2. 抓包分析
- **Maple_Pshark** 是 v83 專用的即時封包截獲工具
- **MaplePacketPuller** + **SpiritIDAPlugin** 是離線分析工具鏈

### 3. Opcode 結構
- v83 Opcode 幾乎全部被映射
- Send/Recv 是兩套不同的 Opcode 表
- 熟悉 `CClientSocket::ProcessPacket` 的分發邏輯

### 4. WZ 修改
- 使用 **HaRepacker-resurrected**（最新 .NET 版本）
- **黃金法則**：禁止刪除/重新導入節點
- 編輯後用 "Private Server..." 導出 XML

### 5. NPC 腳本編寫
- 使用 JavaScript (.js)
- 核心物件：`cm` (NPCConversationManager)
- 狀態機制：`status` 變數追蹤對話流程
