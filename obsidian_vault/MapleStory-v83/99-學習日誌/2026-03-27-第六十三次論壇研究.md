### 3. 登入器/客戶端修改黃金法則

**STREDIT 使用流程**：
```
1. File → Open → 選擇 localhost.exe
2. 詢問「是否 v118+」→ 選「No」（v83 適用）
3. 詢問「IP 搜尋模式」→ 選「Yes」
4. 點擊「Force to a IP」按鈕 → 輸入目標 IP
5. 保存修改後的 localhost.exe
```

**LAN 多人連線錯誤排查**：
| 錯誤碼 | Hex 值 | 意義 | 根本原因 |
|--------|--------|------|---------|
| `-2147467259` | `0x80004005` | Access Denied | 防火牆/權限問題（非網路問題）|

### 4. Opcode結構 — MapleCrypto 三層加密

```
┌─────────────────────────────────────────────────────┐
│ Layer 1: MapleCrypto (頭部混淆)                      │
│   - 4 bytes header (混淆 opcode + length)            │
│                                                        │
│ Layer 2: Shanda Encryption (對稱加密)               │
│   - v83 仍使用，已於 v118+ 移除                     │
│                                                        │
│ Layer 3: AES-OFB (主加密)                           │
│   - v83 主要加密方式                                 │
└─────────────────────────────────────────────────────┘
```

**v83 AES Key 參考 (CMS)**：
```java
// AES UserKey
0x13, 0x08, 0x06, 0xB4, 0x1B, 0x0F, 0x33, 0x52
// IV Transformation Key
0xC65053F2
// 客戶端版本
0x23 (35 decimal)
```
