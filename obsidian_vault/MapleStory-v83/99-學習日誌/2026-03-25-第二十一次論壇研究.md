### 2. Opcode結構 (Opcode Structure)

#### MapleStory封包結構
- 封包標頭(Packet Header)為4位元組
- 使用send IV、封包長度(不含標頭)和版本號生成
- 版本號發送到客戶端時為 -(version + 1)
- **Opcode通常為2位元組(Short類型)**
- 這將最大Opcode數量從255擴展到65,536

#### 抓包工具
- **MaplePacketPuller** (Python): https://github.com/Bratah123/MaplePacketPuller
  - Python實現的Maplestory封包結構分析器
- **Maple_Pshark**: https://github.com/obstriker/Maple_Pshark
  - Mapleroyals v83封包日誌攔截器

### 4. NPC腳本編寫

#### NPC腳本基礎
- NPC腳本用於伺服器端執行互動
- 可使用任何 capable scripting language
- 伺服器需要支援該腳本語言的執行

#### 腳本位置
- 在HeavenMS中，腳本位於 `source/scripts/npc`
- 可參考現有的NPC腳本作為模板

### 5. 伺服器架設 - HeavenMS

#### 重要資源
- **伺服器文件**: https://github.com/ronancpl/HeavenMS
- **客戶端和工具**: https://drive.google.com/drive/folders/0BzDsHSr-0V4MYVJ0TWIxd05hYUk
- **Java 8 SDK & NetBeans**: https://www.oracle.com/technetwork/pt/java/javase/downloads/jdk-netbeans-jsp-3413153-ptb.html

#### 架設步驟
1. 安裝 WampServer (MySQL資料庫)
2. 安裝 Java 8 SDK + NetBeans
3. 下載 HeavenMS 源碼
4. 執行 SQL 腳本 (db_database.sql, db_drops.sql, 可選 db_shopupdate.sql)
5. 配置 `configuration.ini` 中的 IP
6. 在 NetBeans 中建立項目並導入 JAR 核心庫
7. 執行 `launch.bat` 啟動伺服器

#### 客戶端設置
1. 安裝 MapleStory v83
2. 刪除 HShield資料夾、ASPLauncher.exe、MapleStory.exe、Patcher.exe
3. 放入 localhost.exe
4. 用編輯過的WZ文件替換原版WZ文件

### 6. 客戶端IP修改
- 使用十六進制編輯器(如 Neo Hex Editor)修改 localhost.exe
- 搜索 "Type: String" "127.0.0.1" 並替換為目標IP

## 重點筆記

### v83客戶端修改要點
1. 獨立DLL方式是最乾淨的修改方式
2. HD修改涉及記憶體位址修改
3. 客戶端修改通常需要脫殼後進行

### Opcode分析要點
1. v83使用2位元組Opcode
2. 第一個發送的封包通常是明文（未加密）
3. 封包加密使用IV機制

### NPC腳本要點
1. 腳本語言需與伺服器代碼相容
2. 腳本放在伺服器端 `/scripts/npc/` 目錄
3. 可參考現有腳本學習格式
