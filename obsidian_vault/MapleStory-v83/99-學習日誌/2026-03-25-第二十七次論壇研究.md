### MapleEzorsia v2 — 史上第一個獨立 DLL 客戶端方案 (2023-2026)

**GitHub**: https://github.com/444Ro666/MapleEzorsia-v2

這是將官方 Nexon 客戶端轉換為 HD + localhost 私服版本的革命性方案：

| 特性 | 說明 |
|------|------|

| 一體化 | 同時完成：去除保護 + HD 解析度 + localhost 導向 |

| 完全開源 | C++ 源代碼可在 GitHub 查看 |

**使用方式**：
```
1. 從 Nexon 安裝原始 v83 客戶端
2. 將 dinput8.dll 放入遊戲目錄
3. 啟動 MapleStory.exe（自動加載 DLL）
4. 享受 HD 解析度 + localhost 連線
```

### HxD 傳統 IP 修改步驟
```
1. HxD 開啟客戶端 exe（建議先複製備份）
2. Ctrl+F 搜尋 "127.0.0.1"（text-string 模式）
3. 會看到三個連續出現的 127.0.0.1
4. 逐一覆蓋為目標 IP
5. Ctrl+S 儲存
```

### LAN 多人連線常見錯誤 (-2147467259 = 0x80004005 Access Denied)
| 問題 | 解決方案 |
|------|---------|
| 客戶端 IP 指向 127.0.0.1 | 需指向 PC1 的 LAN IP（如 192.168.1.x） |
| 防火牆未開放端口 | `netsh advfirewall firewall add rule name="MapleStory" dir=in action=allow protocol=tcp localport=8484` |
| MySQL 未授權遠端 | `GRANT ALL PRIVILEGES ON *.* TO 'root'@'%' IDENTIFIED BY 'password'; FLUSH PRIVILEGES;` |

### MapleRef Wiki — 封包結構權威參考
https://mapleref.fandom.com/wiki/Packets

> "Actions that are defined by an identifier called an Operation Code (opcode for short), which is the first 2 bytes of the packet"

### 核心原則
> **第一個發送/接收的封包永遠是純文字（未加密）**

封包加密三層架構：
```
Layer 1: MapleCrypto (頭部混淆)     — 4-byte header
Layer 2: Shanda Encryption          — 對稱加密，v118+ 已移除
Layer 3: AES-OFB                   — 256-bit key，版本決定的靜態金鑰
```

### v83 登入結果碼
| 結果碼 | 意義 |
|--------|------|
| 0 | 登入成功 |
| 5 | 帳號或密碼錯誤 |
| 7 | 帳號已停用 |
| 13 | 刪除中（冷靜期） |
| 23 | 登入太頻繁（防機器人延遲） |

## 3. Opcode 結構

### v83 Opcode 現況 (RaGEZONE, 2024-12)
- ✅ 幾乎全部被映射，不需要額外追蹤封包
- ✅ 大多數 opcode 表已在開源專案（Cosmic、HeavenMS）中完整提供
- ⚠️ 唯一需要的反向工程是**修改客戶端時**

### Send vs Recv Opcode 區分
| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

> 同一個 Opcode 數字可能代表不同含義，視方向而定！

### MapleCrypto Header 加密/解密公式

**加密端**：
```java
int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
verify = (verify << 3) | (verify >>> 5);
verify &= 0xFFFF;
```

**解密端（反向）**：
```java
verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
```

### IDA 中 Decode 函數對照
| IDA 函數 | Java 實現 | 讀取大小 |
|---------|---------|---------|
| `Decode1` | `mplew.write(1)` | 1 byte |
| `Decode2` | `mplew.writeShort(1)` | 2 bytes |
| `Decode4` | `mplew.writeInt(1)` | 4 bytes |
| `DecodeStr` | `mplew.writeMapleAsciiString()` | 可變 |
| `DecodeBuffer(x, 8)` | `mplew.writeLong` | 8 bytes |

## 4. WZ 修改

### WZ 編輯黃金法則
> DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs.

**正確流程**：
```
1. HaRepacker 4.2.4 開啟 GMS (old) 加密的客戶端 WZ
2. 使用 UI 直接修改數值（禁止刪除/重新導入節點！）
3. 儲存 WZ
4. 在 HaRepacker 中選擇 "Private Server..." 導出
5. 將導出的 XML 覆蓋到伺服器的 WZ.XML 目錄
```

### Quest 等級限制移除（lvmax Removal）
**來源**: https://forum.ragezone.com/threads/maplestory-v83-method-to-disable-quest-level-restrictions.1250107/

**Notepad++ Regex 批量移除**：
```
尋找：lvmax="\d+"\s*
取代：（留空）
```

### 創建自訂 NPC 完整流程
```
1. HaRepacker 導出 NPC.wz 為 XML
2. 在 XML 中新增自訂 NPC 的 frame 和 origin 設定
3. 複製 0001337.img 作為模板，修改尺寸
4. 將修改後的 img 放入 NPC.wz
5. 在伺服器端編寫對應的 .js 腳本
```

### fonts folder 問題修復
客戶端啟動後文字無法顯示（只有背景圖片）時：
```
Step 1: 從正常客戶端複製 fonts folder
Step 2: 粘貼到目標客戶端目錄（MapleStory/fonts/）
Step 3: 確認 fonts folder 有讀取權限
Step 4: 若問題持續，從 MapleSolaxiaV2 資源包取得完整 fonts
```

## 5. NPC 腳本編寫

### 基本模板
```javascript
var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        if (mode == 1)
            status++;
        else
            status--;
        // 根據 status 處理不同對話
    }
}
```

### 常用 cm 函數速查
| 函數 | 功能 |
|------|------|
| `cm.sendNext(text)` | 下一個對話框 |
| `cm.sendSimple(text)` | 選擇選單 |
| `cm.sendYesNo(text)` | 是/否對話 |
| `cm.gainItem(id, qty)` | 給予物品 |
| `cm.gainMeso(qty)` | 給予/收取楓幣 |
| `cm.openShop(id)` | 開啟商店 |
| `cm.warp(mapId)` | 傳送地圖 |
| `cm.changeJob(jobId)` | 轉職 |
| `cm.dispose()` | 關閉對話 |
| `cm.haveItem(itemId, count)` | 是否持有物品 |
| `cm.getPlayer().getName()` | 取得玩家名稱 |

### 對話格式標籤速查
| 標籤 | 功能 |
|------|------|
| `#b` | 藍色文字 |
| `#r` | 紅色文字 |
| `#h#` | 玩家名稱 |
| `#i[itemid]#` | 物品圖片 |
| `#L0#...#l` | 選項標記 |
| `#c[itemid]#` | 玩家持有數量 |
| `\r\n` | 換行 |

### 音源相容性問題（JDK 7+）
| 問題 | 解決方案 |
|------|---------|
| `importPackage is not defined` | 移除 importPackage，使用完整路徑 |
| `SavedLocationType is not defined` | 檢查音源版本相容性 |
| 不同音源解析器差異 | 使用普遍 `cm.` 函數，避免特定音源專有函數 |

### NPC腳本源間相容性檢查清單
- ✅ 無 `importPackage`（JDK 7+ 已廢棄）
- ✅ 使用 `Packages.xxx` 完整路徑
- ✅ 使用 `cm.` 而非直接類別名
- ✅ 正確的 `status` 狀態機邏輯
