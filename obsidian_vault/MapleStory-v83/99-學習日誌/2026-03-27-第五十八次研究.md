## 🆕 v83 客戶端修改：Hex 編輯 localhost.exe 完整流程 (2026-03 最新)

### 核心原理

v83 客戶端（localhost.exe）默認連接到 `127.0.0.1:8484`。要連接到外部伺服器或實現單機模式，需要修改以下三個地方：

### HxD Hex 編輯步驟

**所需工具**：
- HxD Editor (https://mh-nexus.de/en/hxd/)
- v83 localhost 客戶端

**操作流程**：

```
1. 用 HxD 打開 localhost.exe（或 MapleStory.exe）
2. 按 Ctrl+F，選擇「文字編輯」
3. 搜尋 "127.0.0.1"
4. 將 "127.0.0.1" 替換為目標 IP（如你的 Hamachi IP 或 WAN IP）
5. 儲存檔案

⚠️ 注意事項：
- 需要管理員權限
- 建議先備份原始檔案
- IP 位址長度可能不同，替換時需注意位址對齊
```

### 三種 IP 修改位置

v83 localhost.exe 通常在以下位置儲存 IP 位址：

| 位置 | 用途 | 偏移 |
|------|------|------|
| Login Server IP | 登入伺服器位址 | 通常靠近二進位開頭 |
| Game Server IP | 遊戲伺服器位址 | 可能在另一個偏移 |
| Backup IP | 備用/驗證用 | 第三個位置 |

### 使用 Wireshark 確認修改成功

```
1. 啟動伺服器
2. 啟動客戶端
3. 在 Wireshark 中監控：
   tcp.port == 8484 || tcp.port == 7575
4. 觀察 TCP 連線是否指向正確 IP
5. Hello Packet 應該能正常交換
```

### 常見問題排查

| 問題 | 原因 | 解決方案 |
|------|------|---------|
| 連線不到伺服器 | IP 格式錯誤 | 確認 IP 長度一致（127.0.0.1 是 9 字元） |
| 版本不符錯誤 | 客戶端和伺服器版本不匹配 | 確認雙方都是 v83 |
| 連線後立刻斷線 | WZ 版本問題 | 檢查客戶端和伺服器 WZ 是否匹配 |

## 🆕 MapleProxy v83 — 動態封包編輯器 (RaGEZONE, 2016, 2026 仍然實用)

**來源**: https://forum.ragezone.com/threads/mapleproxy-v83.1111000/

### 核心功能

MapleProxy 允許動態編輯發送到伺服器和客戶端的封包，與靜態封包注入器不同。

**主要功能**：
- 動態封包修改（而非靜態注入）
- 支援封包轉發（如 BungeeCord for Minecraft）
- 可連接多個伺服器

### 使用方式

```
localhost.exe 127.0.0.1 8480
```

**參數說明**：
- `localhost.exe`：代理執行檔
- `127.0.0.1`：目標伺服器 IP
- `8480`：目標伺服器端口

### 功能表操作

- **Trade 按鈕**：開啟功能表
- 世界事件訊息會顯示 "Packet editor is working!"

### 與 Maple_Pshark 的區別

| 工具 | MapleProxy | Maple_Pshark |
|------|-----------|-------------|
| 封包修改 | ✅ 動態修改 | ✅ 靜態修改 |
| 封包轉發 | ✅ 支援 | ❌ |
| 多人伺服器 | ✅ 支援 | ❌ |
| 開源 | ✅ | ✅ |

### 登入相關 Opcode 對照（v83）

| Opcode | S→C 名稱 | C→S 名稱 |
|--------|---------|---------|
| 0x00 | LOGIN_STATUS | - |
| 0x01 | - | LOGIN_PASSWORD |
| 0x02 | ACCEPT_LOGIN | - |
| 0x03 | CHARLIST | - |
| 0x06 | - | SERVERLIST_REQUEST |
| 0x07 | - | CHARLIST_REQUEST |
| 0x08 | - | CHAR_SELECT |
| 0x0B | - | PLAYER_DROP |

### IDA 分析Opcode 步驟

```
1. 找 CClientSocket::ProcessPacket
   → 所有接收封包的總入口
   
2. 分析 switch(opcode) 表
   → 找到對應的 case 處理函數
   
3. 在處理函數中分析 Decode 調用
   → Decode1, Decode2, Decode4, DecodeStr
   
4. 對照開源專案的 encode 實現
   → 確認封包結構
```

### CInPacket Decode 函數對照

| IDA 函數 | 伺服器端方法 | 讀取大小 |
|---------|------------|---------|
| Decode1 | mplew.write(1) | 1 byte |
| Decode2 | mplew.writeShort(1) | 2 bytes |
| Decode4 | mplew.writeInt(1) | 4 bytes |
| Decode8 | mplew.writeLong(1) | 8 bytes |
| DecodeStr | mplew.writeMapleAsciiString() | 可變 |
| DecodeBuffer(x,n) | mplew.write(n bytes) | n bytes |

### v83 WZ 編輯黃金法則

> **DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs.**

**正確流程**：
```
1. HaRepacker 4.2.4 打開 GMS (old) 加密的客戶端 WZ
2. 使用 UI 直接修改數值（禁止刪除/重新導入節點！）
3. 儲存 WZ
4. 在 HaRepacker 中選擇 "Private Server..." 導出
5. 將導出的 XML 覆蓋到伺服器的 WZ.XML 目錄
```

### NPC 腳本與 WZ 編輯的結合

**創建自訂 NPC 完整流程**：
```
1. HaRepacker 導出 NPC.wz 為 XML
2. 在 XML 中新增自訂 NPC 的 frame 和 origin 設定
3. 複製 0001337.img 作為模板，修改尺寸
4. 將修改後的 img 放入 NPC.wz
5. 在伺服器端編寫對應的 .js 腳本
```
