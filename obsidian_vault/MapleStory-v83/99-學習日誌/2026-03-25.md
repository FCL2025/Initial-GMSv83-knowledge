## 本日學習重點

### 1. 登入器/客戶端修改
- 研究了多种 localhost redirector 方案：
  - **Ezorsia v2 DLL**（GitHub: 444Ro666/MapleEzorsia-v2）：史上第一個獨立 v83 dinput8.dll 客戶端，可直接將官方 Nexon exe 轉換為 HD + localhost 版本
  - **MapleClientEditTemplate**（MinimumDelta）：提供完整 WinSock Hook + IJL15 注入框架
  - **localhost.exe + HxD**：傳統方案，需手動 Hex 編輯 IP
- 發現 MapleSyrup（C# 客戶端開發中）和 **MortalClient**（完全開源 v83 客戶端）

### 2. 抓包分析
- 更新了 Maple_Pshark（v83 專用封包 Logger）和 MaplePacketPuller + SpiritIDAPlugin 完整工作流程
- 確認 MapleRef Wiki（mapleref.fandom.com）是最完整的封包結構參考
- v83 封包三層加密：MapleCrypto → Shanda → AES-OFB

### 3. Opcode 結構
- v83 Opcode 幾乎全部被映射，開源專案（Cosmic、HeavenMS）已有完整 Opcode 表
- IDA 分析仍是理解客戶端內部運作的最佳方式
- CClientSocket::ProcessPacket 是所有客戶端接收封包的總入口

### 4. WZ 修改
- **HaRepacker-resurrected** 是最推薦的 WZ 編輯工具（支援 GMS old 加密）
- 跨版本物品 Import 需注意必備屬性（lvlm、STR、DEX 等）
- NX 格式是 WZ 的高效替代方案（檔案更小、載入更快）
- Quest lvmax 限制可用 Notepad++ Regex 批量移除

### 5. NPC 腳本編寫
- 狀態機核心：`status` 追蹤對話進度，`mode` 區分按鈕類型
- 不同音源（Cosmic、Kinoko、HeavenMS）腳本解析器版本存在差異
- 動態 NPC 商店腳本可替代 SQL 商店（不需要重啟伺服器）
- `importPackage` 在 JDK 7+ 已廢棄，應使用完整 Package 路徑
