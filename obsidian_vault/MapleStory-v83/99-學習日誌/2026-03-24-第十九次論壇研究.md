### 1.6 WinSock Hook 核心實作概念
```cpp
// Hook WSAConnect 實現 IP 重定向
typedef int (WSAAPI *WSAConnect_t)(SOCKET, const sockaddr*, int, ...);
WSAConnect_t Original_WSAConnect;

int WSAAPI Hook_WSAConnect(SOCKET s, const sockaddr* name, int namelen, ...) {
    if (name->sa_family == AF_INET) {
        sockaddr_in* addr = (sockaddr_in*)name;
        addr->sin_addr.s_addr = inet_addr("127.0.0.1"); // 改為目標 IP
        addr->sin_port = htons(7575);                    // 改為目標 Port
    }
    return Original_WSAConnect(s, name, namelen, ...);
}
```

## 三、Opcode 結構

### 3.1 封包加密三層架構
```
┌─────────────────────────────────────────────┐
│ Layer 1: MapleCrypto (頭部混淆)             │
│   - 4 bytes header，含混淆 opcode           │
│                                             │
│ Layer 2: Shanda Encryption (對稱加密)       │
│   - v83 仍使用                             │
│                                             │
│ Layer 3: AES-OFB (主加密)                  │
│   - 256-bit key, 4-byte IV                 │
└─────────────────────────────────────────────┘
```

### 3.2 MapleCrypto Header 加密/解密公式
```java
// 加密端
public byte[] getPacketHeader(int opcode, int length, int iv) {
    int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
    verify = (verify << 3) | (verify >>> 5);
    verify &= 0xFFFF;
    return new byte[] {
        (byte) (verify & 0xFF),
        (byte) ((verify >>> 8) & 0xFF),
        (byte) (length & 0xFF),
        (byte) ((length >>> 8) & 0xFF)
    };
}

// 解密端
public int retrieveOpcode(byte[] header, int iv) {
    int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
    verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
    return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
}
```

### 3.3 CClientSocket::ProcessPacket 封包分發流程
```
CClientSocket::ProcessPacket()
    ↓
根據 MapleCrypto header 解密
    ↓
提取 opcode (2 bytes, little endian)
    ↓
switch(opcode)
    ↓
CLogin::OnPacket()    // 登入畫面
    或
CField::OnPacket()    // 遊戲中
```

## 五、NPC 腳本編寫

### 5.1 基本模板
```javascript
var status = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        if (mode == 1)
            status++;
        else
            status--;
        
        if (status == 0) {
            cm.sendNext("你好！");
        } else {
            cm.dispose();
        }
    }
}
```

function action(mode, type, selection) {

### 5.5 腳本源相容性問題 (RaGEZONE, 2026-03)
來源: https://forum.ragezone.com/threads/kinoko-npc-scripts.1261538/

**問題**：不同音源（Cosmic、Kinoko、HeavenMS）的 NPC 腳本解析器版本不同

**解決方案**：
1. 使用普遍支援的 `cm.` 函數
2. 避免使用特定音源專有的 Package 路徑
3. 避免 `importPackage`（JDK 7+ 已廢棄）
4. 先在本地測試再部署

### 5.6 常見錯誤
| 錯誤訊息 | 原因 | 解決 |
|---------|------|------|
| `importPackage is not defined` | JDK 7+ 不支援 | 移除 importPackage 行 |
| `NPC xxx.js is not coding yet` | 腳本路徑錯誤 | 確認檔案在正確目錄 |
| `NullPointerException` | 物件為空 | 檢查 `cm.getPlayer()` 返回值 |
