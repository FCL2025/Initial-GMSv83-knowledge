**架設流程摘要**：
```
1. 安裝 WampServer2.0i → MySQL + Apache
2. 安裝 Hamachi（可選，用於 LAN/遠程連線）

5. 覆蓋 JCE Unlimited Strength Policy JAR
6. 建立 MySQL schema "maplesolaxia"
7. 執行 SQL：db_database.sql → db_drops.sql → db_shopupdate.sql (可選)
8. 設定 configuration.ini 中的 IP（localhost/hamachi IP）
9. 以 NetBeans 開啟專案，Clean and Build
10. 執行 launch.bat 啟動伺服器
```

**客戶端設定流程**：
```
1. 使用 ManagerMsv83.exe 安裝客戶端
2. 刪除：HShield/、ASPLauncher.exe、MapleStory.exe、patcher.exe
3. 替換為 localhost.exe（本地登入器）
4. 覆蓋 WZ 為 client_wz 資料夾中的版本
5. 用 HxD 修改 localhost.exe 中的 IP（Ctrl+F 搜尋 127.0.0.1）
```

### 2. localhost.exe IP 修改詳細步驟（Neo Hex Editor）

**工具**：free-hex-editor-neo.exe

**步驟**：
1. 用 Neo Hex Editor 開啟 `localhost.exe`
2. 按 `Ctrl+F` 搜尋 `127.0.0.1`（文字模式）
3. 找到後直接覆蓋為目標 IP（長度需一致）
4. 通常會有 3 處 IP 地址需要修改（全部替換）
5. 存檔

**注意**：
- IP 長度不同時（如 192.168.1.1 vs 127.0.0.1），需使用 `\0` 填充或找專業工具
- 若長度不符可能導致客戶端崩潰

### 3. localhost.exe 無法啟動的解決方案

**問題**：Windows 8/10 上 localhost.exe 無法正常開啟

**解決方案**：
1. 將 `lolwut.exe` 放到客戶端資料夾
2. localhost.exe 屬性設定：
   - 相容模式：Windows 7
   - 解析度：640x480
   - 以系統管理員身份執行
3. 若仍不行，嘗試提取 `lolwut.exe`（放在客戶端目錄）

### 7. Wireshark 抓包實戰：登入流程詳細解析

**封包序列**（v83）：
```
1. C→S Hello Packet (plaintext)
   - 告知版本、SendIV、RecvIV
   - Opcode = 0x00 (實際上 Hello Packet 沒有 opcode header)

2. S→C Hello Packet (plaintext)
   - 確認版本、回應 IV

3. C→S LOGIN_PASSWORD (encrypted)
   Opcode: 0x01
   Data: 帳號字串 + 密碼字串

4. S→C LOGIN_STATUS (encrypted)
   Opcode: 0x00
   Data: 結果碼 (0=成功, 5=帳密錯誤, 23=過太久)

5. C→S SERVERLIST_REQUEST (encrypted)
   Opcode: 0x06

6. S→C SERVERLIST (encrypted)
   Opcode: 0x0C
   Data: 世界伺服器列表

7. C→S CHARLIST_REQUEST (encrypted)
   Opcode: 0x07

8. S→C CHARLIST (encrypted)
   Opcode: 0x03
   Data: 角色資料陣列

9. C→S CHAR_SELECT (encrypted)
   Opcode: 0x08

10. C→S PLAYER_DROP (encrypted)
    Opcode: 0x0B
```

### 8. v83 登入結果碼（Login Status Code）

| 結果碼 | 意義 |
|--------|------|
| 0 | 登入成功 |
| 5 | 帳號或密碼錯誤 |
| 7 | 帳號已停用 |
| 13 | 刪除中（冷靜期） |
| 23 | 登入太頻繁（防機器人） |
| 其他 | 未知錯誤 |

### 10. 動態 NPC 商店腳本 v83（RaGEZONE 2016）

**來源**：https://forum.ragezone.com/threads/dynamic-npc-shop-script-v83.1089979/

**特點**：
- 不需要修改 SQL 資料庫
- 完全基於 NPC 腳本
- 只支援裝備物品（Equippable items）
- 動態陣列定義商品

**原理**：
- 在 JavaScript 陣列中定義 `[itemId, price, quantity]`
- 腳本計算價格、檢查庫存、給予物品
- 缺少的功能：購買數量選擇（只能一次買一個）
