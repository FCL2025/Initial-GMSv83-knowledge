## 2. 抓包分析

### 🆕 MapleStory 封包加密核心原理 (pkg.go.dev/maplelib, 2026)

**來源**: https://pkg.go.dev/github.com/defaultmagi/maplelib

**重要原理**：
> MapleStory uses two keys: one is used to encrypt sent packets and the other is used to decrypt received packets. The two keys are randomly generated when a client connects and sent in the unencrypted handshake packet which is the first packet.

**兩把鑰匙**：
- **Send Key**：客戶端發送時使用，伺服器接收時解密
- **Recv Key**：伺服器發送時使用，客戶端接收時解密
- 兩把鑰匙在 Hello Packet 中明文交換

### 🆕 完整登入握手流程 (botnored, 2026)

**來源**: https://botnored.weebly.com/blog/maplestory-classes-v83

```
1. C→S: Hello Packet (plaintext)
   - 版本號
   - SendIV (客戶端指定的發送 IV)
   - RecvIV (客戶端指定的接收 IV)
   - 地區標誌

2. S→C: Hello Packet (plaintext)
   - 版本確認
   - RecvIV (伺服器使用的接收 IV)
   - SendIV (伺服器使用的發送 IV)

3. C→S: 登入請求 (encrypted)
   Opcode: 0x01 (LOGIN_PASSWORD)

4. S→C: 登入結果 (encrypted)
   Opcode: 0x00 (LOGIN_STATUS)
```

### 🆕 MapleStory v83 Opcode 結構 (mapleref.fandom.com, 2026)

**來源**: https://mapleref.fandom.com/wiki/Packets

**核心概念**：
> "Packet headers are 4 bytes long, and are generated with the use of the send IV, packet length (without header) and the version of MapleStory. The version is -(version + 1) when sending a packet to the client, otherwise just the plain version. Packet opcodes are usually 2 bytes long."

**封包標頭結構**：
```
[4-byte MapleCrypto Header]
  - 混淆後的長度
  - 混淆後的 opcode

[2-byte Opcode] (little endian)
[Data...]
```

## 3. Opcode 結構

### 🆕 MapleRef Wiki Send vs Recv Opcode 核心概念 (2026)

**來源**: https://mapleref.fandom.com/wiki/Operation_Code

**Send/Recv 區分**：
| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

**重要**：同一個數字的 Opcode 在不同方向的意義完全不同！

### 🆕 v83 常用 Opcode 速查表

**登入相關 (C→S)**：
| Opcode | 名稱 | 說明 |
|--------|------|------|
| 0x01 | LOGIN_PASSWORD | 登入請求 |
| 0x06 | SERVERLIST_REQUEST | 請求伺服器列表 |
| 0x07 | CHARLIST_REQUEST | 請求角色列表 |
| 0x08 | CHAR_SELECT | 選擇角色 |
| 0x0B | PLAYER_DROP | 進入遊戲 |

**登入相關 (S→C)**：
| Opcode | 名稱 | 說明 |
|--------|------|------|
| 0x00 | LOGIN_STATUS | 登入結果 |
| 0x03 | CHARLIST | 角色列表 |
| 0x0C | SERVERLIST | 伺服器列表 |

**遊戲相關**：
| Opcode | 名稱 | 說明 |
|--------|------|------|

### 🆕 MapleCrypto Header 混淆演算法 (v83)

```java
// 加密端
public byte[] getPacketHeader(int opcode, int length, int iv) {
    int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
    verify = (verify << 3) | (verify >>> 5);
    verify &= 0xFFFF;
    return new byte[] {
        (byte) (verify & 0xFF),
        (byte) ((verify >>> 8) & 0xFF),
        (byte) (length & 0xFF),
        (byte) ((length >>> 8) & 0xFF)
    };
}
```

### 🆕 MapleCrypto Header 解密演算法 (v83)

```java
// 解密端
public int retrieveOpcode(byte[] header, int iv) {
    int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
    verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
    return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
}

public int retrieveLength(byte[] header, int iv) {
    int length = (header[2] & 0xFF) | ((header[3] << 8) & 0xFF00);
    length ^= (iv >> 16) & 0xFFFF;
    return length;
}
```

## 5. NPC 腳本編寫

### 🆕 DAMA 技術 - 最基礎 NPC 腳本教學 (2023)

**來源**: https://dama.tw/threads/188/

#### 基本模板
```javascript
function start() {
    cm.sendOk("歡迎來到我的商店！我有各種道具和裝備。");
    cm.dispose();
}
```

#### 帶選項的腳本
```javascript
function start() {
    cm.sendSimple("歡迎來到我的商店！\r\n#b#L0#購買道具#l\r\n#L1#購買裝備#l");
}

function action(mode, type, selection) {
    if (mode == 1) {
        if (selection == 0) {
            cm.openShop(100);
        } else if (selection == 1) {
            cm.openShop(200);
        }
    }
    cm.dispose();
}
```

#### 常用函數速查
```javascript
// 對話
cm.sendOk("文字")
cm.sendNext("文字")
cm.sendSimple("文字\r\n#L0#選項#l")
cm.sendYesNo("文字")

// 玩家操作
cm.getPlayer()        // 取得玩家
cm.getMap()           // 取得地圖
cm.getParty()         // 取得隊伍
cm.getMeso()          // 取得金錢
cm.gainMeso(amount)   // 給予/收取金錢
cm.gainItem(itemId, qty)  // 給予物品
cm.openShop(shopId)   // 開商店
cm.warp(mapId, portal)    // 傳送
cm.dispose()           // 結束對話
```

### 🆕 NPC 腳本狀態機詳解

```
start() → action(1, 0, 0)  // 初始化，status = -1 後變 0

mode == 1 → status++
mode == 0 → status--

status == 0 → 第一個 sendNext/sendSimple
status == 1 → 第二個（玩家回應後）
...
mode == -1 → dispose() 結束
```
