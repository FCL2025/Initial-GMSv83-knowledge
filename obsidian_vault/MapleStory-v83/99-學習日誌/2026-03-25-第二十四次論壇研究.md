```
來源：https://forum.ragezone.com/threads/packet-editing-for-maplestory.1212354/

> "If you truly are looking to learn how the game works internally
> (and not trying to use PE on a server) I suggest you to learn how to use IDA"

當你需要：
- 修改客戶端本身時 → 需要 IDA 分析
- 架設私服時 → 直接用已映射的 Opcode 表

IDA 分析Opcode 的關鍵函數：
1. CClientSocket::ProcessPacket - 接收封包入口
2. CLogin::OnPacket - 登入畫面封包處理
3. CField::OnPacket - 遊戲中封包處理
4. SendPacket / RecvPacket - 封包收發函數
```

## 📌 NPC 腳本研究

### Quest NPC 與 MigrateIn 封包 (RaGEZONE 2015)
```
來源：https://forum.ragezone.com/threads/how-to-script-quest-npcs-v83.1063962/

核心概念：
- Quest NPC 需要發送 MigrateIn 封包才能在 "Other" 選項顯示
- MigrateIn 封包包含 NPC 清單 + FileTime（腳本結束時間）
- 封包在 PlayerLoggedInHandler 中發送

Quest NPC 的 MigrateIn 封包格式：
  int[] npcIds       // NPC ID 陣列
  int fileTime       // 腳本結束時間（毫秒時間戳）

這樣玩家在 NPC 的 "Other" 選項中就能看到自訂腳本
自訂腳本位於 scripts/npc/ 目錄
```

### NPC 腳本源間相容性最新討論 (RaGEZONE 2026-03)
```
來源：https://forum.ragezone.com/threads/kinoko-npc-scripts.1261538/

問題：
- 不同音源（Cosmic、Kinoko、HeavenMS）的腳本解析器版本不同
- 較舊的腳本在新解析器可能不相容
- 主要問題：`importPackage` 和 `SavedLocationType`

最佳實踐：
| 寫法 | 狀態 |
|------|------|
| `importPackage(Packages.xxx)` | ❌ JDK 7+ 廢棄 |
| `var X = Packages.xxx.X` | ✅ 相容 |
| `cm.getXxx()` | ✅ 普遍支援 |

建議：
- 使用普遍支援的 `cm.` 函數
- 避免特定音源專有的 Package 路徑
- 先在本地測試再部署
```

### Dynamic NPC Shop Script 完整範例 (RaGEZONE 2016)
```javascript
/*
 * 動態 NPC 商店腳本 - 不需要 SQL
 * NPC ID: 9010009
 * 優點：每次修改不需重啟伺服器
 */
var status = 0;
var items = Array(
    Array(1302000, 10000, 1),   // [物品ID, 價格, 庫存]
    Array(1302001, 20000, 1),
    Array(1302002, 30000, 1),
    Array(1302003, 50000, 1)
);

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        
        if (mode == 1)
            status++;
        else
            status--;
        
        if (status == 0) {
            var msg = "歡迎光臨！\r\n請選擇商品：\r\n";
            for (var i = 0; i < items.length; i++) {
                msg += "#L" + i + "##i" + items[i][0] + "# - " + items[i][1] + " 金幣#l\r\n";
            }
            cm.sendSimple(msg);
        } else if (status == 1) {
            var item = items[selection];
            if (cm.getMeso() >= item[1]) {
                if (cm.canHold(item[0])) {
                    cm.gainMeso(-item[1]);
                    cm.gainItem(item[0], item[2]);
                    cm.sendOk("購買成功！");
                } else {
                    cm.sendOk("背包空間不足！");
                }
            } else {
                cm.sendOk("金幣不足！");
            }
            cm.dispose();
        }
    }
}
```

## 📌 完整 v83 架設流程摘要 (RaGEZONE 2025-11)

```
來源：https://forum.ragezone.com/threads/video-guide-how-to-make-a-maplestory-v83-private-server-–-full-setup-tutorial.1243566/

Step 1: 準備軟體
- JDK 8 (必要)
- JCE 8 (Java Cryptography Extension)
- MySQL Server 5.5
- v83 localhost 客戶端

Step 2: 架設伺服器
- 選擇音源（Cosmic / HeavenMS / MapleSolaxiaV2）
- 設定 MySQL 資料庫
- 設定 wz 路徑
- 設定端口（7575, 8484）

Step 3: 修改客戶端
- HxD 修改 IP 為 127.0.0.1 (localhost)
- 設定 port 8484
- 測試連線

Step 4: WZ 同步
- 客戶端 WZ → 導出 Private Server XML → 覆蓋伺服器 wz 目錄
- 或使用音源提供的 custom WZ

Step 5: 測試與調整
- 測試登入流程
- 確認封包交換正常
- 調整遊戲設定
```
