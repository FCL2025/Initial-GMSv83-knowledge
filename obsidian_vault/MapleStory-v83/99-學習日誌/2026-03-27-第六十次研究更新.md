## 📝 本次新增內容

### 1) 登入器/客戶端修改

**STREDIT 2.9.2.0 最新用法**:
- 支援 v118+ 的 IP 編輯
- v83 用舊版或 EMS 版本
- 流程: 開啟 → 確認版本 → IP Search → Force to IP → 儲存

**HxD Hex 編輯（通用方法）**:
- Ctrl+F 搜尋 "127.0.0.1"
- 三個 IP 全部替換
- 注意 IP 長度差異

**MapleEzorsia v2 dinput8.dll**:
- 史上第一個獨立 dinput8.dll
- config.ini 配置 IP/解析度/自訂 DLL
- 放入遊戲目錄自動加載

### 2) 抓包分析

**Maple_Pshark** (GitHub: obstriker/Maple_Pshark):
- Hook 加密函數，直接截獲明文
- 封包修改、注入、規則過濾
- Recv/Send 雙向支援

**MapleProxy** (GitHub: tylerhasman/MapleProxy):
- 封包攔截、修改、重放、過濾
- 登入模擬功能

**MapleShark** (GitHub: diamondo25/MapleShark):
- SharpPcap 網路截獲
- 圖形化封包解析

**MaplePE** (GitHub: zhyonc/MaplePE):
- 32-bit 封包編輯器
- 從進程提取 InPacket/OutPacket
- Opcode 過濾

### 3) Opcode 結構

**Send vs Recv 核心區分**:
- S→C = Server Opcode Set
- C→S = Client Opcode Set
- 同一數字可能完全不同含義

**CClientSocket::ProcessPacket 分發流程**:
- 所有封包經過同一入口
- switch(opcode) 分發到 CLogin::OnPacket 或 CField::OnPacket

**MapleCrypto 頭部混淆公式**:
- getPacketHeader: verify = (opcode ^ iv[0]) + (iv[1] << 8), rotate left 3
- retrieveOpcode: 反向操作

**v83 三層加密**:
- Layer 1: MapleCrypto (頭部)
- Layer 2: Shanda (對稱加密, v118+ 移除)
- Layer 3: AES-OFB (主加密)

### 4) WZ 修改

**HaRepacker 核心原則**:
- 不要用伺服器 XML 重新導入客戶端 WZ
- 使用 GMS (old) 加密格式
- 直接 UI 修改，不要刪除/重新導入節點

**lvmax 移除（Notepad++ Regex）**:
- 搜尋: lvmax="\d+"\s*
- 取代: （留空）

**WzImg-MCP-Server** (2026-03-05 最新):
- AI 輔助 WZ 分析工具
- 74 tools across 10 categories
- Claude Desktop MCP 配置

### 5) NPC 腳本編寫

**狀態機核心**:
- mode == -1: 結束
- mode == 1: 下一步
- mode == 0: 返回

**sendXXX API 速查**:
- sendNext, sendOk, sendYesNo, sendSimple
- sendGetNumber, sendGetText, sendAcceptDecline
- 所有最後都要 dispose()

**動態 NPC 商店**:
- 不需 SQL
- Array 存物品和價格
- 完全相容所有音源

**相容性**:
- ✅ 完全不要 importPackage
- ✅ 使用 cm. 通用函數
- ✅ Nashorn ES6 (Cosmic/IntransigentMS)
