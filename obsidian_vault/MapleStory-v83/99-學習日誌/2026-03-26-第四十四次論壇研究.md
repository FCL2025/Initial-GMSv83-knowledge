### v83 完整登入封包序列（實戰總結）

```
1. C→S Hello Packet (plaintext - 無 opcode header)
   - 告知版本、SendIV、RecvIV

2. S→C Hello Packet (plaintext)
   - 伺服器確認版本、回應 IV

3. C→S LOGIN_PASSWORD (encrypted) - Opcode: 0x01
   - 帳號 + 密碼 (AES-OFB 加密)

4. S→C LOGIN_STATUS (encrypted) - Opcode: 0x00
   - 結果碼 + 帳號資訊

5. C→S SERVERLIST_REQUEST (encrypted) - Opcode: 0x06
   - 請求世界伺服器列表

6. S→C SERVERLIST (encrypted) - Opcode: 0x0C
   - 世界伺服器名稱、負載、人數上限

7. C→S CHARLIST_REQUEST (encrypted) - Opcode: 0x07
   - 請求指定世界的角色列表

8. S→C CHARLIST (encrypted) - Opcode: 0x03
   - 角色資料陣列（名稱、等級、職業、外觀）

9. C→S CHAR_SELECT (encrypted) - Opcode: 0x08
   - 選擇角色

10. C→S PLAYER_DROP (encrypted) - Opcode: 0x0B
    - 進入遊戲世界
```

## 3. Opcode 結構

### 三層加密架構（v83 核心知識）

```
┌─────────────────────────────────────────────────────┐
│ Layer 1: MapleCrypto (頭部混淆)                     │
│   - 4 bytes header (混淆 opcode + length)           │
│                                                        │
│ Layer 2: Shanda Encryption (對稱加密)                │
│   - 早期版本使用，已於 v118+ 移除                   │
│                                                        │
│ Layer 3: AES-OFB (主加密)                          │
│   - v83 主要加密方式                                │
│   - 256-bit key, 4-byte IV                          │
│   - 版本決定的靜態金鑰                              │
└─────────────────────────────────────────────────────┘
```

### MapleCrypto 頭部混淆公式

```java
// 加密端
int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
verify = (verify << 3) | (verify >>> 5);  // rotate left 3 bits
verify &= 0xFFFF;
return new byte[] {
    (byte) (verify & 0xFF),
    (byte) ((verify >>> 8) & 0xFF),
    (byte) (length & 0xFF),
    (byte) ((length >>> 8) & 0xFF)
};

// 解密端
int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
```

### CInPacket 封包接收完整流程

```cpp
void __thiscall CClientSocket::ProcessPacket(CClientSocket *this) {
    while (true) {
        // 讀取 4-byte header
        BYTE header[4];
        recv(socket, header, 4);
        
        // MapleCrypto 解混淆
        int packetLen = MapleCrypto::decryptLength(header, ivRecv);
        int opcode = MapleCrypto::decryptOpcode(header, ivRecv);
        
        // 分配 buffer
        byte *buffer = new byte[packetLen];
        recv(socket, buffer, packetLen);
        
        // AES-OFB + Shanda 解密
        MapleCrypto::decrypt(buffer, packetLen, ivRecv);
        
        // 分發到處理函數
        switch (opcode) {
            case 0x00: CLogin::OnPacket(&packet); break;
            case 0x01: CLogin::OnLoginPassword(&packet); break;
            // ...
        }
    }
}
```

### Send vs Receive Opcode 區分

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

**重要**：同一個 Opcode 數字可能代表不同含義，視方向而定！

### 動態配方系統腳本源（最新）

```javascript
/*
 * 動態配方系統 - 玩家提交材料換取成品
 * NPC ID: 9010010
 * 完全相容所有音源
 */
var status = 0;

var recipes = Array(
    {name: "楓之葉地圖", materials: [[4000000, 10], [4000001, 5]], result: [[2300000, 1]], meso: 1000},
    {name: "傳說武器", materials: [[4003000, 100], [4003001, 50]], result: [[1302000, 1]], meso: 50000}
);

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) { cm.dispose(); return; }
    if (mode == 0 && type == 1) { cm.dispose(); return; }
    if (mode == 1) status++; else status--;

    if (status == 0) {
        var msg = "歡迎來到工藝系統！\r\n請選擇配方：\r\n";
        for (var i = 0; i < recipes.length; i++) {
            msg += "#L" + i + "#" + recipes[i].name + "#l\r\n";
        }
        cm.sendNext(msg);
    } else if (status == 1) {
        var recipe = recipes[selection];
        var canCraft = true;
        var msg = "你需要以下材料：\r\n";
        for (var i = 0; i < recipe.materials.length; i++) {
            var mat = recipe.materials[i];
            var have = cm.getQuantityOfItem(mat[0]);
            var need = mat[1];
            msg += "#i" + mat[0] + "# #t" + mat[0] + "#: " + have + "/" + need + "\r\n";
            if (have < need) canCraft = false;
        }
        if (cm.getMesos() < recipe.meso) canCraft = false;
        msg += "加工費: " + recipe.meso + " 金幣\r\n";
        msg += canCraft ? "#g可以製作#k" : "#r材料不足#k";
        cm.sendYesNo(msg);
    } else if (status == 2) {
        var recipe = recipes[selection];
        for (var i = 0; i < recipe.materials.length; i++) {
            cm.gainItem(recipe.materials[i][0], -recipe.materials[i][1]);
        }
        cm.gainMesos(-recipe.meso);
        for (var i = 0; i < recipe.result.length; i++) {
            cm.gainItem(recipe.result[i][0], recipe.result[i][1]);
        }
        cm.sendOk("製作成功！");
        cm.dispose();
    }
}
```

### Quest NPC MigrateIn 封包機制

**問題**：如何讓 Quest NPC 出現在 "Other" 選項中？

**解決方案**：
- 需要發送 `PlayerMigrateInPacket` 封包
- 封包包含：NPC ID 列表 + FileTime
- 這個封包讓客戶端知道哪些 NPC 有自訂腳本

**實現位置**：
```java
// 在 PlayerLoggedInHandler 或 similar handler 中
int fileTime = 自定義腳本結束時間; // 毫秒時間戳
List<Integer> npcIds = Arrays.asList(9000000, 9000001, 9000002);
sendPacket(PlayerMigrateInPacket.encode(npcIds, fileTime));
```
