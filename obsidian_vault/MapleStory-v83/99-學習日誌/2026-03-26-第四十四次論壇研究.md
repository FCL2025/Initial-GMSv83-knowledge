## 一、登入器/客戶端修改 — MapleEzorsia v2 DLL 方案 (RaGEZONE 2023, GitHub)

**GitHub**: https://github.com/444Ro666/MapleEzorsia-v2
**Wiki**: https://github.com/444Ro666/MapleEzorsia-v2/wiki/v83%E2%80%90Client%E2%80%90Setup%E2%80%90and%E2%80%90Development%E2%80%90Guide

### 核心定位
這是**史上第一個 v83 獨立 DLL 客戶端** cr@ck，可將官方 Nexon 預設客戶端直接轉換為 HD 客戶端 + localhost 私服客戶端。

### 重要特性

| 特性 | 說明 |
|------|------|
| 無需注入器 | MapleStory 自動加載 `dinput8.dll`，無需額外工具 |
| 一體化 | 同時完成：去除保護 + HD 解析度 + localhost 導向 |
| 可自訂解析度 | 擺脫 800x600 限制，支援任意解析度 |
| 完全開源 | C++ 源代碼可在 GitHub 查看 |
| 不破壞原廠 exe | 使用 DLL Hook，不修改原始程式 |

### 與傳統 localhost.exe 比較

| 項目 | localhost.exe | MapleEzorsia v2 (dinput8.dll) |
|------|-------------|------------------------------|
| 脫殼 | 需自行脫殼 | DLL 自動完成 |
| 解析度 | 固定 800x600 | 任意解析度 |
| 兼容性 | 需特定版本 | 適用於任何 v83 客戶端 |
| 維護 | 舊版已停更 | 持續更新 |

### 使用方式（超簡單）
```
1. 從 Nexon 安裝原始 v83 客戶端
2. 將 dinput8.dll 放入遊戲目錄
3. 啟動 MapleStory.exe（自動加載 DLL）
4. 享受 HD 解析度 + localhost 連線
```

### HD 客戶端/localhost 版本發布 (Reddit, Oct 2023)

**Reddit**: https://www.reddit.com/r/mapleservers/comments/1756qgk/releasev83_hd_clientlocalhost_release_for_use/

Reddit 用戶分享了 Hendi's 4g client + Cosmic 的 v83 HD 客戶端設定指南，這是目前最優化、最安全的 v83 HD 方案。

### 關鍵發現：dinput8.dll 自動加載原理

**原理**：
- Windows 遊戲程式會自動加載 `dinput8.dll`（DirectInput 8）
- MapleEzorsia v2 透過置換 `dinput8.dll`，在 DLL 中實現：
  1. 記憶體位址修改
  2. 解析度修改
  3. localhost IP 導向
- 這是完全合法的 DLL hijacking，不需要修改遊戲本體

## 三、Opcode結構 — IDA 分析 + RustMS 實現

### MapleRef Wiki Send/Recv Opcode 核心概念

**來源**: https://mapleref.fandom.com/wiki/Operation_Code

> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送到客戶端 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送到伺服器 | Client Opcode Set |

**重要**：同一個 Opcode 數字可能代表不同含義，視方向而定！

### v83 Opcode 現況 (RaGEZONE, 2024-12)

**來源**: https://forum.ragezone.com/threads/packet-opcode-analysis.1238027/

> "If you truly are looking to learn how the game works internally (and not trying to use PE on a server) I suggest you to learn how to use IDA"

**核心結論**：
- v83 的 Opcode 幾乎全部被映射，不需要額外追蹤封包
- 大多數 opcode 表已經在開源專案（Cosmic、HeavenMS）中完整提供
- 唯一需要的反向工程是**修改客戶端時**

### IDA 分析 Opcode 進階技巧 (RaGEZONE)

**來源**: https://forum.ragezone.com/threads/getting-packet-structures-opcodes-using-ida.792436/

**分析關鍵**：
1. 找 `CClientSocket::ProcessPacket` - 客戶端接收封包的總入口
2. 從這裡根據 header 分發到 `CField::OnPacket` 或 `CLogin::OnPacket`
3. 這就是為什麼有 opcode 清單就能輕鬆找到客戶端接收封包的處理函數

## 四、WZ修改 — HaRepacker + Quest LvMax Removal

### WZ 編輯概念 (RaGEZONE, 2013)

**來源**: https://forum.ragezone.com/threads/tutorial-wz-editing-concepts-and-tutorial.838526/

**WZ 編輯的定義**：
- WZ 編輯是指修改 Wizet Files（MapleStory 使用的資源檔案格式）
- 這些檔案包含遊戲的各種資源：地圖、NPC、道具、怪物等

**WZ 編輯的局限性**：
- 並非所有遊戲邏輯都在 WZ 中
- 某些功能需要修改客戶端本身

### Quest 等級限制移除（LvMax Removal）Notepad++ Regex (RaGEZONE, 2025-07)

**來源**: https://forum.ragezone.com/threads/maplestory-v83-method-to-disable-quest-level-restrictions.1250107/

**批量移除所有 lvmax 的 Notepad++ Regex**：
```
尋找：lvmax="\d+"\s*
取代：（留空）
```

**XML 修改範例**：
```xml
<!-- 修改前 -->
<quest id="1000" lvmin="1" lvmax="30">
    ...
</quest>

<!-- 修改後（移除 lvmax） -->
<quest id="1000" lvmin="1">
    ...
</quest>
```

### WZ 編輯黃金法則 (MapleSolaxiaV2 GitHub)

> DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs.

**正確流程**：
1. HaRepacker 4.2.4 開啟 **GMS (old)** 加密的客戶端 WZ
2. **使用 UI 直接修改數值**（禁止刪除/重新導入節點！）
3. 儲存 WZ
4. 在 HaRepacker 中選擇 **"Private Server..."** 導出
5. 將導出的 XML 覆蓋到伺服器的 WZ.XML 目錄

### 創建自訂 NPC 完整流程

```
1. HaRepacker 導出 NPC.wz 為 XML
2. 在 XML 中新增自訂 NPC 的 frame 和 origin 設定
3. 複製 0001337.img 作為模板，修改尺寸
4. 將修改後的 img 放入 NPC.wz
5. 在伺服器端編寫對應的 .js 腳本
```

### NPC Tags WZ 編輯位置

在 HaRepacker 中：`NPC.wz` → `NpcId.img` → `info` → `tags`
- 這是 NPC 名稱下的額外描述文字
- 翻譯時需同步修改 `String.wz` 中對應的條目

### Community WZ/IMG Repack for v83 (RaGEZONE, 2023)

**來源**: https://forum.ragezone.com/threads/community-wz-img-repack-for-v83.1221835/

```
導出步驟：
1. 用 HaRepacker 打開 WZ 檔案
2. 選擇要導出的節點
3. 點擊 Export → 選擇 XML (Private Server) 格式
4. 導出後可使用文字編輯器修改
5. 重新導入 WZ
```

**注意**：
- v83 WZ 加密格式需選擇 `GMS (old)`
- 不要直接刪除/重新導入節點，可能造成數據丟失
- 建議使用 HaRepacker UI 直接編輯數值

### ✅ 相容寫法（所有音源通用）
```javascript
// JavaScript 基本寫法 - 適用所有音源
var status = 0;

function start() {
    cm.sendNext("你好！");
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
        return;
    }
    if (mode == 1) status++;
    else status--;
    
    if (status == 0) {
        cm.sendSimple("選擇：\r\n#b#L0#選項1#l\r\n#L1#選項2#l");
    } else if (status == 1) {
        if (selection == 0) {
            cm.gainItem(1302000, 1);
            cm.sendOk("獲得物品！");
        }
        cm.dispose();
    }
}
```

### ❌ 不相容寫法（JDK 7+ 已廢棄）
```javascript
// ❌ 不相容 (JDK 7+ 已廢棄)
importPackage(Packages.client);
importPackage(Packages.server);

// ✅ 相容寫法 - 完全不要使用 importPackage
// 直接使用 cm. 函數即可
```

### Nashorn ECMAScript 6 進階寫法（Cosmic / IntransigentMS）
```javascript
// 使用 const / let（區塊作用域）
const NPC_ID = 9010000;
let status = 0;

// 使用箭頭函數
const items = [1302000, 1302001, 1302002];

// 使用模板字串
cm.sendNext(`歡迎來到 ${NPC_ID} NPC！`);

// 使用 Java.type（替代 importPackage）
const MapleCharacter = Java.type("net.sf.odinms.client.MapleCharacter");
```
