## 1. 登入器/客戶端修改

### MapleSolaxiaV2 完整架設流程 (2026 最新)

**來源**: https://github.com/P0nk/Cosmic-client

**標準客戶端安裝流程（推薦）**：
```
1. 安裝階段
   下載 MapleGlobal-v83-setup.exe 並執行
   安裝到 C:\Nexon\MapleStory 或自選目錄

2. 清理階段
   刪除 HShield/ 資料夾（整個）
   刪除 ASPLnchr.exe
   刪除 MapleStory.exe
   刪除 Patcher.exe

3. WZ 替換階段
   複製 cosmic-wz 中的所有 WZ 檔案
   覆蓋到安裝目錄

4. 客戶端複製階段
   複製 HeavenMS-localhost-WINDOW.exe
   到安裝目錄

5. IP 設定階段
   （如果需要非 127.0.0.1）
   使用 HxD 修改 localhost.exe 中的 IP

6. 啟動階段
   先啟動伺服器
   然後執行 localhost.exe
```

### localhost.exe Hex 編輯注意事項

| 項目 | 說明 |
|------|------|
| 需找到幾個 IP？ | 通常 3 個（全部需修改） |
| 為什麼長度要一致？ | 否則可能破壞其他資料結構 |
| 如何確認成功？ | 重新搜尋，確認已全數替換 |

### 客戶端被標記為病毒的解決方案

```
問題：Windows Security 將 localhost.exe 標記為威脅並刪除

原因：修改過的 MapleStory 客戶端會被識別為潛在風險

解決方案：
1. Windows Security → Virus & threat protection
2. 管理設定 → 新增排除項目
3. 加入下載資料夾和安裝目錄
```

## 3. Opcode 結構

### MapleCrypto + Shanda + AES-OFB 三層加密架構

```
┌─────────────────────────────────────────────────────┐
│ Layer 1: MapleCrypto (頭部混淆)                     │
│   - 4 bytes header (混淆 opcode + length)           │
│   - 防止靜態分析 opcode                            │
│                                                        │
│ Layer 2: Shanda Encryption (對稱加密)                │
│   - 早期版本使用                                    │
│   - 已於 v118+ 移除                                │
│                                                        │
│ Layer 3: AES-OFB (主加密)                           │
│   - v83 主要加密方式                                │
│   - 256-bit key, 4-byte IV                          │
│   - 版本決定的靜態金鑰                              │
└─────────────────────────────────────────────────────┘
```

### MapleCrypto Header 加密/解密公式

```java
// 加密端 (getPacketHeader)
int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
verify = (verify << 3) | (verify >>> 5);
verify &= 0xFFFF;
return new byte[] {
    (byte) (verify & 0xFF),
    (byte) ((verify >>> 8) & 0xFF),
    (byte) (length & 0xFF),
    (byte) ((length >>> 8) & 0xFF)
};

// 解密端 (retrieveOpcode)
int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
```

### v83 封包加密三層架構

```
發送端 (C→S):
明文資料
  → AES-OFB 加密 (使用版本 Key + IV)
  → Shanda 加密 (對稱加密)
  → MapleCrypto 頭部混淆 (4-byte header)
  → 網路傳輸

接收端 (S→C):
網路接收
  → MapleCrypto 解混淆 (提取 opcode + length)
  → Shanda 解密
  → AES-OFB 解密
  → 明文資料
```

### 排查解密失敗的檢查清單

| 檢查項目 | 正確值 | 症狀 |
|---------|--------|------|
| AES Key | 版本決定的 32-byte key | 解密出乱码 |
| IV | Hello Packet 協商的 IV | opcode 不匹配 |
| Shanda Key | 版本特定的對稱金鑰 | 資料順序錯誤 |
| 版本號 | 0x0053 (83) | Hello Packet 失敗 |
| Opcode 表 | 需與客戶端匹配 | switch case 走错分支 |

### HaRepacker WZ 編輯黃金法則

> **DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate bugs.**

**正確流程**：
```
1. HaRepacker 4.2.4 開啟 GMS (old) 加密的客戶端 WZ

3. 儲存 WZ
4. HaRepacker → "Private Server..." 導出
5. 將導出的 XML 覆蓋到伺服器的 WZ.XML 目錄
```

### NX 格式 - WZ 的高效替代方案

**WzToNx.exe 使用方式**：
```
將 .wz 檔案拖曳到 WzToNx.exe 上
自動在同一目錄生成 .nx 檔案
```

**NX 格式優點**：
| 優點 | 說明 |
|------|------|
| 檔案更小 | 比 WZ 節省 30-50% 空間 |

| 效能更好 | 減少 CPU 和記憶體使用 |
| Git 友好 | 差異更容易追蹤 |

### 配方系統腳本源（動態，無需 SQL）

```javascript
/*
 * 動態配方系統 - 玩家提交材料換取成品
 * NPC ID: 9010010
 */
var status = 0;

var recipes = Array(
    {name: "楓之葉地圖", materials: [[4000000, 10]], result: [[2300000, 1]], meso: 1000}
);

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
        return;
    }
    if (mode == 0 && type == 1) {
        cm.dispose();
        return;
    }
    if (mode == 1) status++;
    else status--;

    if (status == 0) {
        var msg = "歡迎來到工藝系統！\r\n請選擇配方：\r\n";
        for (var i = 0; i < recipes.length; i++) {
            msg += "#L" + i + "#" + recipes[i].name + "#l\r\n";
        }
        cm.sendNext(msg);
    } else if (status == 1) {
        var recipe = recipes[selection];
        var canCraft = true;
        var msg = "你需要以下材料：\r\n";
        for (var i = 0; i < recipe.materials.length; i++) {
            var mat = recipe.materials[i];
            var have = cm.getQuantityOfItem(mat[0]);
            var need = mat[1];
            msg += "#i" + mat[0] + "# #t" + mat[0] + "#: " + have + "/" + need + "\r\n";
            if (have < need) canCraft = false;
        }
        if (cm.getMesos() < recipe.meso) canCraft = false;
        msg += "加工費: " + recipe.meso + " 金幣\r\n";
        msg += canCraft ? "#g可以製作#k" : "#r材料不足#k";
        cm.sendYesNo(msg);
    } else if (status == 2) {
        var recipe = recipes[selection];
        for (var i = 0; i < recipe.materials.length; i++) {
            cm.gainItem(recipe.materials[i][0], -recipe.materials[i][1]);
        }
        cm.gainMesos(-recipe.meso);
        for (var i = 0; i < recipe.result.length; i++) {
            cm.gainItem(recipe.result[i][0], recipe.result[i][1]);
        }
        cm.sendOk("製作成功！");
        cm.dispose();
    }
}
```
