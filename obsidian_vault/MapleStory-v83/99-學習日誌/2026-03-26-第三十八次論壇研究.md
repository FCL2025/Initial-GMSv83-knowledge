## 🆕 Opcode 結構 - 深度更新

### v83 Opcode 現況 (RaGEZONE, 2024-12)

**核心結論**：

### MapleCrypto 頭部混淆公式

**加密端 (getPacketHeader)**：
```java
int verify = (opcode ^ iv[0]) + (iv[1] << 8);
verify = (verify << 3) | (verify >>> 5);  // rotate left 3 bits
verify &= 0xFFFF;
```

**解密端 (retrieveOpcode)**：
```java
int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
return (verify - (iv[1] << 8)) ^ iv[0];
```

### MapleCrypto 雙金鑰機制 (maplelib, 2026-03)

| IV | 擁有者 | 用途 |
|----|--------|------|
| 客戶端 SendIV | 客戶端 | 加密發送到伺服器的封包 |
| 客戶端 RecvIV | 客戶端 | 解密從伺服器收到的封包 |
| 伺服器 SendIV | 伺服器 | 加密發送到客戶端的封包 |
| 伺服器 RecvIV | 伺服器 | 解密從客戶端收到的封包 |

### 三層加密架構

```
┌─────────────────────────────────────────────────────┐
│ Layer 1: MapleCrypto (頭部混淆)                      │
│   - 4 bytes header (混淆 opcode + length)             │
│                                                        │
│ Layer 2: Shanda Encryption (對稱加密)                │
│   - v83 仍使用，已於 v118+ 移除                       │
│                                                        │
│ Layer 3: AES-OFB (主加密)                            │
│   - 256-bit key, 4-byte IV                           │
└─────────────────────────────────────────────────────┘
```

### 動態配方系統腳本源

```javascript
/*
 * 動態配方系統 - 玩家提交材料換取成品
 * NPC ID: 9010010
 */
var status = 0;

var recipes = Array(
    {
        name: "楓之葉地圖",
        materials: Array(Array(4000000, 10), Array(4000001, 5)),
        result: Array(Array(2300000, 1)),
        mesoCost: 1000
    }
);

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else {
        if (mode == 0 && type == 1) {
            cm.dispose();
            return;
        }
        if (mode == 1) status++;
        else status--;

        if (status == 0) {
            var msg = "歡迎來到工藝系統！\r\n請選擇配方：\r\n";
            for (var i = 0; i < recipes.length; i++) {
                msg += "#L" + i + "#" + recipes[i].name + "#l\r\n";
            }
            cm.sendNext(msg);
        } else if (status == 1) {
            var recipe = recipes[selection];
            var canCraft = true;
            var msg = "你需要以下材料：\r\n";
            for (var i = 0; i < recipe.materials.length; i++) {
                var mat = recipe.materials[i];
                var have = cm.getQuantityOfItem(mat[0]);
                var need = mat[1];
                msg += "#i" + mat[0] + "# #t" + mat[0] + "#: " + have + "/" + need + "\r\n";
                if (have < need) canCraft = false;
            }
            if (cm.getMesos() < recipe.mesoCost) canCraft = false;
            msg += "加工費: " + recipe.mesoCost + " 金幣\r\n";
            msg += canCraft ? "#g可以製作#k" : "#r材料不足#k";
            cm.sendYesNo(msg);
        } else if (status == 2) {
            var recipe = recipes[selection];
            for (var i = 0; i < recipe.materials.length; i++) {
                cm.gainItem(recipe.materials[i][0], -recipe.materials[i][1]);
            }
            cm.gainMesos(-recipe.mesoCost);
            for (var i = 0; i < recipe.result.length; i++) {
                cm.gainItem(recipe.result[i][0], recipe.result[i][1]);
            }
            cm.sendOk("製作成功！");
            cm.dispose();
        }
    }
}
```

### NPC 腳本除錯：常見崩潰原因

| 原因 | 解決方案 |
|------|---------|
| `cm.dispose()` 在狀態機中途被呼叫 | 確保 dispose() 只在對話結束時呼叫 |
| 格式標籤無效 | 確保 `#b` → `#k`，`#r` → `#k` 正確配對 |
| 選項數量與 selection 不匹配 | 確保 selection 在有效範圍內 |
