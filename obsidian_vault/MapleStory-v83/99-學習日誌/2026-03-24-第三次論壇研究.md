### 2. 抓包分析

#### 🆕 MapleStory Packet Structure Analyzer
GitHub: https://github.com/Bratah123/MaplePacketPuller
- Python 撰寫的 MapleStory 封包結構分析器
- 適用於多种版本

#### 🆕 Maple_Pshark (v83 專用)
GitHub: https://github.com/obstriker/Maple_Pshark
- MapleRoyals 等私人伺服器使用的封包 Logger
- 專為 v83 設計

#### 第一個封包特性 (重要)
> **第一個發送/接收的封包永遠是純文字（未加密）**
- 用於協商 MapleCrypto 參數 (IV)
- TCP 協定傳輸

#### 封包加密識別徵兆
- 封包長度固定（如每次都 48 bytes）
- 內容看起來像隨機位元組
- 相同操作產生的封包每次都不同（IV 變化）

#### Wireshark 過濾表達式
```
tcp.port == 7575 || tcp.port == 8484
```

#### Wireshark 顏色規則建議
| 顏色 | 條件 | 說明 |
|------|------|------|
| 紅色 | tcp.len > 2000 | 大型封包 |
| 綠色 | ip.src == 127.0.0.1 | 客戶端發送 |
| 藍色 | ip.dst == 127.0.0.1 | 伺服器接收 |
| 黃色 | tcp.port == 7575 | 登入端口 |

### 3. Opcode 結構

#### 🆕 MapleRef WikiOpcode 結構說明
https://mapleref.fandom.com/wiki/Operation_Code

**Opcode 基礎**：
- Opcode 是封包開頭的 2 bytes，用於識別封包類型
- 在伺服器端通常放在枚舉（enum）中方便管理

#### MapleCrypto Header 加密原理
```
封包頭部結構 (4 bytes):
  [2 bytes: 混淆後的長度 + opcode]
  [2 bytes: 實際長度]

Packet Header 生成方式：
  version = -(version + 1)  (發送到客戶端時)
  packetLength = 4 + dataLength
```

#### 封包加密流程
```
1. 客戶端發送 Hello (純文字)
   - 包含：版本號、SendIV、RecvIV、地區標誌

2. 伺服器回應 Hello (純文字)
   - 確認版本、回應 IV

3. 後續封包全部加密 (AES-OFB)
```

#### 🆕 IDA 分析Opcode 的方法
RaGEZONE 教程：https://forum.ragezone.com/threads/getting-packet-structures-opcodes-using-ida.792436/

**IDA 分析關鍵點**：
- 找 `CClientSocket::ProcessPacket` 函數
- 客戶端接收封包流向：`CClientSocket::ProcessPacket → CField::OnPacket / CLogin::OnPacket`
- AOB (Array of Bytes) 搜尋可找到 opcode 表

#### 🆕 IDA 分析封包結構
RaGEZONE: https://forum.ragezone.com/threads/how-does-one-find-a-structure-of-a-packet-via-ida.1162654/

**分析流程**：
1. 找到 SendPacket / RecvPacket 函數
2. 分析 MapleCrypto 邏輯
3. 提取 Opcode 表
4. 對比伺服器端代碼

#### 重要Opcode 參考 (v83)
| Opcode | 名稱 | 說明 |
|--------|------|------|
| 0x00 | LOGIN_STATUS | 登入結果 |
| 0x01 | LOGIN_PASSWORD | 登入請求 |
| 0x03 | CHARLIST | 角色列表 |
| 0x06 | SERVERLIST_REQUEST | 請求伺服器列表 |
| 0x07 | CHARLIST_REQUEST | 請求角色列表 |
| 0x08 | CHAR_SELECT | 選擇角色 |
| 0x0B | PLAYER_DROP | 進入遊戲 |
| 0x14 | MOVE_PLAYER | 移動 |
| 0x15 | CLOSE_RANGE_ATTACK | 近距離攻擊 |
| 0x16 | RANGED_ATTACK | 遠距離攻擊 |
| 0x28 | NPC_TALK | NPC 對話 |

#### 夫妻訊息Opcode (v83)
- 接收：0x79 (OnCoupleMessage)
- 發送：0x88

### 4. WZ 修改

#### 🆕 Cosmic WZ 修改工作流
Cosmic README: https://github.com/P0nk/Cosmic

**重要聲明**：
> ⚠️ 這個伺服器需要自訂 WZ 文件（因為安裝客戶端時可能已包含）。意圖是未來能完全在原始 WZ 上運行。

#### HaRepacker-resurrected 使用流程
https://github.com/lastbattle/Harepacker-resurrected

**編輯客戶端 WZ**：
1. 用 HaRepacker-resurrected 開啟 .wz，encryption: **GMS (old)**
2. 在節點階層中進行修改（複製/貼上節點在罕見情況下可能不可靠）
3. 儲存覆寫原來的 WZ 內容

**匯出到伺服器 XML**：
1. 使用「Private Server」匯出選項
2. 將 XML 覆寫到伺服器的 `wz` 目錄
3. **永遠從客戶端 WZ 匯出到伺服器 XML，不要反向操作！**

**編輯風險**：
- 刪除/重新匯入節點可能導致數據丟失
- 只使用 UI 進行修改，定期備份

#### 🆕 Skill.wz 修改 (Reddit 2024)
Reddit: https://www.reddit.com/r/mapleservers/comments/19cz6eu/v83_skillwz_editing/

> ⚠️ 修改 Skill.wZ 需要使用組合語言。Ollydbg 是免費的 x86 調試器。

#### 🆕 動態 NPC 商店腳本 (RaGEZONE 2016)
https://forum.ragezone.com/threads/dynamic-npc-shop-script-v83.1089979/

**特點**：
- 不需要 SQL 資料庫
- 每次修改無需重啟伺服器
- 使用 JavaScript 陣列定義商品

#### NPC 腳本常見問題

**importPackage 錯誤**：
- `importPackage` 在 JDK 7+ 廢棄
- 解決：移除或註解掉 `importPackage` 這一行

**SavedLocationType 錯誤**：
```
Exception: ReferenceError: "SavedLocationType" is not defined
原因：某些來源使用 JDK 6 編譯
解決：註解掉或移除相關程式碼
```

#### NPC 腳本最佳實踐
1. 永遠在最後一個分支呼叫 `cm.dispose()`
2. 使用完整路徑而非 `importPackage`
3. 狀態機確保邏輯清晰
