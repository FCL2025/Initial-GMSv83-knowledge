### 1. MapleStory v83 架設完整流程更新 (RaGEZONE 2025)

**主要參考**：https://forum.ragezone.com/threads/video-guide-how-to-make-a-maplestory-v83-private-server-–-full-setup-tutorial.1243566/

**完整架設環境需求**：
- JDK 8 + JCE (Java Cryptography Extension)
- MySQL Server 5.5+
- v83 localhost 客戶端
- HxD 或 STREDIT（IP 修改工具）

**客戶端設定流程**：
```
1. 用 ManagerMsv83.exe 安裝 MapleStory 到目錄
2. 刪除 HShield 資料夾、ASPLnchr.exe、MapleStory.exe、patcher.exe
3. 用 localhost.exe 替換啟動器
4. 用修改後的 WZ 檔案覆蓋原版
5. 如需修改 IP，使用 HxD 或 STREDIT 編輯 localhost.exe
```

**伺服器設定流程**：
```
1. MySQL Query Browser 建立 session (localhost, port 3306, user: root)
2. 依序執行 sql/db_database.sql → db_drops.sql → db_shopupdate.sql (可選)
3. 在 accounts 表格手動建立第一個帳號
4. 修改 configuration.ini 中的 IP 設定
5. NetBeans 開啟專案 → 設定 Libraries → Clean and Build
6. 執行 launch.bat 啟動伺服器
```

### 2. MapleEzorsia - DLL 注入式客戶端修改 (2026)

**來源**：https://github.com/izarooni/MapleEzorsia

**技術特點**：
- 使用 Microsoft Detours API 進行記憶體注入
- 無需修改原始 .exe 檔案
- 可自訂解析度、視窗模式、聊天記錄行數等

**安裝方式**：
```
1. 將 nmconew.dll 改名為 nmconew2.dll
2. 將 MapleEzorsia.dll 或 config.ini 放入遊戲目錄
3. 啟動客戶端即可
```

**config.ini 範例**：
```ini
[general]
width=1280
height=720
discord=false
debug=false
```

### 3. WZ 修改與 HaRepacker 最佳實踐 (RaGEZONE 2023)

**來源**：https://forum.ragezone.com/threads/community-wz-img-repack-for-v83.1221835/

**重要原則**：
- v83 WZ 加密格式需選擇 `GMS (old)`
- 不要直接刪除/重新導入節點，可能造成數據丟失
- 建議使用 HaRepacker UI 直接編輯數值

**HaRepacker 導出 XML 步驟**：
```
1. 打開 WZ 檔案
2. 選擇要導出的節點
3. Export → XML (Private Server) 格式
4. 用文字編輯器修改
5. 重新導入 WZ
```

**WZ 編輯器比較**：
| 編輯器 | 優點 | 缺點 |
|--------|------|------|
| HaRepacker 4.2.4 | 最多人用，相容性好 | 介面舊 |
| GMS WZ Editor | 支援新格式 | 需找乾淨版本 |
| 111beyond | 輕量級 | 功能有限 |

### 4. NPC 腳本源間相容性最佳實踐 (RaGEZONE 2026-03)

**來源**：https://forum.ragezone.com/threads/kinoko-npc-scripts.1261538/

**核心問題**：不同音源（Cosmic、Kinoko、HeavenMS）的腳本解析器存在差異

**解決方案**：
1. ✅ 使用普遍支援的 `cm.` 函數
2. ❌ 避免使用特定音源專有的 Package 路徑
3. ❌ 避免 `importPackage`（JDK 7+ 已廢棄）
4. ✅ 先在本地測試再部署

**相容性檢查清單**：
| 檢查項目 | 狀態 |
|---------|------|
| 無 `importPackage` | ✅ |
| 使用完整 Package 路徑 | ✅ |
| 使用 `cm.` 而非直接類別名 | ✅ |
| `SavedLocationType` 已處理 | ✅ |

### 5. v83 完整 Opcode 對照表更新

**主要參考**：RaGEZONE + MapleRef Wiki

#### 登入相關 Opcode
| Opcode (Hex) | 名稱 | 方向 | 說明 |
|-------------|------|------|------|
| 0x00 | LOGIN_STATUS | S→C | 登入結果回應 |
| 0x01 | LOGIN_PASSWORD | C→S | 登入請求（帳號+密碼） |
| 0x02 | ACCEPT_LOGIN | S→C | 同意登入 |
| 0x05 | SELECT_CHARLIST | C→S | 選擇角色列表 |
| 0x06 | SERVERLIST_REQUEST | C→S | 請求世界伺服器列表 |
| 0x07 | CHARLIST_REQUEST | C→S | 請求指定世界的角色列表 |
| 0x08 | CHAR_SELECT | C→S | 選擇角色 |
| 0x0B | PLAYER_DROP | C→S | 進入遊戲世界 |

#### 遊戲內 Opcode
| Opcode (Hex) | 名稱 | 說明 |
|-------------|------|------|
| 0x03 | CHARLIST | 角色列表 |
| 0x04 | CHAR_INFO | 角色資訊 |
| 0x14 | MOVE_PLAYER | 移動封包 |
| 0x15 | CLOSE_RANGE_ATTACK | 近距離攻擊 |
| 0x16 | RANGED_ATTACK | 遠距離攻擊 |
| 0x28 | NPC_TALK | NPC 對話 |
| 0x6A | MOB_APPLY_CONTROL | 怪物生成+HP顯示 |
| 0x6F | SHOP_BUY | 商場購買 |

### 6. 封包加密三層架構總結

```
┌─────────────────────────────────────────────────────┐
│ Layer 1: MapleCrypto (頭部混淆)                     │
│   - 4 bytes header (混淆 opcode + length)           │
│   - verify = (opcode ^ iv[0]) + (iv[1] << 8)       │
│   - verify = (verify << 3) | (verify >> 5)         │
│                                                        │
│ Layer 2: Shanda Encryption (對稱加密)                │
│   - 早期版本使用                                     │
│   - 已於 v118+ 移除                                 │
│                                                        │
│ Layer 3: AES-OFB (主加密)                           │
│   - v83 主要加密方式                                │
│   - 256-bit key, 4-byte IV                          │
└─────────────────────────────────────────────────────┘
```
