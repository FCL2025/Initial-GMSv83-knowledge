## 1. 登入器 / 客戶端修改 — 最新論壇動態

### 1.1 MapleSolaxiaV2 完整客戶端部署流程（詳細）

**來源**: https://github.com/skhzhang/MapleSolaxiaV2

**重要發現 — 客戶端安裝順序至關重要**：

```
標準 v83 客戶端安裝流程：
1. 使用 ManagerMsv83.exe 安裝原始 MapleStory v83
   → 安裝至 e.g. C:\Nexon\MapleStory
   
2. 刪除以下檔案（防作弊檢測）：
   - HShield/ (資料夾)
   - ASPLauncher.exe
   - MapleStory.exe  ← 注意：會被 localhost.exe 替換
   - patcher.exe

3. 解壓 localhost.exe 至客戶端資料夾

4. 用提供的 client_wz 覆蓋原廠 WZ 檔案

5. 若伺服器 IP 非 127.0.0.1：
   → 用 STREDIT 或 Hex Editor 編輯 localhost.exe 內的 IP
```

**重要警告**（RaGEZONE 2026-03）：
> "DO NOT USE the server's XMLs for reimporting into the client's WZ, it WILL generate some kind of bugs afterwards."
> → 必須分開管理 client WZ 和 server WZ XML

### 1.2 STREDIT — localhost.exe IP 修改（v83 標準工具）

**來源**: RaGEZONE - STREDIT Release Thread

**STREDIT v2.9.2.0 功能**：
- 修改 localhost 中的加密 URL 字串
- 修改 IP 地址（支援 Hamachi / 固定 IP）
- 修改 ADS URL / 忘記密碼 URL
- 自動處理 v83 等舊版本的加密格式

**STREDIT 修改 v83 IP 步驟**：
```
1. 下載 STREDIT (RaGEZONE f921)
2. 開啟 STREDIT → File → Open → 選擇 localhost.exe
3. 出現提示「是否 v118+？」→ 選「No」（v83 使用舊加密）
4. 出現提示「IP 搜尋模式？」→ 選「Yes」
5. 點擊「Force to a IP」按鈕
6. 輸入目標 IP（如 192.168.1.100 或 Hamachi IP）
7. 保存並關閉
```

**無法使用 STREDIT 時的替代方案**（Hex Editor）：
```
1. 用 16 進制編輯器開啟 localhost.exe
2. 搜尋字串 "127.0.0.1"（可能以 Unicode 或加密形式儲存）
3. 找到後替換為目標 IP（保持相同位元組長度）
4. 保存並關閉
```

```go
initVector := [4]byte{}
rand.Read(initVector[:])
crypt := maplelib.NewCrypt(initVector, 62)  // version=62

p := maplelib.NewPacket()
```


### 2.5 MapleCrypto 加密機制深入解析

**來源**: maplelib 原始碼 + mapleref.fandom.com

**關鍵概念**：
- **兩組金鑰**：客戶端發送用一組，接收用另一組
- **隨機 IV**：連線時生成，透過第一個未加密封包交換
- **頭部混淆**：前 4 bytes 為加密頭部（包含長度 + 混淆後 opcode）

**封包頭部格式**：
```
偏移  大小  內容
0x00  4     MapleCrypto 混淆頭部
      (包含：封包長度 + verify 值)
Opcode 位置 2 bytes (v83) 或 1 byte (更舊版本)
```

**Packet Header 生成規則**：
```
version = -(version + 1)  // 發送到客戶端時
version = version         // 發送到伺服器時
verify = (opcode ^ iv[0]) + (iv[1] << 8)
verify = (verify << 3) | (verify >> 13)  // rotate left 3 bits
```

## 3. Opcode 結構 — v83 封包通信底層

### 3.1 Opcode 基礎概念

**來源**: mapleref.fandom.com + RaGEZONE 研究

**Opcode 定義**：
> "An Operation Code (aliases: Opcode, Header) is the first bytes in a packet to help identify which packet is received or sent in MapleStory"

**v83 Opcode 特性**：
- **長度**：2 bytes (little endian)
- **格式**：例如 `0x0001` 在封包中為 `01 00`
- **版本差異**：v83 已高度文檔化，但每個私服可能自定義

### 3.2 IDA 分析 Opcode 處理流程

**來源**: RaGEZONE - "How does one find a structure of a packet via IDA"

**客戶端接收封包流程**（CClientSocket）：
```
CClientSocket::ProcessPacket
  → 讀取 header
  → 根據 opcode 分發到 CField::OnPacket / CLogin::OnPacket
  → 各自處理函數
```

**伺服器端封包處理（Java 風格）**：
```
PacketProcessor (舊式, v83 常用)
  → 枚舉所有 handler
  → 根據 opcode 查找對應 handler
  → 執行 handle() 方法
```

### 3.3 MapleProtocol — Opcode 枚舉與定義

**GitHub**: https://github.com/diamondo25/MultiShark/blob/master/Protocols/MapleStory/MapleProtocol.cs

**Opcode 命名規範（以 MapleProtocol.cs 為例）**：
```csharp
// C→S (Client to Server) opcodes
LOGIN_REQUEST = 0x0001,
LOGIN_CHECK_PASSWORD = 0x0002,
ENTER_SERVER = 0x0003,
CHAR_SELECT = 0x0004,
...

// S→C (Server to Client) opcores
LOGIN_STATUS = 0x0001,
SERVERLIST = 0x0002,
CHARLIST = 0x0003,
ENTER_PORTAL = 0x0004,
...
```

**檔案路徑命名約定**：
```
packet.Locale + "/" + packet.Version + "/" 
  + (packet.Outbound ? "Outbound" : "Inbound") 
  + "/0x" + Opcode.ToString("X4") + ".txt"
```

### 3.4 PacketErrorHandler — 錯誤封包識別

**來源**: RaGEZONE - v8x PacketErrorHandler Thread

**功能**：在伺服器端識別處理失敗的封包

**v83 已知錯誤 opcode**：
```
0x19 = 玩家地圖 + 錯誤碼
（用於診斷客戶端/伺服器同步問題）
```

### 3.5 第一個封包一定是純文字（Hello Packet）

**來源**: botnored.weebly.com MapleStory classes v83

**關鍵發現**：
> "The first packet sent / received will always be in plain-text (not encrypted)"

**意義**：
- 可以直接用 Wireshark 看到第一個封包的內容
- 第一個封包用於交換加密 IV（金鑰協商）
- 之後所有封包都經過 MapleCrypto 加密

## 4. WZ 修改 — 工具與方法

### 4.1 HaRepacker — 標準 WZ 編輯器

**GitHub**: https://github.com/lastbattle/Harepacker-resurrected

**v83 WZ 加密格式**：
- 必須使用 **GMS (old)** 加密類型（HaRepacker 4.2.3+）
- v117+ 使用 **BMS** 加密
- 混用加密會導致 WZ 無法讀取

**HaRepacker 標準編輯流程**：
```
1. File → Load → 選擇 .wz 檔
2. 導航至目標 img（e.g., String.wz → NPC → 00001337.img）
3. 修改屬性值（int, float, string, canvas 等）
4. File → Save → 覆蓋原檔
5. 若需匯出 XML → Export → Private Server...
```

### 4.2 WZ 編輯注意事項（v83）

**來源**: MapleSolaxiaV2 README

**客戶端 vs 伺服器 WZ 管理**：
```
⚠️ 重要原則：
- 客戶端 WZ 和伺服器 WZ XML 是分開的！
- 絕對不要用伺服器的 XML 匯入客戶端 WZ
- 會造成不可預期的 bug

正確流程：
Client WZ 修改：HaRepacker → 直接編輯 → 保存
Server WZ XML：單獨管理，匯出時用 "Private Server" 模式
```

**WZ 結構概覽**：
```
MapleStory.wz/           — 主要遊戲資源
  Character.wz/          — 角色、裝備屬性
  Item.wz/               — 物品定義
  Mob.wz/                — 怪物定義
  Npc.wz/                — NPC 圖像
  Skill.wz/              — 技能數據
  String.wz/             — 文字字串（NPC名、物品名、任務描述等）
  Map.wz/                — 地圖瓷磚與物件
  Sound.wz/              — 音效
  etc.
```

**流程**：

```
HaRepacker → Load String.wz
```

### 4.4 物品屬性修改（Character.wz / Item.wz）

```javascript
// 基礎對話框
```

### 5.2 NPC 狀態機制 — status 變數

```javascript
var status = -1;

function action(mode, type, selection) {
    if (mode == -1) {
        // 關閉對話框（按 X 或 ESC）
        cm.dispose();
        return;
    }
    
    if (mode == 0) {
        // 按「返回」/「否」
        status--;
    } else {
        // 按「下一步」/「是」
        status++;
    }
    
    if (status == 0) {
        cm.sendNext("這是第一句話...");
    } else if (status == 1) {
        cm.sendYesNo("你想要任務嗎？");
    } else if (status == 2) {
        // 處理選擇結果
        cm.gainItem(4000000, 1);  // 給予任務物品
        cm.gainExp(100);          // 給予經驗值
        cm.sendOk("任務完成！");
    } else {
        cm.dispose();
    }
}
```

### 5.3 Quest NPC 腳本模式

**來源**: RaGEZONE - "How to script Quest NPCs v83"

**Quest NPC vs 普通 NPC 的區別**：
- Quest NPC 會有路燈泡圖示（任務可接取提示）
- 需要在 WZ 中設置 `quest` 標誌
- 伺服器端需要發送 ` MigrateIn` / `PlayerLoggedInHandler` 封包，包含 NPC 腳本終止時間

**Quest NPC 腳本結構**：
```javascript
var status = -1;

function start() {
    status = 0;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
        return;
    }
    
    if (mode == 0) {
        status--;
    } else {
        status++;
    }
    
    if (status == 0) {
        if (cm.getPlayer().getQuestNAdd(\$id_to_check).getStatus() != 1) {
            // 任務未開始
            cm.sendYesNo("你想要接受 #r任務名稱#k 嗎？");
        } else {
            cm.sendOk("任務還在進行中。");
            cm.dispose();
        }
    } else if (status == 1) {
        // 玩家選擇了 Yes
        cm.startQuest(\$quest_id);  // 開始任務
        cm.sendNext("去吧！收集 #t4000000# x 10");
    } else {
        cm.dispose();
    }
}
```

### 5.4 動態 NPC 商店腳本（非 SQL）

**來源**: RaGEZONE - "Dynamic NPC Shop Script (v83)"

**優點**：
- 不需要修改 SQL 資料庫
- 不需要重啟伺服器
- 隨時更新商店內容

**實現方式**：
- 在 NPC 腳本中直接定義商品列表
- 使用 `cm.openShop(shopId)` 或自訂 UI

**範例**：
```javascript
var shopItems = [
    [2000005, 1000, 1],   // [物品ID, 價格, 數量]
    [2000006, 2000, 1],
    [4000000, 500, 10]
];

function action(mode, type, selection) {
    if (mode == 1) {
        // 處理購買
        var itemId = shopItems[selection][0];
        var price = shopItems[selection][1];
        if (cm.getMeso() >= price) {
            cm.gainMeso(-price);
            cm.gainItem(itemId, 1);
        } else {
            cm.sendOk("楓幣不足！");
        }
    } else {
        cm.dispose();
    }
}
```

### 5.5 NPC Type 值含義

| Type 值 | 客戶端行為 | 腳本需求 |
|---------|-----------|---------|
| `0` | 普通 NPC（點擊觸發對話） | 需要 `.js` 腳本 |
| `1` | 商店 NPC（直接開商店介面） | 可能需要 `.js` |
| `2-6` | 拍賣/特殊功能 NPC | 通常不需要腳本 |

**查看位置**：HaRepacker → `NPC.wz` → `XXX.img` → `info` → `type`

### 5.6 NPC 腳本除錯技巧

```
常見錯誤：
1. 忘記 cm.dispose() → 對話框無法關閉
2. status 溢位（過多狀態）→ 無限迴圈
3. cm.gainItem() 物品ID錯誤 → 玩家收不到物品
4. 任務狀態判斷失誤 → 選項按鈕邏輯錯誤

除錯方法：
- 在 cm.sendNext() 加入 cm.getPlayer().dropMessage(6, "DEBUG: status=" + status);
- 檢查伺服器日誌中的 Script Error
- 使用 Wireshark 確認客戶端確實收到對話封包
```
