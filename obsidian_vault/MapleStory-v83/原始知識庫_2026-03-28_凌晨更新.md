## 二、Opcode結構 — MaplePE 技術實現深度解析

### MaplePE 完整技術棧（GitHub 原文 2026-03-28 直接抓取）

**GitHub**: https://github.com/zhyonc/MaplePE

**技術架構**：
```
MaplePE.exe (UI + UDP Server)
    ↓
Packet.dll (注入目標進程)
    ↓ Hook 封包讀寫函數
UDP Client 發送封包資訊
```

**技術棧**：
| 組件 | 技術 |
|------|------|
| UI | C++ MFC + MFC-GridCtrl |
| 網路 | WinSock2 |
| Hook | Detours (微軟官方) |
| 編譯 | Visual Studio 2022 (x86 & x64) |

**需要 Hook 的函數完整列表**：

| 接收端 (CInPacket) | 發送端 (COutPacket) | Socket |
|-------------------|-------------------|--------|
| Decode1 | Encode1 | ProcessPacket |
| Decode2 | Encode2 | SendPacket |
| Decode4 | Encode4 | |
| Decode8 (optional) | Encode8 (optional) | |
| DecodeStr | EncodeStr | |
| DecodeBuffer | EncodeBuffer | |
| | MakeBufferList | |

**崩潰防護機制（BB+ 版本）**：

> "Since some versions after BB add hook-detection in CClientSocket::SendPacket, Hook MakeBufferList instead of SendPacket to avoid client crash when sending OutPacket after DLL injection"

使用 Cheat Engine 將崩潰條件填為 `90` (NOP) 來禁用：
- `(int)&loc_7FFFFE + 4198402` 範圍外的 retaddr → `JUMPOUT(0)` → 第1次崩潰
- `retaddr == m_pLock` → `MEMORY[0](this, a2)` → 第2次崩潰
- 之後 3~6 次崩潰同樣模式

**版本位址參考**：
- Google Sheets: https://docs.google.com/spreadsheets/d/1-LqA-OWgOuT-Xy3i73Jy5F_uVmogaGvmYpH4mLDSPhg

**類似工具**：
- [RirePE](https://github.com/Riremito/RirePE) — 類似功能
- [KaedeEditor](https://github.com/Riremito/KaedeEditor) — AOB 掃描提供函數位址
