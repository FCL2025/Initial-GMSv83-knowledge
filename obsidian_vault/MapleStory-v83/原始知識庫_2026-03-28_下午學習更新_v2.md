## 五、NPC腳本編寫

### 核心重點
- 使用 **JavaScript/ECMAScript**（不是Java！）— JDK 8+ 的 Nashorn引擎
- `status` = 追蹤對話進度狀態機
- `mode` = 處理按鈕點擊

### NPC TYPE值
| TYPE | 說明 |
|------|------|
| 0 | 普通NPC（需要.js腳本）|
| 1 | 商店NPC |
| 2-6 | 特殊NPC（不需要腳本）|

### 方法列表（NPCConversationManager）
- `sendNext()`, `sendSimple()`, `sendYesNo()`
- `gainItem()`, `warp()`
- `startQuest()`, `completeQuest()`

### 兼容性注意
- ✅ **Cosmic/Nashorn** = 最佳兼容性
- ❌ `importPackage` 已廢棄（JDK 7+）
- ✅ ES6支援：`const`, `let`, template strings, arrow functions

### 動態NPC商店
- 不需要SQL，直接用.js陣列（RaGEZONE 2016）
- 參考：https://forum.ragezone.com/threads/dynamic-npc-shop-script-v83.1089979/

### 重要封包
- **Quest lvmax removal**: https://forum.ragezone.com/threads/maplestory-v83-method-to-disable-quest-level-restrictions.1250107/
- **Quest NPC MigrateIn packet** = "Other"分頁NPC可見性必備

### 參考腳本
- IntransigentMS (GitHub): 完整ES6 Nashorn參考
