### 客戶端 IP 修改 — Hex Editor 標準流程

**URL**: https://forum.ragezone.com/threads/how-to-make-a-maplestory-private-server-v83.759889/

#### v83 localhost.exe IP 修改步驟
```
1. 使用 Neo Hex Editor 或 HxD 開啟 localhost.exe
2. 搜尋 "Text String" 中的 "127.0.0.1"
   → Hex: 31 37 32 2E 30 2E 30 2E 31
3. 找到 3 個 IP 位址，全部替換為目標 IP
   → 若目標 IP 比 127.0.0.1 長，剩餘空間填 00
4. 保存並關閉
```

**重要**：「IP 位址中的點（.）在 Hex Editor 中表示為 2E，而非 00」

## 🆕 抓包分析 — 最新工具與技術 (2026-03-28)

### 封包加密/解密雙向流程圖

```
發送端 (COutPacket):
  Encode1/Encode2/Encode4/EncodeStr
       ↓
  MapleCrypto (IV shift + 混淆)
       ↓
  AES-OFB 加密 (MapleAES)
       ↓
  Socket 傳輸

接收端 (CInPacket):
  Socket 接收
       ↓
  AES-OFB 解密 (MapleAES)
       ↓
  MapleCrypto (反向 IV shift)
       ↓
  Decode1/Decode2/Decode4/DecodeStr
```

### 握手指餐 (Hello Packet) — 第一個封包是明文

根據 maplelib 文件 (Go Packages):

> "MapleStory uses two keys: one is used to encrypt sent packets and the other is used to decrypt received packets. The two keys are randomly generated when a client connects and sent in the unencrypted handshake packet which is the first packet."

**握手指餐**：
```
1. 客戶端連線到 localhost:8484
2. 伺服器發送 handshake（含版本號 + IV）
3. 客戶端回應（更換 IV）
4. 之後所有通訊使用 MapleStory 自訂加密 + AES
```

## 🆕 Opcode 結構 — 詳細技術分析 (2026-03-28)

### v83 Opcode 基礎

根據 MapleRef Wiki:

> "Actions that are handled by packets on MapleStory are defined by an identifier called an Operation Code (opcode for short), which is the first 2 bytes of the packet (a short), though in earlier versions it used to be the first byte (a byte)."

**v83 = 2 bytes Opcode**

### 封包格式

```
[Header: 4 bytes][Opcode: 2 bytes][Data: 可變]

Header = [長度(2 bytes)][混淆後的 Opcode(2 bytes)]
Opcode = 識別封包類型的 2 bytes 值
Data   = 實際內容，根據封包類型不同而變化
```

### Maple_Pshark Opcode 過濾功能

Maple_Pshark 支援在 Hook 層面過濾特定 Opcode：
- 減少日誌雜訊
- 只關注感興趣的封包
- 適合分析特定功能（如 NPC 對話、物品操作）

### IDA 獲取 Opcode 的方法

**靜態分析**：
1. IDA 打開 MapleStory.exe
2. 找到 `CLogin::OnPacket` (switch case)
3. 分析每個 case 對應的 opcode

**動態分析**：
1. 在 `CInPacket::Decode2` 設斷點
2. 觀察每次 Decode 的實際 opcode 值
3. 交叉驗證靜態分析的結果

### RustMS 加密實現 — 學習範本

**URL**: https://github.com/neeerp/RustMS

RustMS 開源實現了完整的 MapleStory 加密：

| 加密層 | Rust 實現 | 說明 |
|--------|----------|------|
| MapleAES (OFB) | `maple_aes.rs` | AES-OFB 模式主要加密 |
| Shanda | `shanda.rs` | 對稱加密演算法 |
| bcrypt | Rust bcrypt crate | 密碼雜湊 |

### v83 登入相關 Opcode (參考)

| Opcode | 名稱 | 方向 | 說明 |
|--------|------|------|------|
| 0x01 | LOGIN_REQUEST | C→S | 登入請求 |
| 0x02 | LOGIN_RESPONSE | S→C | 登入回應 |
| 0x03 | CHECK_PASSWORD | C→S | 密碼驗證 |
| 0x04 | CHECK_PASSWORD_RESULT | S→C | 密碼驗證結果 |

> ⚠️ 具體 Opcode 值可能因私服而異，需使用工具實際抓包確認。

### ✅ 通用相容寫法

```javascript
// 所有音源通用 - 建議使用
var status = 0;

function start() {
    cm.sendNext("你好！");
}

function action(mode, type, selection) {
    if (mode == -1) { cm.dispose(); return; }
    if (mode == 1) status++; else status--;
    
    if (status == 0) {
        cm.sendSimple("選擇：\r\n#b#L0#選項1#l\r\n#L1#選項2#l");
    } else if (status == 1) {
        if (selection == 0) {
            cm.gainItem(1302000, 1);
            cm.sendOk("獲得物品！");
        }
        cm.dispose();
    }
}
```

### ❌ 不相容寫法（JDK 7+ 已廢棄）

```javascript
// ❌ 不相容 - 移除這些
importPackage(Packages.client);
importPackage(Packages.server);

// ✅ 替代方案
var MapleCharacter = Packages.client.MapleCharacter;
```

### NPC 腳本核心觀念

| 概念 | 說明 |
|------|------|
| `status` | 狀態機，追蹤對話進度 |
| `mode == 1` | 玩家點擊「下一步」/「是」 |
| `mode == 0` | 玩家點擊「返回」/「否」 |
| `mode == -1` | 玩家關閉對話框 |
| `cm.dispose()` | 結束腳本，必須在最後呼叫 |

### NPC 腳本除錯：常見崩潰原因

| 問題 | 原因 | 解決方案 |
|------|------|---------|
| 客戶端崩潰 | `cm.dispose()` 在狀態機中途被呼叫 | 確認 dispose 只在對話結束時呼叫 |
| 無效格式標籤 | `#b` 沒有 `#k` 重置 | 確保格式標籤正確配對 |
| 選項範圍錯誤 | selection 超出範圍 | 確認選項數量與處理匹配 |
| 無窮對話迴圈 | 忘記 status++ | 每個分支都需要狀態遞增 |
