### 2.2 localhost.exe IP 修改（jonnylin13/Maple83）

**完整流程**（來源: Maple83 GitHub README）：

1. 使用 `ManagerMsv83.exe` 安裝 MapleStory 到 `C:\Nexon\MapleStory`
2. 刪除：`HShield`資料夾、`ASPLauncher.exe`、`MapleStory.exe`、`patcher.exe`
3. 用 `localhost.exe` 替換
4. 用 `client_wz` 覆蓋遊戲目錄中的WZ檔案
5. **HEX編輯IP**：
   - 用 `Neo Hex Editor` 或 `HxD` 開啟 `localhost.exe`
   - `Ctrl+F` → 選擇 "Text String" → 搜索 `127.0.0.1`
   - 找到後替換為目標IP（長度需相同，如需不同長度用 `00` 填充）

## 三、抓包分析

### 3.1 MaplePacketPuller — IDA 偽代碼分析工具

**GitHub**: https://github.com/Bratah123/MaplePacketPuller
**Python 3.8+ | IDA Pro 7.0+**

這是一個 Python 工具，用於分析 IDA 生成的 C 偽代碼，提取封包結構。

#### 核心功能
| 功能 | 說明 |
|------|------|
| 關鍵詞讀取 | 從 IDA 偽代碼格式化為服務端 encode 格式 |
| 封包結構解析 | 顯示封包的完整結構 |
| Opcode 搜索 | 在 Header 中搜索 Opcode |
| 輸出檔案 | 輸出到 `IDA Maple Script/FuncOutput/` |

#### 使用流程
```
1. 在 IDA 中對封包處理函數進行反編譯
2. 複製偽代碼到 .txt 文件（位於 MaplePacketPuller/IDA Maple Script/Functions/）
3. 運行 main.py（位於 src/main/python/）
4. 選擇分析類型
5. 程式輸出封包結構到 FuncOutput/
```

#### 限制
- 不處理 `while()` 迴圈嵌套（可設 `GET_ALL_DECODES = true` 改善）
- 不顯示 switch case（需用戶自行添加）
- IDA 6.8 不相容（使用 `os.scandir()`）

#### 配套工具
建議配合 **SpiritIDAPlugin** 使用：
- GitHub: `Bratah123/SpiritIDAPlugin`
- 自動格式化 IDA 輸出為服務器友好的格式

### 3.2 Maple_Pshark — v83 專用封包攔截器

**GitHub**: https://github.com/obstriker/Maple_Pshark

| 功能 | 說明 |
|------|------|
| Recv訊息攔截 | 監控伺服器發送給客戶端的封包 |
| 訊息修改 | 動態修改封包內容 |
| 封包注入 | 向 Send/Recv 通道注入封包 |
| 規則過濾 | 基於規則的封包過濾與操作 |
| 自動Opcode識別 | 不需要手動解析Opcode |

**原理**：鉤住加密函數（`IOBufferManipulator.Encrypt`），在加密前/後攔截訊息。

### 3.3 封包加密三層架構（v83）

```
┌──────────────────────────────────────────────┐
│ 1. MapleCrypto (頭部混淆)                    │
│    Header 4 bytes = 長度 + 混淆Opcode        │
│                                              │
│ 2. Shanda Encryption (IOBufferManipulator)   │
│    在 AES 之前應用                           │
│    v118+ 已移除                              │
│                                              │
│ 3. AES-OFB (主加密)                          │
│    主要資料加密，IV 每次更新                  │
└──────────────────────────────────────────────┘
```

## 四、Opcode 結構

### 4.1 v83 Opcode 特點

- **長度**: 2 bytes (little endian)
- **位置**: Header 之後
- **混淆**: MapleCrypto 頭部混淆
- **文檔化程度**: v83 Opcode 幾乎全部被映射

### 4.2 MapleCrypto 頭部混淆原理

```
原始Opcode: 0x0001
IV: [0x7A, 0x52, ...]

Step 1: XOR + SHIFT
  verify = (opcode ^ iv[0]) + (iv[1] << 8)
        = (0x0001 ^ 0x7A) + (0x52 << 8)
        = 0x007B + 0x5200
        = 0x527B

Step 2: rotate left 3 bits
  verify = (verify << 3) | (verify >> 13)
        = 0x527B << 3 | 0x527B >> 13
        = 0x29BD8 | 0x00
        = 0x29BD8

Step 3: 取低位 16 bits
  verify = 0x29BD8 & 0xFFFF = 0x9BD8
```

### 4.3 第一個封包（Hello Packet）

- 連接建立後第一個封包為**明文**（未加密）
- 用於交換 AES 初始化向量 (IV)
- 客戶端和伺服器各自生成隨機密鑰

## 五、WZ 修改

### 5.1 Maple83 項目 WZ 修改流程（jonnylin13/Maple83）

**來源**: https://github.com/jonnylin13/Maple83

#### 編輯器推薦
- **HaRepacker 4.2.4**
- 加密模式：`GMS (old)`

#### 編輯流程
```
1. 用 HaRepacker 開啟要編輯的 WZ 檔案
2. 在 UI 中進行修改
3. 保存，覆蓋客戶端目錄中的原檔案
4. 重新導出（Export）XML 到服務端的 WZ.XML
```

#### ⚠️ 重要警告
- **禁止**使用服務端的 XML 重新導入客戶端 WZ！
- 這會導致各種 bug
- 正確做法是用 HaRepacker UI 直接編輯

### 5.2 服務端/客戶端同步原則

```
服務端 WZ.XML ≠ 客戶端 WZ
     ↓              ↓
  服務器代碼引用   客戶端顯示引用
     ↓              ↓
   保持同步        可獨立編輯
```

### 5.3 String.wz 修改範例（MonsterBook）

jonnylin13 提供了 MobBookUpdate 專案：
1. 用 HaRepacker 開啟 `String.wz`（客戶端）
2. 刪除 `MonsterBook.img` 節點
3. 從服務端導入更新後的 `MonsterBook_updated.img.xml`
4. 保存覆蓋

**危險操作**：這個操作是危險的，因為刪除/導入節點可能導致數據丟失。請務必備份！

### 6.2 NPC 腳本基本結構（JavaScript）

```javascript
var status = -1;

function start() {
    cm.sendNext("你好，我是測試NPC。");
}

function action(mode, type, selection) {
    if (mode == -1) {
        // 關閉對話框
        cm.dispose();
    } else {
        if (mode == 1) {
            status++;
        } else {
            status--;
        }
        
        if (status == 0) {
            cm.sendYesNo("你想要開始任務嗎？");
        } else if (status == 1) {
            if (selection == 1) {
                cm.startQuest(1000);
                cm.sendOk("任務已開始！");
            } else {
                cm.sendOk("也許下次吧。");
            }
        } else {
            cm.dispose();
        }
    }
}
```
