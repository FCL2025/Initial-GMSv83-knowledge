### 4G Flag 設置（允許客戶端使用 4GB 記憶體）

**修改步驟**：
```
1. 使用 Hex Editor 打開 MapleStory.exe
2. 找到 PE Header 中的特定偏移: 0x0000013E
3. 預設值: 0x0F → 修改為: 0x2F
4. 保存並關閉
```

**目的**：允許客戶端使用最大 4GB 虛擬記憶體，對於大型 WZ 文件（如 HD 客戶端）非常有用。

### LAN 多人連線常見錯誤排查

**錯誤碼分析**：

**排查檢查清單**：
- [ ] 確認兩台 PC 在同一 LAN 網段
- [ ] 確認伺服器 listen 0.0.0.0（而非 127.0.0.1）
- [ ] 確認防火牆開啟 7575 和 8484 埠口

- [ ] config.yaml HOST 設定為 LAN IP（不是 127.0.0.1）
- [ ] WZ 檔案在兩台 PC 上完全一致

## 三、Opcode結構 — 深度解析

### Send vs Recv Opcode 核心區分

> "Transmitting from the server and receiving on the client has one set of agreed upon Opcodes, where transmitting from the client and receiving on the server has yet another."

| 方向 | 說明 | Opcode 集合 |
|------|------|------------|
| **S→C (Send)** | 伺服器發送，客戶端接收 | Server Opcode Set |
| **C→S (Recv)** | 客戶端發送，伺服器接收 | Client Opcode Set |

**⚠️ 重要**：同一個 Opcode 數字在不同方向代表**完全不同**的含義！

### MapleCrypto 頭部混淆公式

```java
// 加密端
int verify = (opcode ^ (iv & 0xFF)) + ((iv >> 8) & 0xFF) * 0x100;
verify = (verify << 3) | (verify >>> 5);
verify &= 0xFFFF;

// 解密端
int verify = (header[0] & 0xFF) | ((header[1] << 8) & 0xFF00);
verify = (verify >>> 3) | ((verify << 5) & 0xFFFF);
return (verify - ((iv >> 8) & 0xFF) * 0x100) ^ (iv & 0xFF);
```

### v83 LAN 連線問題 Opcode 視角排查

```
正常連線流程（Opcode 視角）：
1. C→S: 0x01 (LOGIN_PASSWORD) - 加密封包
2. S→C: 0x00 (LOGIN_STATUS) - 登入結果

連線失敗時：
→ 若完全看不到任何封包 → 防火牆/網路問題
→ 若 Hello Packet 正常但登入封包異常 → 版本/IV 不匹配
→ 若 Hello Packet 就失敗 → 伺服器未啟動或 listen 問題
```

### 登入結果碼（Login Status Code）

| 結果碼 | 意義 |
|--------|------|
| 0 | 登入成功 |
| 5 | 帳號或密碼錯誤 |
| 7 | 帳號已停用 |
| 13 | 刪除中（冷靜期） |
| 23 | 登入太頻繁（防機器人延遲） |

## 八、端口配置要點（RaGEZONE, Maple83 GitHub）

- MapleSolaxia 需要開放大端口：`7575 + (channel數)` 和 `8484`
- 建立新的自訂服務，啟用 TCP/UDP 兩種協定
- 預設配置：
  - Login Port: 8484
  - Channel 1: 7575
  - Channel 2: 7576
  - ... 依此類推
