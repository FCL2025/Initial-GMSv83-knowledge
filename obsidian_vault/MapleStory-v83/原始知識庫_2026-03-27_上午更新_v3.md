5. 保存並退出

**客戶端修改前置步驟 (以 HeavenMS 為例)**
1. 安裝 MapleStory (使用 ManagerMsv83.exe)
2. 刪除: `HShield/` 資料夾, `ASPLnchr.exe`, `MapleStory.exe`, `Patcher.exe`
3. 複製 `localhost.exe` 到客戶端資料夾

## 2. 抓包分析 (Packet Analysis)

### Maple_Pshark (GitHub: obstriker/Maple_Pshark)
專為 v83 設計的封包攔截工具

**功能特色:**
- Hook 到加密函數，直接攔截訊息
- 攔截 Recv 訊息
- 修改訊息內容
- 注入 send/recv 封包
- 基於規則的封包操作/過濾
- 可在 localhost 直接使用 (監聽 127.0.0.1:8484)

**使用流程:**
1. 啟動 MapleStory localhost 客戶端
2. 啟動 Maple_Pshark
3. 觀察封包交換
4. 自動解密並解析 opcode

### MaplePacketPuller (GitHub: Bratah123/MaplePacketPuller)
- Python 封包結構分析工具
- 自動解析封包格式並識別 opcode

```
tcp.port == 7575 || tcp.port == 8484
```

- 第一個封包永遠是明文 (Hello Packet)

- 客戶端和伺服器各自生成隨機密鑰

## 3. Opcode 結構 (Opcode Structure)

### v83 Opcode 特性
- **長度**: 2 bytes (little endian)
- **版本**: v83 的 Opcode 已幾乎全部被社群映射
- **加密**: 使用 MapleCrypto + Shanda + AES 三層加密

### 封包頭部結構
- Header 為 4 bytes (包含長度 + 混淆 opcode)
- 第一個封包 (Hello Packet) 永遠是明文
- 使用 AES-OFB (Advanced Encryption Standard using Output Feed Buffer)

### 加密流程
1. **明文資料** → AES 加密 → 更新 IV shift
2. **Shanda 加密** (v83 仍有)
3. 最終封包包含混淆後的 opcode

### MapleStory Reference Wiki
- https://mapleref.fandom.com/wiki/Packets
- 完整的封包格式文檔
- Opcode 列表可供參考

### IDA/逆向工程獲取 Opcode (RaGEZONE)
- 使用 IDA 分析客戶端 DLL
- 找到封包處理函數
- 動態分析比靜態分析更有效
- 教程: https://forum.ragezone.com/threads/getting-packet-structures-opcodes-using-ida.792436/

## 4. WZ 修改 (WZ File Modification)

### 主要工具

**Harepacker-resurrected** (GitHub: lastbattle/Harepacker-resurrected)
- 官方推薦的 WZ 編輯器
- All-in-one .wz 檔案編輯器
- 可用於地圖、道具、NPC 等修改
- 必須在關閉 MapleStory 的情況下編輯

**使用方法:**
1. 開啟 HaRepacker
2. 點擊 "Load .wz" 載入目標 WZ 檔案
3. 導航到要修改的項目 (如 Map.wz → physics → walkspeed)
4. 修改數值後保存

### 常見 WZ 檔案用途
| WZ 檔案 | 用途 |
|---------|------|
| Character.wz | 角色造型、發型、染色 |
| Item.wz | 道具屬性 |
| Map.wz | 地圖結構 |
| Mob.wz | 怪物屬性 |
| Npc.wz | NPC 圖像 |
| Skill.wz | 技能屬性 |
| Quest.wz | 任務資料 |

### WZ 編輯概念 (RaGEZONE 2013)
- WZ 檔案包含遊戲各種資料
- 並非所有內容都在 WZ 中，有些需要服務端修改
- WZ 編輯需要配合伺服器端的 wz.xml 設定

### XML 導出/導入流程
1. 在 HaRepacker 中導出 WZ 為 XML
2. 在伺服器 wz 資料夾中替換
3. 使用 WZClear 或 XML Parser 處理

### v83 社群 WZ 整合包 (RaGEZONE 2023)
- 可將新地圖/內容整合到 v83
- 導出 XML 到伺服器 wz 資料夾
- 替換原有檔案後重啟伺服器

## 5. NPC 腳本編寫 (NPC Scripting)

### 腳本位置
`scripts/npc/` 目錄，每個 NPC ID 對應一個 `.js` 檔案

### 核心函數結構
```javascript
var status = 0;

function start() {
    cm.sendNext("歡迎來到我的商店！");  // status = 0
}

function action(mode, type, selection) {
    if (mode == 1) {
        // 按下 "下一步"
        status++;
    } else {
        // 按下 "結束" 或取消
        cm.dispose();
        return;
    }
    
    if (status == 1) {
        cm.sendSimple("請選擇:\n #b#L0#購買道具#l\n #L1#出售道具#l");
    } else if (status == 2) {
        if (selection == 0) {
            cm.sendNext("感謝購買！");
        } else {
            cm.sendNext("感謝光臨！");
        }
    } else {
        cm.dispose();
    }
}
```

### 常用 cm 方法
| 方法 | 用途 |
|------|------|
| cm.sendNext() | 下一頁對話 |
| cm.sendSimple() | 選擇選單 |
| cm.haveItem(id, count) | 檢查物品 |
| cm.gainItem(id, count) | 給予物品 |
| cm.gainMesos(amount) | 給予楓幣 |
| cm.changeJob(jobId) | 轉職 |
| cm.warp(mapId) | 傳送 |
| cm.openShop(shopId) | 開店 |
| cm.startQuest/completeQuest | 任務操作 |
| cm.getPlayer() | 獲取玩家物件 |
| cm.getEvent() | 獲取事件物件 |

### MSI NPC 腳本格式
```javascript
var status = 0;
var selected = 1;
var wui = 0;

function start() {
    status = -1;
    action(1, 0, 0);
}

function action(mode, type, selection) {
    if (mode == -1) {
        cm.dispose();
    } else if (mode == 1) {
        status++;
    } else {
        status--;
    }
    
    if (status == 0) {
        cm.sendYesNo("要進入神秘商店嗎？");
    } else if (status == 1) {
        cm.openShop(9030000);
        cm.dispose();
    }
}
```

### 動態商店腳本 (v83)
- 可不使用 SQL 表格的商店系統
- 透過修改 server 端檔案動態定義商品
- 範例: db_shopupdate.sql 可更新 NPC 商店內容

### Quest NPC 腳本
- Quest 腳本使用與一般 NPC 不同的觸發機制
- 需要發送特定封包才能觸發自訂腳本
- 腳本位置: `scripts/quest/`

### NPC 變量使用
```javascript
var var1 = cm.getEventInstance().getIntProperty("var1");
cm.getEventInstance().setIntProperty("var1", var1 + 1);
```
