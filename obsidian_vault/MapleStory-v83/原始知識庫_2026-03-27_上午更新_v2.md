## 2. 抓包分析工具

### Maple_Pshark (GitHub: obstriker/Maple_Pshark)
**專為 v83 設計的封包 Logger**

功能特色：
- Hook 到加密函數，直接攔截訊息
- 攔截 Recv 訊息
- 修改訊息內容
- 注入 send/recv 封包
- 基於規則的封包操作/過濾
- 可在 localhost 直接使用（監聽 127.0.0.1:8484）

使用流程：
1. 啟動 MapleStory localhost 客戶端
2. 啟動 Maple_Pshark
3. 觀察封包交換
4. 自動解密並解析 opcode

### MaplePacketPuller (GitHub: Bratah123/MaplePacketPuller)
- Python 封包結構分析工具
- 自動解析封包格式並識別 opcode

### Wireshark 過濾表達式
```
tcp.port == 7575 || tcp.port == 8484
```

## 3. Opcode 結構

### v83 Opcode 特性
- **長度**: 2 bytes (little endian)
- **版本**: v83 的 Opcode 已幾乎全部被映射
- **加密**: 使用 MapleCrypto + Shanda + AES 三層加密

### 封包頭部結構
- Header 為 4 bytes（包含長度 + 混淆 opcode）
- 第一個封包永遠是明文（Hello Packet）

### MapleStory Reference Wiki (Fandom)
- https://mapleref.fandom.com/wiki/Packets
- 完整的封包格式文檔
- Opcode 列表

## 5. NPC 腳本編寫

### 腳本位置
`scripts/npc/` 目錄，每個 NPC ID 對應一個 `.js` 檔案

### 核心函數
```javascript
function start() {
    cm.sendNext("文字");  // status = 0
}

function action(mode, type, selection) {
    if (mode == 1) {
        // 按下 "下一步"
    }
    status++;
}
```

### 常用 cm 方法
| 方法 | 用途 |
|------|------|
| cm.sendNext() | 下一頁對話 |
| cm.sendSimple() | 選擇選單 |
| cm.haveItem(id, count) | 檢查物品 |
| cm.gainItem(id, count) | 給予物品 |
| cm.gainMesos(amount) | 給予楓幣 |
| cm.changeJob(jobId) | 轉職 |
| cm.warp(mapId) | 傳送 |
| cm.openShop(shopId) | 開店 |
| cm.startQuest/completeQuest | 任務操作 |

### Quest NPC 腳本
- Quest 腳本使用與一般 NPC 不同的�發機制
- 需要發送特定封包才能觸發自訂腳本

### 動態商店腳本 (v83)
- 可不使用 SQL 表格的商店系統
- 透過修改 server 端檔案動態定義商品

## 學習建議

1. **入門順序**: 
   - 先架設本地伺服器 (HeavenMS/MapleSolaxiaV2)
