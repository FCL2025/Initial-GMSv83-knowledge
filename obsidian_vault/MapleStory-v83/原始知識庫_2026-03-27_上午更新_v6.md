## 二、客戶端修改 — 最新發現

### 2.1 localhost.exe 修改流程（HeavenMS）

根據 HeavenMS 官方文檔，最新的客戶端修改流程如下：

**所需工具：**
- `ManagerMsv83.exe` — 安裝官方客戶端
- `localhost.exe` — 反向工程後的登入器（用於連接私服）
- `Neo Hex Editor` 或 `HxD` — HEX編輯器

**步驟：**
1. 使用 ManagerMsv83 安裝 MapleStory 到 `C:\Nexon\MapleStory`
2. 刪除：`HShield` 資料夾、`ASPLnchr.exe`、`MapleStory.exe`、`Patcher.exe`
3. 用 `localhost.exe` 替換原執行檔
4. 用提供的 WZ 檔案覆蓋原版（`commit397_wz` 或 `current_wz`）
5. **如需修改IP**：用 HEX 編輯器搜索 `127.0.0.1`（Type: String），找到所有位置後替換為目標IP

**重要：localhost.exe 會被殺毒軟件標記**，這是因為反向工程方法導致的正常現象，檔案本身無害。

## 三、抓包分析 — 工具與方法

### 3.1 主要抓包工具

#### Maple_Pshark（推薦 v83專用）
- GitHub: `obstriker/Maple_Pshark`
- 專為v83設計，鉤住加密函數攔截訊息
- 功能：
  1. 攔截Recv訊息
  2. 修改訊息
  3. 注入Send/Recv封包
  4. 基於規則的封包過濾

#### MapleShark
- GitHub: `diamondo25/MapleShark`
- 使用 SharpPcap 進行網卡層抓包
- 可分析所有客戶端與伺服器之間的TCP通訊

#### MaplePacketPuller
- GitHub: `Bratah123/MaplePacketPuller`
- Python 工具，用於IDA反編譯 pseudocode 轉換為 packet 結構
- 需要 IDA Pro 生成 pseudocode
- 支援搜尋 InHeader opcodes

#### Wireshark（通用方案）
- 設置 TCP port 8484 過濾
- 缺點：加密後的封包需要手動解密

### 3.2 封包加密機制

- **第一個發送/接收的封包始終是明文（不加密）**
- 封包通過 TCP socket 傳輸
- MapleStory 使用特有的加密演算法（基於IV）
- 封包格式為 little-endian

### 3.3 封包基本結構

```
[Header 4 bytes] [Opcode 2 bytes] [Data...]
```

- Header：包含長度和版本信息
- Opcode：2 bytes，區分不同封包類型
- Data：可變長度，根據Opcode有不同的結構

## 四、Opcode結構 — 核心概念

### 4.1 Opcode 版本變遷

- **早期版本**：Opcode 為 1 byte（最大255個）
- **後來版本**：Opcode 改為 2 bytes short（最大65536個）
- v83 使用 2 bytes Opcode

### 4.2 Header 生成機制

```
Header = f(version, packet length, send IV)
```

- version = `-(version + 1)` 發送給客戶端時
- 接收時使用明文 version

### 4.3 Opcode 分析工具

1. **MaplePacketPuller** — IDA pseudocode 分析
2. **手動分析** — 在伺服器原始碼中搜尋 handle 函數
3. **MapleRef Wiki** — 公共Opcode資料庫（mapleref.fandom.com）

### 4.4 常見Opcode範例

| Opcode | 名稱 | 用途 |
|--------|------|------|
| 0x001F | `PONG` | 心跳響應 |
| 0x000B | `PING` | 心跳請求 |
| 0x007B | `DUPE` | 交易請求 |
| 0x009C | `WARP_TO_FM` | 進入自由市場 |
| 0x00A8 | `MAP_CRASH` | 地圖崩潰 |

## 五、WZ修改 — 進階主題

### 5.1 WZ 檔案結構

- `Character.wz` — 角色相關數據
- `Item.wz` — 物品定義
- `Mob.wz` — 怪物屬性
- `Map.wz` — 地圖配置
- `Npc.wz` — NPC定義
- `Skill.wz` — 技能數據
- `Quest.wz` — 任務數據

### 5.2 WZ編輯注意事項

- 使用 `Wz.json` 或專用編輯器（如GMS管理器）
- 修改前先備份原始檔案
- v83的WZ格式相對較舊，許多現代工具可能不相容
- 建議使用HeavenMS提供的編輯版WZ檔案

### 5.3 怪物的關鍵屬性（From maplesecrets blog）

```xml
rx0 → 最左X座標
rx1 → 最右X座標
type → M (Monster) / N (NPC)
x → 出生X座標
y → 出生Y座標
```

## 六、NPC腳本編寫 — 進階指南

### 6.1 NPC腳本語言

- MapleStory NPC 腳本使用 **JavaScript** 語法
- 與 Java 不同，不能假設兩者相同
- 腳本存放在伺服器端 `scripts/npc/` 目錄
- 檔案名稱對應 NPC 的 ID

### 6.2 腳本基本結構

```javascript
/* NPC ID: 9010000 酒館老闆 */
var status = 0;

function start() {
    cm.sendNext("歡迎來到杉木酒館！");
}

function action(mode, type, selection) {
    if (mode === 1) {
        status++;
    } else {
        status--;
    }
    
    if (status === 1) {
        cm.sendNext("今天想要來點什麼？");
    } else if (status === 2) {
        // 處理選擇
    } else {
        cm.dispose();
    }
}
```

### 6.3 動態商店腳本（Dynamic NPC Shop）

根據 2016 年 RaGEZONE 教程，動態商店腳本可以：
- 不需要每次修改都重啟伺服器
- 動態定義商品和價格
- 適合活動期間的臨時商店

### 6.4 任務NPC（Quest NPC）

任務NPC需要：
1. 在 `Quest.wz` 中定義任務
2. 在 `Npc.wz` 中設置任務標記
3. 使用 `sendNextPrev` 等函數推進劇情

### 6.5 常用cm函數

| 函數 | 用途 |
|------|------|
| `cm.sendNext(text)` | 發送下一段對話 |
| `cm.sendAcceptDecline(text)` | 發送接受/拒絕選項 |
| `cm.gainItem(id, count)` | 給予物品 |
| `cm.getPlayer()` | 獲取玩家對象 |
| `cm.startQuest(questId)` | 開始任務 |
| `cm.completeQuest(questId)` | 完成任務 |

## 七、伺服器架設 — HeavenMS 最新流程

### 7.1 環境需求

- Java 8 SDK + NetBeans（或任何Java IDE）
- WampServer（MySQL）
- mysql-query-browser（可選，視覺化DB）
- Hamachi 或 Port Forwarding（用於遠程連接）

### 7.2 數據庫初始化

執行SQL腳本順序：
1. `db_database.sql` — 建立 schema
2. `db_drops.sql` — 掉落數據
3. `db_shopupdate.sql`（可選）— 商店商品

### 7.3 構建與運行

```bash
# 1. 在IDE中導入項目，選擇 "with existing code"
# 2. 指定 src/ 作為源碼目錄
# 3. 添加 cores/ 中的JAR依賴
# 4. Clean and Build
# 5. 執行 launch.bat
```

### 7.4 帳號創建

默認開啟自動註冊，直接輸入帳密即可創建帳號。

使用 `@commands` 指令可查看所有可用GM指令。
