## 🆕 MapleClientEditTemplate — MapleStory 客戶端編輯框架 (GitHub, 持續更新中)

**GitHub**: https://github.com/oung/MapleClientEditTemplate

**作者**: Minimum Delta (oung)

### 專案定位

一個越來越通用、全面的 MapleStory 客戶端編輯框架，專為製作新的客戶端重定向器/修改而設計。

### 核心功能

| 功能 | 說明 |
|------|------|
| Windows Library Hooks | 包含所有需要的 Windows 庫鉤子 |
| 熱插拔設計 | 可開關的鉤子，視版本需求調整 |
| 無需等待解殼 | v83 localhost 等未虛擬化的客戶端可即時記憶體編輯 |
| VS 2019+ 相容 | C++ Template 專案，需編譯為 x86 |

### 架構說明

```
MapleClientEditTemplate/
├── Common/                    # 通用鉤子（非版本特定）
│   └── commonconfig.h        # 所有通用配置
├── GenericLauncher/          # 通用啟動器
│   └── GenericLauncher.exe   # 啟動 MapleStory 並注入 DLL
├── LEN.dll/                  # 主要編輯 DLL（本體）
├── detours.lib               # 需要放入 Debug 資料夾
└── commonconfig.h            # 配置文件
```

### commonconfig.h 主要設定

| 設定 | 說明 |
|------|------|
| `MAPLE_INJECT_USE_IJL` | 啟用 IJL15 proxy 自動注入（取代啟動器） |
| `MAPLE_INSTAJECT` | 立即注入（不等待客戶端解殼） |
| 開關鉤子 | WinSock、記憶體編輯等可單獨開關 |

### 使用方法

**方法 1：GenericLauncher（通用注入）**
```
1. 編譯 LEN.dll
2. 將編譯出來的 DLL 重命名為 LEN.dll（或設定的目標名）
3. 執行 GenericLauncher.exe
4. 自動啟動 MapleStory 並注入 DLL
```

**方法 2：IJL15 Proxy 自動注入**
```
1. 在 commonconfig.h 中定義 MAPLE_INJECT_USE_IJL
2. 將編譯好的 DLL 重命名為 ijl15.dll
3. 放入 MapleStory 目錄
4. 直接啟動 MapleStory，自動注入
```

### v83 localhost 使用範例

```cpp
// 在 commonconfig.h 中針對 v83 的設定
#define MAPLE_INJECT_USE_IJL 0    // v83 建議用 GenericLauncher
#define MAPLE_INSTAJECT 1         // 立即注入（不需要等待解殼）
#define MAPLE_HOOK_WINSOCK 1      // 啟用 WinSock 鉤子
#define MAPLE_HOOK_MEMORY 1       // 啟用記憶體編輯
```

### 與傳統 Hex 編輯的對比

| 方式 | 優點 | 缺點 |
|------|------|------|
| Hex 編輯 | 簡單直接 | 每次改 IP 需重新編輯 |
| MapleClientEditTemplate | 可動態修改，即時見效 | 需 C++ 編譯環境 |
| localhost.exe 修改 | 簡單 | 功能有限 |

### 編譯注意事項

1. **VS 必須設為 x86**（非 x64！）
   - x64 會導致 detours.lib 連結錯誤
2. **Include 目錄設定**：在專案屬性中加入 `$(SolutionDir)Common;%(AdditionalIncludeDirectories)`
3. **detours.lib 必須放在 Debug 資料夾**

## 🆕 v83 客戶端修改：IP 十六進位修改位置速查 (maplesecrets 2013)

**URL**: http://maplesecrets.blogspot.com/2013/04/how-to-hex-edit-clients-easily.html

### 快速定位 IP 位址

使用 Hex 編輯器（HxD、Neo Hex Editor）在 `localhost.exe` 中搜尋 `127.0.0.1`：

```
搜尋：31 32 37 2E 30 2E 30 2E 31  (127.0.0.1 的 Hex 表示)

替換：xx xx xx xx xx xx xx xx xx  (目標 IP 的 Hex)
```

### 標準 IP 位置（v83 localhost）

| 地址偏移 | 內容 | 說明 |
|---------|------|------|
| 約 +0x1000 | 127.0.0.1 | 主要 IP |
| 約 +0x2000 | 127.0.0.1 | 備用 IP |
| 約 +0x3000 | 127.0.0.1 | 第三處 |

### Hex 編輯器推薦

| 工具 | 特色 | 費用 |
|------|------|------|
| **HxD** | 輕量快速，支援大檔案 | 免費 |
| **Neo Hex Editor** | 現代 UI，離散位址檢視 | 免費版可用 |
| **WinHex** | 專家級，功能全面 | 付費 |

### 步驟

```
1. 右鍵 localhost.exe → Hex edit
2. Ctrl+F 搜尋 127.0.0.1
3. 找到後，選中並輸入目標 IP
4. 儲存
5. 若有 3 處 IP，全部替換
```
