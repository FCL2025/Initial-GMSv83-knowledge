## 🆕 Kinoko NPC Scripts 相容性問題 (RaGEZONE, 2026-03 最新)

**來源**: https://forum.ragezone.com/threads/kinoko-npc-scripts.1261538/

### 問題核心
不同私服音源（Cosmic、Kinoko、HeavenMS）的 NPC 腳本解析器版本不同，較舊的腳本在新解析器可能不相容。

### 相容性速查表（2026-03 最新）

| 音源 | 解析器 | importPackage | ECMAScript 6 | JDK 支援 | 穩定性 |
|------|--------|-------------|-------------|---------|--------|
| **Cosmic (P0nk)** | Nashorn | ❌ 已移除 | ✅ | JDK 8+ | ⭐⭐⭐⭐⭐ 最穩定 |
| **IntransigentMS** | Nashorn | ❌ | ✅ | JDK 8+ | ⭐⭐⭐⭐ |
| **Kinoko** | 混合 | ⚠️ 警告 | ⚠️ 混合 | 混合 | ⭐⭐⭐ |
| **HeavenMS** | 較舊 | ✅ 存在 | ❌ | JDK 6-7 | ⭐⭐ 已停止維護 |
| **MapleSolaxiaV2** | 較舊 | ✅ 存在 | ❌ | JDK 6-7 | ⭐ 已停止維護 |

### 最佳實踐建議

```
✅ 推薦做法：
  1. 使用 Cosmic 作為主要音源（最活躍維護）
  2. 避免 importPackage（JDK 7+ 已廢棄）
  3. 使用 cm. 函數（通用於所有音源）
  4. 避免 JDK 版本特定功能
  5. 先在本地測試再部署

❌ 避免做法：
  1. 不要使用 importPackage
  2. 不要假設所有音源都有相同函數
  3. 不要忽視 JDK 版本差異
```

### Quest NPC 與 MigrateIn 封包

**問題**：如何讓 Quest NPC 出現在 "Other" 選項中？

**解決方案**：
- 需要發送 `PlayerMigrateInPacket` 封包
- 封包包含：NPC ID 列表 + FileTime
- 這個封包讓客戶端知道哪些 NPC 有自訂腳本

## 🆕 v83 LAN 多人連線錯誤 0x80004005 解決方案 (RaGEZONE, 2026-03 最新)

**來源**: https://forum.ragezone.com/threads/need-help-v83-lan-server-works-on-pc1-but-pc2-gets-error-2147467259.1262187/

### 錯誤分析

| 錯誤碼 | Hex 值 | 意義 | 根本原因 |
|--------|--------|------|---------|
| `-2147467259` | `0x80004005` | Access Denied | **防火牆問題（非網路問題）** |

### 排查流程

```
PC2 連線失敗
    ↓
確認錯誤碼是否為 0x80004005 ✅
    ↓
確認兩台 PC 在同一 LAN 網段 ✅
    ↓
確認伺服器 listen 0.0.0.0（而非 127.0.0.1）✅
    ↓
檢查 Windows 防火牆
    ↓
    開啟 控制台 → Windows Defender 防火牆 → 允許應用
    ↓
    確認 java.exe 和客戶端 exe 都能通過 ✅
    ↓ 或
    暫時關閉防火牆測試：
    netsh advfirewall set allprofiles state off
    ↓
    若使用 Hamachi → 確認同一 Hamachi 網路
    ↓
    若在路由器後 → Port Forward: 7575 (login), 8484 (game)
```

### 防火牆快速設定

```powershell
# PowerShell 管理员权限 - 開啟 MapleStory 必要連接埠
New-NetFirewallRule -DisplayName "MapleStory Login" `
  -Direction Inbound `
  -Protocol TCP `
  -LocalPort 7575,8484 `
  -Action Allow
```

### 伺服器 Listen 位址設定

若伺服器預設綁定 `127.0.0.1`，PC2 無法連線：
```
檢查 ServerConstants.java 或 config.yaml
確認 listen address 設為 0.0.0.0
而非 127.0.0.1
```

## 🆕 MaplePacketPuller + SpiritIDAPlugin 最新狀態 (2026-03)

**GitHub**:
- MaplePacketPuller: https://github.com/Bratah123/MaplePacketPuller
- SpiritIDAPlugin: https://github.com/Bratah123/SpiritIDAPlugin

### 現況
- Python 跨平台分析工具
- IDA Pro 插件版（SpiritIDAPlugin）可直接在 IDA 中格式化封包結構
- 支援 Python 3.8.5、3.6.12

### 完整工作流程
```
Step 1: IDA Pro 分析
  → 用 IDA 打開 MapleStory.exe
  → 生成目標函數的 C-style pseudocode
  → 複製 pseudocode 到 .txt 文件
  → 將 .txt 檔案放入 Functions/ 目錄

Step 2: SpiritIDAPlugin 格式化
  → 執行 MaplePacketPuller/IDA Maple Script/src/main/python/main.py
  → 選擇 Analysis type
  → 輸出格式化後的封包結構到 FuncOutput/

Step 3: MaplePacketPuller 解析
  → 將格式化後的結構輸入 MaplePacketPuller
  → 生成對應的 Java/C# 程式碼
```

## 🆕 RustMS - Rust 實現 MapleStory v83 (2026-03 重啟)

**GitHub**: https://github.com/neeerp/RustMS

### 專案現況
- **2026 年 3 月重啟**，使用 AI coding assistant 輔助開發
- 目前僅包含基本可用的 **Login Server**（監聽 localhost:8484）
- 完全使用 Rust 語言從頭編寫
- 支援加密協商（不需要禁用加密）

### 技術架構
```
rust-ms-login
├── net::io      → 封包讀寫、加密/解密、Session 管理
├── net::packet  → handle/ (處理器) + build/ (構建器)
├── crypt        → MapleAES (OFB) + Shanda + bcrypt
└── db           → Diesel ORM + PostgreSQL
```

### RustMS vs Java 模擬器

| 特性 | RustMS | Java 模擬器 (Cosmic/HeavenMS) |
|------|--------|-------------------------------|
| 語言 | Rust | Java |
| 記憶體效率 | ✅ 極佳（無 GC） | ⚠️ JVM 開銷 |
| 執行速度 | ✅ 優秀 | ⚠️ JVM JIT 需預熱 |
| 生態完整度 | ⚠️ 僅 Login Server | ✅ 全功能 |
| 學習資源 | ⚠️ 匱乏 | ✅ 豐富 |
| 維護難度 | ⚠️ Rust 陡峭 | ✅ Java 簡單 |
