## ☕ JDK 安裝

### 版本需求（重要更新 2026-03）
| 專案 | 最低 JDK 版本 | 推薦版本 | 備註 |
|------|-------------|---------|------|
| **Cosmic** | **Java 21** | **Java 21 (Amazon Corretto)** | ⚠️ 重大更新！ |
| **HeavenMS** | JDK 7/8 | JDK 8 | 舊版專案 |
| **MapleSolaxiaV2** | JDK 7 | JDK 7u79 | 需要 JDK 7 |

### 安裝步驟（Cosmic）

1. 下載 JDK
   - ** Amazon Corretto (Cosmic 推薦)**：https://aws.amazon.com/corretto/
   - **Adoptium**：https://adoptium.net/ （無需帳號）
   - **Oracle**：https://www.oracle.com/java/technologies/javase-downloads.html

2. **Cosmic 必須使用 Java 21**，舊版 JDK 無法編譯

3. 設定環境變數
   ```
   JAVA_HOME = C:\Program Files\Amazon Corretto\jdk21.x.x-hotspot
   PATH = %JAVA_HOME%\bin
   ```

### Java 版本差異說明
- **Java 21 (Cosmic)**：現代版本，支援 Records、Pattern Matching、Virtual Threads
- **Java 8 (HeavenMS)**：經典版本，廣泛支援，Nostalgia 私服常用
- **Java 7 (MapleSolaxiaV2)**：古老版本，需要特定 JDK 7u79

## 🗄️ MySQL 安裝

### 版本需求（重要更新 2026-03）
| 專案 | MySQL 版本需求 | 備註 |
|------|---------------|------|
| **Cosmic** | **MySQL 8+** | ⚠️ 必需 MySQL 8 |
| **HeavenMS** | 5.5+ | 寬鬆 |
| **MapleSolaxiaV2** | 5.5+ | 寬鬆 |

### 安裝方式

#### 方式一：獨立安裝 MySQL Server (推薦)
1. 下載：https://dev.mysql.com/downloads/mysql/
2. 選擇 `mysql-installer-community-8.x.msi`
3. 安裝時選擇 **"Server only"** 或 **"Full"**
4. 設定 root 密碼（**務必記住！**）

#### 方式二：使用 HeidiSQL (資料庫客戶端)
1. 下載：https://www.heidisql.com/download.php
2. 安裝後建立新連線：
   - 主機：`127.0.0.1`
   - 使用者：`root`
   - 密碼：（你設定的密碼）
3. 建立資料庫：
   - 右鍵 → 建立新資料庫 → 名稱：`cosmic`

### Cosmic 完整資料庫架設流程 (GitHub, 2024-2025)

**來源**: https://github.com/P0nk/Cosmic

```
步驟 1: 安裝 MySQL 8+
  → 下載 MySQL Community Server 8+
  → 設定 root 密碼（務必記住！）

步驟 2: 安裝 HeidiSQL
  → https://www.heidisql.com/download.php
  → 連線到 MySQL 8

步驟 3: 建立資料庫
  → 在 HeidiSQL 中右鍵 → 建立新資料庫
  → 名稱設為: cosmic
  → 完成後無需手動建立表格（Cosmic 啟動時自動建立）

步驟 4: Clone Cosmic 專案
  → git clone https://github.com/P0nk/Cosmic.git
  → 使用 IntelliJ IDEA 開啟

步驟 5: 設定資料庫連線
  → 修改專案中的 db.properties:
    username=root
    password=<你的密碼>
    url=jdbc:mysql://localhost:3306/cosmic
```

## 🐳 Docker 安裝 (可選)

### 使用 Docker 架設 MySQL
```bash
# 啟動 MySQL 8 容器
docker run -d \
  --name mysql8 \
  -e MYSQL_ROOT_PASSWORD=your_password \
  -e MYSQL_DATABASE=cosmic \
  -p 3306:3306 \
  mysql:8

# 驗證
docker exec -it mysql8 mysql -u root -p
```

## ⚙️ 資料庫設定

### 1. 建立資料庫
使用 HeidiSQL：
```
右鍵 → 建立新資料庫 → 名稱：cosmic → 編碼：utf8mb4
```

### 2. 設定資料庫連線
編輯 `Cosmic/scripts/database/database.properties` 或設定檔：
```properties
db.host=127.0.0.1
db.port=3306
db.name=cosmic
db.user=root
db.password=your_password
```

### 3. 初始化資料庫
首次啟動時，Cosmic 會自動建立所有必要的表格。

## 🔨 編譯伺服器

### 使用 Gradle (Cosmic)
```bash
# 進入 Cosmic 目錄
cd Cosmic

# 編譯 (會自動下載依賴)
./gradlew build

# 或直接執行 (首次會編譯)
./gradlew run
```

### 輸出位置
```
Cosmic/build/libs/
├── cosmic-x.x.x.jar       # 主程式
└── scripts/                # 腳本資料夾
```

### 快速啟動指令
```bash
# Windows
.\gradlew.bat run

# Linux/macOS
./gradlew run
```

## 🐾 常見問題

### Q: Gradle 編譯失敗
A: 確認JDK版本正確，嘗試 `./gradlew clean build`

### Q: MySQL 連線失敗 (Communications link failure)
A: 確認 MySQL 服務已啟動，檢查 port 3306 是否被占用

### Q: 資料庫錯誤 (Unknown database 'cosmic')
A: 手動建立 cosmic 資料庫，確保編碼為 utf8mb4

### Q: 啟動後立刻斷線
A: 檢查 AES 金鑰和 IV 是否與客戶端匹配
